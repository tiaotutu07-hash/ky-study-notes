# 模块二：5.3 二叉树的遍历与线索二叉树

## 一、二叉树遍历——递归版（地基中的地基）

遍历的本质：把**非线性**的树结构变成**线性**序列。对二叉树的每个结点，都要决定**根(N)、左子树(L)、右子树(R)**三者的访问顺序。

以这棵树为例贯穿全部讲解：

```
           A
          / \
        B     C
       / \     \
      D   E     F
     /
    G
```

### 1. 先序遍历（NLR）—— 根→左→右

```c
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);              // 先访问根
        PreOrder(T->lchild);   // 再递归左子树
        PreOrder(T->rchild);   // 最后递归右子树
    }
}
```

手动推演过程（**先序序列的第一个一定是根**）：

```
访问A → 进左子树B
  访问B → 进左子树D
    访问D → 进左子树G
      访问G → 左空,右空,返回
    D右空,返回
  B进右子树E
    访问E → 左空,右空,返回
  B返回
A进右子树C
  访问C → 左空,进右子树F
    访问F → 左空,右空,返回
  C返回

先序序列：A B D G E C F
```

### 2. 中序遍历（LNR）—— 左→根→右

```c
void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);    // 先递归左子树
        visit(T);              // 再访问根
        InOrder(T->rchild);    // 最后递归右子树
    }
}
```

```
中序序列：G D B E A C F
```

**跨章关联：** 你有CSAPP基础，想想每次递归调用发生了什么——函数栈帧压栈！`InOrder(A)`的栈帧里保存着A的上下文，调用`InOrder(B)`时B的栈帧压在A上面，一路压到`InOrder(G)`，G的左子树为空返回，**弹栈回到G**，访问G，G的右子树为空返回，**弹栈回到D**……这就是为什么非递归遍历要用**显式栈**来模拟。

### 3. 后序遍历（LRN）—— 左→右→根

```c
void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);  // 先递归左子树
        PostOrder(T->rchild);  // 再递归右子树
        visit(T);              // 最后访问根
    }
}
```

```
后序序列：G D E B F C A
```

**后序序列的最后一个一定是根。**

### 4. 层序遍历（BFS）—— 用队列

```c
void LevelOrder(BiTree T) {
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);                    // 根入队
    while (!IsEmpty(Q)) {
        DeQueue(Q, p);                // 出队
        visit(p);                     // 访问
        if (p->lchild != NULL)
            EnQueue(Q, p->lchild);    // 左孩子入队
        if (p->rchild != NULL)
            EnQueue(Q, p->rchild);    // 右孩子入队
    }
}
```

```
层序序列：A B C D E F G
```

**跨章关联：** 这就是你在AcWing刷过的**BFS**！第3章学的队列在这里直接用上了。每出队一个结点，就把它的孩子入队——这和BFS搜图是完全同构的。

### 三种递归遍历的对比表

| | 先序(NLR) | 中序(LNR) | 后序(LRN) |
|---|---|---|---|
| 访问根的时机 | **最先** | 左子树之后 | **最后** |
| 序列特征 | 第一个是根 | 根分割左右子树 | 最后一个是根 |
| 递归调用栈深度 | O(h) | O(h) | O(h) |
| 时间复杂度 | O(n) | O(n) | O(n) |

⚠️ **关键认知：三种遍历的递归代码结构完全相同，区别仅在于 `visit(T)` 的位置！** 左右子树的递归顺序始终是先左后右。

### 举一反三（PPT补充）

若想实现**逆层序遍历**（从下到上、从右到左），只需在层序遍历的基础上借助一个**栈**：层序遍历时每个出队结点不直接访问而是压栈，最后依次弹栈访问即可。

## 二、非递归遍历（用显式栈模拟）——算法题高频考点

递归本质是系统帮你用调用栈，非递归就是你自己用栈来模拟。

### 中序遍历的非递归（最核心，必须手写）

思路：沿着左子树一路压栈到底→弹栈访问→转向右子树

```c
void InOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !IsEmpty(S)) {
        if (p) {                    // 左子树一路压栈
            Push(S, p);
            p = p->lchild;
        } else {                    // 左子树到底了
            Pop(S, p);              // 弹栈
            visit(p);               // 访问
            p = p->rchild;          // 转向右子树
        }
    }
}
```

用上面的树跑一遍：

```
p=A, 压A, p=B
p=B, 压B, p=D
p=D, 压D, p=G
p=G, 压G, p=NULL
弹G, 访问G, p=G->rchild=NULL
弹D, 访问D, p=D->rchild=NULL
弹B, 访问B, p=B->rchild=E
p=E, 压E, p=NULL
弹E, 访问E, p=E->rchild=NULL
弹A, 访问A, p=A->rchild=C
p=C, 压C, p=C->lchild=NULL
弹C, 访问C, p=C->rchild=F
p=F, 压F, p=NULL
弹F, 访问F, p=NULL
栈空且p=NULL, 结束

输出：G D B E A C F  ✓
```

### 先序遍历的非递归

只需把 `visit` 移到压栈前：

```c
void PreOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !IsEmpty(S)) {
        if (p) {
            visit(p);              // 先访问再压栈！
            Push(S, p);
            p = p->lchild;
        } else {
            Pop(S, p);
            p = p->rchild;
        }
    }
}
```

### 后序遍历的非递归（最难，需要标记）

难点：弹栈时要区分"从左子树返回"还是"从右子树返回"——只有右子树也处理完了才能访问根。

```c
void PostOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T, r = NULL;       // r记录最近访问的结点
    while (p || !IsEmpty(S)) {
        if (p) {                   // 左子树一路压栈
            Push(S, p);
            p = p->lchild;
        } else {
            GetTop(S, p);         // 看栈顶（不弹出！）
            if (p->rchild && p->rchild != r) {
                // 右子树存在且未被访问
                p = p->rchild;
            } else {
                // 右子树不存在或已访问，可以访问根了
                Pop(S, p);
                visit(p);
                r = p;            // 标记为最近访问
                p = NULL;         // 重要！置空以继续弹栈
            }
        }
    }
}
```

⚠️ **后序非递归的核心：栈中保存的是从根到当前结点的路径！** 这个性质在408算法题中极其有用（比如求两个结点的最近公共祖先）。

### 非递归遍历总结

| 遍历 | 核心思路 | visit时机 |
|---|---|---|
| 先序非递归 | 左路压栈，访问后压栈 | 压栈前 |
| 中序非递归 | 左路压栈，弹栈时访问 | 弹栈时 |
| 后序非递归 | 左路压栈+标记右子树是否已访问 | 右子树处理完后 |

⚠️ **零壹讲义的例题（非常好）：** 在后序非递归遍历中，若p是q的祖先，则栈中存在从p到q的**干净路径**——因为后序遍历中，访问q时，q的所有祖先都还在栈中未弹出。先序和中序做不到这一点。

## 三、由遍历序列还原二叉树（年年考！）

### 核心规则

**必须有中序序列 + 另外一种，才能唯一确定二叉树。**

| 组合 | 能否唯一确定？ |
|---|---|
| 先序 + 中序 | ✅ |
| 后序 + 中序 | ✅ |
| 层序 + 中序 | ✅ |
| 先序 + 后序 | ❌ |
| 先序 + 层序 | ❌ |
| 后序 + 层序 | ❌ |

### 方法：先序+中序 还原（以王道例题为例）

先序：`A B C D E F G H I`，中序：`B C A E D G H F I`

```
Step 1: 先序第一个 A 是根
        中序中找A → 左子树中序{B,C}, 右子树中序{E,D,G,H,F,I}
        对应先序：左子树先序{B,C}, 右子树先序{D,E,F,G,H,I}

Step 2: 左子树先序第一个 B 是左子树的根
        中序{B,C}中B在最前 → B无左子树, 右子树中序{C}
        → B的右孩子是C

Step 3: 右子树先序第一个 D 是右子树的根
        中序{E,D,G,H,F,I}中D的位置 → 左子树{E}, 右子树{G,H,F,I}
        → D的左孩子是E

Step 4: 继续递归处理{G,H,F,I}...
        先序中对应{F,G,H,I}, 根是F
        中序中F的位置 → 左子树{G,H}, 右子树{I}
        ...
```

最终：
```
            A
          /   \
        B       D
         \     / \
          C   E   F
             / \   \
            G   H   I
       (实际G是F的左子树再分)
```

### 方法：后序+中序 还原

后序**最后一个**是根，其余思路相同——从后序末尾取根，用中序分割左右。

### 方法：层序+中序 还原

层序**第一个**是根，在中序中分左右后，层序的**第二个元素**如果在左子树中序中则是左子树根，否则是右子树根——以此递归。

⚠️ **为什么先序+后序不行？** 王道给了反例：两棵不同的树——一棵只有左孩子B，一棵只有右孩子B——先序都是AB，后序都是BA，层序都是AB。没有中序就**无法区分左右**。

### 例题速解套路

```
先序: A B C D E F
中序: C B A E D F

Step 1: 根=A (先序首), 中序分割 → 左{C,B} 右{E,D,F}
Step 2: 左子树: 先序{B,C}, 根=B, 中序{C,B} → C在B左边 → C是B的左孩子
Step 3: 右子树: 先序{D,E,F}, 根=D, 中序{E,D,F} → E是D左孩子, F是D右孩子

       A
      / \
     B   D
    /   / \
   C   E   F

后序序列: C B E F D A  ✓(讲义答案A)
```

## 四、线索二叉树（难点+重点）

### 为什么需要线索二叉树？

模块一讲过：n个结点的二叉链表有 **n+1个空指针**。这些空指针白白浪费了。

同时，普通二叉链表只能体现**父→子**的关系，无法直接找到某结点在遍历序列中的**前驱**和**后继**——必须重新遍历一遍才行，效率低。

**线索二叉树的核心思想：** 利用空指针存放前驱/后继信息！

### 结点结构

```
| lchild | ltag | data | rtag | rchild |

ltag=0: lchild指向左孩子
ltag=1: lchild指向前驱（线索）

rtag=0: rchild指向右孩子  
rtag=1: rchild指向后继（线索）
```

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;    // 0=孩子, 1=线索
} ThreadNode, *ThreadTree;
```

### 中序线索二叉树（最重要！）

以这棵树为例，中序序列为 `D B E A F C G`：

```
           A
          / \
        B     C
       / \   / \
      D   E F   G
```

线索化过程：检查每个结点的空指针，空左指针→指向中序前驱，空右指针→指向中序后继。

```
D: 无左孩子 → lchild指向NULL(D是中序第一个,无前驱)
   无右孩子 → rchild指向B(中序后继)
   ltag=1, rtag=1

E: 无左孩子 → lchild指向B(中序前驱)
   无右孩子 → rchild指向A(中序后继)
   ltag=1, rtag=1

F: 无左孩子 → lchild指向A(中序前驱)
   无右孩子 → rchild指向C(中序后继)
   ltag=1, rtag=1

G: 无左孩子 → lchild指向C(中序前驱)
   无右孩子 → rchild指向NULL(G是中序最后一个,无后继)
   ltag=1, rtag=1

A,B,C: 左右孩子都有, ltag=0, rtag=0, 不变
```

### 中序线索化的代码（必须掌握）

```c
void InThread(ThreadTree &p, ThreadTree &pre) {
    if (p != NULL) {
        InThread(p->lchild, pre);       // 递归线索化左子树
        
        if (p->lchild == NULL) {        // 左子树空→建前驱线索
            p->lchild = pre;
            p->ltag = 1;
        }
        if (pre != NULL && pre->rchild == NULL) {
            pre->rchild = p;            // 前驱的右子树空→建后继线索
            pre->rtag = 1;
        }
        
        pre = p;                        // pre跟进
        InThread(p->rchild, pre);       // 递归线索化右子树
    }
}

// 主过程
void CreateInThread(ThreadTree T) {
    ThreadTree pre = NULL;
    if (T != NULL) {
        InThread(T, pre);
        pre->rchild = NULL;     // 处理最后一个结点
        pre->rtag = 1;
    }
}
```

**理解要点：** 线索化的本质就是在**中序遍历过程中**，顺便把空指针填上前驱/后继。`pre`始终指向**刚刚访问过的结点**（即当前结点p的中序前驱）。

### 中序线索二叉树的遍历（不需要栈！）

这是线索化的核心价值——可以像遍历链表一样遍历二叉树：

```c
// 找中序序列的第一个结点（最左下结点）
ThreadNode *Firstnode(ThreadNode *p) {
    while (p->ltag == 0) p = p->lchild;
    return p;
}

// 找结点p在中序序列中的后继
ThreadNode *Nextnode(ThreadNode *p) {
    if (p->rtag == 0)                    // 有右子树
        return Firstnode(p->rchild);     // 后继是右子树的最左下结点
    else
        return p->rchild;               // rtag==1, 直接返回后继线索
}

// 中序遍历（非递归，不用栈！）
void Inorder(ThreadTree *T) {
    for (ThreadNode *p = Firstnode(T); p != NULL; p = Nextnode(p))
        visit(p);
}
```

**跨章关联：** 这段代码的结构和你第2章遍历链表 `for(p=head; p!=NULL; p=p->next)` 如出一辙！线索二叉树本质上就是把树"穿成了一条链"。

### 三种线索二叉树的能力对比（核心考点！）

| 线索类型 | 找前驱 | 找后继 |
|---|---|---|
| **先序**线索 | ❌ 需要栈 | ✅ 可以 |
| **中序**线索 | ✅ 可以 | ✅ 可以 |
| **后序**线索 | ✅ 可以 | ❌ 需要栈（或三叉链表） |

**为什么先序线索找不了前驱？**

先序是NLR。考虑结点p，它的先序前驱要么是p的双亲，要么是双亲左子树中最后被先序访问的结点。但线索二叉树**没有指向双亲的指针**，如果p是其双亲的左孩子，我们根本无法通过p的指针回到双亲。

**为什么后序线索找不了后继？**

后序是LRN。结点p的后序后继可能是p的双亲（如果p是右孩子或唯一孩子），或者双亲右子树的"后序第一个结点"（如果p是左孩子且双亲有右子树）。同样需要双亲指针，但线索二叉链表没有。

⚠️ **结论：中序线索二叉树是最实用的——前驱后继都能O(1)找到。** 这也是408考得最多的。

## 五、先序序列对应不同二叉树的个数（PPT例题）

先序序列为 `abcd` 的不同二叉树个数 = **卡特兰数** C(2n, n)/(n+1)

n=4 时：C(8,4)/5 = 70/5 = **14**

这个公式和第3章栈的出栈序列个数是一样的！因为先序遍历的递归过程本质上等价于一个栈的入栈出栈序列。

## 【模块二总结】

| 知识点 | 核心要点 |
|---|---|
| 递归遍历 | 三种只差visit位置，时间O(n)空间O(h) |
| 非递归中序 | 左路压栈→弹栈访问→转右，背住模板 |
| 非递归后序 | 需要标记r记录最近访问，栈中=祖先路径 |
| 层序遍历 | 队列BFS，和AcWing的BFS同构 |
| 还原二叉树 | **必须有中序**+另一种，递归分割 |
| 线索二叉树 | 利用n+1个空指针存前驱/后继 |
| 中序线索 | 前驱后继都能找，最实用 |
| 先序线索 | 能找后继，不能找前驱 |
| 后序线索 | 能找前驱，不能找后继 |
| 卡特兰数 | n个结点的不同二叉树 = C(2n,n)/(n+1) |
