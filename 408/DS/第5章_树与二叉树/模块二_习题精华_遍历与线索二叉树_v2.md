> 基于王道5.3节45道选择题 + 17道综合应用题，含11道统考真题（2009-2024）

## 一、题型分类与解题方法

### 【题型1】遍历序列的性质判断（第01、02、06、07、08、19、43、45题）

**核心知识：三种遍历的不变量**

| 性质 | 适用范围 | 典型题目 |
|---|---|---|
| 三种遍历中，**叶结点的相对顺序完全相同** | 所有二叉树 | 第06题(B) |
| 三种遍历中，**左子树的结点一定排在右子树的结点前面** | 所有二叉树 | 第02题(C) |
| 先序第一个=根，后序最后一个=根 | 所有非空二叉树 | 基础常识 |
| 先序中祖先一定在子孙**前面**，后序中祖先一定在子孙**后面** | 所有二叉树 | 第19题(B) |
| 中序中，祖先和子孙的前后关系**不确定**（取决于在左/右子树） | 所有二叉树 | 第19题(B) |

**第01题精析（C）——"中序最后一个结点"的陷阱：**

中序遍历的最后一个结点p，一定是从根沿右子女指针走到底的结点。分两种情况：
- p是叶结点 → p同时也是前序遍历该子树的最后一个（叶结点在前序和中序中的末尾位置一致）→ **C正确**
- p不是叶结点（有左子树）→ 它有左子女q，q是叶结点，q才是前序最后一个，不是p → D错误

⚠️ **易错点：** A选项"中序最后一个结点是前序最后一个"看似对称，但不成立。反例：根只有右孩子时，中序最后是右孩子，前序最后也是右孩子，看似成立；但根有左右子树时就不一定了。**必须区分"某子树的中序最后"和"整棵树的前序最后"。**

**第19题精析（2015真题前置知识）——祖先子孙在三种遍历中的位置规律：**

a是b的祖先时：

| 遍历 | a和b的位置关系 | 原因 |
|---|---|---|
| 先序(NLR) | pre(a) < pre(b) **一定成立** | 祖先先于子孙访问 |
| 后序(LRN) | post(a) > post(b) **一定成立** | 祖先后于子孙访问 |
| 中序(LNR) | **不确定** | b在a左子树→in(b)<in(a)；b在a右子树→in(b)>in(a) |

→ 所以 post(a) < post(b) **不可能出现**，选B。

⚠️ **推论（超级有用）：** 如果结点x在先序中在y**前面**，且在后序中在y**后面**，那么x一定是y的**祖先**。这个推论反复出现在第11、37等题中。

**第43题精析（2022统考真题）——中序相邻结点的关系分析：**

p和q在中序中相邻，且p在q前。分析p和q的四种可能关系：

| 关系 | 是否可能 | 构造反例/证明 |
|---|---|---|
| I. q是p的双亲 | ✅ 可能 | p是q的左孩子，p无右子树 |
| II. q是p的右孩子 | ✅ 可能 | q无左子树，q紧跟p |
| III. q是p的右兄弟 | ❌ 不可能 | p和q之间必须有它们的公共双亲F，F在中序中夹在p和q中间 |
| IV. q是p的双亲的双亲 | ✅ 可能 | p是某结点的右子树最右结点，q是该结点的双亲的双亲 |

→ III不可能，答案B。

⚠️ **核心理解：** 中序是"左根右"，如果q是p的右兄弟，那么它们的双亲F在中序中一定在p和q之间（先遍历p→再访问F→再遍历q所在子树），所以p和q**不可能相邻**。

**第45题精析（2024统考真题）——中序序列中v的左右邻居：**

中序为 `...p, v, q,...`，v有两个孩子：
- p紧挨v左边 → p是v**左子树中序遍历的最后一个结点** → p是v左子树的**最右结点** → **p没有右孩子**
- q紧挨v右边 → q是v**右子树中序遍历的第一个结点** → q是v右子树的**最左结点** → **q没有左孩子**

→ 答案A：p没有右孩子，q没有左孩子。

### 【题型2】遍历序列之间的位置关系推断（第03、04、11、12题）

**核心方法：画树分析法——把结点放到树上，看遍历顺序**

**第03题（C）——中序中n在m前的条件：**

中序是LNR。n在m前的三种可能（画图！）：
```
(a) m是根，n在左子树     (b) n是根，m在右子树     (c) x是祖先，n在左子树，m在右子树
    (m)                      (n)                      (x)
   /                          \                      /   \
  (n)                         (m)                  (n)   (m)
```
三种情况的共同点：**n在m的左方**（n在某个公共祖先的左子树中，m在该祖先或右子树中）。

→ 答案C：n在m左方。

**第04题（D）——后序中n在m前的充分条件：**

后序是LRN。n在m前的充分条件是 **n是m的子孙**（不是充要条件！n也可能在m的左兄弟子树中）。

⚠️ **第03题和第04题对比（极易混淆）：**

| 题目 | 问的是 | 答案 | 关键区别 |
|---|---|---|---|
| 第03题 | 中序中n在m前的**条件** | n在m左方 | 中序的位置由左右关系决定 |
| 第04题 | 后序中n在m前的**充分条件** | n是m子孙 | 后序的位置由祖先/子孙+左右共同决定 |

**第11题（C）——先序X在Y前 + 后序X在Y后：**

由第19题的推论：先序中X在Y前，后序中X在Y后 → **X是Y的祖先**。

→ 答案C：X是Y的祖先。

**第12题（C）——先序...a...b... + 中序...b...a...：**

先序中a在b前：a可能是b的祖先，或a在b的"左边"
中序中b在a前：b在a的左子树中，或b和a有共同祖先且b在左侧

综合两个条件，画出所有可能的情况（见答案解析的6张图），**共同点是b在a的左子树中**。

→ 答案C：b在a的左子树中。

### 【题型3】由遍历序列还原/确定二叉树（第05、09、10、13、16、17、18、37、41、44题）

**这是出题频率最高的题型，几乎年年考！**

**解题模板：**

```
Step 1: 从先序/后序/层序中确定根
        先序 → 第一个是根
        后序 → 最后一个是根
        层序 → 第一个是根

Step 2: 在中序中找到根，将中序分为左子树和右子树

Step 3: 根据左右子树的结点数，在先序/后序/层序中划分对应部分

Step 4: 递归处理左右子树
```

**各组合的解题要点：**

| 组合 | 根的确定 | 左右划分 | 典型题目 |
|---|---|---|---|
| **先序+中序** | 先序第一个=根 | 中序中根的位置划分左右 | 第17题 |
| **后序+中序** | 后序最后一个=根 | 中序中根的位置划分左右 | 第16题 |
| **层序+中序** | 层序第一个=根 | 中序划分后，在层序中找左右子树的根 | 第18题 |
| **后序+树形** | 后序最后一个=根 | 树形已知，直接填入 | 第41、44题 |

**第05题精析（C）——顺序存储的完全二叉树求后序：**

关键：顺序存储中下标从0开始，结点i的左孩子=2i+1，右孩子=2i+2。

```
数组: a(0) b(1) c(2) _(3) d(4) e(5) f(6) _(7) _(8) g(9) _(10) h(11)

画出树:
          a(0)
         /    \
       b(1)    c(2)
      /    \      \
    _(3)   d(4)   f(6)    ← 注意3号位置为空！
           / \     /
         _(7) _(8) h(11)  ← 注意e在5号=b的右孩子的... 
           g(9)
```

⚠️ **注意：下标从0开始时，父子关系公式不同！** 这里的数组下标从0开始，左孩子=2i+1，右孩子=2i+2。仔细画出树后做后序遍历。

**第09题精析（D）——前序A,B,C + 后序C,B,A → 4棵不同二叉树：**

前序ABC（A是根，B在C前），后序CBA（A是根，B在C后）。
没有中序无法区分左右，所以有多种可能：
```
(1) A          (2) A          (3) A          (4) A
   /               /               \              \
  B               B                 B              B
 /                 \               /                \
C                   C             C                  C
```
第1种：B是A左孩子，C是B左孩子
第2种：B是A左孩子，C是B右孩子
第3种：B是A右孩子，C是B左孩子
第4种：B是A右孩子，C是B右孩子

→ 4棵，但只有当每个中间结点只有一个孩子时才出现歧义。第5种A有左右孩子B和C的情况：先序ABC✓，后序应该是BCA或CBA，只有当C是B的子孙时后序才是CBA，所以A有两个孩子的情况不满足后序CBA。

→ 答案D：4棵。

**第10题精析（C）——完全二叉树后序→先序：**

7个结点的完全二叉树形状唯一（3层满二叉树），后序为CDBFGEA → 填入固定树形：
```
         A(根=后序最后)
        / \
       B    E
      / \  / \
     C  D F  G
```
先序遍历：ABCDEFG → 答案C。

⚠️ **技巧：完全二叉树的形状由结点数唯一确定！** 知道形状后，后序/先序/层序任何一个都能直接还原。

**第37题精析（2012统考真题）——前序+后序确定祖先关系：**

前序 `a,e,b,d,c`，后序 `b,c,d,e,a`

由"先序中X在Y前 + 后序中X在Y后 → X是Y祖先"的规律：
- a在先序最前、后序最后 → a是根 ✓
- e在先序中在b,d,c前面；e在后序中在b,c,d后面 → **e是b,d,c的祖先**
- 所以根a的孩子**只有e**

→ 答案A：只有e。

⚠️ **重要推论：** 前序+后序虽然不能唯一确定二叉树，但能确定**祖先-子孙关系**！具体地，前序为XY且后序为YX时，X是Y的祖先。

### 【题型4】遍历序列的唯一确定性（第14题）

**必背结论：**

| 组合 | 能否唯一确定 | 原因 |
|---|---|---|
| 先序 + 中序 | ✅ | 先序定根，中序分左右 |
| 后序 + 中序 | ✅ | 后序定根，中序分左右 |
| 层序 + 中序 | ✅ | 层序定根，中序分左右 |
| **先序 + 后序** | ❌ | 都能定根，但**无法划分左右子树** |
| 先序 + 层序 | ❌ | 无法划分左右 |
| 后序 + 层序 | ❌ | 无法划分左右 |

**本质：必须有中序才能确定！** 因为只有中序能用根的位置把结点划分为左子树和右子树。

⚠️ **例外：** 如果已知树是**满二叉树**，则先序+后序可以唯一确定（因为满二叉树每个非叶结点都有两个孩子，不存在只有一个孩子的歧义）。这就是综合题第12题的考点。

### 【题型5】遍历序列相同/相反的特殊树形（第15、32、33、42题）——统考高频！

**这类题每年都可能以不同形式出现，必须烂熟于心。**

| 条件 | 树的形态 | 原因 | 题目 |
|---|---|---|---|
| **先序 = 后序的逆序** | 每层只有一个结点（高度=结点数） | NLR与LRN互逆 → L和R都为空 → 每个结点最多一个孩子，且树退化为链 | 第32题(B) |
| **先序 = 中序** | 任意结点**只有右子树**（无左孩子）→ 右单支 | NLR=LNR → L为空 | 第42题(B) |
| **中序 = 后序** | 任意结点**只有左子树**（无右孩子）→ 左单支 | LNR=LRN → R为空 | 第15题(B) |
| **先序 = 中序的逆序** | 任意结点**无右孩子** → 左倾单支 | NLR与RNL相同 → R为空 | 第33题(D) |
| **先序 = 后序** | 只有根结点（或空树） | NLR=LRN → L和R都空，只剩N | - |
| **中序 = 层序** | 任意结点**无左孩子** → 右单支 | 类似分析 | - |

**推导方法（以"中序=后序"为例）：**
```
中序: L N R
后序: L R N
要使 LNR = LRN
→ 去掉相同的L，需要 NR = RN
→ 这只有在R为空时成立（否则N和R的相对顺序不同）
→ 任意结点没有右子树
→ 加上空树的边界情况：空树或任一结点没有右子树
```

⚠️ **第33题易错点：** "先序和中序正好相反"问"下列正确的是"：
- A. 只有一个结点 → 太强了，不一定
- B. 只有一个叶结点的一定满足 → 不对，还要看方向
- C. 任意一个结点无左孩子的一定满足 → 这是先序=中序的条件，不是先序=中序的逆序
- **D. 任意一个结点无右孩子的一定满足** ✓

### 【题型6】遍历方式的选择/应用（第20、21、34题）

**核心：根据任务需求选遍历方式**

| 任务 | 最适合的遍历 | 原因 | 题目 |
|---|---|---|---|
| **删除结点并释放空间** | 后序遍历 | 先删子树再删根，不会丢失子树指针 | 第20题(C) |
| **交换左右子树使中序变降序** | 先序遍历（先处理根再递归）或后序遍历 | 都能实现；中序不行（交换后左右颠倒导致重复/遗漏） | 第21题(B) |
| **编号满足"根>左右孩子，左<右"** | 后序遍历 | 后序先访问左子树再右子树再根，编号递增 → 根编号最大，左<右 | 第07题(C) |
| **编号满足BST规则（左<根<右）** | 先序遍历 | 先序中编号越大越靠后 → 先左子树再右子树再根 | 第08题(B) |
| **求树的高度（非递归）** | 层序遍历 | 用last指针标记每层结尾 | 综合题03 |
| **判断是否完全二叉树** | 层序遍历 | 遇到空结点后不应再有非空结点 | 综合题04 |
| **打印所有祖先** | 后序非递归 | 栈中元素=当前结点的所有祖先 | 综合题09 |
| **求最近公共祖先** | 后序非递归 | 两个栈比对找第一个相同元素 | 综合题10 |
| **求树的宽度** | 层序遍历 | 记录每个结点所在层次 | 综合题11 |

**第21题精析（B）——交换左右子树不能用中序遍历：**

```
中序遍历顺序: 左 → 根 → 右
如果在"根"时交换左右子树:
交换后原来的右子树变成了左子树，但中序接下来要遍历"右子树"
→ 实际遍历的是已经交换过来的原左子树 → 原左子树被遍历了两次！原右子树一次都没遍历！
```

所以答案选B（后序遍历）：先递归交换左子树 → 再递归交换右子树 → 最后交换当前结点的左右孩子。先序也可以。

**第34题精析（2009统考真题）——非标准遍历方式RNL：**

题目给了一棵树，遍历后的结点序列是3175624。分析发现根结点1在中间被访问，且**右子树结点在左子树之前**。这不是标准的三种遍历，而是**RNL**（先右子树→再根→再左子树）。

⚠️ **考生陷阱：** 看到遍历序列不要死套NLR/LNR/LRN，要灵活分析根的位置和左右子树的访问顺序。408也会考非标准遍历。

### 【题型7】线索二叉树基础概念（第22、23、24、25、26题）

**必背知识点汇总：**

| 知识点 | 内容 | 题目 |
|---|---|---|
| 引入线索的目的 | **加快查找前驱和后继的速度** | 第22题(A) |
| 线索二叉树是什么结构 | **物理结构**（是二叉树在计算机内部的一种存储表示） | 第23题(C) |
| n个结点有多少线索 | **n+1个**（空指针数=n+1，每个空指针变成一条线索） | 第24题(C) |
| 判断有右孩子的条件 | **p->rtag==0**（tag=0表示指针指向孩子，tag=1表示线索） | 第25题(C) |
| 线索化后的空链域 | 取决于具体情况，需要具体分析 | 第26题(D) |

**第24题推导过程：**
```
n个结点 → 总指针数 = 2n
非空指针 = n-1 (每条边对应一个非空指针)
空指针 = 2n - (n-1) = n+1
每个空指针变成一条线索
→ 线索数 = n+1
```

**第26题精析（D）——左子树为空的二叉树先序线索化后的空链域：**

"左子树为空"意味着根没有左子树。先序线索化后：
- 根的左指针为空（根是先序第一个，没有前驱）→ ltag=1, lchild=NULL → **1个空链域**
- 先序遍历的最后一个结点（最右叶子）的右指针为空（没有后继）→ rtag=1, rchild=NULL → **1个空链域**

→ 共**2个**空链域，答案D。

⚠️ **规律：任何线索二叉树，线索化后恰好有2个空链域**——先序/中序/后序第一个结点的前驱为空，最后一个结点的后继为空。（如果加了头结点则可以消除这2个空链域。）

### 【题型8】线索二叉树的能力与局限（第27、28、29、30、31题）——核心难点！

**三种线索二叉树能力对比（扩展版，考试必背）：**

| 能力 | 先序线索 | 中序线索 | 后序线索 |
|---|---|---|---|
| 找**前驱** | ❌ 需要双亲指针 | ✅ 左子树最右/直接线索 | ✅ 右孩子或右兄弟/直接线索 |
| 找**后继** | ✅ 左孩子或直接线索 | ✅ 右子树最左/直接线索 | ❌ 需要双亲指针 |
| 不用栈遍历 | ✅（顺着后继走） | ✅（顺着后继走） | ❌（找后继需要栈） |
| 逆向遍历（不用栈） | ❌（找前驱需要栈） | ✅（顺着前驱走） | ✅（顺着前驱走） |

**为什么先序线索找不了前驱？（必须理解原理）**

先序是NLR。考虑结点p：
- 如果p是某结点的**左孩子**：p的先序前驱是它的**双亲**（因为NLR中，访问完根N就进入左子树L）
- 如果p是某结点的**右孩子**且双亲有左子树：p的先序前驱是双亲左子树中**最后被先序访问的结点**
- 两种情况都需要知道p的**双亲**，但二叉链表中**没有双亲指针**！

**为什么后序线索找不了后继？（对称理解）**

后序是LRN。考虑结点p：
- 如果p是某结点的**右孩子**（或唯一孩子）：p的后序后继是它的**双亲**
- 如果p是某结点的**左孩子**且双亲有右子树：p的后序后继是双亲右子树中**最先被后序访问的结点**
- 同样需要双亲指针，但二叉链表没有。

**第27题精析（D）——"每个结点都能直接找到前驱和后继"是错的：**

即使在中序线索树中，如果结点有左/右孩子（tag=0），找前驱/后继也需要沿着子树走，不是"直接"通过线索找到。更关键的是，先序线索找不了前驱，后序线索找不了后继。

→ D选项"每个结点都能通过线索直接找到前驱和后继"不正确。

**第29题精析（C）——中序线索树中有左孩子的结点，其前驱是什么？**

在中序线索树中，如果结点X有左孩子（ltag=0），那么X的中序前驱不在线索中，需要自己找：
- 中序前驱 = X的**左子树中最右的结点**（沿着lchild进入左子树，然后一路沿rchild走到底）

⚠️ 注意：是左子树中最**右**的结点，**不一定是最右的叶结点**（可能是有左孩子但没右孩子的分支结点）。

**第30题精析（A）——后序线索中叶结点X有左兄弟Y：**

X是叶结点（两个指针都是线索），X有左兄弟Y。在后序遍历中：
- 后序是LRN，X的双亲的后序后继：先遍历左子树(Y为根)→再遍历右子树(X)→再访问双亲
- **X的后序后继就是X的双亲**
- X的右线索（rtag=1）指向后序后继 = **X的双亲**

→ 答案A：X的双亲。

**第31题精析（C）——哪种线索树遍历仍需栈？**

后序线索树！因为后序线索找不到后继（需要双亲指针），所以用后序线索树做后序遍历时仍然需要栈来辅助。

### 【题型10】卡特兰数（第40题）

**第40题（2015统考真题）：先序序列为a,b,c,d的不同二叉树个数**

公式：**C(2n, n) / (n+1)** = 卡特兰数

n=4时：C(8,4)/5 = 70/5 = **14**

⚠️ **跨章关联：** 这和第3章"n个不同元素入栈，有多少种不同的出栈序列"是**同一个公式**！因为先序遍历的递归过程等价于一个栈的入栈出栈过程：先序序列=入栈顺序，中序序列=出栈顺序。

**第13题解法2就用了这个关联：** 先序1234567看作入栈序列，中序看作出栈序列。判断某个出栈序列是否合法等价于判断该中序序列是否可能。

### 高频算法代码（必背3个）

**1. 层序遍历求高度（综合题03）：**
```c
int Btdepth(BiTree T) {
    if (!T) return 0;
    int front = -1, rear = -1;
    int last = 0, level = 0;
    BiTree Q[MaxSize], p;
    Q[++rear] = T;
    while (front < rear) {
        p = Q[++front];
        if (p->lchild) Q[++rear] = p->lchild;
        if (p->rchild) Q[++rear] = p->rchild;
        if (front == last) {    // 处理到该层最右结点
            level++;
            last = rear;        // last指向下一层最右
        }
    }
    return level;
}
```

**2. 判断完全二叉树（综合题04）：**
```c
bool IsComplete(BiTree T) {
    InitQueue(Q);
    if (!T) return true;
    EnQueue(Q, T);
    while (!IsEmpty(Q)) {
        DeQueue(Q, p);
        if (p) {
            EnQueue(Q, p->lchild);  // 空结点也入队！
            EnQueue(Q, p->rchild);
        } else {
            // 遇到空结点，检查后面是否还有非空
            while (!IsEmpty(Q)) {
                DeQueue(Q, p);
                if (p) return false;  // 空后有非空→不完全
            }
        }
    }
    return true;
}
```

**3. 后序非递归打印祖先（综合题09）——利用"后序栈=祖先路径"的性质：**
```c
// 后序非递归遍历，找到x时栈中所有元素就是x的祖先
void Search(BiTree bt, ElemType x) {
    stack s[];
    top = 0;
    while (bt != NULL || top > 0) {
        while (bt != NULL && bt->data != x) {
            s[++top].t = bt;
            s[top].tag = 0;
            bt = bt->lchild;
        }
        if (bt != NULL && bt->data == x) {
            // 找到x，栈中元素即为祖先
            for (i = 1; i <= top; i++)
                printf("%d", s[i].t->data);
            exit(1);
        }
        while (top != 0 && s[top].tag == 1)
            top--;
        if (top != 0) {
            s[top].tag = 1;
            bt = s[top].t->rchild;
        }
    }
}
```

## 三、12条易错清单

| 编号 | 易错点 | 正确理解 | 相关题目 |
|---|---|---|---|
| 1 | "中序+先序能确定"就以为"先序+后序也能" | 先序+后序**不能**唯一确定（无法分左右），但能确定祖先关系 | 14, 37 |
| 2 | 线索二叉树的每个结点都能直接找到前驱后继 | **不能！**先序找不了前驱，后序找不了后继；有孩子的结点也需要沿子树找 | 27, 28 |
| 3 | 线索数 = n-1 或 2n | 线索数 = **n+1**（空指针数） | 24 |
| 4 | rtag=0 表示线索 | **rtag=0 表示有右孩子**，rtag=1 才是线索 | 25 |
| 5 | 中序=后序 → 只有根结点 | 中序=后序 → **任意结点无右子树**（可以有多个结点） | 15 |
| 6 | 先序=后序逆序 → 任意结点度≤1 | 应该是**每层只有一个结点**（高度=结点数），但每个结点可以有左孩子或右孩子 | 32 |
| 7 | 后序遍历适合删除，所以所有"先处理再释放"都用后序 | 交换左右子树用**先序或后序都行**，但**不能用中序**（会导致重复交换） | 21 |
| 8 | 完全二叉树的顺序存储下标总是从1开始 | 第05题下标**从0开始**！此时左孩子=2i+1，右孩子=2i+2 | 05 |
| 9 | 中序相邻的两个结点一定是父子或兄弟 | 中序相邻可以是父子、祖孙等关系，**但不可能是兄弟**（双亲会夹在中间） | 43 |
| 10 | 先序中a在b前就说明a是b的祖先 | 还需要**后序中a在b后面**才能确定祖先关系；仅先序在前不够（a可能只是在b左边） | 11, 19 |
| 11 | 后序线索树的遍历不需要栈 | **后序线索树遍历仍需栈！**（找后继需要双亲指针） | 31 |
| 12 | 层序+中序不能还原二叉树 | **可以还原！**层序第一个=根，中序分左右，层序中按序找左右子树的根 | 18 |
