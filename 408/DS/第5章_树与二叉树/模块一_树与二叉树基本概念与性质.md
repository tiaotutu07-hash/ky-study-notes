## 一、树的定义——递归是灵魂

树是 n (n≥0) 个结点的有限集。n=0 时为空树。任意一棵非空树满足：有且仅有一个根结点；其余结点可分为 m 个互不相交的有限集 T₁, T₂, ..., Tₘ，每个集合本身又是一棵树（根的子树）。

**跨章关联：** 你在第2章学的链表是**线性**的一对一关系，树是**一对多**的层次关系。但树的定义本身是递归的——这和你在第4章KMP里见到的递归思想、以及后面遍历用到的递归调用栈是一脉相承的。

树的两个核心特点：
- 根结点没有前驱，其余结点有且仅有一个前驱（双亲）
- 所有结点可以有零个或多个后继（孩子）
- n个结点的树有 **n-1** 条边（每个非根结点贡献一条到双亲的边）

## 二、基本术语（结合图来记）

```
            A          ← 第1层 (根, 深度1)
          / | \
        B   C   D      ← 第2层
       / \   \  /|\
      E   F   G H I J  ← 第3层
     / \        |
    K   L       M      ← 第4层
```

**亲缘关系类：**
- **双亲/孩子**：A是B的双亲，B是A的孩子
- **兄弟**：B、C、D互为兄弟（同一双亲）
- **堂兄弟**：E和G、H互为堂兄弟（双亲在同一层但不同）
- **祖先/子孙**：A到K路径上的所有结点都是K的祖先

**度量类：**
- **结点的度**：该结点的孩子个数。A的度=3，B的度=2，K的度=0
- **树的度**：所有结点度的最大值。上图树的度=3（A或D）
- **叶子结点**：度为0的结点（K、L、F、G、J、M）
- **分支结点**：度>0的结点

**层次类：**
- **层次**：根为第1层，根的孩子为第2层……（⚠️ 有的教材从0开始，408统考默认**从1开始**）
- **结点的深度**：从**根**往下数，根的深度=1
- **结点的高度**：从**叶子**往上数，最底层叶子高度=1
- **树的高度（深度）**：结点的最大层次。上图高度=4

**路径类：**
- **路径**：从结点到结点经过的结点序列
- **路径长度**：路径上经过的**边**的个数
- ⚠️ 树中的路径是**从上向下**的（双亲→孩子），同一双亲的兄弟之间**不存在路径**

**其他：**
- **有序树/无序树**：子树有无左右次序之分
- **森林**：m (m≥0) 棵互不相交的树的集合。把根删了，子树就构成森林；给森林加一个根，就变成树。

## 三、树的性质（必背公式）

### 性质1：结点数 = 总度数 + 1

即 `n = 所有结点的度之和 + 1`。理解：每条边对应一个孩子结点的度贡献，n-1条边 = 总度数。

**经典套路题：** 在一棵度为4的树中，有20个度为4的结点，10个度为3的结点，1个度为2的结点，10个度为1的结点，求叶结点个数。

```
n = n₀ + n₁ + n₂ + n₃ + n₄
总度数 = 0·n₀ + 1·n₁ + 2·n₂ + 3·n₃ + 4·n₄
       = 0 + 10 + 2 + 30 + 80 = 122
n = 122 + 1 = 123
n₀ = 123 - 10 - 1 - 10 - 20 = 82
```

### 性质2：度为m的树第i层至多有 m^(i-1) 个结点 (i≥1)

第1层1个（根），第2层至多m个，第3层至多m²个……

### 性质3：高度为h的m叉树至多有 (m^h - 1)/(m - 1) 个结点

就是等比数列求和：1 + m + m² + ... + m^(h-1)

### 性质4：具有n个结点的m叉树最小高度为 ⌈log_m(n(m-1)+1)⌉

每层都满时高度最小。

⚠️ **PPT补充的重要区分——"度为m的树" vs "m叉树"：**

| | 度为m的树 | m叉树 |
|---|---|---|
| **定义** | 树中至少有一个结点的度=m | 每个结点的度≤m，可以没有度=m的 |
| **可否为空** | 至少3个结点(m≥2) | 可以是空树 |
| **最小结点数** | m+1（根+m个孩子） | 0（空树） |

## 四、二叉树的定义

二叉树是 n (n≥0) 个结点的有限集，它或者是空集，或者由一个根结点和两棵互不相交的、分别称为左子树和右子树的二叉树组成。

关键：**每个结点至多有2个孩子，且严格区分左右**。

⚠️ **PPT补充的重要区分——"二叉树" vs "度为2的有序树"：**

| | 二叉树 | 度为2的有序树 |
|---|---|---|
| 可否为空 | 可以为空 | 至少3个结点 |
| 只有一个孩子时 | 必须区分是左还是右 | 左右是相对的，无需区分 |

二叉树的5种基本形态：空树、只有根、只有左子树、只有右子树、左右子树都有。

## 五、二叉树的性质（5条核心，必须滚瓜烂熟）

**性质1：** 非空二叉树第k层至多有 **2^(k-1)** 个结点 (k≥1)

**性质2：** 高度为h的二叉树至多有 **2^h - 1** 个结点 (h≥1)

**性质3：** n个结点的二叉树高度至少为 **⌈log₂(n+1)⌉**

**性质4（最重要！）：** 对任意二叉树，若叶结点数为n₀，度为2的结点数为n₂，则 **n₀ = n₂ + 1**

```
证明：
  设度为1的结点数为n₁，总结点数 n = n₀ + n₁ + n₂
  总边数 = n - 1（树的基本性质）
  从度的角度看总边数 = 0·n₀ + 1·n₁ + 2·n₂ = n₁ + 2n₂
  所以 n₀ + n₁ + n₂ - 1 = n₁ + 2n₂
  化简得 n₀ = n₂ + 1   ✓
```

**性质5（完全二叉树编号性质）：** 对完全二叉树按从上到下、从左到右编号1, 2, ..., n，对任意结点i：
- 双亲编号：**⌊i/2⌋**（i≠1时）
- 左孩子编号：**2i**（2i≤n时存在）
- 右孩子编号：**2i+1**（2i+1≤n时存在）
- 判叶子：**i > ⌊n/2⌋** 则为叶子

⚠️ 这个性质要求编号**从1开始**！如果从0开始，公式变为：双亲 ⌊(i-1)/2⌋，左孩子 2i+1，右孩子 2i+2。

## 六、几种特殊二叉树

### 满二叉树

```
         1
       /   \
      2     3
     / \   / \
    4   5 6   7
```

高度为h，结点数恰好为 2^h - 1。每层都是满的。每个结点要么度为2要么度为0（叶子），所有叶子都在最后一层。

### 完全二叉树

```
         1
       /   \
      2     3
     / \   /
    4   5 6
```

按层序编号后，编号与同高度满二叉树一一对应（最后一层可以不满，但必须**从左到右连续**）。

**完全二叉树的6条重要性质：**
1. 编号与满二叉树一一对应
2. i > ⌊n/2⌋ 则为叶子
3. 叶子只出现在最后一层和倒数第二层
4. **至多有1个度为1的结点，且该结点只有左孩子**
5. n为奇数→n₁=0（无度为1的结点）；n为偶数→n₁=1
6. 高度 h = ⌈log₂(n+1)⌉ = ⌊log₂n⌋ + 1

⚠️ **性质5是选择题陷阱重灾区！** 记住：n为奇数则所有非叶结点度都是2；n为偶数则恰有一个度为1的结点。

### 二叉排序树（BST）、平衡二叉树（AVL）

这两个是**应用**层面的特殊二叉树，后面5.5模块会详细讲，这里先有个定义印象：
- **BST**：左子树所有结点值 < 根 < 右子树所有结点值，中序遍历是递增序列
- **AVL**：在BST基础上，任意结点左右子树高度差绝对值≤1

## 七、二叉树的存储结构

### 1. 顺序存储（数组）

```
编号:  1   2   3   4   5   6   7
数组: [1] [2] [3] [4] [5] [6] [7]
```

用结点编号作为数组下标，利用性质5的父子编号关系实现**随机访问**。

⚠️ 数组下标**必须从1开始**（下标0空着不用），否则 2i/2i+1 的关系不成立。

**适用场景：** 完全二叉树——空间利用率高。
**不适用：** 稀疏二叉树（比如右单支树），空间浪费严重，需要补很多空结点。

**跨章关联：** 顺序存储就是第2章学的顺序表思想。而这种用数组存完全二叉树的方法，正是后面**堆排序**（第8章）的基础！

### 2. 链式存储（二叉链表）

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;
```

每个结点有 data + 左指针 + 右指针。

```
         [A]
        /   \
     [B]     [C]
     / \       \
   [D] [E]    [F]
   
链表视角：
  A: data=A, lchild→B, rchild→C
  B: data=B, lchild→D, rchild→E
  C: data=C, lchild=NULL, rchild→F
  D: data=D, lchild=NULL, rchild=NULL
  ...
```

⚠️ **重要结论：n个结点的二叉链表有 n+1 个空指针。**

```
证明：总指针数 = 2n（每个结点2个指针）
      非空指针 = n-1（每条边对应一个非空指针，共n-1条边）
      空指针 = 2n - (n-1) = n+1
```

**跨章关联：** 这 n+1 个空指针是"浪费"的——后面的**线索二叉树**正是利用这些空指针来存储前驱/后继信息，化腐朽为神奇！

## 【模块一总结】

| 概念 | 核心要点 |
|---|---|
| 树的定义 | 递归定义，n个结点n-1条边 |
| n₀ = n₂ + 1 | 任意二叉树都成立，不只是完全二叉树 |
| 完全二叉树编号 | 从1开始！双亲⌊i/2⌋, 左孩子2i, 右孩子2i+1 |
| 完全二叉树n₁ | n奇→n₁=0, n偶→n₁=1 |
| 顺序存储 | 适合完全二叉树，堆排序基础 |
| 链式存储空指针 | n+1个，线索二叉树的切入点 |
| m叉树 vs 度为m的树 | 前者可以为空，后者至少有度=m的结点 |
| 二叉树 vs 度为2有序树 | 二叉树可空+严格区分左右 |
