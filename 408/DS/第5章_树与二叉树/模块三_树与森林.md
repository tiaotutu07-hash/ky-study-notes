# 模块三：5.4 树与森林

## 一、树的存储结构（三种表示法）

树不像二叉树那样每个结点最多两个孩子，树的度不固定，所以存储方式更灵活。

### 1. 双亲表示法（顺序存储）

每个结点存 `data` + `parent`（双亲在数组中的下标），根结点的parent = -1。

```
树:          R
           / | \
          A   B   C
         / \      |
        D   E     F
                / | \
               G   H   K

数组表示:
下标  data  parent
 0     R     -1
 1     A      0
 2     B      0
 3     C      0
 4     D      1
 5     E      1
 6     F      3
 7     G      6
 8     H      6
 9     K      6
```

```c
#define MAX_TREE_SIZE 100
typedef struct {
    ElemType data;
    int parent;          // 双亲位置域
} PTNode;
typedef struct {
    PTNode nodes[MAX_TREE_SIZE];
    int n;               // 结点数
} PTree;
```

**优点：** 找双亲 O(1)——直接读parent字段

**缺点：** 找孩子 O(n)——必须遍历整个数组找parent等于自己下标的

**跨章关联：** 这种"每个元素存储其父节点下标"的思想，就是后面**并查集**的核心数据结构！并查集的 `f[i]` 数组本质就是双亲表示法。

### 2. 孩子表示法（顺序+链式）

每个结点有一个**孩子链表**，把所有孩子串成一条链。

```
下标  data  → 孩子链表
 0    R    → [1] → [2] → [3] → ^
 1    A    → [4] → [5] → ^
 2    B    → ^
 3    C    → [6] → ^
 4    D    → ^
 5    E    → ^
 6    F    → [7] → [8] → [9] → ^
 7    G    → ^
 8    H    → ^
 9    K    → ^
```

**优点：** 找孩子方便——直接遍历孩子链表

**缺点：** 找双亲 O(n)——需要遍历所有链表

**跨章关联：** 这不就是第2章学的"顺序表 + 链表"的混合结构吗？和图的**邻接表**存储（第6章）也是同构的。

### 3. 孩子兄弟表示法（链式，最重要！）

每个结点两个指针：`firstchild`（第一个孩子）+ `nextsibling`（右兄弟）。

```c
typedef struct CSNode {
    ElemType data;
    struct CSNode *firstchild, *nextsibling;
} CSNode, *CSTree;
```

```
树:       R                    孩子兄弟表示(二叉链表):
        / | \
       A   B   C                    R
      / \      |                   /
     D   E     F                  A → B → C
                / | \            /         |
               G  H  K         D → E      F
                               /
                              G → H → K
```

⚠️ **关键洞察：孩子兄弟表示法的存储结构和二叉链表完全相同！** 左指针=第一个孩子，右指针=右兄弟。这就是**树与二叉树转换的物理基础**。

**优点：** 方便实现树↔二叉树的转换，找孩子方便

**缺点：** 找双亲不方便（可以加parent指针解决）

### 三种存储结构对比

| | 双亲表示法 | 孩子表示法 | 孩子兄弟表示法 |
|---|---|---|---|
| 找双亲 | ✅ O(1) | ❌ O(n) | ❌ 需遍历 |
| 找孩子 | ❌ O(n) | ✅ O(度) | ✅ O(度) |
| 实现树↔二叉树 | ❌ | ❌ | ✅ 天然支持 |
| 应用场景 | 并查集 | 图的邻接表思路 | 树/森林与二叉树转换 |

## 二、树、森林与二叉树的转换（核心考点）

### 树 → 二叉树（三步口诀：**连兄弟、删多余、旋转**）

```
原树:          A                Step1: 兄弟间加线
             / | \             B-C-D间加横线
            B   C   D
           / \     / | \       Step2: 每个结点只保留与第一个孩子的连线
          E   F   G  H  I     删除A→C, A→D, D→H, D→I的线
                               (保留A→B, B→E, D→G)

                               Step3: 以根为轴顺时针旋转45°
```

结果：

```
转换后的二叉树:
         A
        /
       B
      / \
     E   C
      \   \
       F   D
          /
         G
          \
           H
            \
             I
```

⚠️ **规律：树转二叉树后，根结点一定没有右子树！** 因为根没有兄弟。

**本质理解：** 左指针 = firstchild（第一个孩子），右指针 = nextsibling（右兄弟）。所以：
- 在二叉树中，**左孩子** = 原树中的**第一个孩子**
- 在二叉树中，**右孩子** = 原树中的**右兄弟**

### 二叉树 → 树（逆过程，前提：根无右子树）

```
Step1: 把所有右指针"旋转"为横线（恢复兄弟关系）
Step2: 删除横边，把所有孩子连到各自双亲上
```

### 森林 → 二叉树

```
森林:  T1     T2     T3
       A       E      G
      / \     / \
     B   C   F   H
     |       |
     D       I
```

**方法：**
1. 先把每棵树各自转成二叉树
2. 各棵树的根可视为兄弟（它们在森林中并列），在各二叉树的根之间加右指针
3. 第一棵树的根作为整棵二叉树的根

```
T1的二叉树:    T2的二叉树:    T3的二叉树:
    A              E              G
   /              /
  B → C          F → H
  |              |
  D              I

连接（各根成兄弟）:
         A
        / \
       B   E
        \  / \
        C F   G
        |  \
        D   H → (无)
            |
            I
    
最终:
         A
        / \
       B   E
        \  / \
        C F   G
          |
          I → H    (H是F的右兄弟)
... (具体看王道图5.23)
```

⚠️ **规律：如果二叉树的根有右子树，则对应森林；根无右子树，则对应单棵树。**

### 二叉树 → 森林

```
Step1: 递归断开根的右子树（每断一次就分出一棵树的二叉树形式）
Step2: 把每棵二叉树各自还原成树
```

## 三、树和森林的遍历（与二叉树遍历的对应关系——必考！）

### 树的遍历（两种）

**先根遍历：** 先访问根→再按先根遍历依次访问每棵子树

**后根遍历：** 先按后根遍历依次访问每棵子树→最后访问根

```
树:          A
           / | \
          B   C   D
         / \     / | \
        E   F   G  H  I

先根序列: A B E F C D G H I   (先访问根A,再先根遍历B子树,C子树,D子树)
后根序列: E F B C G H I D A   (先后根遍历B子树,C子树,D子树,再访问A)
```

⚠️ 树**没有**"中根遍历"——因为树的孩子数不固定，"根在中间"没有明确定义。

### 森林的遍历（两种）

**先序遍历：** 先先序遍历第一棵树→再依次先序遍历剩下的树

**中序遍历：** 先中序遍历第一棵树的子树森林→访问第一棵树的根→再中序遍历剩余的树

⚠️ **命名陷阱：** 王道书说"森林的中序遍历"，有的教材叫"森林的后序遍历"（因为根确实是最后才访问的）。严蔚敏教材按"中序"命名，408统考以王道为准理解即可，但要**知道两种叫法指同一件事**。

### 三者遍历的对应关系（核心表格，必背！）

| 树 | 森林 | 对应的二叉树 |
|---|---|---|
| **先根遍历** | **先序遍历** | **先序遍历** |
| **后根遍历** | **中序遍历** | **中序遍历** |

**为什么是这样？** 理解了孩子兄弟表示法的本质就秒懂：

树的先根遍历 = 访问根 → 先根遍历第一棵子树 → 先根遍历其余子树
二叉树先序 = 访问根(N) → 先序左子树(L) → 先序右子树(R)

左子树 = firstchild = 第一棵子树
右子树 = nextsibling = 其余子树（兄弟们）

所以两者完全同构！

树的后根遍历 = 后根遍历第一棵子树 → 后根遍历其余子树 → 访问根
二叉树中序 = 中序左子树(L) → 访问根(N) → 中序右子树(R)

等等，这看起来顺序不同？其实是对的——树的后根遍历"后根遍历其余子树"中包含了"访问各子树的根"，展开后和二叉树中序遍历的访问顺序完全一致。

⚠️ **这个对应关系直接出过真题：** 2019、2020统考都考了树/森林遍历与二叉树遍历的对应。讲义里第8题就用了这个：知道森林的先根和后根（=二叉树的先序和中序），直接还原二叉树求后序。

## 四、讲义例题精讲

### 例题7：森林转二叉树后，u是v的爷爷结点，原森林中u和v什么关系？

在二叉树中u是v的双亲的双亲，v可能在4个位置（V₁V₂V₃V₄）：

```
二叉树中:
     u
    / \
   ●   ●
  /\ /\ 
V₁V₂V₃V₄
```

转回森林后分析每种情况：
- V₂位置（u→左→右）：左=u的第一个孩子，右=该孩子的兄弟 → **v是u的孩子** → 父子关系 ✅
- V₄位置（u→右→右）：右=u的兄弟，右=兄弟的兄弟 → **v是u的兄弟** → 兄弟关系 ✅
- V₁位置（u→左→左）：左=u的第一个孩子，左=该孩子的第一个孩子 → **v是u的孙子** → 也是父子关系的延伸（不在选项中，但属于I的泛化）
- V₃位置（u→右→左）：右=u的兄弟，左=兄弟的第一个孩子 → v和u无直接父子/兄弟关系

答案选 **B（I和II）**。

### 例题8：已知森林F的先根序列和中根序列，求对应二叉树T的后序

森林的先根 = 二叉树的先序 = `a b c d e f`
森林的中根 = 二叉树的中序 = `b a d f e c`

直接用先序+中序还原二叉树：
```
根=a, 中序分割: 左{b}, 右{d,f,e,c}
左子树: 只有b
右子树: 先序{c,d,e,f}, 根=c, 中序{d,f,e,c}→左{d,f,e}, 右空
  左子树: 先序{d,e,f}, 根=d, 中序{d,f,e}→左空, 右{f,e}
    右子树: 先序{e,f}, 根=e, 中序{f,e}→左{f}, 右空

     a
    / \
   b   c
      /
     d
      \
       e
      /
     f

后序: b f e d c a → 答案C: bfedca  ✓
```

## 【模块三总结】

| 知识点 | 核心要点 |
|---|---|
| 双亲表示法 | 找双亲O(1)，找孩子O(n)，并查集的基础 |
| 孩子表示法 | 找孩子方便，找双亲O(n) |
| 孩子兄弟表示法 | 左=第一个孩子，右=右兄弟，与二叉链表同构 |
| 树→二叉树 | 连兄弟、删多余、旋转；根无右子树 |
| 森林→二叉树 | 各树转二叉树后根间连右指针 |
| 根有右子树 | 对应森林；根无右子树对应单棵树 |
| 遍历对应 | 树先根=森林先序=二叉树先序 |
| | 树后根=森林中序=二叉树中序 |

## 【串联助记框架】

```
树与森林
│
├── 存储结构
│   ├── 双亲表示法 → f[i]存parent → 并查集(5.5)
│   ├── 孩子表示法 → 顺序表+链表 → 图的邻接表(第6章)
│   └── 孩子兄弟表示法 → 左孩子右兄弟 → 二叉链表同构!
│       └── 这就是树↔二叉树转换的物理基础
│
├── 转换
│   ├── 树 → 二叉树: 连兄弟+删多余+旋转 (根无右子树)
│   ├── 森林 → 二叉树: 各树转完后根间连右指针 (根有右子树)
│   └── 逆过程: 断右链→恢复兄弟→连双亲
│
└── 遍历对应 (必背!)
    ┌─────────┬─────────┬──────────┐
    │   树     │  森林    │  二叉树   │
    ├─────────┼─────────┼──────────┤
    │ 先根遍历 │ 先序遍历 │ 先序遍历  │
    │ 后根遍历 │ 中序遍历 │ 中序遍历  │
    └─────────┴─────────┴──────────┘
    关键: 左指针=firstchild, 右指针=nextsibling
```
