# 遍历序列位置关系全对比速查表

> 核心问题：在某种遍历中"n在m前"，能推出n和m在树中是什么结构关系？

---

## 一、四种遍历中"n在m前"的所有可能情况

### 【约定】
- LCA(n,m) 表示n和m的最近公共祖先
- "n在m左方" = n在LCA的左子树中，m在LCA的右子树中（或m就是LCA）
- 下面的分析对**任意二叉树**成立

---

### 1. 先序(NLR)中 n在m前

访问顺序：根→左子树→右子树

| 可能的结构关系 | 为什么n在m前 | 示意图 |
|---|---|---|
| **n是m的祖先** | n作为根先被访问，m在n的子树中后访问 | `n` → `...m...` |
| **n在m左方**（n在LCA左子树，m在LCA右子树） | LCA先递归左子树(含n)，再递归右子树(含m) | `LCA` → L(`n`) → R(`m`) |
| **n在m左方**（n在LCA左子树，m就是LCA） | 不可能！LCA作为根先于左子树被访问 | ❌ |

**总结：先序n在m前 ⟺ n是m的祖先 ∨ n在m左方（LCA左vs右子树）**

```
情况A: n是m的祖先          情况B: n在m左方
    (n)                        (LCA)
   / \                        /    \
  ... ...                   (..n..) (..m..)
    m
```

---

### 2. 中序(LNR)中 n在m前

访问顺序：左子树→根→右子树

| 可能的结构关系 | 为什么n在m前 | 示意图 |
|---|---|---|
| **n在m的左子树中**（m是n的祖先） | n在m的左子树中先被访问，然后访问m | L(`n`) → `m` |
| **m在n的右子树中**（n是m的祖先） | 访问n后，递归n的右子树访问到m | `n` → R(`m`) |
| **n在m左方**（n在LCA左子树，m在LCA右子树） | LCA先递归左子树(含n)，再递归右子树(含m) | L(`n`) → LCA → R(`m`) |
| **n在m左方**（n在LCA左子树，m就是LCA） | LCA先递归左子树(含n)，再访问LCA=m | L(`n`) → `m=LCA` |
| **n在m左方**（n就是LCA，m在LCA右子树） | 访问LCA=n，再递归右子树(含m) | `n=LCA` → R(`m`) |

**总结：中序n在m前 ⟺ n在m的"左方"**

```
统一理解: 存在某个结点x（可能是n或m或LCA），
使得n在x的左子树中(或n就是x)，m在x的右子树中(或m就是x)

所有情况本质: n的中序位置 < m的中序位置
即: n在整棵树的"更左边"
```

⚠️ **中序位置完全由左右结构决定，与祖先/子孙关系无关！**

---

### 3. 后序(LRN)中 n在m前

访问顺序：左子树→右子树→根

| 可能的结构关系 | 为什么n在m前 | 示意图 |
|---|---|---|
| **n是m的子孙**（m是n的祖先） | n在m的子树中先被访问，m作为根最后访问 | `...n...` → `m` |
| **n在m左方**（n在LCA左子树，m在LCA右子树） | LCA先递归左子树(含n)，再递归右子树(含m) | L(`n`) → R(`m`) → LCA |

**总结：后序n在m前 ⟺ n是m的子孙 ∨ n在m左方**

```
情况A: n是m的子孙          情况B: n在m左方
    (m)                        (LCA)
   / \                        /    \
  ... ...                   (..n..) (..m..)
  n
```

⚠️ 注意和先序的**对称性**：
- 先序n在m前 → n可能是m的**祖先** 或 n在m左方
- 后序n在m前 → n可能是m的**子孙** 或 n在m左方

---

### 4. 层序(BFS)中 n在m前

| 可能的结构关系 | 为什么n在m前 |
|---|---|
| **n的层次 < m的层次** | 层序按层扫描，浅层先访问 |
| **n和m同层，n在m左边** | 同层从左到右扫描 |

**总结：层序n在m前 ⟺ n的层次更浅 ∨ (同层且n在左边)**

⚠️ 层序位置与祖先/子孙关系**没有直接对应**！n在m前不代表n是m的祖先。

---

## 二、四种遍历"n在m前"的大对比表

| 遍历方式 | n在m前的所有可能 | 关键决定因素 | 易错点 |
|---|---|---|---|
| **先序NLR** | ①n是m祖先 ②n在m左方 | 祖先关系 + 左右位置 | n在m前≠n一定是m的祖先 |
| **中序LNR** | n在m的"左方"（统一） | **纯粹由左右位置决定** | 与祖先子孙无关！ |
| **后序LRN** | ①n是m子孙 ②n在m左方 | 子孙关系 + 左右位置 | n在m前≠n一定是m的子孙 |
| **层序BFS** | ①n层更浅 ②同层n在左 | 层次 + 同层左右 | 与祖先/子孙无直接对应 |

## 三、两种遍历组合推断（最核心！）

### 组合1: 先序n在m前 + 后序n在m后

```
先序n在m前 → n是m祖先 ∨ n在m左方
后序n在m后 → m是n子孙(=n是m祖先) ∨ m在n左方(=n在m右方)

取交集:
  "n是m祖先"在两个条件中都满足 ✓
  "n在m左方"满足先序条件，但后序中n在m后需要n在m右方或n是m祖先
    → "n在m左方"不满足后序条件(因为左方的n在后序中应该在m前)
    → 矛盾！排除

结论: n一定是m的祖先 ✅
```

### 组合2: 先序n在m前 + 后序n在m前

```
先序n在m前 → n是m祖先 ∨ n在m左方
后序n在m前 → n是m子孙 ∨ n在m左方

取交集:
  "n是m祖先"(先序) ∩ "n是m子孙"(后序) → 只有n=m时同时成立，排除
  "n是m祖先"(先序) ∩ "n在m左方"(后序) → 祖先不在左方(祖先在上方)，排除
  "n在m左方"(先序) ∩ "n是m子孙"(后序) → 子孙不在左方(子孙在下方)，排除
  "n在m左方"(先序) ∩ "n在m左方"(后序) → ✓

结论: n在m左方（n在LCA左子树，m在LCA右子树） ✅
```

### 组合3: 先序n在m前 + 中序n在m前

```
先序n在m前 → n是m祖先 ∨ n在m左方
中序n在m前 → n在m左方

这只能缩小到: n在m左方 ∨ (n是m祖先 ∧ n在m左方)

但n是m的祖先时，中序n在m前 → m在n的右子树中 → 此时n也算"在m左方"

结论: n在m左方（但包含"n是m祖先且m在n的右子树中"这种特殊情况）
```

⚠️ 这就是第12题的情况：先序a在b前 + 中序b在a前 → b在a的左子树中

### 组合4: 先序n在m前 + 中序m在n前（第12题！）

```
先序n在m前 → n是m祖先 ∨ n在m左方
中序m在n前 → m在n左方

取交集:
  "n在m左方"(先序) ∩ "m在n左方"(中序) → 矛盾！排除
  "n是m祖先"(先序) ∩ "m在n左方"(中序)
    → n是m祖先，中序中m在n前 → m在n的左子树中 ✓

结论: n是m的祖先，且m在n的左子树中 ✅
```

### 全部组合速查表

| 条件1 | 条件2 | 结论 | 对应题号 |
|---|---|---|---|
| 先序n在m**前** | 后序n在m**后** | **n是m的祖先** | 第11题 |
| 先序n在m**后** | 后序n在m**前** | **n是m的子孙**（对称） | — |
| 先序n在m**前** | 后序n在m**前** | **n在m左方**（非祖先子孙） | — |
| 先序n在m**前** | 中序n在m**前** | n在m左方（含n是m祖先且m在右子树） | — |
| 先序n在m**前** | 中序m在n**前** | **n是m祖先，m在n左子树中** | 第12题 |
| 中序n在m**前** | 后序n在m**前** | n在m左方（含n是m子孙且n在m左子树） | — |
| 先序n在m**前** | 层序n在m**前** | 无法确定唯一关系 | — |

## 四、单一遍历中能/不能推出的结论

### 能推出的 ✅

| 已知 | 能推出 |
|---|---|
| 先序**第一个**是X | X是根 |
| 后序**最后一个**是X | X是根 |
| 层序**第一个**是X | X是根 |
| 中序中X把序列分成L和R | L在X左子树，R在X右子树 |
| 先序n在m前 + 后序n在m后 | n是m的祖先 |
| 先序n在m前 + 后序n在m前 | n在m左方（不是祖先子孙） |

### 不能推出的 ❌

| 已知 | 不能推出 | 原因 |
|---|---|---|
| 先序n在m前 | n是m的祖先 | n也可能只是在m左方 |
| 后序n在m前 | n是m的子孙 | n也可能只是在m左方 |
| 中序n在m前 | n是m的祖先/子孙 | 中序只反映左右位置 |
| 层序n在m前 | n是m的祖先 | n可能只是层更浅或同层靠左 |
| 先序+后序 | 唯一确定二叉树 | 无法区分左右子树 |

## 五、第03/04/11/12题速解

### 第03题: 中序中n在m前的条件

中序(LNR)中n在m前 ⟺ **n在m左方**。

三种具体情况：
```
(a) n在m的左子树中    (b) m在n的右子树中    (c) n在LCA左子树，m在LCA右子树
      (m)                   (n)                    (LCA)
     /                       \                    /    \
   (..n..)                 (..m..)             (..n..) (..m..)
```

→ 答案C：n在m左方

### 第04题: 后序中n在m前 → n是m的子孙？

后序(LRN)中n在m前 → n是m子孙 **∨** n在m左方

所以"n是m的子孙"只是**充分条件**，不是充要条件！

→ 答案D："n是m的子孙"是充分条件（n是m子孙→后序n在m前✓，但反过来不一定）

### 第11题: 先序X在Y前 + 后序X在Y后

由组合表直接查：**X是Y的祖先** → 答案C

### 第12题: 先序a在b前 + 中序b在a前

先序a在b前 → a是b祖先 ∨ a在b左方
中序b在a前 → b在a左方 → b在a的左子树中(如果a是b祖先)

组合 → **a是b祖先，且b在a的左子树中** → 答案C：b在a的左子树中

## 六、终极记忆口诀

```
先序前 → 可能是祖先（N先于L、R）
后序前 → 可能是子孙（N最后才访问）
中序前 → 就是在左边（L、N、R的顺序）
层序前 → 就是层更浅或同层靠左

先前+后后 → 确定是祖先（黄金定律！）
先前+后前 → 只是在左边（非祖先子孙）

口诀: "先祖后孙中左右，先后组合定乾坤"
```

| 口诀拆解 | 含义 |
|---|---|
| 先祖 | 先序在前 → 可能是**祖**先 |
| 后孙 | 后序在前 → 可能是子**孙** |
| 中左右 | 中序位置 = 纯粹的**左右**关系 |
| 先后组合定乾坤 | 先序+后序组合 → 能唯一确定祖先/子孙/左右关系 |
