## 一、易混淆概念辨析

### 1. "度为m的树" vs "m叉树"

| 对比项 | 度为m的树 | m叉树 |
|---|---|---|
| 定义 | **至少存在**一个度为m的结点 | 每个结点的度**≤m** |
| 可否为空 | ❌ 至少有m+1个结点 | ✅ 可以是空树 |
| 最少结点数 | m+1 | 0 |
| 最少非叶结点 | 至少1个度=m的结点 | 可以没有度=m的结点 |
| 最大高度（n个结点） | n-m+1 | n（退化为单支） |
| 最小高度（n个结点） | ⌈log_m(n(m-1)+1)⌉ | ⌈log_m(n(m-1)+1)⌉ |
| **做题关键** | 算极端情况时必须保证有一个度=m的结点 | 不需要保证 |

**⚠️ 典型陷阱（第04题）：** "对于一棵具有n个结点、度为4的树"求最大高度时，不能直接答n，而是n-3（因为必须有1个度=4的结点，最后一层至少4个孩子，前面每层1个）。

### 2. "二叉树" vs "度为2的有序树"

| 对比项 | 二叉树 | 度为2的有序树 |
|---|---|---|
| 可否为空 | ✅ 可以 | ❌ 至少3个结点 |
| 只有1个孩子时 | **必须区分**是左孩子还是右孩子 | 左右是相对的，只有1个就不分左右 |
| 结点数下限 | 0（空树） | 3（根+2个孩子） |

**⚠️ 第01题易错：** "度为2的有序树就是二叉树"→ **错！** 区别就在上表第1、2行。

### 3. "二叉树为空" vs "二叉树不存在"

- "二叉树为空"：存在这棵树，只是结点数n=0 → 类似空表≠不存在
- 不存在：没有定义过

### 4. "高度" vs "深度"

| | 高度 | 深度 |
|---|---|---|
| 方向 | 从**叶子**往上数 | 从**根**往下数 |
| 根结点 | 高度 = 树高 | 深度 = 1 |
| 叶结点 | 高度 = 1 | 深度 = 所在层次 |
| **树的高度 = 树的深度** | 结点的最大层次 | 结点的最大层次 |

### 5. "路径" vs "路径长度" vs "带权路径长度"

| 概念 | 含义 | 注意 |
|---|---|---|
| 路径 | 从结点A到结点B经过的结点序列 | 只能**从上往下**，兄弟之间无路径 |
| 路径长度 | 路径上**边**的个数 | 不是结点数！ |
| 树的路径长度 | 根到每个结点的路径长度的**总和** | 不是最大值！(第03题) |
| 带权路径长度(WPL) | Σ(权值×路径长度)，只算**叶子** | 哈夫曼树用的 |

### 6. "完全二叉树" vs "满二叉树"

| | 满二叉树 | 完全二叉树 |
|---|---|---|
| 定义 | 每层都满（达到最大结点数） | 最后一层从左到右连续，前h-1层满 |
| 结点数 | **恰好** 2^h - 1 | 范围：2^(h-1) ≤ n ≤ 2^h - 1 |
| 叶子位置 | 全在最后一层 | 在最后一层和倒数第二层 |
| 度为1的结点 | **0个** | **至多1个**（且只有左孩子） |
| 是否为完全二叉树 | ✅ 满二叉树一定是完全二叉树 | 完全二叉树不一定是满二叉树 |
| 顺序存储 | 不浪费空间 | 不浪费空间 |

## 二、各类二叉树性质大对比

### 【通用性质 — 任意二叉树都成立】

| 编号 | 性质 | 公式 | 备注 |
|---|---|---|---|
| P1 | 叶结点与度2结点关系 | **n₀ = n₂ + 1** | 最核心公式！ |
| P2 | 第k层最多结点数 | 2^(k-1) | k≥1 |
| P3 | 高h至多结点 | 2^h - 1 | h≥1 |
| P4 | n个结点最小高度 | ⌈log₂(n+1)⌉ | 完全二叉树时取到 |
| P5 | n个结点最大高度 | n | 单支树时取到 |
| P6 | 总边数 | n - 1 | |
| P7 | 总边数(从度看) | n₁ + 2n₂ | = n - 1 |
| P8 | 链式存储空指针 | **n + 1** | = 2n - (n-1) |
| P9 | 三叉链表空指针 | **n + 2** | = 3n - (n-1) - (n-1+有根指向) |

### 【完全二叉树专用性质】

| 编号 | 性质 | 公式/结论 |
|---|---|---|
| C1 | 高度 | h = ⌈log₂(n+1)⌉ = ⌊log₂n⌋ + 1 |
| C2 | 父子编号关系 | 双亲⌊i/2⌋, 左孩子2i, 右孩子2i+1 |
| C3 | 判叶子 | i > ⌊n/2⌋ 则为叶子 |
| C4 | 度为1的结点 | **至多1个**，且只有左孩子 |
| C5 | n₁与n的奇偶关系 | n奇→n₁=0; n偶→n₁=1 |
| C6 | 叶结点个数 | n₀ = ⌈n/2⌉ |
| C7 | 叶子位置 | 只在最后一层和倒数第二层 |

**⚠️ C5+C6的推导（第14题必用）：**
```
n = n₀ + n₁ + n₂ = n₀ + n₁ + (n₀-1) = 2n₀ + n₁ - 1
完全二叉树中 n₁ ∈ {0, 1}
当n为奇数: n₁=0, n = 2n₀-1, n₀ = (n+1)/2
当n为偶数: n₁=1, n = 2n₀, n₀ = n/2
统一公式: n₀ = ⌈n/2⌉
```

### 【满二叉树专用性质】

| 性质 | 公式 |
|---|---|
| 总结点 | 2^h - 1 |
| 叶结点 | 2^(h-1) |
| 分支结点 | n - m = 2^(h-1) - 1 |
| n₁ | 0（没有度为1的结点） |
| 高度 | h = log₂(n+1) |

## 三、极端情况速查（最大/最小高度·结点数）

### 二叉树

| 条件 | 最小高度 | 最大高度 |
|---|---|---|
| n个结点的二叉树 | ⌈log₂(n+1)⌉（完全二叉树时） | n（单支树） |
| n个结点只有度0和度2 | h_min = ⌈log₂(n+1)⌉ | h = (n+1)/2（锯齿形） |
| 高h的完全二叉树 | 最少 2^(h-1) 个结点 | 最多 2^h - 1 个结点 |

### 一般树（度为m）

| 条件 | 最小值 | 最大值 | 说明 |
|---|---|---|---|
| n结点度m树高度 | ⌈log_m(n(m-1)+1)⌉ | n-m+1 | 最大时：前面单链+最后1个有m个孩子 |
| 高h度m树结点 | h+m-1 | (m^h-1)/(m-1) | 最小时同上反推；最大时满m叉树 |
| 度4高h最少结点 | h+3 | (4^h-1)/3 | 第05题 |
| 度4高h最多结点 | h+3 | 1+4+...+4^(h-1) | 满4叉树 |

### m叉树

| 条件 | 最小高度 | 最大高度 |
|---|---|---|
| n个结点m叉树 | ⌈log_m(n(m-1)+1)⌉ | n（和度为m的树不同！可以是单链） |

**⚠️ 度为m的树最大高度是n-m+1，而m叉树最大高度是n，区别在于度为m的树必须有一个结点度=m。**

## 四、给定n个结点能构成的不同二叉树

### n个结点高度为h的二叉树有多少种？

**总数公式：** n个不同结点能构成的二叉树总数 = **卡特兰数** = C(2n,n)/(n+1)

| n | 不同二叉树数 |
|---|---|
| 1 | 1 |
| 2 | 2 |
| 3 | 5 |
| 4 | 14 |
| 5 | 42 |

### n个结点且高度为n的二叉树（高=n的二叉树）有多少种？

高h=n意味着每层只有1个结点。除根结点外，每个结点要么是左孩子要么是右孩子→ **2^(n-1)** 种（第06题的结论）。

### 具有n个结点且高度为h的二叉树数目

这个需要递推，无简洁公式，但两个极端好记：
- 高度最小（完全二叉树）：形态**唯一**（或至多少数几种）
- 高度最大（单支链）：**2^(n-1)** 种

## 五、常考题型分类 + 解法套路

### 题型1：结点数与度的关系计算

**核心公式：**
```
① 总结点数 = n₀ + n₁ + n₂ + ... + nₘ
② 总分支数 = 1·n₁ + 2·n₂ + 3·n₃ + ... + m·nₘ
③ 总结点数 = 总分支数 + 1
```

**解题模板：** 列出①②③三个等式，联立求解。

**经典例：** 度为4的树，n₄=20, n₃=10, n₂=1, n₁=10，求n₀？
```
总分支 = 10 + 2 + 30 + 80 = 122
总结点 = 122 + 1 = 123
n₀ = 123 - 20 - 10 - 1 - 10 = 82
```

**推广公式（m叉树，第08题）：**
```
n₀ = 1 + Σ(i-1)·nᵢ   (i从2到m)
```

### 题型2：完全二叉树求叶结点/总结点

**解法：** 利用 n₁∈{0,1} + n₀=n₂+1

**第14题：** n=1001个结点的完全二叉树，叶结点？
```
n=1001为奇数 → n₁=0
n = 2n₀ - 1 → n₀ = 501
```

**第26题：** n=768的完全二叉树叶结点？
```
n=768为偶数 → n₁=1
n = 2n₀ → n₀ = 384
```

### 题型3：完全二叉树第k层有叶结点，求总结点

**思路：** 第k层有叶结点 → 树高度可能是k或k+1

**求最多（高度=k+1）：**
```
前k层是满二叉树 = 2^k - 1
第k层有x个叶子 → 第k层有 2^(k-1) - x 个分支结点
第k+1层结点 = 2×(2^(k-1) - x) = 2^k - 2x
总结点 = 2^k - 1 + 2^k - 2x = 2^(k+1) - 2x - 1
```

**求最少（高度=k）：**
```
前k-1层满 = 2^(k-1) - 1
第k层就是这x个叶子
总结点 = 2^(k-1) - 1 + x
```

### 题型4：空指针数计算

| 存储方式 | 空指针数 | 推导 |
|---|---|---|
| 二叉链表 | **n + 1** | 2n - (n-1) |
| 三叉链表 | **n + 2** | 3n - 2(n-1) |
| 三叉链表（有根指向根的额外指针） | 需具体分析 | |

**第19题（三叉链表）：** 每个结点有3个指针（左孩子、右孩子、双亲）
- 总指针3n，非空指针 = (n-1)个孩子指针 + (n-1)个双亲指针 = 2(n-1)
- 等等！根结点没有双亲→非空双亲指针只有n-1个
- 非空孩子指针也是n-1个（每条边对应一个）
- 但根结点还被一个额外指针指向？不一定
- 空指针 = 3n - 2(n-1) = n+2 ✅

### 题型5：顺序存储的最近公共祖先

**第21题思路：** 反复取 i=⌊i/2⌋ 和 j=⌊j/2⌋ 直到 i==j

```c
while (i != j) {
    if (i > j) i = i / 2;
    else j = j / 2;
}
// i就是最近公共祖先的编号
```

**第21题：** 17的祖先链: 17→8→4→2→1，19的祖先链: 19→9→4→2→1 → LCA=4

### 题型6：判断两结点是否同层

**第20题：** 编号为i的结点层次 = ⌊log₂i⌋ + 1

两结点p,q同层 ⟺ **⌊log₂p⌋ = ⌊log₂q⌋** （选A）

### 题型7：顺序存储的空间需求

**第28题：** 高度5、10个结点的二叉树顺序存储，最少需要多少存储单元？

最少 = 使编号尽量连续紧凑。高度5意味着第5层至少有1个结点，即编号≥16。若为完全二叉树形态下标最大为n，但本题高度=5强制第5层有结点→需要存到高度5的满二叉树的编号空间。

10个结点最好情况：前4层满(15个位置)但只有10个→至少第5层有结点编号为16→数组至少开到下标16→但有些中间位置空着。

关键：必须存到最大编号处。最坏时最大编号=2^5-1=31→31个单元。最好时尽量让10个结点编号连续→但高度5要求编号≥16存在→最少也需要16个单元？

正确分析：10个结点高度5，要使存储最少→让前4层尽量满，第5层左边有结点。前4层最多15个结点，但我们只有10个。所以前3层满(7个)+第4层3个=10个，但这样高度只有4！要高度5，必须第5层有结点。所以至少前4层取最少使高度5可达：前面放9个使某条路径到第4层，第5层放1个。最大编号 = 那个第5层结点的编号。让它尽量小→放在第5层最左→编号16。但前面第1-4层对应路径上的结点编号(1,2,4,8)必须存在，4个结点+第5层1个=5个，剩余5个放在编号1-15中尽量靠前。总之数组要开到至少下标16+几个→答案A=31（要能容纳高度5的满二叉树编号空间）。

**答案31** = 2^5 - 1。顺序存储必须按满二叉树的编号分配空间。

## 六、易错点清单（⚠️ 标记）

| 编号 | 易错点 | 正确理解 |
|---|---|---|
| ⚠️1 | n₀=n₂+1只对二叉树成立 | 一般树不成立！一般树用总结点=总分支+1 |
| ⚠️2 | 完全二叉树编号从1开始 | 从0开始时公式变：左孩子2i+1,右孩子2i+2,双亲⌊(i-1)/2⌋ |
| ⚠️3 | 完全二叉树高度公式 | ⌈log₂(n+1)⌉ **等价于** ⌊log₂n⌋+1（可用n=4验证：⌈log₂5⌉=3=⌊log₂4⌋+1=3 ✓） |
| ⚠️4 | "二叉树为空"≠"不存在" | 空二叉树是合法的(第02题) |
| ⚠️5 | 完全二叉树"第i个结点的左孩子是2i" | 只有**当2i≤n时**左孩子才存在！(第03题D选项陷阱) |
| ⚠️6 | 度为m的树的最大高度 | **n-m+1**，不是n！（因为至少有一个度=m的结点占了m个孩子） |
| ⚠️7 | 顺序存储空间 | 必须按**最大编号**分配，中间空位也要占空间(第28题) |
| ⚠️8 | "高度为h的完全二叉树最少结点" | 2^(h-1)，不是2^h-1(那是最多) |
| ⚠️9 | 满二叉树中n与m的关系 | n=2^h-1, m=2^(h-1), 所以n=2m-1 |
| ⚠️10 | 路径是**有向**的 | 只能从祖先到子孙，兄弟之间无路径 |
| ⚠️11 | 三叉链表的空指针 | 是n+2不是n+1，多了一个parent空指针（根的parent） |
| ⚠️12 | 树的路径长度 | 是**总和**不是最大值(第03题) |

## 八、速算技巧

### 1. 完全二叉树快速判断n₁

```
n为奇数 → n₁=0（所有非叶都是度2）
n为偶数 → n₁=1（恰好一个度1）
```

### 2. 完全二叉树叶结点速算

```
n₀ = ⌈n/2⌉
例：n=1001 → n₀=501; n=768 → n₀=384
```

### 3. 层次快速计算

```
编号i的层次 = ⌊log₂i⌋ + 1
例：编号17 → ⌊log₂17⌋+1 = 4+1 = 5
```

### 4. 第k层最多结点

```
二叉树: 2^(k-1)
m叉树: m^(k-1)
```

### 5. 高h最多结点（满树）

```
二叉树: 2^h - 1
m叉树: (m^h - 1)/(m - 1)
```
