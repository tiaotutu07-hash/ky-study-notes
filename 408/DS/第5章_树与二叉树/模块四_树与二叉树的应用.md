# 模块四：5.5 树与二叉树的应用

这是第5章的"集大成"模块，包含四个应用：哈夫曼树、并查集、二叉排序树（BST）、平衡二叉树（AVL）。每个都是高频考点。

## 一、哈夫曼树与哈夫曼编码

### 1.1 基本概念

**权(Weight)：** 结点被赋予的数值（通常代表频率/概率）

**带权路径长度(WPL)：** 所有**叶结点**的 权×路径长度 之和

$$WPL = \sum_{i=1}^{n} w_i \cdot l_i$$

其中 wᵢ 是第i个叶结点的权值，lᵢ 是该叶结点到根的路径长度（边数）。

**哈夫曼树（最优二叉树）：** 在含有n个带权叶结点的所有二叉树中，**WPL最小**的那棵。

王道例子对比（权值7,5,2,4）：

```
(a) WPL = 7×2 + 5×2 + 2×2 + 4×2 = 36
(b) WPL = 4×2 + 7×3 + 5×3 + 2×1 = 46  
(c) WPL = 7×1 + 5×2 + 2×3 + 4×3 = 35  ← 最小，哈夫曼树
```

**直觉：** 权值大的叶子离根近（路径短），权值小的叶子离根远——这样总的WPL最小。

### 1.2 哈夫曼树的构造（贪心算法）

**构造步骤：**
1. 将n个权值作为n棵只有根的树，构成森林F
2. 从F中选取**权值最小的两棵树**合并为新树（新根的权=两子树权之和）
3. 删除被选中的两棵树，把新树加入F
4. 重复2-3，直到F中只剩一棵树

**例：** 权值 {7, 5, 2, 4}

```
Step 1: 选最小的 2 和 4，合并为 6
        剩余: {7, 5, 6}

Step 2: 选最小的 5 和 6，合并为 11
        剩余: {7, 11}

Step 3: 选 7 和 11，合并为 18
        完成

最终哈夫曼树:
            18
           /  \
          7    11
         (A)  / \
             5   6
            (B) / \
               2   4
              (C) (D)

WPL = 7×1 + 5×2 + 2×3 + 4×3 = 7+10+6+12 = 35
```

### 1.3 哈夫曼树的性质（必背！）

1. **每个初始结点最终都是叶结点**，权值越小离根越远
2. 构造过程共新建 **n-1** 个结点，总结点数 = **2n-1**
3. **不存在度为1的结点**（每次都是两棵树合并，新结点度必为2）
4. 哈夫曼树形状**不唯一**（权值相等时选哪个、谁左谁右都可以），但**WPL一定相同且最优**

⚠️ 性质3意味着：哈夫曼树中只有度为0（叶子）和度为2的结点。由 n₀ = n₂ + 1 得 n₂ = n-1，总数 = n + (n-1) = 2n-1 ✓

### 1.4 哈夫曼编码

**编码问题：** 给字符集设计二进制编码，使得编码后总长度最短。

**固定长度编码：** 每个字符用相同位数。n个字符需要 ⌈log₂n⌉ 位。

**可变长度编码：** 高频字符用短码，低频字符用长码——总长度更短。

**前缀编码：** 没有任何一个编码是另一个编码的前缀。这保证了**唯一可译性**。
- 例：A=0, B=10, C=110 是前缀编码 ✅
- 反例：A=0, B=01 不是前缀编码 ❌（A是B的前缀）

**哈夫曼编码 = 用哈夫曼树生成的前缀编码**

方法：将字符频率作为权值构造哈夫曼树，**左分支标0，右分支标1**（或反过来），从根到叶子的路径即编码。

**例：** 字符串 "AAABBACCCDEEA"，各字符频率：A=5, B=2, C=3, D=1, E=2

```
构造哈夫曼树（总权=13）:
Step 1: 选D(1)和B(2)→3    剩{5,3,3,2}  [A,C,新,E]
Step 2: 选E(2)和新(3)→5    剩{5,3,5}
Step 3: 选C(3)和旧5→8      剩{5,8}  (这里策略可能不同)
Step 4: 合并5和8→13

一种可能的树:
           13
          /  \
         A:5   8
              / \
            C:3   5
                 / \
               B:2   3
                    / \
                  D:1  E:2

编码:
A: 0            (1位)
C: 10           (2位)
B: 110          (3位)
D: 1110         (4位)
E: 1111         (4位)

WPL = 1×5 + 2×3 + 3×2 + 4×(1+2) = 5+6+6+12 = 29
等长编码: 3×13 = 39 位
压缩率: 29/39 ≈ 74%
```

**哈夫曼编码的核心特点：**
1. **天然是前缀编码**——因为字符都在叶子，根到任何叶子的路径不可能是到另一叶子路径的前缀
2. **是最短的前缀编码**——WPL最小直接等价于编码总长度最短
3. **编码不唯一**——树形状不唯一 + 左0右1的约定可以反过来

⚠️ 王道注意框：左0右1没有强制规定，所以构造出的编码不唯一，但WPL一定相同且最优。

## 二、并查集

### 2.1 概念

并查集是一种处理**不相交集合**的合并与查询的树型数据结构，用**森林**表示。

三种基本操作：
- **Initial(S)：** 初始化，每个元素自成一个单元素集合
- **Find(S, x)：** 找x所在集合的**根结点**（代表元）
- **Union(S, Root1, Root2)：** 合并两个集合

**跨章关联：** 存储结构就是模块三讲的**双亲表示法**！用数组 `S[]` 存每个结点的双亲下标，根结点存负数（或自身下标）。

### 2.2 基本实现

```c
#define SIZE 100
int S[SIZE];   // 双亲指针数组

// 初始化：每个元素自成集合，根标记为-1
void Initial(int S[]) {
    for (int i = 0; i < SIZE; i++)
        S[i] = -1;
}

// 查找：沿parent指针一路向上到根
int Find(int S[], int x) {
    while (S[x] >= 0)    // 根的S值为负数
        x = S[x];
    return x;            // 返回根
}

// 合并：让一棵树的根指向另一棵树的根
void Union(int S[], int Root1, int Root2) {
    if (Root1 == Root2) return;  // 已在同一集合
    S[Root2] = Root1;            // Root2挂到Root1下面
}
```

**Find的时间复杂度 = O(d)** ，d是树的深度。最坏情况下树退化成链，O(n)。

### 2.3 优化一：按秩合并（小树合并到大树）

合并时让**结点少的树**挂到**结点多的树**下面，保持树矮。根的负数绝对值存结点数。

```c
void Union(int S[], int Root1, int Root2) {
    if (Root1 == Root2) return;
    if (S[Root2] > S[Root1]) {   // Root2的结点数更少(负数绝对值小)
        S[Root1] += S[Root2];    // 累加结点总数
        S[Root2] = Root1;        // 小树挂到大树
    } else {
        S[Root2] += S[Root1];
        S[Root1] = Root2;
    }
}
```

按秩合并后树的深度不超过 **⌊log₂n⌋ + 1**。

### 2.4 优化二：路径压缩（Find时顺便把路径上所有结点直接挂到根下）

```c
int Find(int S[], int x) {
    int root = x;
    while (S[root] >= 0)       // 先找到根
        root = S[root];
    while (x != root) {        // 路径压缩
        int t = S[x];          // 记住x的父结点
        S[x] = root;           // x直接挂到根下
        x = t;                 // 继续处理下一个
    }
    return root;
}
```

零壹讲义的递归版（更简洁）：

```c
int find(int x) {
    if (f[x] == x) return x;
    else {
        f[x] = find(f[x]);   // 递归+路径压缩
        return f[x];
    }
}
```

```
压缩前:           压缩后:
    5                 5
    |               / | \ \
    4              4   3  1  2
    |
    3
    |
    1
    |
    2
```

路径压缩后，Find的均摊复杂度接近 **O(α(n))**，α是反阿克曼函数，对任何实际n都≤4，可视为常数。

### 并查集的应用

1. **维护无向图的连通性**（判断两点是否在同一连通分量）
2. **Kruskal最小生成树算法**（第6章会用到）

## 三、二叉排序树（BST）

### 3.1 定义

BST或者是空树，或者满足：
- 左子树所有结点的值 **<** 根结点的值
- 右子树所有结点的值 **>** 根结点的值
- 左右子树本身也是BST

**核心性质：BST的中序遍历是递增有序序列。**

```
         8
        / \
       4    9
      / \     \
     2   6    12
        /    /
       5   10

中序遍历: 2, 4, 5, 6, 8, 9, 10, 12  ← 递增
```

### 3.2 查找

```c
BiTNode* BST_Search(BiTree T, int key) {
    if (T == NULL)
        return NULL;
    if (key == T->key)
        return T;
    else if (key < T->key)
        return BST_Search(T->lchild, key);  // 小于根，去左子树
    else
        return BST_Search(T->rchild, key);  // 大于根，去右子树
}
```

**跨章关联：** 这个查找过程和第1章学的**折半查找**思想一致——每次排除一半，但BST是在树结构上实现的。

### 3.3 插入（新结点一定是叶子！）

```c
int BST_Insert(BiTree &T, int k) {
    if (T == NULL) {                    // 找到空位，插入
        T = (BiTree)malloc(sizeof(BiTNode));
        T->key = k;
        T->lchild = T->rchild = NULL;
        return 1;
    }
    else if (k == T->key) return 0;     // 已存在，不插入
    else if (k < T->key)
        return BST_Insert(T->lchild, k);
    else
        return BST_Insert(T->rchild, k);
}
```

### 3.4 构造

依次插入即可。**相同的关键字，插入顺序不同，得到的BST形状不同！**

```
插入序列 {2,1,4,3}:        插入序列 {1,2,3,4}:
       2                        1
      / \                        \
     1   4                        2
        /                          \
       3                            3
                                     \
  ASL = (1+2+2+3)/4 = 2              4
                              ASL = (1+2+3+4)/4 = 2.5
```

### 3.5 删除（三种情况）

**情况1：** 删除叶结点→直接删

**情况2：** 删除只有一棵子树的结点→子树顶替

**情况3：** 删除有两棵子树的结点→用**中序直接前驱**（左子树最右结点）或**中序直接后继**（右子树最左结点）替代，然后删除替代结点

```
删除结点4（有两棵子树）:

原树:       8              用直接后继5替代4:
           / \                    8
          4    9                 / \
         / \    \               5    9
        2   6   12             / \    \
           /   /              2   6   12
          5  10                  /   /
                                (空) 10
```

⚠️ **删除后再插入相同的值，得到的BST可能与原来不同！**

### 3.6 查找效率（ASL）

BST的ASL取决于树的形态：

| 情况 | 树形 | ASL |
|---|---|---|
| 最好 | 平衡（类似折半查找） | **O(log₂n)** |
| 最坏 | 单支（退化为链表） | **O(n)** |
| 平均 | | **O(log₂n)** |

为了避免最坏情况→引出**平衡二叉树（AVL）**。

## 四、平衡二叉树（AVL）

### 4.1 定义

AVL树是一种特殊的BST，满足：任意结点的**左右子树高度差的绝对值不超过1**。

**平衡因子(BF) = 左子树高度 - 右子树高度**，取值只能是 **{-1, 0, 1}**。

```
平衡二叉树:              不平衡的树:
       1                      3
      / \                    / \
    -1   -1                -2   0
    / \    \               /
   0   1    0             2
       |                  |
       0                  1
                          |
                          0
```

### 4.2 重要推论

**(1) 高度为h的最小AVL树的结点数 Nₕ：**

```
N₀ = 0, N₁ = 1, N₂ = 2
Nₕ = Nₕ₋₁ + Nₕ₋₂ + 1
```

这和**斐波那契数列**形式一致！所以AVL树不会太高。

**(2) 含n个结点的AVL树最大深度为 O(log₂n)**，平均查找长度也是 O(log₂n)。

### 4.3 四种失衡与旋转调整

插入结点后，从插入点往上找到**第一个**|BF|>1的结点A（最小不平衡子树的根），根据失衡类型做旋转。

#### LL型（右单旋）

在A的**左孩子**的**左子树**上插入，A的BF从1变为2。

```
插入前:        插入后(失衡):      右旋后(平衡):
    A(1)          A(2)              B(0)
   / \           / \               / \
  B(0) C        B(1) C            D   A(0)
 / \           / \               |   / \
D   E         D   E              F  E   C
              |
              F(新)

操作: B上旋成根, A成B的右孩子, B的原右子树E成A的左子树
```

#### RR型（左单旋）

在A的**右孩子**的**右子树**上插入，A的BF从-1变为-2。与LL**镜像对称**。

```
    A(-2)           C(0)
   / \             / \
  B   C(-1)       A(0) E
     / \         / \    |
    D   E       B   D   F
        |
        F(新)

操作: C上旋成根, A成C的左孩子, C的原左子树D成A的右子树
```

#### LR型（先左旋后右旋）

在A的**左孩子**的**右子树**上插入。需要**两次旋转**。

```
    A(2)              A(2)              E
   / \      左旋B    / \     右旋A     / \
  B(-1) C  ──→    E   C   ──→      B   A
 / \              / \               / \   \
D   E            B   F(新)         D   F   C
    |           /
    F(新)      D

操作: 先对B做左旋(E上旋), 再对A做右旋(E上旋成根)
```

⚠️ LR的最终结果：**E成为新根**，B是E的左孩子，A是E的右孩子，E的原左右子树分别分配给B的右和A的左。

#### RL型（先右旋后左旋）

在A的**右孩子**的**左子树**上插入。与LR**镜像对称**。

```
    A(-2)              A(-2)            D
   / \      右旋C    / \    左旋A     / \
  B   C(1)  ──→    B   D   ──→     A   C
     / \                 \         /   / \
    D   E                 C       B   F   E
    |                    / \
    F(新)               F   E

操作: 先对C做右旋(D上旋), 再对A做左旋(D上旋成根)
```

### 旋转判断速查表

| 失衡类型 | 判断条件 | 旋转方式 | 记忆口诀 |
|---|---|---|---|
| **LL** | A的BF=2, A左孩子BF=1 | 右单旋 | 左左→右旋 |
| **RR** | A的BF=-2, A右孩子BF=-1 | 左单旋 | 右右→左旋 |
| **LR** | A的BF=2, A左孩子BF=-1 | 先左后右 | 左右→先左后右 |
| **RL** | A的BF=-2, A右孩子BF=1 | 先右后左 | 右左→先右后左 |

**记忆技巧：** 看失衡路径的前两步方向（LL/RR/LR/RL），旋转方向取**反**（L→右旋，R→左旋）。双旋先处理孩子再处理A。

### 4.4 完整构造示例

插入序列 {5, 3, 9, 7, 8}：

```
Step 1: 插入5      Step 2: 插入3      Step 3: 插入9
    5                  5(1)              5(0)
                      /                 / \
                     3                 3   9

Step 4: 插入7                Step 5: 插入8
    5(-1)                        5(-2)  ← 失衡！
   / \                          / \
  3   9(1) ← BF=1             3   9(2)
     /                           /
    7                           7(-1)
                                 \
                                  8

9的BF=2(左子树高2,右子树高0)? 不对, 让我重算:
插入8后: 7的BF=-1, 9的BF=2? 

重新算: 9的左子树高度=2(7→8), 右子树高度=0, BF=2
这是9的RL型? 不,是9的左孩子7的BF=-1, 所以是LR型!

不对, 从5往下看:
5的BF: 左高1,右高3 → BF=-2
找最小不平衡子树: 9的BF: 左高2,右高0 → BF=2 → 9是最小不平衡子树

9的左孩子7, 7的BF=-1(右子树8)
→ 9是LL? 不, 9的BF=2(左重), 7的BF=-1(右重) → LR型!

对9做LR旋转: 先左旋7, 再右旋9
     先左旋7:  9          再右旋9:  8
              /                   / \
             8                   7   9
            /
           7

整棵树:
        5
       / \
      3   8
         / \
        7   9
```

## 【模块四总结】

| 知识点 | 核心要点 |
|---|---|
| 哈夫曼树构造 | 每次选最小的两棵合并，贪心 |
| 哈夫曼树性质 | 总结点2n-1，无度为1的结点，WPL最优 |
| 哈夫曼编码 | 天然前缀码，最短前缀码，编码不唯一但WPL唯一 |
| 并查集 | 双亲表示法，Find沿parent上溯，Union合并根 |
| 路径压缩 | Find时把路径上结点直接挂根下，近似O(1) |
| BST定义 | 左<根<右，中序递增序列 |
| BST删除 | 叶子直删/单子树顶替/前驱(后继)替代 |
| BST效率 | 最好O(log n)，最坏O(n) |
| AVL定义 | BST+|BF|≤1 |
| AVL旋转 | LL右旋/RR左旋/LR先左后右/RL先右后左 |
| AVL查找 | 保证O(log₂n) |
