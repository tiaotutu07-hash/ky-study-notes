# 遍历序列相同/相反 → 倒推树形结构 · 终极总结

> 核心问题：两种遍历序列相同或相反时，二叉树一定是什么形状？

---

## 一、推导方法论（统一套路）

所有题目用同一个方法解决：**看根在两种序列中的位置，推出哪棵子树必须为空，然后递归**。

```
先序 NLR → 根一定在【最前】
中序 LNR → 根在【中间】（左子树之后、右子树之前）
后序 LRN → 根一定在【最后】
层序 BFS → 根一定在【最前】
```

**通用推导步骤：**
1. 找到根在两个序列中的位置
2. 如果位置矛盾 → 某棵子树必须为空才能消除矛盾
3. 对剩下的子树递归应用同样的推理
4. 得出"每个结点都满足XXX"的结论

## 二、先序/中序/后序 两两组合（6种核心情况）

### 【情况1】先序 = 中序 → 每个结点无左孩子（右单支链）

```
推导:
  先序第1个 = 根
  中序中，根前面的部分 = 左子树
  
  如果先序=中序，那么根也是中序第1个
  → 中序中根前面没有元素 → 左子树为空
  → 递归：每个结点都没有左孩子

验证: A→(右)B→(右)C
  先序 = ABC (N→R→R)
  中序 = ABC (根A无左，先访问A；然后B无左，先访问B；然后C)  ✓
```

### 【情况2】先序 = 中序的逆序 → 每个结点无右孩子（左单支链）⭐ 第33题

```
推导:
  先序第1个 = 根
  先序逆序的最后1个 = 根 = 中序最后1个
  
  中序中，根后面的部分 = 右子树
  根是中序最后一个 → 右子树为空
  → 递归：每个结点都没有右孩子

验证: A→(左)B→(左)C
  先序 = ABC
  中序 = CBA = 先序逆序  ✓
```

### 【情况3】中序 = 后序 → 每个结点无右孩子（左单支链）

```
推导:
  后序最后1个 = 根
  中序 = 后序 → 中序最后1个也是根
  → 中序中根后面没有元素 → 右子树为空
  → 递归：每个结点都没有右孩子

验证: A→(左)B→(左)C
  中序 = CBA
  后序 = CBA  ✓

⚠️ 注意：情况2和情况3结论相同！都是左单支链。
   但条件不同：一个是先序=中序逆，一个是中序=后序。
```

### 【情况4】中序 = 后序的逆序 → 每个结点无左孩子（右单支链）

```
推导:
  后序逆序的第1个 = 根
  中序 = 后序逆序 → 中序第1个也是根
  → 中序中根前面没有元素 → 左子树为空
  → 递归：每个结点都没有左孩子

验证: A→(右)B→(右)C
  中序 = ABC
  后序 = CBA, 后序逆 = ABC  ✓

⚠️ 注意：情况1和情况4结论相同！都是右单支链。
```

### 【情况5】先序 = 后序 → 至多一个结点（空树或只有根）

```
推导:
  先序第1个 = 根 = 后序最后1个
  
  若n≥2, 先序第2个 = 左子树的根(如果有左子树)或右子树的根
                后序第1个 = 左子树最深左叶(或最先后序完成的结点)
  
  先序从根开始往下，后序从最深叶开始往上
  → 方向完全相反，n≥2时不可能逐元素相等

反例验证:
  A→(左)B: 先序=AB, 后序=BA → AB≠BA ✗
  A→(右)B: 先序=AB, 后序=BA → AB≠BA ✗

  只有n=0(空树)或n=1(单结点)时 先序=后序 ✓
```

### 【情况6】先序 = 后序的逆序 → 每个结点至多一个孩子（单支链）⭐ 第32题

```
推导:
  先序: NLR
  后序逆序: NRL (把后序LRN倒过来)
  
  如果先序 = 后序逆序，即 NLR = NRL
  → 根相同 ✓
  → L和R的遍历结果互换后相等
  → 但L和R是不同子树的不同结点，不可能互换后相等
  → 除非L和R中至少一个为空！
  
  递归：每个结点至多有一个孩子（左或右都行）
  → 树是单支链 → 高度 = 结点数

验证:
  A→(左)B→(右)C:
    先序 = ABC
    后序 = CBA, 后序逆 = ABC  ✓
  
  A→(左)B→(左)C:
    先序 = ABC
    后序 = CBA, 后序逆 = ABC  ✓
  
  A→(左)B, A→(右)C (两个孩子):
    先序 = ABC
    后序 = BCA, 后序逆 = ACB  ✗ → 两个孩子不行！
```

## 三、6种情况大汇总表

| 条件 | 结论（树的形状） | 关键推导 | 对应题号 |
|---|---|---|---|
| **先序 = 中序** | 无左孩子（右单支链） | 根是中序第1个→左子树空 | 习题15C |
| **先序 = 中序逆序** | 无右孩子（左单支链） | 根是中序最后→右子树空 | **第33题D** |
| **中序 = 后序** | 无右孩子（左单支链） | 根是中序最后→右子树空 | 习题15D |
| **中序 = 后序逆序** | 无左孩子（右单支链） | 根是中序第1个→左子树空 | — |
| **先序 = 后序** | **空树或只有根** | n≥2必矛盾 | — |
| **先序 = 后序逆序** | 每个结点至多1个孩子（链） | 有两个孩子→LR互换矛盾 | **第32题B** |

### 记忆口诀

```
"先中同→无左，先中反→无右"
"中后同→无右，中后反→无左"
"先后同→只一，先后反→单链"
```

**更好的记忆方式——看根被"挤"到哪端：**

| 如果两个序列让根同时出现在… | 则… |
|---|---|
| 都在**最前** | 左子树必空（中序中根前面没东西） |
| 都在**最后** | 右子树必空（中序中根后面没东西） |
| 一前一后（正常） | 没有特殊限制 |
| 完全相同（先=后） | 只能是平凡情况 |

## 四、补充情况：涉及层序的组合

### 先序 = 层序

先序(DFS)和层序(BFS)重合的条件比较复杂：

```
简单情况: 每个结点至多1个孩子 → 先序=层序 ✓ (单链从上到下)

特殊情况: 某些有分支的树也可能满足，例如:
  A(左=B叶, 右=C叶): 先序=ABC, 层序=ABC ✓
  
但: A(左=B, B有左孩子D; 右=C):
  先序=ABDC, 层序=ABCD ✗
  
  先序深入左子树后才访问右子树，而层序是逐层扫描
  → 一旦左子树深度>1且有右兄弟，两者就不同
```

**408考试中，层序的组合题很少出现，了解即可。**

### 后序 = 层序逆序

类似分析，也比较复杂，了解即可。

## 五、第32题详解

> 某二叉树的先序序列和后序序列正好相反，则该二叉树一定是（ ）。
> A. 空或只有一个结点　　B. 高度等于其结点数
> C. 任意一个结点无左孩子　D. 任意一个结点无右孩子

**分析：** 先序 = 后序逆序（情况6）

由汇总表：每个结点至多1个孩子 → 树是单支链 → **高度 = 结点数**

逐项排除：
- A "空或只有一个结点"：太严格。3个结点的左单支链也满足 → A错
- B "高度等于其结点数"：单支链的特征 → **B对 ✓**
- C "任意一个结点无左孩子"：A→(左)B→(左)C 也满足条件 → C错
- D "任意一个结点无右孩子"：A→(右)B→(右)C 也满足条件 → D错

⚠️ **C和D的陷阱：** 单支链可以是全左、全右、或者左右混合（如A左→B右→C），所以不能说"一定无左"或"一定无右"，只能说"每个结点至多一个孩子"。

**答案：B**

## 六、第33题详解

> 某非空二叉树的先序序列和中序序列正好相反，则下列叙述中正确的是（ ）。
> A. 该二叉树一定只有一个结点
> B. 只有一个叶结点的二叉树一定满足条件
> C. 任意一个结点无左孩子的二叉树一定满足条件
> D. 任意一个结点无右孩子的二叉树一定满足条件

**分析：** 先序 = 中序逆序（情况2）→ 每个结点无右孩子（左单支链）

逐项分析：
- A "一定只有一个结点"：A→(左)B 就满足（先序AB，中序BA=先序逆）→ A错
- B "只有一个叶结点的二叉树一定满足"：A→(右)B 只有一个叶B，先序AB，中序AB≠BA → B错
  ⚠️ **关键：只有一个叶结点=单支链，但可能是右单支链，右单支链不满足！**
- C "无左孩子的二叉树一定满足"：这是右单支链，先序=中序（不是逆序）→ C错
- D "无右孩子的二叉树一定满足"：左单支链，先序=中序逆序 → **D对 ✓**

**⚠️ B选项的陷阱分析（很多人选B）：**
```
"只有一个叶结点" = 单链（每个非叶结点恰好1个孩子）
但单链分两种:
  左单支链 A→(左)B→(左)C: 先序ABC, 中序CBA=先序逆 ✓
  右单支链 A→(右)B→(右)C: 先序ABC, 中序ABC=先序同 ✗ ← 不满足！
  
所以"只有一个叶结点"不是充分条件!
```

**答案：D**

## 七、第21题详解

> 某二叉树T采用二叉链表存储，T的中序为升序序列，要求对T进行某种操作
> 得到新的二叉树T'，要求T'的中序为降序序列。

**本质：** T是BST（中序=升序），要把中序变成降序 → **镜像翻转**（交换每个结点的左右子树）

### 哪种遍历最合适？

```
先序（NLR）—— ✅ 可以
  visit(T): swap(T->left, T->right)  // 先交换
  mirror(T->lchild)                   // 再递归（此时lchild是原来的rchild）
  mirror(T->rchild)                   // 递归
  → 没有问题，交换后再递归，不会漏掉子树

后序（LRN）—— ✅ 可以
  mirror(T->lchild)                   // 先递归
  mirror(T->rchild)                   // 再递归
  visit(T): swap(T->left, T->right)  // 最后交换
  → 没有问题，先递归完再交换

中序（LNR）—— ❌ 有问题！
  mirror(T->lchild)                   // 处理左子树
  visit(T): swap(T->left, T->right)  // 交换左右
  mirror(T->rchild)                   // 处理"右子树"
  
  ⚠️ 交换之后，T->rchild变成了原来的lchild（已经处理过了！）
  原来的rchild现在变成了T->lchild，永远不会被处理！
  → 漏掉了一棵子树！中序不行！
```

### 逐项分析

- A "采用中序遍历的方法最合适" → **错！中序会导致漏处理子树**
- B "采用后序遍历的方法最合适" → 后序可以，但先序也可以，"最合适"有争议
- C "T'中的根结点还是原T中的根结点" → **对 ✓ 镜像翻转不改变根**
- D "T'中的叶结点不一定是原T中的叶结点" → 错！镜像翻转保持叶/非叶性质不变

**答案：C**

### 延伸思考：为什么中序有问题而先序后序没有？

```
本质原因：中序的visit在两次递归之间（L → N → R）

当visit(N)改变了树的结构（交换左右子树）后，
第二次递归(R)看到的树已经不是原来的树了——
右子树变成了已处理的旧左子树。

而先序（N → L → R）在递归前就完成了操作，
后序（L → R → N）在递归后才操作，
都不会出现"改了结构再递归到错误子树"的问题。

⚠️ 这个bug和中序线索化时需要pre指针是同一类问题：
中序遍历中，当前操作会影响后续递归访问的对象。
```

## 八、终极对比表（考前看这一张就够）

### 两种序列相同

| 相同的两种序列 | 树的形状 | 记忆 |
|---|---|---|
| 先序 = 中序 | 无左孩子（右链） | 根挤到中序最前→左空 |
| 中序 = 后序 | 无右孩子（左链） | 根挤到中序最后→右空 |
| 先序 = 后序 | 空树或单结点 | 方向完全相反，n≥2不可能 |

### 两种序列互为逆序

| 互逆的两种序列 | 树的形状 | 记忆 |
|---|---|---|
| 先序 = 中序逆 | 无右孩子（左链） | 根挤到中序最后→右空 |
| 中序 = 后序逆 | 无左孩子（右链） | 根挤到中序最前→左空 |
| 先序 = 后序逆 | 单支链（高度=n） | 每个结点至多1个孩子 |

### 一句话总结规律

```
中序序列决定左右：
  根在中序最前 → 左子树空 → 无左孩子（右链）
  根在中序最后 → 右子树空 → 无右孩子（左链）

先序和后序的关系决定链的约束强度：
  相同 → 约束最强（只能空或单结点）
  互逆 → 约束最弱（只要求每个结点至多1孩子，左右都行）
```

## 九、做题决策树（遇到这类题直接走流程）

```
题目说"遍历X和遍历Y相同/相反"
         │
         ▼
    涉及中序吗？
    ├── 是 → 看根被挤到中序的哪端
    │       ├── 根在中序最前 → 无左孩子（右链）
    │       └── 根在中序最后 → 无右孩子（左链）
    │
    └── 否（先序和后序的关系）
            ├── 相同 → 空树或单结点
            └── 互逆 → 每个结点至多1个孩子（任意单链）

⚠️ 特别注意：
  "单链"包括左链、右链、左右混合链
  所以不能说"一定无左孩子"或"一定无右孩子"
  只能说"高度=结点数"或"至多1个孩子"
```
