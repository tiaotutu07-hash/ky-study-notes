## 🗺️ 全景导图

排序按思想分四大类——插入、交换、选择、归并基数。上一节的插入排序靠"挪窝"找位置,这一节的核心动作换成**对换两个元素**。基础款冒泡(O(n²)、稳定)简单但慢,真正的考点是它的进化版——**快速排序**:基于分治、平均 O(nlog₂n)、是所有内部排序中**平均性能最优**的算法,也是 408 算法题的常驻嘉宾。学完本节,你应能:① 默写两份代码;② 手动跑一遍 Partition 双指针推演;③ 精准判断"k 趟后哪些元素已就位"这类高频选择题;④ 看到"基本有序"立刻反应过来这是快排的最坏情况。

## 一、交换排序总览

讲第二章排序前先提一个问题: 既然插入排序已经能完成排序任务,为什么还要再搞一类排序?答案是——**插入排序的"挪窝"成本太高了**。每插一个元素,有序区里所有比它大的元素都得集体后挪一格,这是 O(n²) 移动操作的根源。

如果换个思路: 不再"为新元素腾位置",而是**直接对换两个错位元素**呢?这就是交换排序的核心思想。

> **零壹 PPT 直观图**:序列 `[31, 42, 46, 61, 16, 84, 28, 75, 38]`,因为 `38 < 75`,把它俩对换一下 → `[31, 42, 46, 61, 16, 84, 28, 38, 75]`。这一步就叫**一次交换**。

### 1. 什么是"交换"?

**交换**: 根据序列中两个元素关键字的比较结果,对换这两个记录在序列中的位置。

听起来朴素,但所有交换排序都是把这个朴素动作**重复 N 次**——区别只在于"挑哪两个交换"和"按什么顺序交换"。

### 2. 两兄弟的战力对比

基于交换思想的算法很多,408 重点考查两个:

| 维度 | 冒泡排序 | 快速排序 |
|---|---|---|
| 思路 | 相邻两两比较 | 分治 + 双指针划分 |
| 平均时间 | O(n²) | O(nlog₂n) |
| 稳定性 | 稳定 | **不稳定** |
| 408 考频 | 简单题偶尔考 | **重灾区**,大题常客 |

**📦 总览收束小结**: 交换排序 = "对换两个错位元素",冒泡是基础款(简单稳定但慢),快排是进阶款(平均最快但不稳定),后者才是 408 算法大题最爱的考点。

## 二、冒泡排序 (BubbleSort)

### 1. 基本思想:水里的气泡

为什么叫"冒泡"?想象一缸水,水里有大大小小的气泡——**小气泡轻,会逐渐往上飘到水面**;大石头重,会沉到水底。冒泡排序就是这个比喻的算法版:

> **从后往前**两两比较相邻元素,若为逆序(`A[j-1] > A[j]`)就交换它们,直到这一趟扫完。这一趟下来,**最小的元素**像气泡一样"漂浮"到了序列的**最前面**(它的最终位置)。下一趟从倒数第二个位置开始扫(因为最前面那个已经定了),如此**最多 n-1 趟**就能把所有元素排好序。

⚠️ **方向可以反过来**: 也可以"从前往后"扫,把最大的"沉"到最后。两种方向都对,408 选择题里要看清楚题目用的哪种。

### 2. 王道经典图解(8 数 5 趟全过程)

王道图 8.3 用序列 `49, 38, 65, 97, 76, 13, 27, 49̄` 演示冒泡(注意有两个 49,加横线的那个是后一个,用来区分稳定性)。

```
位置  初始  第1趟后  第2趟后  第3趟后  第4趟后  第5趟后(终态)
 1    49    13      13       13       13       13
 2    38    49      27       27       27       27
 3    65    38      49       38       38       38
 4    97    65      38       49       49       49
 5    76    97      65       49̄       49̄       49̄  ← 49 与 49̄ 相对顺序未变,稳定
 6    13    76      97       65       65       65
 7    27    27      76       97       76       76
 8    49̄    49̄      49̄       76       97       97
```

**第 1 趟从后往前扫的细节**(王道原文):  
27 < 49̄,不交换;13 < 27,不交换;76 > 13,**交换**;97 > 13,**交换**;65 > 13,**交换**;38 > 13,**交换**;49 > 13,**交换**。一趟下来 13 漂到了最前面。

**第 5 趟没有发生任何交换** → 算法提前结束(flag 优化的功劳,见下面代码)。

讲完图解,关键问题来了: 怎么用代码把这个过程写出来?

### 3. 代码实现(含 flag 优化)

```c
void BubbleSort(ElemType A[], int n) {
    for (int i = 0; i < n-1; i++) {        // 外层:总共最多 n-1 趟
        bool flag = false;                 // 标志:本趟是否发生过交换
        for (int j = n-1; j > i; j--) {    // 内层:从后往前两两比较
            if (A[j-1] > A[j]) {           // 若逆序
                swap(A[j-1], A[j]);        // 调用 swap 函数交换
                flag = true;               // 标记本趟有交换
            }
        }
        if (flag == false)                 // 本趟一次交换都没有
            return;                        // 说明已全局有序,直接退出
    }
}
```

**逐行细节解读**:

- **外层 `i < n-1`**: 为什么是 n-1 不是 n? 因为 n-1 个元素都归位了,最后那个一定也在最终位置(没东西跟它换了)。
- **内层 `j > i` 不是 `j >= i`**: ⚠️ 这是经典坑点!`i` 之前的位置(0~i-1)在前几趟已经确定是最小的几个元素,**不需要再比较**。如果写成 `j >= i`,会多比一次而且可能出错(`A[j-1]` 越界到 `A[i-1]`,但这个位置已是定值,会浪费时间)。
- **`A[j-1] > A[j]`**: 严格大于,等于不换 → 这是冒泡稳定性的来源。
- **flag 优化**: 如果某一趟一次交换都没发生,说明序列已经完全有序,直接 return。这就是冒泡的最好情况能做到 O(n) 的关键。

⚠️ **flag 必须放在内层 for 之前、外层 for 之内**: 每趟开始时重置为 false,趟尾检查。位置错了优化就废了。

### 4. 性能分析(口诀: 最好 n、最坏 n²、稳定不挑结构)

讲完代码,我们来推一推复杂度。这一段是 408 选择题的高频考点。

**时间效率**:

| 情况 | 比较次数 | 移动次数 | 时间复杂度 |
|---|---|---|---|
| 最好(初始有序) | n-1 | 0 | **O(n)** |
| 最坏(初始逆序) | n(n-1)/2 | **3n(n-1)/2** | **O(n²)** |
| 平均 | — | — | **O(n²)** |

**最坏移动次数为什么是 3n(n-1)/2,不是 n(n-1)/2?**

⚠️ 还记得 swap 函数怎么写的吗?  
```c
int temp = a;  a = b;  b = temp;  // 三条赋值 = 一次交换需要 3 次移动
```

正是因为一次 swap 内部要做 3 次赋值,所以最坏情况下比较 n(n-1)/2 次、每次都触发交换、每次交换 3 次移动 → **总移动 = 3n(n-1)/2**。这一点零壹 PPT 没强调,但王道明确写了,408 选择题考过类似计算。

**空间效率**: 只用了常数个辅助变量(temp、flag、i、j) → **O(1)**。

**稳定性**: ⚠️ 代码里 `if (A[j-1] > A[j])` 是**严格大于**,相等时**不交换** → 相等元素相对顺序不变 → **稳定**。

**适用性**: 顺序存储和**链式存储**都适用(因为只交换相邻元素,链表也能实现)。

### 5. 关键特性:有序子序列是"全局有序"的

⚠️ **这是 408 选择题最爱挖的坑**,也是冒泡和直接插入的本质区别。

**跨章关联**: 还记得 8.2 节直接插入排序吗?它每一趟扩大有序区,但**这个有序区只是相对前 i 个元素有序,不一定是全局有序**——后面可能藏着比有序区更小的值。

冒泡排序则相反:

| 排序 | 第 k 趟后的"有序部分" | 性质 |
|---|---|---|
| 直接插入 | 前 k 个元素相对有序 | **不一定全局有序**(后面可能有更小的) |
| **冒泡** | 前 k 个(或后 k 个)元素 | **一定全局有序**!(已就位,不再变动) |

**为什么冒泡是全局有序的?** 因为每一趟冒泡都把"剩下未排序部分的最小值"**漂浮到序列最前面**——这个最小值小于等于剩余所有元素,所以它放到的位置就是它的**最终位置**。多趟下来,前面的有序段就是全局最小的那 k 个,且彼此有序。

**408 选择题模板**:  
> "对序列 [...] 用冒泡排序进行 k 趟后,以下哪个序列可能是结果?"  
> 答题套路: 看前 k 个(或后 k 个)位置上的元素,**它们必须是原序列中最小(或最大)的 k 个,且已排好序**。如果不满足 → 排除。

**📦 冒泡收束小结**: 三步法记骨架(后往前扫 → 逆序就换 → 一趟漂一个);flag 优化让最好情况降到 O(n);最坏移动 3n(n-1)/2 别忘三倍系数;**有序子序列是"全局有序"** 是它跟直接插入最大的区别,也是 408 选择题的命门。

## 三、快速排序 (QuickSort)

🚨 **重头戏来了**。如果说冒泡是 408 排序里的"小学生题",快排就是"高考压轴题"——2014、2019、2023 都直接考它的中间过程,2016 考划分应用,2010 考递归次数,2024 考思想。这一章你要把每一行代码都吃透。

### 1. 分治思想:先分两边,再各自排

冒泡是"两两交换",一次只能让一个元素归位,效率太低。能不能**一次让一个元素直接跳到最终位置,同时把整个序列分成"小的一堆 + 大的一堆"**?这正是快排的灵魂——**Partition 操作**(一次划分)。

> **快排基本思想**(基于分治):  
> ① **选枢轴 pivot**: 在待排序表 L[1..n] 中任取一个元素作为枢轴,通常取**首元素**。  
> ② **一次划分(Partition)**: 通过一趟排序把表分成独立的两部分 L[1..k-1] 和 L[k+1..n],使得**左半部所有元素 < pivot,右半部所有元素 ≥ pivot**,枢轴 pivot 放到了它的最终位置 L[k] 上。  
> ③ **递归两边**: 分别对左、右两个子表重复上述过程,直到每个子表只剩一个元素或为空。

⚠️ **每趟划分必有 ≥1 个枢轴归位,但有序子序列不会形成**(王道注意框)——这跟冒泡完全不同!冒泡每趟产生有序段,快排每趟只产生"枢轴归位 + 左小右大"的格局。

讲完抽象思想,我们用具体数字跑一遍。

### 2. Partition 双指针推演(零壹 PPT 8 数完整版)

序列: `38, 27, 54, 86, 65, 11, 16, 38̄`(两个 38,加横线的是第二个,用来证明不稳定)

设 `low=0, high=7`,取首元素 `pivot = A[low] = 38`,然后开始**双指针交替搜索**:

```
初始:   38   27   54   86   65   11   16   38̄
        ↑                                  ↑
       low                                high
       pivot=38(暂存,A[low] 位置视为"空坑")

【步 1】high 从右往前找 < pivot 的元素 → A[7]=38̄ 不<38(大于等于 pivot),high--
       continue 循环条件 while(low<high && A[high]>=pivot) high--
        38   27   54   86   65   11   16   38̄
        ↑                              ↑
       low                            high (此时 A[6]=16<38,跳出 while)

【步 2】把 16 填到 low 位置: A[low] = A[high]
        16   27   54   86   65   11   16   38̄
        ↑                              ↑
       low                            high

【步 3】low 从左往后找 > pivot 的元素 → A[1]=27<=38,low++ → A[2]=54>38,跳出
       while(low<high && A[low]<=pivot) low++
        16   27   54   86   65   11   16   38̄
                  ↑                   ↑
                 low                 high (A[2]=54)

【步 4】把 54 填到 high 位置: A[high] = A[low]
        16   27   54   86   65   11   54   38̄
                  ↑                   ↑
                 low                 high

【步 5】high 从右往前找 → A[6]=54>=38,high--; A[5]=11<38,跳出
        16   27   54   86   65   11   54   38̄
                  ↑              ↑
                 low            high (A[5]=11)

【步 6】把 11 填到 low 位置: A[low] = A[high]
        16   27   11   86   65   11   54   38̄
                  ↑              ↑
                 low            high

【步 7】low 从左往后找 → A[3]=86>38,跳出
        16   27   11   86   65   11   54   38̄
                       ↑         ↑
                      low       high (A[3]=86)

【步 8】把 86 填到 high 位置: A[high] = A[low]
        16   27   11   86   65   86   54   38̄
                       ↑         ↑
                      low       high

【步 9】high 从右往前找 → A[5]=86>=38,high--; A[4]=65>=38,high--; low==high,跳出主循环
        16   27   11   86   65   86   54   38̄
                       ↑↑
                    low,high

【步 10】最终: A[low] = pivot,把 38 放到 low==high 的位置,这就是它的最终位置!
        16   27   11   38   65   86   54   38̄
        └────小────┘   ↑   └────大────┘
                    pivot 归位
        return low;  // 返回枢轴最终位置 = 3
```

🎯 **一趟 Partition 完成**: 38 归位在第 3 号位置,左边 [16,27,11] 都 <38,右边 [65,86,54,38̄] 都 ≥38。

⚠️ **观察 38̄**: 它原来在第 7 位(38 之后),现在还在右半区。**虽然这一趟它没动**,但只要后续右子区一划分,它就可能跳到 38 的左边——**这就是快排不稳定的来源**(后面会专门讲)。

### 3. QuickSort 递归 + 王道二叉树图

一趟 Partition 完成后,枢轴归位,左右两个子表各自递归调用 QuickSort 即可。

```c
void QuickSort(ElemType A[], int low, int high) {
    if (low < high) {                          // 递归终止:子表只剩 1 个元素或为空
        int pivotpos = Partition(A, low, high); // 划分,得到枢轴最终位置
        QuickSort(A, low, pivotpos - 1);        // 递归排左子表
        QuickSort(A, pivotpos + 1, high);       // 递归排右子表
    }
}

int Partition(ElemType A[], int low, int high) {
    ElemType pivot = A[low];                   // 取首元素为枢轴(暂存)
    while (low < high) {                       // 主循环:双指针未相遇
        while (low < high && A[high] >= pivot) // 右指针找 < pivot 的元素
            high--;
        A[low] = A[high];                      // 把 < pivot 的塞到左空位
        while (low < high && A[low] <= pivot)  // 左指针找 > pivot 的元素
            low++;
        A[high] = A[low];                      // 把 > pivot 的塞到右空位
    }
    A[low] = pivot;                            // 主循环结束,low==high 即枢轴最终位置
    return low;                                // 返回枢轴位置
}
```

**逐行精读**:

- **`if (low < high)`**: ⚠️ 这是递归出口,不是 `if (low <= high)`!当 `low == high` 时子表只剩一个元素,本身有序,直接返回。
- **`pivot = A[low]`**: 取首元素,A[low] 这个位置在循环中被视为"空坑",可以被覆盖。
- **`A[high] >= pivot`**: ⚠️ **大于等于**,不是大于!如果写成 `A[high] > pivot`,遇到等于 pivot 的元素 high 不会下移,会陷入死循环或错误归位。**等于 pivot 的元素留在右边**(或左边,取决于另一个 while 的写法)。
- **`A[low] <= pivot`**: ⚠️ 同理,**小于等于**。两个 while 一个用 `>=`、一个用 `<=`,**缺一不可**。
- **`A[low] = pivot`**: 主循环结束时,low == high,这个位置就是 pivot 该去的最终位置。

**🌳 王道图 8.4 · 用二叉树形式描述递归调用**:

接着上面那组 `[16, 27, 11] 38 [65, 86, 54, 38̄]`,递归往下:

```
            第1层划分后: 38 归位
                   38
                  /  \
              [16,27,11]  [65,86,54,38̄]
                  ↓             ↓ 各自调用 Partition
            第2层: 27 归位      54 归位(假设)
                  27                 54
                 /  \               /  \
              [16] [11]          [38̄]  [65,86]
                  ↓                       ↓
            第3层: 16/11 直接返回       第3层: 65 归位
                                          65
                                         /  \
                                       []   [86]
                  ↓
            第4层: 86 直接返回(子表只剩1个)

最终: 11  16  27  38  38̄  54  65  86
```

观察这棵递归树:**树的深度 = 递归调用栈的最大深度 = 快排空间复杂度的来源**。这是下一节性能分析的伏笔。

### 4. 性能分析(空间 = 递归深度,这是核心)

🎯 **【命题追踪】2010 年考点**: 快排中递归次数的影响因素 → 答案: 与初始序列有关,与划分是否对称有关。

**空间效率**: ⚠️ 这是和插入/冒泡完全不同的特点!

> 快排是递归的,需要借助一个**递归工作栈**保存每层递归的信息,**栈的容量 = 递归调用的最大层数**。

| 情况 | 递归树形态 | 栈深度 | 空间复杂度 |
|---|---|---|---|
| **最好** | 完美平衡(每次划分两边等长) | log₂n | **O(log₂n)** |
| 平均 | 接近平衡 | log₂n | **O(log₂n)** |
| **最坏** | 退化成链(每次只分出 1 个) | n-1 | **O(n)** |

**跨章关联**: 这个"递归调用栈"就是 → 第 3 章 · 栈的"系统栈"。考研里常说的"递归 → 栈"就是这个机制。

**时间效率**:

| 情况 | 时间复杂度 | 触发条件 |
|---|---|---|
| 最好 | **O(nlog₂n)** | 每次 Partition 都对半分(如随机序列) |
| 平均 | **O(nlog₂n)** | 接近平均 |
| **最坏** | **O(n²)** | ⚠️ **初始基本有序或基本逆序** |

⚠️ **快排最坏情况发生在"已经有序"时**——这极其反直觉!

**为什么?** 当序列已升序、且取首元素为枢轴时,首元素就是当前最小值,Partition 划分后**左子表为空,右子表 n-1 个元素**——递归树退化成右斜链,深度 n-1,每层做 O(n) 工作,总时间 O(n²)。

🚨 **结论**: **快排是所有内部排序算法中,平均性能最优的排序算法**(王道原话)。这句话 408 选择题考过原话辨析。

### 5. 稳定性(不稳定)与适用性(仅顺序)

**稳定性**: ⚠️ **不稳定**!

**反例**(王道原话简化版): 表 L = `{3, 2, 2̄}`,以 3 为枢轴。

```
初始:   3   2   2̄         pivot=3, low=0, high=2
        ↑       ↑
       low     high

high 找 <3 的: A[2]=2̄<3 → A[low]=A[high] → 2̄ 2 2̄
low 找 >3 的: A[1]=2<=3, low++; A[2]=2̄<=3, low++ (碰到 high) → 跳出
A[low] = pivot = 3 → 2̄ 2 3

最终序列: 2̄  2  3
```

原序列里 2 在 2̄ 之前,排序后 2̄ 在 2 之前——**相对顺序变了**,所以不稳定。

**适用性**: 🎯 **【命题追踪】2011 年考点**: 快排适合的存储方式 → **仅适用于顺序存储的线性表**。

**为什么链表不行?** 快排的 Partition 依赖 low/high 双指针**双向逼近**,需要 `high--`(从右往前移)。链表只有 next 指针,没有 prev,无法高效后退,所以**链表上无法实现快排**(或者说效率会退化得没意义)。

### 6. 优化方法(让最坏几乎不发生)

既然最坏情况是"基本有序",而实际场景中"基本有序"的输入很常见(比如往一个有序数组里加几个新元素再排),那快排岂不是常常退化?于是有了优化:

| 方法 | 思路 | 效果 |
|---|---|---|
| **三数取中** | 从首、尾、中三个位置取元素,选**中位数**作为枢轴 | 避开极端值,划分更平衡 |
| **随机化枢轴** | 在 [low, high] 中随机选一个位置的元素作枢轴 | 让恶意输入失效,期望复杂度 O(nlog₂n) |
| **小区间用插入** | 子表长度 < 某阈值(如 10)时改用直接插入 | 避免递归常数开销 |

**正是因为**最坏情况太恶心,**所以**实战中(包括 C++ STL 的 sort)都用三数取中 + 随机化的混合策略。考研一般默认"取首元素"作枢轴。

**📦 快排收束小结**: 分治三步走(选 → 划 → 递),Partition 双指针填坑(while 内层条件 `>=` / `<=` 缺一不可),递归终止 `low<high`,空间=栈深度=最坏 O(n)、最好 O(log₂n),时间最坏 O(n²)发生在已有序时(反直觉),不稳定 + 仅顺序存储,**平均性能最优**是它的金字招牌。

## 四、冒泡 vs 快排 全维度对比

讲完两兄弟,408 大题最爱让你做对比。把下面这张表死记硬背:

| 维度 | 冒泡排序 | 快速排序 |
|---|---|---|
| 思想 | 相邻两两交换,逐趟漂浮 | 分治 + 双指针划分 |
| 时间(最好) | O(n)(序列已有序) | O(nlog₂n)(对半划分) |
| 时间(平均) | O(n²) | **O(nlog₂n)** ✨ |
| 时间(最坏) | O(n²) | **O(n²)(基本有序!)** ⚠️ |
| 空间 | O(1) | O(log₂n) ~ O(n) |
| 稳定性 | **稳定** | **不稳定** |
| 适用结构 | 顺序 + **链式** | **仅顺序** |
| 一趟产生的有序区 | **一定全局有序** | 仅有 1 个枢轴归位,无有序区 |
| 408 考查重点 | 趟数判断、稳定性 | 中间过程、递归深度、思想 |

**易错点速查**(蓝箭头 = 反直觉):
- ⚠️ 冒泡最坏移动 = **3**n(n-1)/2(swap 三次赋值)
- ⚠️ 快排最坏 = **基本有序**(不是基本逆序)
- ⚠️ 快排"一趟"只产生 1 个枢轴归位,**不形成**有序子序列
- ⚠️ 冒泡"一趟"产生的有序段是**全局有序**的(不同于直接插入)

## 五、本节总结表

| 知识点 | 核心要点 |
|---|---|
| 交换的定义 | 根据比较结果对换两个记录的位置 |
| 冒泡思想 | 后往前两两换,一趟漂一个,最多 n-1 趟 |
| 冒泡代码内层 | `for(j=n-1; j>i; j--)`,j>i 不是 j>=i |
| flag 优化 | 趟前置 false,本趟无交换则 return,最好 O(n) |
| 冒泡时间 | 最好 O(n) / 最坏平均 O(n²) / 最坏移动 3n(n-1)/2 |
| 冒泡空间稳定 | O(1) · 稳定 · 顺序+链式都行 |
| 冒泡有序段特性 | **一定是全局有序**(与直接插入的关键区别) |
| 快排思想 | 选枢轴 → Partition 左小右大 → 递归两侧 |
| Partition 算法 | 双指针填坑,low/high 交替搜索,low==high 时枢轴归位 |
| Partition 边界 | `A[high]>=pivot` 与 `A[low]<=pivot`,缺一不可 |
| QuickSort 终止 | `if(low<high)`,子表只剩 1 个元素时返回 |
| 快排时间 | 最好/平均 O(nlog₂n),**最坏 O(n²) 发生在基本有序时** |
| 快排空间 | =递归栈深度,最好 O(log₂n),最坏 O(n) |
| 快排稳定性 | **不稳定**(L={3,2,2̄} → {2̄,2,3}) |
| 快排适用性 | **仅顺序存储**(链表不行) |
| 快排地位 | **所有内部排序中平均性能最优** |
| 快排优化 | 三数取中、随机枢轴、小区间用插入 |
| 一趟划分特性 | 仅 1 个枢轴归位,**不产生有序子序列** |
