## 🗺️ 全景导图

排序四大类讲到第三类——**选择排序**。它的灵魂动作是"**每一趟从未排序部分挑出关键字最值,放到有序段末尾**",一趟定一位。基础款是**简单选择排序**(O(n²)、不稳定、链表也能用),朴素直接但效率不够;真正的考点是它的进阶款——**堆排序**:用一棵完全二叉树(数组层序存储)把"找最值"的成本压到 O(log₂n),最终 **O(nlog₂n) 时间 + O(1) 空间**。堆排序是 408 大题的"必考嘉宾"——堆的建立、插入、删除、调整、Top-k 应用全是命题点。学完本节,你应能:① 默写大根堆建堆 + HeapAdjust 代码;② 手动跑一遍 Floyd 建堆(从 ⌊n/2⌋ 起逆序调整);③ 区分"插入(向上调整)"与"删除(向下调整)"两套流程;④ 应对"求第 k 大/小元素"的海量数据题。

## 一、选择排序总览

讲完前两节排序,我们已经见识了两套思路:**插入派**靠"找位置 + 挪窝",**交换派**靠"对换两个错位元素"。这一节登场的**选择派**,思路更朴素:不绕弯,**直接挑出最值放到该去的位置**。

### 1. 选择排序的本质动作

> **选择排序**(零壹 PPT 原话): 每趟在后面的 n-i+1 (i=1,2,…,n-1) 个待排序元素中**选取关键字最小**(以升序为例)的元素,作为有序子序列的第 i 个元素,直到 n-1 趟做完,完成 n 个元素的排序。

```
   有序序列 L[0..i-1]      |      无序序列 L[i..n-1]
   ┕━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┙
                     ↓ 从无序部分挑出最小元素 L[k]
   有序序列 L[0..i-1] | L[i..k-1]  L[k]  L[k+1..n-1]
                              ↓ L[k] 与 L[i] 交换
   有序序列 L[0..i]            |     无序序列 L[i+1..n-1]
```

**关键观察**:每趟选择都让有序段长度 +1,无序段长度 -1。**有序段是全局有序的**(已就位,不再变动)——这一点跟冒泡相同,跟直接插入不同。

### 2. 两兄弟战力对比

基于"挑最值"的算法有两种:

| 维度 | 简单选择排序 | 堆排序 |
|---|---|---|
| 找最值的成本 | **O(n)**(全扫一遍) | **O(log₂n)**(借助堆结构) |
| 时间复杂度 | O(n²) | **O(nlog₂n)** ✨ |
| 稳定性 | **不稳定** | **不稳定** |
| 408 考频 | 偶尔考 | **重灾区**,统考重点 |

**📦 总览收束小结**: 选择排序 = "每趟挑最值放前面",简单选择是 O(n²) 朴素版(找最值靠暴力扫),堆排序是 O(nlog₂n) 高级版(找最值靠堆结构),后者是 408 历年统考考查的重点。

## 二、简单选择排序 (SelectSort)

### 1. 基本思想:朴素直接

> **算法思想**(王道原话): 假设排序表为 L[1..n],第 i 趟排序即从 L[i..n] 中选择关键字最小的元素与 L(i) 交换,每趟排序可以确定一个元素的最终位置,这样经过 n-1 趟排序就可使得整个排序表有序。

简单选择的"简单"二字不是贬义,是**真的简单**——没有花哨的数据结构,没有递归,就是双重 for 循环硬扫:外层管"第几趟",内层从无序段里**线性扫描找最小值的位置**,最后跟无序段开头那个元素交换。

### 2. 零壹 PPT 5 数全程推演 (2,4,1,5,3)

```
初始:   [2]  [4]  [1]  [5]  [3]      ← 全是无序的(红色)

第1趟: 从全部5个元素中找最小=1(下标 2),与 A[0]=2 交换
       【1】 [4]  [2]  [5]  [3]      ← 1 已就位(绿色)

第2趟: 从 A[1..4] 中找最小=2(下标 2),与 A[1]=4 交换
       【1】【2】 [4]  [5]  [3]      ← 2 已就位

第3趟: 从 A[2..4] 中找最小=3(下标 4),与 A[2]=4 交换
       【1】【2】【3】 [5]  [4]      ← 3 已就位

第4趟: 从 A[3..4] 中找最小=4(下标 4),与 A[3]=5 交换
       【1】【2】【3】【4】【5】     ← 4 已就位,5 自动就位

完成。共 n-1 = 4 趟。
```

⚠️ **注意**: 第 4 趟做完后,最后一个元素 5 不需要单独再选(它已经是剩下唯一的元素了),所以**总共 n-1 趟而不是 n 趟**。

### 3. 代码实现(min 变量记位置)

```c
void SelectSort(ElemType A[], int n) {
    for (int i = 0; i < n-1; i++) {       // 一共 n-1 趟
        int min = i;                       // 假设当前 i 位置就是最小
        for (int j = i+1; j < n; j++) {    // 在 A[i+1..n-1] 中扫描
            if (A[j] < A[min])             // 发现更小的
                min = j;                   // 更新最小元素位置
        }
        if (min != i)                      // 最小不在 i 位置才需要交换
            swap(A[i], A[min]);            // swap 内部 3 次赋值
    }
}
```

**逐行精读**:

- **`int min = i`**: 用 min 变量记录"当前认为最小的元素的**下标**",不是值!这是细节。
- **`if (A[j] < A[min])`**: ⚠️ 严格小于,等于不更新 min → 这是简单选择**保持相对位置**的关键……但稳定性还是会因为最后那次 swap 被破坏(见下文)。
- **`if (min != i)`**: 这一句是优化,如果 min 还是 i(本趟没找到更小的),就不调用 swap,**省 3 次赋值**。零壹 PPT 没写这个判断,王道写了——养成习惯写上。

讲完代码,关键问题来了: 简单选择跟前面的冒泡、直接插入相比,在性能上有什么独特之处?

### 4. 性能分析(比较恒定 · 移动极少 · 不稳定)

**时间效率**:

| 项目 | 次数 | 说明 |
|---|---|---|
| 比较次数 | **n(n-1)/2**(恒定) | ⚠️ **与初始状态无关**!这是简单选择的标志性特征 |
| 移动次数(最好) | 0 | 序列已有序,每趟 min==i,跳过 swap |
| 移动次数(最坏) | ≤ **3(n-1)** | 每趟最多 swap 一次 = 3 次赋值,共 n-1 趟 |
| 时间复杂度 | **O(n²)** | 由比较次数主导 |

⚠️ **这是 408 选择题的高频考点**: 简单选择的比较次数**与序列初态无关**,始终是 n(n-1)/2 次。这是因为内层 for 必须扫完整段无序区才能确定最小值——不能像冒泡那样"看序列是否有序就提前结束"。

⚠️ **移动次数 ≤ 3(n-1) 而不是 3n(n-1)/2**: 简单选择每趟**最多只 swap 一次**(冒泡每趟可能 swap n-1 次),所以移动总数远小于冒泡。这也是简单选择"虽然慢但不冤"的优点——比较多但移动少。

**空间效率**: O(1)。

**稳定性**: ⚠️ **不稳定**!

> **王道反例**: L = {2, 2̄, 1}。第 1 趟从 L[0..2] 选最小=1(下标 2),与 L[0]=2 交换 → {1, 2̄, 2}。两个 2 的相对顺序发生了变化。

**零壹 PPT 反例 {4, 4̄, 2}** 同理:第 1 趟选最小 2 与第一个 4 交换,4̄ 跑到了 4 前面。

**适用性**: 顺序存储和**链式存储**都适用(只是找最小后做交换,链表也能实现)。

**📦 简单选择收束小结**: 朴素三步法(挑最小→记位置→换到前面),比较次数恒定 n(n-1)/2(与初态无关 → 408 考点),移动次数 ≤3(n-1)(比冒泡少得多),不稳定 + 顺序+链式都行。它的"简单"是优点也是缺点:实现简单,但没法用初态加速 → 这就是为什么我们需要堆排序。

## 三、堆的概念与性质

简单选择最大的瓶颈是**每趟都要 O(n) 扫描找最值**——明明前 i 趟做的工作里,大量的"谁比谁大"信息被丢弃了!如果能把这些比较结果**记录在某个数据结构里**,下一趟直接复用,效率就能跃升。

这个数据结构就是——**堆**。

### 1. 堆的定义(数学版)

> **堆的定义**(王道原话): n 个关键字序列 L[1..n] 称为堆,当且仅当该序列满足:  
> ① L(i) ≥ L(2i) 且 L(i) ≥ L(2i+1) → **大根堆**(大顶堆)  
> ② L(i) ≤ L(2i) 且 L(i) ≤ L(2i+1) → **小根堆**(小顶堆)  
> 其中 1 ≤ i ≤ ⌊n/2⌋

注意下标从 **1 开始**(王道惯例),这个细节后面写代码时要谨慎处理。

### 2. 大小根堆 + 完全二叉树形式

把数组按层序遍历"摆"到完全二叉树上,堆就有了几何形态:

```
小根堆: A = [1, 4, 7, 5, 6, 8, 9]      大根堆: A = [9, 7, 4, 6, 5, 3, 2]
            1                                   9
          /   \                              /     \
         4     7                            7       4
        / \   / \                          / \     / \
       5   6 8   9                        6   5   3   2

性质:根节点最小(全局最小)              性质:根节点最大(全局最大)
```

**核心观察**(零壹 PPT 第 14 页原话):
- **小根堆**构建的完全二叉树,**根节点比左右子树小**。
- **大根堆**构建的完全二叉树,**根节点比左右子树大**。

🎯 **【命题追踪】2020 年考点**: 堆的性质与特点(选择题 04: "下列哪个是一个堆"——必须把序列在脑中翻成完全二叉树检查)。

### 3. 堆的四条核心性质

**跨章关联**: 还记得第 5 章学的完全二叉树和层序遍历吗?堆**就是**完全二叉树的数组层序存储 → 第 5 章 · 完全二叉树性质。

| 性质 | 内容 |
|---|---|
| **存储方式** | 用数组下标 1 开始层序存储,逻辑上是完全二叉树 |
| **父子下标关系** | 节点 i 的左孩子 = 2i,右孩子 = 2i+1,父节点 = ⌊i/2⌋ |
| **最值在堆顶** | 大根堆 A[1] = 最大值;小根堆 A[1] = 最小值 |
| **叶子分布** | i > ⌊n/2⌋ 的节点都是叶子(没有孩子) |

⚠️ **2020 真题原话改编**: "在含有 n 元素的小根堆中,关键字最大的元素**可能存在于哪个位置**?"  
→ 答: 因为最大元素一定不在小根堆的内部节点(否则违反堆性质),只能在**叶子**中。叶子从 ⌊n/2⌋+1 起到 n,所以选项中通常给 n、n/2 之类,正确答案是位于叶子中的那个。

### 4. 升序大根堆 / 降序小根堆(反直觉)

⚠️ **这是 408 最爱挖的坑**(零壹 PPT 第 49 页用红字强调):

> **对 n 个元素进行升序排列,要使用大根堆!**  
> **对 n 个元素进行降序排列,要使用小根堆!**

**为什么反着?** 因为堆排序的输出方式是"**反复把堆顶元素扔到堆尾**":
- 大根堆:每趟把"最大值"扔到末尾 → 末尾从右往左累积:max, 次大, ..., min → 最终**升序** ✅
- 小根堆:每趟把"最小值"扔到末尾 → 末尾累积:min, 次小, ..., max → 最终**降序** ✅

记不住就背口诀:**升序大、降序小,记反就完蛋**。

**📦 堆概念收束小结**: 堆 = 完全二叉树 + 数组层序存储,大根堆顶最大、小根堆顶最小,父子下标 (i, 2i, 2i+1)。**升序用大堆、降序用小堆**这条反直觉规则务必背熟。

## 四、堆的两个基本操作:插入 + 删除

讲完静态结构,接下来是动态操作。**插入**和**删除**(实际上是"删堆顶")是堆的两个基本动作,也是堆排序的零件。这两个操作的核心都是**调整**——每次操作都可能破坏堆结构,需要"沿一条路径修复"。

### 1. 插入元素(向上调整 · shift-up)

🎯 **【命题追踪】2009、2011 年考点**: 堆的插入操作及调整操作分析。

**思想**: 新元素先放到堆底末尾(数组末尾),然后**沿单侧路径向上**与父节点比较,如果违反堆性质就上浮,直到不违反或到达根。

**零壹 PPT 4 步推演**(原大根堆 9-7-6-5-2-4-3,插入 10):

```
步骤 0(原大根堆):                  步骤 1(放堆底):
        9                                 9
       / \                              /   \
      7   6                            7     6
     / \ / \                          / \   / \
    5  2 4  3                        5   2 4   3
                                    /
                                   10  ← 新元素先置于堆底

步骤 2(10 与父 5 比较 → 10>5,上浮):  步骤 3(10 与父 7 比较 → 10>7,上浮):
        9                                 9
       / \                              /   \
      7   6                           10     6
     / \ / \                         / \    / \
    10  2 4  3                      7   2  4   3
   /                               /
  5                               5

步骤 4(10 与父 9 比较 → 10>9,上浮 → 调整完成):
        10
       /  \
      9    6
     / \  / \
    7   2 4  3
   /
  5
```

⚠️ **关键细节**(零壹 PPT 红框反复强调): **不需要与兄弟节点比较**!插入时只跟**父节点**比,因为父节点已经是局部最值,只要新元素比父大(在大根堆中),就该上浮 → 父节点和它兄弟的关系不会被破坏。

**插入代价**: 沿一条根到叶的路径走,**最多 ⌊log₂n⌋ 次比较与交换**,时间复杂度 **O(log₂n)**。

讲完插入,自然要问:插入只跟父比一边路径,删除呢?

### 2. 删除元素(向下调整 · shift-down)

🎯 **【命题追踪】2015、2024 年考点**: 堆的删除操作及调整操作分析。

⚠️ "删除"在堆里特指**删除堆顶**(最值),不是任意位置删除——因为堆排序需要的就是反复输出最值。

**思想**: 把**堆底**最后一个元素**移到堆顶**(覆盖掉被删除的根),然后**自顶向下**比较两个孩子,选**较大者**(大根堆)交换,直到不违反或到达叶子。

**零壹 PPT 4 步推演**(原大根堆 9-7-6-5-2-4,删除 9):

```
步骤 0(原大根堆):                  步骤 1(堆底 3 移到堆顶):
        9                                 3
       / \                              /   \
      7   6                            7     6
     / \ / \                          / \   /
    5  2 4  3                        5   2 4
    
步骤 2(3 vs 两孩子 7,6 → 7 更大,3↔7):  步骤 3(3 vs 两孩子 5,2 → 5 更大,3↔5):
        7                                 7
       / \                              /   \
      3   6                            5     6
     / \ /                            / \   /
    5  2 4                           3   2 4

调整完成。
```

⚠️ **关键细节**(零壹 PPT 红框): 删除调整时**两个孩子节点都要比**(选较大者交换),这跟插入只比父亲不同。

**删除代价**: 同样沿一条路径走,**O(log₂n)**。

**📦 插入删除收束小结**: 插入 = 放堆底 + 向上单侧调整(只比父);删除 = 堆底搬到堆顶 + 向下两边调整(选较大孩子)。两者都 O(log₂n)。**沿单侧树高调整**是这两个操作的共性,也是堆排序之所以能 O(nlog₂n) 的根源。

## 五、堆排序的核心:HeapAdjust(向下筛选)

讲完插入、删除两个零件,现在拼装真正的引擎——**HeapAdjust**(向下筛选算法)。它是堆排序中**所有调整操作**的核心,既用于建堆,也用于堆排序主循环。

### 1. 思想:沿子树高度向下筛

> **HeapAdjust(A, k, n)**: 对以 k 为根的子树进行筛选,使其满足大根堆性质。沿 key 较大的孩子向下走,直到当前节点比两个孩子都大,或者走到叶子。

注意:**HeapAdjust 假设左右子树都已经是大根堆,只有根节点 k 可能违反性质**。这是它能 O(log₂n) 完成调整的前提。

### 2. 王道版代码(逐行注释)

```c
void HeapAdjust(ElemType A[], int k, int n) {
    A[0] = A[k];                            // A[0] 暂存待筛选元素(哨兵,不是数据!)
    for (int i = 2*k; i <= n; i *= 2) {     // 沿 key 较大的子节点向下筛选
        if (i < n && A[i] < A[i+1])          // 取 key 较大的孩子下标
            i++;                             // i 现在指向较大那个孩子
        if (A[0] >= A[i])                    // 待筛元素 ≥ 较大孩子 → 已就位
            break;
        else {
            A[k] = A[i];                     // 较大孩子上提到双亲位置
            k = i;                           // k 跟着下移,继续向下筛
        }
    }
    A[k] = A[0];                             // 待筛元素放到最终位置
}
```

### 3. 关键细节(每一处都是考点)

⚠️ **`A[0] = A[k]` 暂存**: A[0] 不存储有效数据(数组下标从 1 开始),它充当"哨兵"——既保存待筛元素,也避免反复 swap 三赋值。这个技巧跟直接插入排序的 A[0] 哨兵思路一致。

**跨章关联**: A[0] 暂存技巧 → 第 8.2 节·直接插入排序的哨兵 A[0]。

⚠️ **`i *= 2` 不是 `i++`**: 这是树形结构的精髓!`i*=2` 代表"走到左孩子",而 `i++` 是"走到右孩子"——两者结合就是"先去左孩子,如果右孩子更大再 i++ 移过去"。

⚠️ **`if (i < n && A[i] < A[i+1]) i++`**: `i < n` 防止越界(右孩子可能不存在),`A[i] < A[i+1]` 决定是否切到右孩子。这一行同时处理"取较大孩子"和"边界检查",写错任意一处都会导致错误。

⚠️ **`if (A[0] >= A[i]) break`**: ⚠️ **大于等于**!如果写成 `>`,等于时不 break 会多余下沉,不影响正确性但浪费时间;但反过来 `<=` 写成 `<` 会出错——总之记成 `>=` 最安全。

⚠️ **结尾 `A[k] = A[0]`**: 暂存的待筛元素放到循环结束时 k 指向的位置(而不是原位)——这就是为什么 break 时不需要交换、只需要把暂存的搬过去。

**📦 HeapAdjust 收束小结**: 假设左右子树已是堆,只有根可能违反 → 沿较大孩子向下筛 → A[0] 暂存避免反复 swap → 时间 O(log₂n) = O(树高)。这一段代码是堆排序的灵魂,务必能默写。

## 六、建堆:Floyd 法 O(n)

讲完 HeapAdjust 这个零件,我们用它来组装第一台机器:**建堆**。

### 1. 为什么从 ⌊n/2⌋ 开始?

🎯 **【命题追踪】2018、2021 年考点**: 初始建堆的操作。

> **王道原话**: 建堆思路是从后往前检查所有分支结点,看是否满足堆的要求,若不满足,则对以该分支结点为根的子树进行调整。n 个结点的完全二叉树,最后一个结点是第 ⌊n/2⌋ 个结点的孩子。对以第 ⌊n/2⌋ 个结点为根的子树筛选……向前依次对各结点(⌊n/2⌋-1 ~ 1)为根的子树进行筛选。

**为什么从 ⌊n/2⌋ 开始而不是从 1 或 n?**
- 从 1 开始: 根的左右子树还不是堆,不满足 HeapAdjust 的前提,会出错。
- 从 n 开始: ⌊n/2⌋+1 ~ n 都是叶子,**叶子本身就是堆**(只有自己一个节点),无需调整 → 浪费时间。
- 从 ⌊n/2⌋ 开始: 第一个**非叶子节点**,从这里逆序往前调,每次调整时左右子树都已是堆,前提满足 ✅。

### 2. 王道图 8.6 五步建堆推演

序列: `(53, 17, 78, 9, 45, 65, 87, 32)`,n=8,⌊n/2⌋=4。

### 3. Floyd 建堆 O(n) 的直觉

⚠️ 反直觉点: 看起来是 n/2 次 O(log₂n) 的调整,应该是 O(nlog₂n)?为什么是 O(n)?

> **王道原话**: 调整的时间与树高有关,为 O(h)。在建含 n 个元素的堆时,关键字的比较总次数不超过 4n,时间复杂度为 O(n)。

**直觉解释**:越靠近底层,**节点数越多**(占整棵树一半节点是叶子)但**调整高度越短**(叶子调整高度为 0、倒数第二层为 1...)。求和后是等比级数,收敛到 O(n)。

### 4. 建堆代码

```c
void BuildMaxHeap(ElemType A[], int n) {
    for (int i = n/2; i > 0; i--)            // 从最后一个非叶子节点开始
        HeapAdjust(A, i, n);                 // 逆序调整每个分支节点
}
```

⚠️ **`i--` 不是 `i++`**: 必须**逆序**(从 n/2 倒着到 1),保证每次 HeapAdjust 时左右子树已是堆。

**📦 建堆收束小结**: Floyd 法从 ⌊n/2⌋ 起逆序调用 HeapAdjust,O(n) 时间建成大根堆。**起点 n/2、方向 i--、依赖 HeapAdjust 前提**——三个细节都是 408 命题点。

## 七、堆排序主流程

建堆是 0-to-1,堆排序是 1-to-100——**反复利用堆顶取最值**。

### 1. n-1 趟:堆顶↔堆尾 → 缩堆 → 重建

> **堆排序思路**(以大根堆为例,升序排序):  
> ① 将 n 个元素建成初始大根堆;  
> ② 把堆顶(最大值)与当前堆尾交换 → 最大值落到末尾的最终位置;  
> ③ 堆长度 -1,把"剩余 i-1 个元素"重新调整为大根堆;  
> ④ 重复 ②③ 共 n-1 次。

```
初始(假设已建大根堆): [9, 7, 6, 5, 2, 4, 3]
                       ↑
                    堆顶 9 = 最大值

第1趟: 堆顶 9 ↔ 堆尾 3 → [3, 7, 6, 5, 2, 4 | 9]
       9 已就位,缩堆为前 6 个,HeapAdjust(A, 1, 6) 重建大根堆 → [7, 5, 6, 3, 2, 4 | 9]

第2趟: 堆顶 7 ↔ 堆尾 4 → [4, 5, 6, 3, 2 | 7, 9]
       重建 → [6, 5, 4, 3, 2 | 7, 9]

第3趟: 堆顶 6 ↔ 堆尾 2 → [2, 5, 4, 3 | 6, 7, 9]
       重建 → [5, 3, 4, 2 | 6, 7, 9]

第4趟: 堆顶 5 ↔ 堆尾 2 → [2, 3, 4 | 5, 6, 7, 9]
       重建 → [4, 3, 2 | 5, 6, 7, 9]

第5趟: 堆顶 4 ↔ 堆尾 2 → [2, 3 | 4, 5, 6, 7, 9]
       重建 → [3, 2 | 4, 5, 6, 7, 9]

第6趟: 堆顶 3 ↔ 堆尾 2 → [2 | 3, 4, 5, 6, 7, 9]
       仅剩 1 个,无需调整。

最终升序: [2, 3, 4, 5, 6, 7, 9]
```

### 2. HeapSort 主算法代码

```c
void HeapSort(ElemType A[], int n) {
    BuildMaxHeap(A, n);                      // 一次性 O(n) 建初始大根堆
    for (int i = n; i > 1; i--) {            // n-1 趟交换 + 调整
        swap(A[i], A[1]);                    // 堆顶 ↔ 当前堆尾
        HeapAdjust(A, 1, i-1);               // 缩堆后重建,长度 i-1
    }
}
```

⚠️ **`for (int i = n; i > 1; i--)`**: 循环到 i=2 终止(i>1),不到 i=1。因为只剩一个元素时本身就是有序的,无需再处理。

⚠️ **`HeapAdjust(A, 1, i-1)` 的第三个参数**: 是 i-1 不是 n!每趟堆都在缩短,新的有效堆长度是 i-1。

### 3. 性能分析

🎯 **【命题追踪】2009、2011、2015、2018、2021、2024 年考点**: 堆排序复杂度计算。

| 项目 | 复杂度 | 说明 |
|---|---|---|
| 建堆 | **O(n)** | Floyd 法 |
| n-1 次调整 | (n-1) × O(log₂n) = **O(nlog₂n)** | 每趟 HeapAdjust 树高 |
| **总时间** | **O(nlog₂n)** | 建堆相比之下可忽略 |
| 空间 | **O(1)** | 仅常数辅助变量 |
| 稳定性 | **不稳定** | 见反例 |
| 适用性 | **仅顺序存储** | 链表无法 O(1) 访问父子节点 |

⚠️ **稳定性反例**(王道原话): L = {1, 2, 2̄}。建初始堆时可能把 **2̄** 交换到堆顶,此时 L = {2̄, 1, 2},最终排序序列为 L = {1, 2̄, 2},2 与 2̄ 的相对次序已发生变化 → 不稳定。

🎯 **【命题追踪】2022 年考点**: 堆在海量数据中选出最小 k 个数的应用。

**📦 堆排序收束小结**: 一次建堆 + n-1 次"换尾 + 调整",总时间 O(nlog₂n)、空间 O(1)、不稳定、仅顺序存储。**升序用大根堆、降序用小根堆**这条规则务必记住。

## 八、堆的应用:优先队列 + Top-k

⚠️ **2026 年大纲新增考点**: 王道明确写"最新 408 统考大纲增加了考点'堆的应用',堆的应用只有两个:**堆排序**和**优先队列**"。

### 1. 优先队列 (Priority Queue)

**优先队列**是一种特殊的队列,出队顺序按"优先级"而非 FIFO——优先级高的先出。**用堆实现优先队列是天作之合**:
- 大根堆 → 最大优先级队列(每次出队 = 删堆顶)
- 小根堆 → 最小优先级队列

**操作复杂度**: 入队 O(log₂n)(插入)、出队 O(log₂n)(删堆顶)。

**跨章关联**: 优先队列在算法里随处可见 → 第 7 章·图算法(Dijkstra 最短路径、Prim 最小生成树都用到优先队列)。

### 2. Top-k 海量数据问题(2022 真题)

**问题**: 在 1 亿个数中选出**最大的 100 个**(N=1 亿,k=100,N >> k)。

**朴素法**: 全排序 → O(N log₂N),数据量太大跑不动。

**堆解法**:
1. 用前 100 个数建一个**小根堆**(注意是小根堆,反着想!)
2. 依次读入剩余 N-100 个数:
   - 若 ≤ 堆顶 → 舍弃(不可能进 Top-100)
   - 若 > 堆顶 → 替换堆顶,**自顶向下**调整(O(log₂k))
3. 全部读完后,堆中 100 个数即为 Top-100。

**复杂度**: O(N log₂k),k=100 时 log₂k≈7,远小于 log₂N≈27。**空间 O(k)**,内存友好。

⚠️ **为什么取 Top-k 大用小根堆**?反着想:
- 小根堆顶是堆中**最小**的——它代表"当前 k 个数里的门槛"。
- 新元素只有比这个门槛大才有资格进堆,进堆后要踢掉门槛(堆顶)。
- **一句话**: 小根堆顶充当"最低准入线"。

**对偶规则**:
- 求 Top-k **大** → 用 **小**根堆
- 求 Top-k **小** → 用 **大**根堆
- 求第 k **小** → 用大小为 k 的**大**根堆,堆顶就是答案
- 求第 k **大** → 用大小为 k 的**小**根堆,堆顶就是答案

**📦 应用收束小结**: 优先队列(2026 大纲新增)= 堆的别名,Top-k 问题用 k 大小的相反堆 O(N log₂k) 解决。**反着想**(取大用小堆)是这类题的灵魂。

## 九、简单选择 vs 堆排序 全维度对比

讲完两兄弟,408 大题最爱让你做对比:

| 维度 | 简单选择排序 | 堆排序 |
|---|---|---|
| 思想 | 暴力扫描找最值 | 借助堆结构找最值 |
| 找最值成本 | O(n) | **O(log₂n)** |
| 时间(最好/平均/最坏) | 全部 **O(n²)** | 全部 **O(nlog₂n)** |
| 比较次数 | n(n-1)/2 **(与初态无关)** | 建堆 ≤4n + 排序 (n-1)O(log₂n) |
| 移动次数(最坏) | ≤ **3(n-1)** | n-1 次 swap + 调整中的赋值 |
| 空间 | O(1) | O(1) |
| 稳定性 | **不稳定** | **不稳定** |
| 适用结构 | 顺序 + **链式** | **仅顺序** |
| 408 考查重点 | 比较次数与初态无关、不稳定反例 | 建堆过程、调整过程、Top-k 应用 |

**易错点速查**(蓝箭头 = 反直觉):
- ⚠️ 简单选择**比较次数恒定**(与初态无关),不像冒泡能利用有序性
- ⚠️ 简单选择**移动次数 ≤3(n-1)**,远少于冒泡的 3n(n-1)/2
- ⚠️ 堆排序**升序用大根堆、降序用小根堆**(反着记)
- ⚠️ Floyd 建堆从 **⌊n/2⌋** 起逆序,**O(n) 不是 O(nlog₂n)**
- ⚠️ Top-k **大**用**小**根堆(反着记)
- ⚠️ HeapAdjust 中 **A[0] 是哨兵不是数据**
