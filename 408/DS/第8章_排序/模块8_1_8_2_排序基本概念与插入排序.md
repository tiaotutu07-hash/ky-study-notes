## 🗺️ 全景导图

排序这一章可以拆成两个问题:**"什么叫排好序"** 和 **"怎么把它排好"**。8.1 回答第一个——给出"稳定性""内部/外部""比较+移动"等共同语言,这些是后面**每一个**算法都要反复用到的尺子。8.2 回答第二个的第一种思路:**像你打牌时整理手牌那样**,左手已经有序的牌不动,右手抽一张,从右往左找位置塞进去——这就是**插入排序**的全部思想。围绕这一思想王道讲了三个变种:

- **直接插入**:最朴素的"边比边移";

## 一、排序的基本概念(王道 8.1)

### 1.1 排序到底在排什么?

很多人以为排序就是"把数字按大小排一排"——这话只对了一半。**排序排的是关键字(key),而不是数据本身**。比如学生成绩表里,你按"学号"排和按"姓名拼音"排,得到的两份表完全不同,但**数据记录本身**没变,变的只是它们按哪个字段对齐。

精确定义如下:

> **输入**:n 个记录 R₁, R₂, ..., Rₙ,对应的关键字为 k₁, k₂, ..., kₙ。
> **输出**:输入序列的一个**重排** R'₁, R'₂, ..., R'ₙ,使得 k'₁ ≤ k'₂ ≤ ... ≤ k'ₙ。

注意输出是"重排"而不是"新构造"——所有 R 还是那批 R,只是顺序变了。

> 📌 **本书惯例**:王道全书默认排序结果为**非递减(升序)序列**,后面所有代码、推演也都按升序写。

### 1.2 算法的稳定性 —— 一条容易栽跟头的定义

**定义**:若待排序表中存在两个元素 R_i 和 R_j,它们的关键字相同(key_i = key_j),且**排序前 R_i 在 R_j 前面**;若使用某排序算法排序后 **R_i 仍然在 R_j 前面**,则称该算法是**稳定的**;否则是**不稳定**的。

通俗说:**相等的元素,排序前后相对位置不变就是稳定**。

```
排序前:  ... 49 ... 49̄ ...    (49̄ 表示后出现的那个 49)
            ↑       ↑
            R_i     R_j

稳定排序: ... 49 49̄ ...   (R_i 仍在 R_j 前 ✓)
不稳定:   ... 49̄ 49 ...   (相对位置颠倒 ✗)
```

⚠️ **第一个大坑——稳定性 ≠ 算法优劣**。王道原文说得很死:"算法是否具有稳定性并不能衡量一个算法的优劣,它主要是对算法的性质进行描述。" 还记得第 7 章里我们怎么比较查找算法吗?那里看的是 ASL,这里看的是稳定性,但**评判一个排序好不好用的根本指标始终是时空复杂度**,稳定性只是某些应用场景(比如多关键字排序、按学号排好的表再按成绩二次排)才在乎的属性。

⚠️ **第二个坑——证明一个算法不稳定,只需举一个反例**。这是王道明确的注。比如希尔排序,你只要构造出 `[2, 4, 1, 2̄]` 让两个 2 颠倒过来,就证完了,不需要证"所有情况都不稳定"。

⚠️ **第三个坑——关键字唯一时,稳定性无所谓**。如果待排表里没有相等的关键字,排序结果是**唯一**的,那么稳不稳定根本不影响最终序列。这条结论会在某些选择题里用来排除选项。

**跨章关联**:稳定性这个概念会在本章后续每一节反复出现,后面我们会给出最终的稳定性总览表(冒泡稳、简单选择不稳、快排不稳、堆不稳、归并稳、基数稳),这里先记住"举反例就能证不稳定"这一招就够了。

### 1.3 内部排序 vs 外部排序

划分依据只有一句话:**数据是否全部驻留在内存中**。

| 类别 | 数据位置 | 关心什么 |
|---|---|---|
| **内部排序** | 排序期间所有元素**全部**在内存 | 比较次数 + 移动次数(决定时间) |
| **外部排序** | 元素无法全部装入内存,排序过程要在内、外存间不断搬运 | I/O 次数(决定时间) |

8.2–8.6 讲的全是**内部排序**(直接/折半/希尔/冒泡/快排/选择/堆/归并/基数),8.7 才是外部排序(多路归并、败者树、置换-选择)。本节属于内部排序。

> 📌 **王道注**:大多数内部排序算法**更适用于顺序存储的线性表**。原因很简单——很多算法需要按下标跨步访问(比如希尔的分组、堆排序的父子下标计算、折半插入的 mid),链表跨步访问代价是 O(n),会把算法直接拖垮。后面每一个算法我们都会单独标"适用存储"。

### 1.4 内部排序的两种核心操作

执行任何一个基于比较的内部排序时,都离不开两件事:

1. **比较**——确定两个关键字的前后关系;
2. **移动**——把元素挪到新位置。

时间复杂度本质上由**比较次数 + 移动次数**决定。后面每讲一个算法,都会分别分析这两个指标——比如折半插入排序为什么"比直接插入省"?省的就是**比较次数**(从 O(n²) 降到 O(nlog n)),而**移动次数没变**(还是 O(n²)),所以总复杂度依然是 O(n²)。这一点是 2012 真题原题考点,我们到 8.2.2 再细说。

⚠️ 并非所有内部排序都基于比较——**基数排序**就不基于比较,它基于"分配-收集"。这一点在 8.6 会展开。

**📦 8.1 收束小结**:8.1 给的全是"共同语言"——稳定性的定义、内/外部的划分、比较+移动的核心操作。这些不是用来直接做计算题的,而是后面**每一个**具体算法的判分依据。把"稳定性 ≠ 优劣""举反例证不稳定""内部排序更适用顺序存储"这三句话刻在脑子里,后面的章节就不会被概念绊倒。

## 二、插入排序总论(承上启下)

王道在 8.2 开篇给出了插入排序的统一思想:

> **基本思想**:每次将一个待排序的记录,按其关键字大小**插入到前面已排好序的子序列**中,直到全部记录插入完成。

这个统一骨架是这样的:

```
排序中某一时刻的数组状态:
┌──────────────────────────┬─────┬──────────────────────────┐
│ 有序子序列 L[0..i-1]    │ L[i]│ 无序序列 L[i+1..n-1]    │
└──────────────────────────┴─────┴──────────────────────────┘
                              ↑
                         待插入的元素
                              │
                              ▼
┌────────────┬─────┬──────────┬──────────────────────────┐
│ L[0..k-1] │ L[i]│L[k..i-1]│ 无序序列 L[i+1..n-1]    │
└────────────┴─────┴──────────┴──────────────────────────┘
                              已扩大的有序区
```

每一趟插入都做两件事:
- **a. 找位置**:在 L[0..i-1] 中找到 L[i] 应该插入的位置 k;
- **b. 腾位置**:把 L[k..i-1] 后移一位,把 L[i] 复制到 L[k]。

**正是因为**这两件事可以独立优化,才衍生出三个变种:
- 找位置用**顺序查找**+腾位置一边比一边移 → **直接插入排序**
- 找位置用**折半查找**(优化 a)+腾位置统一后移(分离 b) → **折半插入排序**
- 找位置不变,但**先把序列分组**让其"基本有序"再做直接插入 → **希尔排序**

下面分别展开。

## 三、直接插入排序

### 3.1 三步法 —— 把"打牌摸排"写成代码

回到那个比喻——**整理手牌**:左手已经按大小排好了几张,右手抽一张新牌,你下意识做的事就是:
1. **从右往左**比,找到该插入的位置;
2. **把比新牌大的那些**整体右移一位;
3. **把新牌**塞到空出来的位置。

王道把这个过程精确化为三步:

> a) 查找出 L[i] 在 L[0..i-1] 中的插入位置 k;
> b) 将 L[k..i-1] 中所有元素**依次后移一个位置**;
> c) 将 L[i] 复制到 L[k]。

注意第二步是"**从后往前**逐个往后挪",而不是一次性 memmove——因为我们正是在边比边挪。这正是直接插入排序"边比较边移动"的特征。

### 3.2 哨兵代码 —— A[0] 一身二职

```c
void InsertSort(ElemType A[], int n) {
    int i, j;
    // 数组下标从 1 开始用,A[1] 视为已排好的子序列(初始有序区)
    for (i = 2; i <= n; i++) {                // 依次将 A[2]..A[n] 插入前面已排好的序列
        if (A[i] < A[i-1]) {                  // ⚠️ 是否需要往前比较的快速判断
            A[0] = A[i];                      // 哨兵:A[0] 暂存待插入元素,不存放真实数据
            for (j = i - 1; A[0] < A[j]; j--) // 从后往前查找插入位置
                A[j+1] = A[j];                // 比 A[0] 大的元素统统后移一位
            A[j+1] = A[0];                    // 把待插入元素复制到正确位置
        }
    }
}
```

**为什么要用 A[0] 当哨兵?** 它干了两件事:
1. **暂存**待插入的值(避免被后移的元素覆盖);
2. **充当循环边界**——内层 `for` 的判断条件是 `A[0] < A[j]`,当 j 一路减到 0 时,因为 A[0] = 待插值,自身比较自然为假,循环自动终止,**不需要单独写 `j >= 1` 的边界判断**,代码更简洁、还能少一次比较。

**跨章关联**:这种"哨兵"技巧在第 7 章 7.2 顺序查找里见过——把待查值放到 A[0],查找循环就不用判越界。同样的招数,这里又用了一次。

⚠️ **大坑**:数组从 **A[1]** 开始用,**A[0] 不存真实数据**。所以输入是 n 个元素时,数组开 A[0..n] 共 n+1 个空间,A[0] 是哨兵位。考试时如果题目要你模拟代码运行,千万不要把 A[0] 当成第一个待排元素。

⚠️ **大坑**:if(A[i] < A[i-1]) 这个外层判断**不是为了节省移动**,而是为了"如果 A[i] 已经比前驱大,就跳过整段插入逻辑"——这正是直接插入"输入敏感"的根源:序列越接近有序,这个 if 命中得越多,整个排序越快。

### 3.3 手动推演 —— 跑一遍王道原例

待排序列(8 个数,带下划线的 49̄ 是和前面的 49 相等的"重复元素",用来观察稳定性):

```
初始:  [49]   38   65   97   76   13   27   49̄
        └─有序区─┘   └────────无序区────────┘
```

每趟把绿色元素插进左边的方括号里。**带括号的部分始终是已排好的子序列**:

```
i=2:  (38   49)   65   97   76   13   27   49̄    ← 38 插到 49 前
i=3:  (38   49   65)   97   76   13   27   49̄    ← 65 已比 49 大,直接进
i=4:  (38   49   65   97)   76   13   27   49̄    ← 97 已比 65 大,直接进
i=5:  (38   49   65   76   97)   13   27   49̄    ← 76 插到 97 前
i=6:  (13   38   49   65   76   97)   27   49̄    ← 13 一路退到最左
i=7:  (13   27   38   49   65   76   97)   49̄    ← 27 退到 13 后面
i=8:  (13   27   38   49   49̄   65   76   97)    ← 49̄ 插到 49 后面 ⭐
```

**重点观察 i=8 这一步**:49̄ 在原序列里就在 49 后面,排序完成后**仍然在 49 后面**——这是因为内层循环用的是 `A[0] < A[j]`(严格小于),当 A[0] 等于 A[j] 时循环停止,49̄ 不会越过 49 跑到前面去。这就是直接插入排序**稳定**的图形化证明。

⚠️ **如果把内层条件改成 `A[0] <= A[j]`**(加了等号),49̄ 就会越过 49 跑到前面,算法立刻变成不稳定。出题人喜欢在这种"一行代码改稳定性"的细节上做文章。

### 3.4 性能分析

| 维度 | 结论 | 说明 |
|---|---|---|
| **空间复杂度** | **O(1)** | 仅用 A[0] + 几个循环变量,常数辅助空间 |
| **最好时间** | **O(n)** | 序列已升序,每个 A[i] 跟 A[i-1] 比一次就跳过(if 条件不成立),0 次移动 |
| **最坏时间** | **O(n²)** | 序列逆序,第 i 趟比 i-1 次、移 i+1 次 |
| **平均时间** | **O(n²)** | 比较次数和移动次数均约 n²/4 |
| **稳定性** | **稳定** | 内层用严格小于 `<`,相等元素不越过 |
| **适用存储** | **顺序 + 链式** | 链式存储时**无须移动元素**,但比较次数不变 |

⚠️ **王道明确指出**:直接插入排序的时间效率**取决于待排序表的初始状态**——这就是 PPT 里说的"**输入敏感**"。最好情况 O(n) 和最坏情况 O(n²) 差了整整一个数量级,这是直接插入区别于"无脑 O(n²)"算法(比如简单选择排序)的关键特征。

**📦 直接插入收束小结**:三步法记骨架(查位置→腾位置→塞进去),哨兵记代码细节(A[0] 双重作用、A[1] 起步),"输入敏感 + 稳定 + 链表也行"是它跟另外两兄弟最大的区别。

## 四、折半插入排序

### 4.1 改进的动机 —— 把两件事拆开

讲完直接插入,我们回头看王道总结的两步:① 找插入位置 ② 移动元素。直接插入是**边比边移**,两步揉在一个内层循环里。但仔细想想——既然有序区已经排好了,**找位置这件事完全可以用折半查找做啊**!从 O(i) 降到 O(log i)。**这就是折半插入排序**。

**跨章关联**:折半查找的全部细节在第 7 章 7.2.2 已经吃透——`while(low<=high)` 循环、`mid=(low+high)/2` 取中点、根据比较结果调整 low/high。这里直接拿来用,核心算法**一字不改**,只是把"等于就返回找到"改成"等于就继续往右找"以保证稳定性。

### 4.2 关键代码 —— 一个等号定生死

```c
void InsertSort(ElemType A[], int n) {
    int i, j, low, high, mid;
    for (i = 2; i <= n; i++) {
        A[0] = A[i];                          // 暂存(此处不是真"哨兵",只是缓存)
        low = 1;  high = i - 1;               // 折半查找的范围:已排序区
        while (low <= high) {                 // ⚠️ 折半查找(默认升序)
            mid = (low + high) / 2;           // 取中点
            if (A[mid] > A[0])  high = mid - 1;  // 中间值大,往左半找
            else                 low  = mid + 1;  // ⚠️ 中间值 ≤ A[0],往右半找(含等号!)
        }
        // 循环结束时 high+1 就是插入位置
        for (j = i - 1; j >= high + 1; --j)   // 统一后移
            A[j+1] = A[j];
        A[high+1] = A[0];                     // 插入
    }
}
```

⚠️ **这个 else 分支为什么必须含等号?** —— 关键到极致。如果遇到 `A[mid] == A[0]` 时让 high=mid-1,那 A[0] 会被插到与它相等的元素**前面**,稳定性立刻就被破坏。代码里 `if(A[mid] > A[0])` 严格大于走左,**等于走右**,保证待插值始终插到所有相等元素的最右边——这才是折半插入**稳定**的代码层面原因。

⚠️ 注意:这里的 A[0] **不是真正意义的哨兵**(它没有用作循环边界),只是一个**暂存单元**(防止被后移的元素覆盖)。希尔排序的 A[0] 也是同样性质。"哨兵"这个词严格来说只属于直接插入。

⚠️ **循环结束时 low > high,插入位置是 high+1**,不是 mid、不是 low。这一点考代码题时容易写错。

### 4.3 手动推演 —— 跟着零壹 PPT 跑 i=1→5

序列:`[2, 5, 1, 3, 6, 4]`(下标 1..6)。设 i=1 时 A[1]=2 已是有序区。

#### 第一趟 i=2(待插 5):
```
[ 2 ]  5   1   3   6   4
  ↑
 low=1, high=1
 mid=1, A[1]=2 < 5(A[0]),走 low=mid+1=2
 → low=2 > high=1,循环结束
 → 插入位置 = high+1 = 2,即 5 留在原位
结果:[ 2   5 ]  1   3   6   4
```

#### 第二趟 i=3(待插 1):
```
[ 2   5 ]  1   3   6   4
  ↑   ↑
 low=1 high=2
 mid=1, A[1]=2 > 1,high=0
 → low=1 > high=0,结束
 → 插入位置 = high+1 = 1
 → A[1..2] 后移一位,1 插到 A[1]
结果:[ 1   2   5 ]  3   6   4
```

#### 第三趟 i=4(待插 3):
```
[ 1   2   5 ]  3   6   4
 mid=2, A[2]=2 < 3,low=3
 mid=3, A[3]=5 > 3,high=2
 → 插入位置 = 3
 → A[3] 后移,3 插到 A[3]
结果:[ 1   2   3   5 ]  6   4
```

#### 第四趟 i=5(待插 6):
```
[ 1   2   3   5 ]  6   4
 mid=2, A[2]=2 < 6,low=3
 mid=3, A[3]=5 < 6,low=4
 mid=4, A[4]=5 < 6,low=5
 → 插入位置 = 5,6 留在原位
结果:[ 1   2   3   5   6 ]  4
```

#### 第五趟 i=6(待插 4):
```
[ 1   2   3   5   6 ]  4
 mid=3, A[3]=3 < 4,low=4
 mid=4, A[4]=5 > 4,high=3
 mid=4? high=3,low=4 → 循环结束
 → 插入位置 = high+1 = 4
 → A[4..5] 后移,4 插到 A[4]
结果:[ 1   2   3   4   5   6 ]  ✓
```

### 4.4 性能分析 —— 一个被低估的"伪改进"

| 维度 | 折半插入 | 对比直接插入 |
|---|---|---|
| **空间复杂度** | O(1) | 一样 |
| **比较次数** | **O(n log₂n)**,**与初态无关** | 直接插入比较次数取决于初态(O(n) ~ O(n²)) |
| **移动次数** | O(n²),取决于初态 | 一样 |
| **总时间复杂度** | **O(n²)** | 一样 |
| **稳定性** | **稳定** | 一样 |
| **适用存储** | **仅顺序存储** | 直接插入两种都行 |

⚠️ **2012 年 408 真题考点(命题追踪)**:折半插入排序与直接插入排序相比,**减少的是比较次数,而不是移动次数**;且**比较次数与初始状态无关**。这两句话是高频选择题原话。

⚠️ **为什么折半插入只能用顺序存储?** —— 因为折半查找需要 `mid = (low+high)/2` 然后 O(1) 跳过去,链表做不到这一点。所以折半查找本身就要求顺序存储,折半插入自然继承这个限制。

**为什么改进了比较次数,总复杂度还是 O(n²)?** ——因为时间瓶颈被**移动次数**主导。折半插入只优化了"找位置"那一半,"挪位置"那一半完全没动。**正是因为**移动是 O(n²),所以哪怕比较降到 O(n log n),总时间还是 O(n²)。这是一个非常典型的"瓶颈分析"思路,跨章关联到时间复杂度分析的一般原则。

**📦 折半插入收束小结**:它的核心价值不是"更快",而是"**比较次数稳定且较少**",对**数据量不大**的表(比如 n 几百)有奇效。三大特征记住:① 仅减少比较 ② 比较次数与初态无关 ③ 仅顺序存储。

## 五、希尔排序(缩小增量排序)

### 5.1 思想引入 —— 钻直接插入的最大空子

还记得直接插入排序的最好情况是什么吗?**O(n) —— 当序列已经基本有序**。希尔(Donald Shell,1959)就盯上了这个性质:既然直接插入对"基本有序"特别友好,那我**先想办法让序列变得基本有序**,最后一趟再做直接插入收尾,不就快了?

怎么"让序列基本有序"?希尔的天才之处在于**分组**——把相距某个步长 d 的元素放进同一个子表,**只在子表内部做直接插入**。第一次步长大(比如 d=4),每个子表只有 4–5 个元素,排起来飞快;然后步长减半 d=2,子表更长但已"局部有序";最后 d=1,整张表做一次直接插入,此时整张表已经非常接近有序了——直接插入跑成最好情况 O(n)。

> **王道原话**:希尔排序的基本思想是,先将待排序表分割成若干形如 L[i, i+d, i+2d, ..., i+kd] 的"特殊"子表(即把相隔某个"增量"的记录组成一个子表),对各个子表分别进行直接插入排序,当整个表中的元素已呈"基本有序"时,再对全体记录进行一次直接插入排序。

这个思路在算法史上有个外号叫"**缩小增量排序**"(diminishing increment sort)。

### 5.2 分组规则与增量序列

**操作流程**:
1. 先取一个小于 n 的步长 d₁,把表的元素按"距离为 d₁"分成 d₁ 组——所有距离为 d₁ 的倍数的元素放在同一组;
2. **同一组内**做直接插入排序;
3. 然后取第二个步长 d₂ < d₁,重复上述过程;
4. 直到所取 d=1,即所有元素在同一组,再做最后一次直接插入。

王道给的常用增量是 **d₁ = ⌊n/2⌋,d_(i+1) = ⌊d_i / 2⌋**,直到 d=1。比如 n=20,增量序列就是 10, 5, 2, 1。零壹 PPT 用的例子 n=19,取了 4, 2, 1 三个增量(注意零壹这里没用 n/2 而是直接给了 4——增量序列**没有统一规定**,这是希尔排序至今**没有最优解**的数学未解决问题,王道明示)。

⚠️ **大坑**:增量序列不止一种,常考的就两种:① 王道默认的 n/2 折半法;② 题目自己给一组(比如"d₁=5, d₂=3, d₃=1")。**做题时严格按题目给定的增量序列来**,别擅自换 n/2。

### 5.3 代码 —— 把直接插入的 1 换成 dk

```c
void ShellSort(ElemType A[], int n) {
    // ⚠️ A[0] 只是暂存单元,不是哨兵——当 j<=0 时,插入位置已到
    int dk, i, j;
    for (dk = n/2; dk >= 1; dk = dk/2)               // 增量变化(无统一规定)
        for (i = dk + 1; i <= n; ++i)                // 依次扫描每个分组的待插元素
            if (A[i] < A[i-dk]) {                    // 需要将 A[i] 插入有序增量子表
                A[0] = A[i];                         // 暂存到 A[0]
                for (j = i - dk; j > 0 && A[0] < A[j]; j -= dk)
                    A[j+dk] = A[j];                  // 同组内向后移一个 dk 位置
                A[j+dk] = A[0];                      // 插入到正确位置
            }
}
```

把它和直接插入的代码并排看,你会发现**结构几乎一模一样**——只是把所有的"1"换成了"dk":
- 步长从 +1/-1 变成 +dk/-dk
- 起始 i 从 2 变成 dk+1

这就是希尔为什么叫"插入排序"族的原因——它的内核就是直接插入,只是"距离"变了。

⚠️ **A[0] 在希尔里不是真哨兵**——内层循环条件是 `j > 0 && A[0] < A[j]`,显式判断了 j>0,所以 A[0] 只起暂存作用。这一点和折半插入一致,和直接插入(真哨兵)不一致。

### 5.4 手动推演 —— 跟着零壹 PPT 跑

序列(19 个数):
```
1  3  5  8  11  31  27  16  92  24  45  10  20  91  22  58  65  23  50  18
```
增量序列:**d₁=4, d₂=2, d₃=1**

#### 第一趟 d₁=4

把所有距离为 4 的元素分到一组,共 4 组(用颜色或下标区分):

```
组 0:  1   11  92  20  22  50            (下标 1, 5, 9, 13, 17)
组 1:  3   31  24  91  58  18            (下标 2, 6, 10, 14, 18)
组 2:  5   27  45  22  65                (下标 3, 7, 11, 15, 19)
组 3:  8   16  10  58  23                (下标 4, 8, 12, 16, 20)
```

每组内部做直接插入排序(组内已经基本有序的就基本不动)。结果合并回原序列:

```
1  3  5  8  11  23  22  10  20  24  27  16  65  31  45  18  92  91  50  58
```

(零壹 PPT 给出的答案是 `1 3 5 8 11 23 22 10 20 24 27 16 65 31 45 18 92 91 50 58`)

#### 第二趟 d₂=2

距离为 2 的元素分成 2 组,组内做直接插入。结果:

```
1  3  5  8  11  10  20  16  22  18  27  23  45  24  50  31  65  58  92  91
```

#### 第三趟 d₃=1

整张表做一次直接插入排序——此时表已经"基本有序",直接插入跑得飞快。最终结果:

```
1  3  5  8  10  11  16  18  20  22  23  24  27  31  45  50  58  65  91  92
```

⚠️ **观察重点**:第一趟中,假设原序列里有两个相同的关键字(比如下标 5 和 11 都是 22),它们会被分到**不同的组**(因为 11-5=6 ≠ 4),组内独立排序后,它们之间的相对位置就**很可能颠倒**——这就是希尔**不稳定**的根源。**用一个反例就能证明不稳定**(回想 8.1 那条规则)。

⚠️ **2014、2018 真题原题**(命题追踪):给出希尔排序某一**中间状态**,反推**当前用的增量是多少**或**是第几趟之后**。判断方法是:看每隔 d 的元素是否构成有序子序列——若是,则当前增量就是 d。

⚠️ **2015 真题**(命题追踪):希尔排序的各子序列中,采用的是**直接插入排序**(不是折半,也不是别的)——这是定义里写死的。

### 5.5 性能分析

| 维度 | 结论 | 备注 |
|---|---|---|
| **空间复杂度** | **O(1)** | 几个循环变量 + A[0] 暂存 |
| **时间复杂度(常用)** | 约 **O(n^1.3)** | 当 n 在某特定范围时 |
| **时间复杂度(最坏)** | **O(n²)** | 增量取得不好时 |
| **时间复杂度(平均)** | 数学未解决 | 王道:**一般不考查希尔的时间复杂度** |
| **稳定性** | **不稳定** | 同 key 元素可能被分到不同组而交换相对位置 |
| **适用存储** | **仅顺序存储** | 需要按下标跨步访问 |

⚠️ **零壹讲义重点提示**:希尔排序的性能分析比较复杂,**不需要掌握具体推导**,只需要知道它是不稳定算法、空间 O(1)、最坏 O(n²)、最好约 O(n^1.3) 即可。

⚠️ **希尔三大特征同时成立**:① 不稳定 ② 仅顺序存储 ③ 突破 O(n²)(平均情况)。同时具备这三条的,在 8.2 这一节里**只有希尔**——直接插入和折半插入都稳定,直接插入还能用链表。

**📦 希尔收束小结**:希尔利用了"直接插入对基本有序极快"这个性质,通过"分组+缩小增量"让序列**逐渐变得基本有序**。它是 8.2 三兄弟里唯一打破 O(n²)(平均)、唯一不稳定、唯一只能顺序存储的"叛徒",但也是性能最好的一个。

## 六、三种插入排序终极对比

### 6.1 全方位对照表

| 维度 | 直接插入排序 | 折半插入排序 | 希尔排序 |
|---|---|---|---|
| **基本思想** | 边比边移 | 折半找位置,统一后移 | 分组+缩小增量,组内直接插入 |
| **空间复杂度** | O(1) | O(1) | O(1) |
| **最好时间** | **O(n)**(已序) | O(n log n)(比较)+ O(n)(移动) ≈ O(n) | 约 O(n^1.3) |
| **最坏时间** | O(n²) | O(n²) | O(n²) |
| **平均时间** | O(n²) | O(n²) | 约 **O(n^1.3)** |
| **比较次数** | 取决于初态 | **与初态无关**(O(n log n)) | 取决于增量序列 |
| **移动次数** | 取决于初态 | 取决于初态 | 取决于增量序列 |
| **稳定性** | **✅ 稳定** | **✅ 稳定** | **❌ 不稳定** |
| **适用存储** | **顺序 + 链式** | **仅顺序** | **仅顺序** |
| **典型考点** | 哨兵、稳定性证明 | 减少比较不减少移动(2012) | 反推增量(2014/2018)、子序列用直接插入(2015) |

### 6.3 选择策略(给做题人的建议)

| 题目情景 | 推荐答案 |
|---|---|
| n 较小、对稳定性有要求 | 直接插入或折半插入 |
| 链表存储 | **只能用直接插入**(其它两个都不行) |
| n 中等、要求快、不在乎稳定 | 希尔排序 |
| 待排表"基本有序" | 直接插入(O(n))或希尔最后一趟 |
| 题目给了"中间状态"问算法 | 看每 d 间隔是否有序 → 希尔 |
