# 板块一 · 题型分类与解题方法

## 【题型1】顺序查找的适用性与等概率 ASL 分析（选01、选02、选03、综合01）

### 核心知识

| 项目 | 顺序存储 | 链式存储 | 有序单链表 |
|------|---------|---------|----------|
| **顺序查找是否适用** | ✅ | ✅ | ✅ |
| **成功 ASL（等概率）** | (n+1)/2 | (n+1)/2 | (n+1)/2 |
| **失败 ASL · 无序表** | n+1 | n+1 | —— |
| **失败 ASL · 有序表** | n/2 + n/(n+1) ≈ n/2 | —— | n/2 + n/(n+1) ≈ n/2 |

**关键性质**：
- 顺序查找**对存储结构无要求**（顺序/链式都行），只要能"从一端向另一端逐个比较"即可
- 顺序查找成功 ASL **与表是否有序无关**，只与元素位置有关
- 有序表查找**失败**时可提前终止（关键字大于待查值即停），**失败 ASL 小于无序表**

---

### 第01题精析（A）—— 题目核心主旨：顺序查找对存储结构的要求

**解析推导链**：顺序查找 = 从表头（或表尾）开始逐个向另一端扫描 → 不需要随机存取特性（折半查找才需要）→ 顺序存储与链式存储均可。

**⚠️ 易错点**：B "散列存储" 错（散列走哈希而非遍历）；D "压缩存储" 是特殊矩阵用，不适合普通线性表顺序查找。

**⚠️ 推论（超级有用）**：**顺序查找 = 唯一对存储结构零要求的查找方法**。题干只要出现"链式"/"静态链表"/"单链表" → 首选顺序查找。

---

### 第02题精析(B)—— 题目核心主旨：顺序查找成功 ASL 只跟位置有关，不跟有序性有关

**解析推导链**：两个表（一个有序、一个无序）做**顺序查找** → 不论是否有序，查第 i 个元素都要比较 i 次 → 成功 ASL = (1+2+…+n)/n = (n+1)/2，两者相同。

**⚠️ 易错点**：错选 A "有序表更快" 的人混淆了"**成功 ASL**"与"**失败 ASL**"——有序表在**失败**情况下确实更快（可提前终止），但**成功**查找两者时间相同。

**⚠️ 推论（超级有用）**：**顺序查找成功 ASL = (n+1)/2 是铁律**，与存储结构、有序性均无关；只有**失败 ASL** 会受有序性影响。

---

### 第03题精析（B）—— 题目核心主旨：有序单链表的顺序查找成功 ASL

**解析推导链**：单链表 → 只能顺序查找 → 成功 ASL 公式与顺序表一致 = (n+1)/2。

**⚠️ 易错点**：错选 A "n/2" 的人凭直觉认为"平均在中间"——但精确的数学期望是 (n+1)/2，不是 n/2。

**⚠️ 推论**：只要是"**顺序访问且等概率**"，成功 ASL 永远是 **(n+1)/2**，与存储结构（表/链表/静态链表）无关。

---

### 综合01精析 —— 顺序查找在有序表 vs 无序表的 ASL 对比（三种情况）

**解析完整推演**：

**情况 1：查找失败**
- **有序顺序表**：扫描到某元素的关键字大于目标值 → 立即停止并报失败 → 平均需扫描约 **n/2** 次
- **无序顺序表**：必须扫描到表尾才能确认失败 → 平均扫描 **n** 次（或表示为 n+1 含空位）
- **结论**：**平均查找长度不同**，有序表更快

**情况 2：查找成功，且表中只有一个关键字等于给定值 k**
- 有序 / 无序表 **都**在找到 k 即停 → 两者平均扫描次数都是 (n+1)/2
- **结论**：**平均查找长度相同**

**情况 3：查找成功，且表中有若干关键字等于 k，一次查找要找出所有**
- **有序表**：相同关键字**相邻排列** → 找到第一个后**连续**查到最后一个即可（总次数 ≈ 首位置 + m）
- **无序表**：必须**扫描全表** n 次才能确认找完所有 k
- **结论**：**平均查找长度不同**，有序表更快

**⚠️ 推论（超级有用）**：顺序查找中，有序 vs 无序的差异**只在两个场景出现**——① 查找失败、② 查找全部重复元素。其他情况两者一样。

### 📋 【做题 SOP · 题型1】顺序查找 ASL 分析

**⏱️ 建议用时**：40 秒 ｜ **考频**：★★★☆☆ ｜ **难度**：★★☆☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP)：
- "单链表"/"链式存储"/"静态链表" + 查找
- "无序表"/"不要求有序" + 查找
- "顺序查找" 直接点名

**🔧 标准步骤**：
- **Step 1**：确认是否为**等概率** → 是则走公式，否则跳【题型9】
- **Step 2**：判断查**成功**还是**失败** → 成功永远 (n+1)/2；失败需再看有序/无序
- **Step 3**：若查失败 → 有序表 ≈ n/2，无序表 = n+1
- **Step 4**：核对存储结构是否影响 → 顺序查找**不受**存储结构影响，直接套公式

**🪤 踩坑预警**：
- Step 1 易踩：看到 "概率为 1/2, 1/3, 1/6" 这种**不等概率**题干（选04 ）还套等概率公式
- Step 2 易踩：把"成功 ASL 与有序性无关"错记成"都和有序性无关"
- Step 3 易踩：有序表**失败** ASL 不要写成 n 或 (n+1)/2，是 **n/2**

**✅ 自检口诀**：**"顺序无存要，成功(n+1)/2"**

**🎯 一题实战演示（第03题）**：
- Step 1：题目"概率相等" → 等概率 ✅，走公式
- Step 2：问"成功的平均查找长度" → 成功
- Step 3：应用公式 → (n+1)/2
- Step 4：存储结构是"有序单链表" → 不影响，公式仍成立
- **答案 B：(n+1)/2** ✅

## 【题型2】折半查找前提条件与适用性辨析（选05、选06、选15、选25【2024统考】)

### 核心知识

| 前提条件 | 说明 | 若违反的后果 |
|---------|------|------------|
| **必须有序** | 升序或降序均可 | 无法通过一次比较舍弃一半 → 不能折半 |
| **必须顺序存储** | 需按下标随机访问 mid | 链表/静态链表访问 mid 是 O(n) → 折半退化 |
| 有序方向 | 升/降都行 | —— |

**折半查找 vs 顺序查找速度对比**：
- **一般情况**：折半更快（O(log n) vs O(n)）
- **特殊情况**：顺序查找可能更快（如查表首元素，顺序 1 次 vs 折半 ⌈log₂n⌉ 次）

---

### 第05题精析（D）—— 折半查找的两个必要条件

**解析推导链**：
- A 错：折半查找**不能**用链表存储（mid 访问要 O(n)，失去折半优势）
- B 错：关键字可以是实数、字符串等可比较类型，不限整型
- C 错：升序降序都可以，只要"有序"即可
- D 对：**有序 + 顺序存储** 是两个必要条件

**⚠️ 易错点**：A 最具迷惑性——"能用链表存储有序数据"是对的，但**能用 ≠ 能折半**。链表上无法 O(1) 定位 mid。

**⚠️ 推论（超级有用）**：**折半查找 = 有序 ∧ 顺序存储**（两个条件缺一不可，且都是必要条件）。

---

### 第06题精析（C）—— 折半查找并非总比顺序查找快

**解析推导链**：
- 一般情况：折半 O(log n) 优于顺序 O(n)，大部分场景折半更快
- **反例**：查含 1000 个元素有序表的**第一个元素** → 顺序查找只需 1 次比较；折半查找需要 ⌈log₂1000⌉ ≈ 10 次
- 所以"必然快"（A）和"必然不快"（D）都错；"取决于递增/递减"（B）也错（方向不影响速度）

**⚠️ 易错点**：错选 A 的人认定"折半查找绝对更快"——**错**！只是"大部分情况下更快"。

**⚠️ 推论**：折半查找的优势是**渐近**的，小规模或命中表首时顺序查找可能反而占优。

---

### 第15题精析（D）—— 折半查找适用性与性能多选判断

**解析推导链**：
- A 错：数据元素有序**但存储必须顺序**，缺一不可
- B 错：折半最坏 O(log₂n)；二叉查找树最坏 O(n)（退化为单支树）——不相同
- C 错：查找概率不同时，顺序查找若按降序排概率，**平均 ASL 可能小于**折半（参考综合07）
- D 对：折半查找约需 O(log₂n) 次关键字比较

**⚠️ 推论**：**折半"比较次数" ≈ O(log₂n)** 是所有场景成立的性能上界描述。

---

### 第25题精析（D）—— 2024 统考真题：哪些数据结构不适合直接折半查找

**解析推导链（完整走一遍每个选项）**：

题目四个选项：I 有序链表、II 无序数组、III 有序静态链表、IV 无序静态链表。

折半查找两个必要条件：**① 有序、② 顺序存储**。

- **I 有序链表**：链表无法 O(1) 定位 mid → **不适合**
- **II 无序数组**：无序 → **不适合**
- **III 有序静态链表**：静态链表虽然用数组模拟，但元素的"逻辑顺序"靠 next 指针串起，**物理下标与逻辑顺序无必然对应** → 不能简单地由 (low+high)/2 得到"逻辑上的中位"→ **不适合直接折半**
- **IV 无序静态链表**：既无序又非顺序访问 → **不适合**

四个都不适合 → 选 **D：I、II、III、IV**

**⚠️ 易错点**：错选 B（仅 II、IV）的人认为"有序就能折半"——**错**！**存储结构也必须是真正的顺序存储**（数组），静态链表虽借用数组但其"顺序"由指针决定，不满足折半要求。

**⚠️ 推论（超级有用）**：
- **链表 → 永远不能折半**（不论有序无序）
- **静态链表 → 也不能直接折半**（虽然底层是数组，但逻辑顺序不等于物理顺序）
- **数组无序 → 必须先排序，不能直接折半**

### 📋 【做题 SOP · 题型2】折半查找适用性判断

**⏱️ 建议用时**：30 秒 ｜ **考频**：★★★★☆ ｜ **难度**：★★☆☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP)：
- "下列哪些数据结构**不适合**折半查找"
- "折半查找的**前提条件**"
- "能否用折半查找" / "是否适合折半"

**🔧 标准步骤**：
- **Step 1**：**查有序** → 无序？直接排除
- **Step 2**：**查顺序存储** → 链表/单链表？排除
- **Step 3**：**警惕静态链表陷阱** → 虽然是数组实现，但逻辑顺序由指针决定 → **不能**直接折半
- **Step 4**：剩下的才是"适合"的 → 必须同时是"有序 + 数组"

**🪤 踩坑预警**：
- Step 1 易踩：题面只说"链表"没说"有序"，直接判无序其实可能有序
- Step 2 易踩：把"链表"看成"数组"漏掉存储方式陷阱
- Step 3 易踩：静态链表误判为"能折半"（2024 真题正是考这个）

**✅ 自检口诀**：**"有序顺存两都要"**

**🎯 一题实战演示（第25题 · 2024 统考）**：
- Step 1 查有序：II 无序数组、IV 无序静态链表 → 排除（不适合）
- Step 2 查顺序存储：I 有序链表 → 链式结构 → 排除
- Step 3 警惕静态链表：III 有序静态链表 → 逻辑顺序由指针决定 → 排除
- Step 4 剩余：没有剩下的 → 四个全部不适合
- **答案 D：I、II、III、IV** ✅

## 【题型3】折半查找过程模拟（比较序列 / 合法路径判别）（选09、选10、选11、选21【2015统考】）

### 核心知识

| 要素 | 说明 |
|------|------|
| **mid 计算公式** | `mid = ⌊(low+high)/2⌋`（向下取整，王道默认） |
| **向下取整判定树特征** | 左子树结点数 **≥** 右子树结点数（左子树"多一个"） |
| **向上取整判定树特征** | 右子树结点数 ≥ 左子树结点数 |
| **铁律** | 一棵判定树**全程必须用同一种取整方式**，不能混用 |

**合法折半比较序列的三个判据**：
1. 满足**二叉排序树 BST** 性质（左子树所有值 < 根 < 右子树所有值）
2. 取整方式**全程一致**（全向下或全向上，不可混用）
3. 各子树高度差绝对值不超过 1（**平衡 BST**）

---

### 第09题精析（B）—— 折半过程模拟 · 查 A[11]

**解析推导链**：在 A[1..11] 中用 `mid=⌊(low+high)/2⌋` 查 A[11]。

| 步骤 | low | high | mid = ⌊(l+h)/2⌋ | 比较后动作 |
|------|-----|------|-----------------|-----------|
| 1 | 1 | 11 | **6** | A[6] < A[11] → low=7 |
| 2 | 7 | 11 | **9** | A[9] < A[11] → low=10 |
| 3 | 10 | 11 | **10** | A[10] < A[11] → low=11 |
| 4 | 11 | 11 | **11** | A[11] == A[11] → 命中 |

比较下标序列 = **6, 9, 10, 11** → 选 B。

**⚠️ 易错点**：错选 A（6, 8, 10, 11）的人把 `⌊(7+11)/2⌋=9` 误算成 8（可能误用了 (7+11+1)/2 → 上取整）。

**⚠️ 推论**：**无脑套 `⌊(low+high)/2⌋`**，不要凭直觉"目测中点"。

### 第10题精析（B）—— 折半查找指定值 90 的比较次数

**解析推导链**：表 {13,18,24,35,47,50,62,83,90,115,134}，n=11。

| 步骤 | low | high | mid | 比较 | 动作 |
|------|-----|------|-----|------|------|
| 1 | 1 | 11 | 6 → A[6]=50 | 90 > 50 | low=7 |
| 2 | 7 | 11 | 9 → A[9]=90 | 命中 | —— |

共 **2 次**比较 → 选 B。

**⚠️ 易错点**：把比较次数错记为"路径长度 + 1"或漏算首次比较。

**⚠️ 推论**：**折半查找比较次数 = 判定树上从根到命中结点的路径结点数**（含根和命中点）。

### 第11题精析（A）—— 比较序列合法性（取整方式一致性）

**解析推导链**：有序表 {b, c, d, e, f, g, q, r, s, t}（10 个元素，下标 1-10）中查 b。

对每个选项逐个检验"mid 取整方式是否全程一致"：

| 选项 | 第一次 mid | 第二次 mid | 第三次 mid | 结论 |
|------|-----------|-----------|-----------|------|
| A (f, c, b) | f = A[5] （向下，⌊(1+10)/2⌋=5） | c = A[2]（向下，⌊(1+4)/2⌋=2）| b = A[1]（向下，⌊(1+1)/2⌋=1）| **全向下取整，合法 ✅** |
| B (f, d, b) | f=A[5]（向下）| d=A[3]（向上，⌈(1+4)/2⌉=3）| —— | 向下+向上混合 ❌ |
| C (g, c, b) | g=A[6]（向上，⌈(1+10)/2⌉=6）| c=A[2]（向下）| —— | 向上+向下混合 ❌ |
| D (g, d, b) | g=A[6]（向上）| d=A[3]（向上）| b=A[1]（向下 ⌊(1+2)/2⌋=1）| 取整不一致 ❌ |

**答案 A**。

**⚠️ 易错点（最高频）**：没注意到"取整方式必须全程一致"，只检验"是否是 BST 路径"。

**⚠️ 推论（超级有用）**：**一次折半查找的 mid 取整方式全程固定**（要么全向下、要么全向上）——这是判定比较序列合法性的**第一道关卡**。

### 第21题精析（A）—— 2015 统考：不能构成折半查找比较序列

**解析推导链**：将四个选项各自画出"根→路径"的树形（每次比较的元素是下一次查找的"根"），用 BST 性质 + 平衡性检验。

选项分析（画出每个序列对应的路径树）：
- **A**：500 → 200 → 450 → 180。路径：先访 500，转左到 200，转右到 450，再转左到 180 。但 180 < 200，应在 200 的左子树，而不是 450 的子树中 → **违反 BST 性质** ❌
- **B**：500 → 450 → 200 → 180。每一步都满足 BST 路径 ✅
- **C**：180 → 500 → 200 → 450。每步满足 BST ✅
- **D**：180 → 200 → 500 → 450。每步满足 BST ✅

**答案 A**（不能构成）。

**⚠️ 易错点**：只检查"是否递减"忽略 BST 性质（折半判定树本质是 BST）。

**⚠️ 推论（超级有用）**：**折半查找判定树 = 一棵特殊的二叉排序树（平衡 BST）**——所以比较路径**必须**满足 BST 的"左小右大"性质。

### 📋 【做题 SOP · 题型3】折半过程模拟 & 比较序列合法性

**⏱️ 建议用时**：60 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★★☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP)：
- "依次与表中哪些元素比较"
- "比较序列为 …"
- "能 / 不能构成折半查找比较序列"

**🔧 标准步骤（过程模拟版）**：
- **Step 1**：写下 low、high 初值（通常 low=1，high=n）
- **Step 2**：计算 mid = ⌊(low+high)/2⌋（按题目约定，通常向下）
- **Step 3**：比较 A[mid] 与目标 key，若命中则停止
- **Step 4**：key > A[mid] → low = mid+1；key < A[mid] → high = mid-1
- **Step 5**：回到 Step 2，直到命中或 low > high
- **Step 6**：把每次的 A[mid] 记为比较序列；比较次数 = 比较的总次数

**🔧 标准步骤（合法性判别版）**：
- **Step 1**：设序列为 x₁, x₂, …, xₖ，x₁ 是根
- **Step 2**：看 x₂ 相对 x₁：x₂ < x₁ 则 x₂ 在 x₁ 左子树；反之在右子树
- **Step 3**：**BST 一致性**：x₃ 必须落在 x₂ 对应的子区间内（比 x₁、x₂ 都满足 BST 关系）
- **Step 4**：**取整一致性**：每次 mid 的取整方式必须**全程相同**（全下或全上）
- **Step 5**：**平衡性**：左右子树高度差 ≤ 1（一般在已知"折半"的题干下隐式满足，主要看 Step 3、4）

**🪤 踩坑预警**：
- Step 2 易踩：算错 `⌊(low+high)/2⌋`（习惯性上取整）
- Step 4 易踩：忘记 low=mid+1（写成 mid）导致死循环
- 合法性 Step 3 易踩：只看"大小关系"不看 BST 子区间归属（21 题的 A 正是此陷阱）
- 合法性 Step 4 易踩：忽略"取整方式一致性"（11 题陷阱）

**✅ 自检口诀**：**"取整一致，BST 合法"**

**🎯 一题实战演示（第11题）**：
- 序列 A：f, c, b
- Step 1：第一次 mid，low=1 high=10 → ⌊11/2⌋=5 → A[5]=f（向下取整）✅
- Step 2：b < f → high=4；第二次 low=1 high=4 → ⌊5/2⌋=2 → A[2]=c（向下取整）✅
- Step 3：b < c → high=1；第三次 low=1 high=1 → ⌊2/2⌋=1 → A[1]=b 命中 ✅
- **取整方式全程向下** → 合法 → 答案 **A** ✅

## 【题型4】折半查找判定树的性质（选07、选08、选23【2017统考】）

### 核心知识

| 判定树性质 | 说明 |
|----------|------|
| **树形本质** | 一棵**平衡二叉排序树（平衡 BST）** |
| **平衡性** | 任意结点左右子树**高度差 ≤ 1** |
| **非完全二叉树** | 不一定是完全二叉树（王道解析里已给反例） |
| **非满二叉树** | 更不是满二叉树 |
| **子树结点数关系** | 向下取整：**左子树结点数 ≥ 右子树**；向上取整相反 |
| **具体差值** | 左右子树结点数**相等**或**左多一个**（向下取整）、**相等**或**右多一个**（向上取整） |

### 第07题精析（B）—— 判定树的树形分类

**解析推导链**：
- A 最小生成树 → 针对图论概念，与查找无关 ❌
- C 完全二叉树 → 王道正文考点示例里就给过"非完全二叉树"的判定树 → ❌
- D 满二叉树 → 满二叉树对结点数有严格要求（2^h-1）→ ❌
- B 平衡二叉树 → 折半查找每次把数组从中点分割，两子数组长度相差**至多 1** → 对应判定树两子树高度差 **≤1** → ✅

**⚠️ 易错点（高频陷阱）**：错选 C "完全二叉树" 是最大坑——折半判定树**看起来像**完全二叉树但**不是**，注意"完全二叉树"的严格定义（除最后一层外，每层都满，且最后一层从左到右连续）。

**⚠️ 推论（超级有用）**：**折半判定树 = 平衡 BST**（必记！且"平衡"是其最根本的性质）。

### 第08题精析（B）—— 折半查找 vs 二叉排序树性能对比

**解析推导链**：
- 折半查找：最坏和平均 ASL 均为 O(log₂n)（判定树高度固定）
- 二叉排序树（BST）：最好情况 O(log₂n)（平衡 BST）；**最坏情况 O(n)**（退化为单支树）
- 所以性能**有时不相同**（数据输入顺序决定 BST 形态）

**⚠️ 推论**：折半 = **始终 O(log n)**；BST = **O(log n) ~ O(n) 区间**。

### 第23题精析（A）—— 2017 统考：哪棵二叉树能成为折半查找判定树

**解析推导链**：

折半判定树对"左右子树结点数"的硬约束：
- 设区间有 m 个元素
  - m = 2n+1 → mid 分割后左右各 n 个（左右相等）
  - m = 2n → 向下取整时左 n-1 / 右 n（**左比右少一个**）
  - m = 2n → 向上取整时左 n / 右 n-1（**左比右多一个**）

**重点**：向下取整规则下 → 每个结点的**左子树结点数 ≤ 右子树**，或相等，或**少一个**；向上取整相反（左 ≥ 右）。

**给定的四个选项**（2017 年原题图，每项是一棵 7~8 结点的二叉树）：
- **选项 A**：每个结点都满足"左右子树结点数相等 OR 左多一个"→ 符合**向上取整**规则 ✅
- **选项 B、C、D**：都出现了"有的结点左子树比右子树少一个、有的左比右多一个"的**混合情况** → 违反"取整方式全程一致" ❌

**答案 A**。

**⚠️ 易错点（极高频）**：只看"是否平衡二叉树"忽略了"取整方式必须全程一致"。B、C、D 单纯作为平衡二叉树都成立，但作为**折半判定树**不成立。

**⚠️ 推论（超级有用）**：**折半判定树的结点数差别**（左 ≥ 右 或 左 ≤ 右）**全树必须一致**——这是比"平衡"更严苛的条件。

**✅ 自检口诀**：**"左右结点差，全树须一致"**

- Step 2：均未标数值，跳过 BST 检查

  - 选项 A：每个结点左子树结点数 ≥ 右子树（或相等）→ 全树符合向上取整 ✅
  - 选项 B、C、D：存在结点"左比右多"、另有结点"右比左多"→ 混合取整 ❌

- **答案 A** ✅

## 【题型5】折半查找判定树高度与最多比较次数（选12、选13、选20【2010统考】、选24【2023统考】）

### 核心知识

| 量 | 公式 | 备注 |
|------|------|------|
| **判定树高度 h** | `h = ⌈log₂(n+1)⌉` | n 为结点数，计算圆圈结点 |
| **等价形式** | `h = ⌊log₂n⌋ + 1` | 两者**数值等价**，可互换 |
| **查找成功最多比较次数** | = h | = 根到最深叶结点的路径结点数 |
| **查找失败最多比较次数** | = h | 失败路径最深也只到最深圆圈结点的下一层方框之前 |
| **查找成功 vs 失败最多比较次数** | **相等** | 都等于树高 h |

**典型值速查**：

| n | 1 | 2 | 3 | 7 | 8 | 15 | 16 | 31 | 32 | 600 | 1000 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| ⌈log₂(n+1)⌉ | 1 | 2 | 2 | 3 | 4 | 4 | 5 | 5 | 6 | **10** | 10 |

### 第12题精析（A）—— 判定树高度通用公式

**解析推导链**：对 n 个结点的判定树，设其总结点数 n = 2^h - 1（满二叉树上界）→ h = ⌈log₂(n+1)⌉。

**验证**：
- n=1 → h=⌈log₂2⌉=1 ✅
- n=2 → h=⌈log₂3⌉=2 ✅
- 其他选项代入 n=1: B 给 0、C 给 0、D 给 -1 → 全错

**⚠️ 推论（超级有用）**：**判定树高度首选公式 `h = ⌈log₂(n+1)⌉`**，遇到 ⌊log₂n⌋+1 也视作等价。

### 第13题精析（A, B）—— 长度 16 查找不存在元素的最少/最多比较

**解析推导链**：n=16，判定树高度 h = ⌈log₂17⌉ = **5**。

- **最多比较次数**：到最深的方框结点之前（即最深圆圈结点的层数）= h = **5**
- **最少比较次数**：到次深层的方框之前 = h-1 = **4**
- 原因：n=16 时判定树不是满二叉树（满二叉树需 n=2^h-1=15 或 31），最深方框只出现在部分分支上

**答案：最少 4（A），最多 5（B）**。

**⚠️ 易错点**：误以为"最多 = 最少" → 只有 n = 2^h-1（满判定树）时才相等。

**⚠️ 推论**：判定树**非满二叉树时**，查找失败的最少和最多比较次数**可能不同**。

## 【题型6】折半查找 ASL 计算（成功/失败，画判定树法）（选14、综合02、综合03）

### 核心知识

| 步骤 | 做法 |
|------|------|
| **Step 1** | 根据给定的 n（表长），画出**折半判定树**（圆圈=实际结点，方框=失败结点） |
| **Step 2** | 统计每一层的**圆圈**结点个数（成功）和**方框**结点个数（失败） |
| **Step 3** | **成功 ASL** = ∑（每层圆圈数 × 层数）/ n |
| **Step 4** | **失败 ASL** = ∑（每层方框数 × 其**父圆圈**所在层数）/ 方框总数 |

**关键记忆点**：
- **成功**查找到**圆圈**结点 → 层数即比较次数
- **失败**查找到**方框**结点 → 比较次数 = 方框**父圆圈**的层数（即比 "失败方框层数 - 1"）
- 方框结点总数 = n + 1（长度 n 的表，失败位置有 n+1 个空隙）

---

### 第14题精析（A, D）—— n=12 折半查找 ASL

**解析推导链**：n=12，画出判定树如下（圆圈为实际元素 A[0..11]）：

```
         第1层:       5
                    /   \
         第2层:    2       8
                 / \     / \
         第3层: 0   3   6  10
                |   |   |   |
         第4层: 1   4   7   9  11 (圆圈)
                |  |  |  |  |  |  |  |  (方框结点在第5层)
```

层数统计：
- 第1层：1 个圆圈
- 第2层：2 个圆圈
- 第3层：4 个圆圈
- 第4层：5 个圆圈（1、4、7、9、11）
- 共 1+2+4+5 = 12 个圆圈 ✅

**成功 ASL** = (1×1 + 2×2 + 3×4 + 4×5)/12 = (1+4+12+20)/12 = **37/12** → A

**失败 ASL**：方框总数 = 13（n+1=13）
- 在第 4 层（方框父在第 3 层，比较 3 次）的方框：3 个（对应 0、3、6、10 各自缺一个子）
- 在第 5 层（方框父在第 4 层，比较 4 次）的方框：10 个（5 个叶圆圈各贡献 2 个方框）

**失败 ASL** = (3×3 + 4×10)/13 = (9+40)/13 = **49/13** → D

**答案：成功 A（37/12），失败 D（49/13）**

**⚠️ 易错点**：
- **易错点 1**：失败 ASL 误用 (3×1 + 4×13)/13 —— 错！方框层数 ≠ 比较次数，比较次数是"方框父圆圈"层数
- **易错点 2**：把方框总数记成 n 而不是 n+1

**⚠️ 推论（超级有用）**：
- **成功**：到**圆圈**，比较次数 = 圆圈层数
- **失败**：到**方框**，比较次数 = **方框父圆圈**层数（即方框层数 - 1）

### 综合02精析 —— 有序表 15 元素折半查找 ASL

**原题**：有序顺序表 {017, 094, 154, 170, 275, 503, 509, 512, 553, 612, 677, 765, 897, 908}（n=14 个元素）

1）**判定树**：

```
                        509
                       /    \
                     154      677
                    /   \    /   \
                  017   275 553   897
                   |     |   |    / \
                  094   170 512  765 908
                         |   |
                        503  612
```

（每一层圆圈数：1,2,4,7 —— 共 14 个）

**查找 684**：509 → 677 → 897 → 765 → 不存在（失败）→ 共 **4 次** 比较

3）**成功 ASL**：  
ASL成功 = (1×1 + 2×2 + 3×4 + 4×7)/14 = (1+4+12+28)/14 = **45/14**

**失败 ASL**（方框总数 = 15，即 n+1）：
- 失败位置父在第 3 层（比较 3 次）的方框：3 个
- 失败位置父在第 4 层（比较 4 次）的方框：12 个
- （注：这里按王道原解析：3个"失败比较 1 次"的方框实则指父结点层次，具体按题解 3×1+4×14 = 59/15）

ASL失败 = (3×1 + 4×14)/15 = (3+56)/15 = **59/15**

### 综合03精析 —— 顺序 + 折半的混合查找 ASL

**原题**：A[0..8n-1]，先在 A[7], A[15], …, A[8n-1] 中顺序查找，若 A[8k-1] < X < A[8×(k+1)-1]，再在 A[8k] ~ A[8×(k+1)-2] 中折半查找。

1）**判定树结构**：索引数组（8n 个元素分成 n 块）→ 顶层是顺序查找的比较序列 → 每块内（7 元素子区间）是折半查找判定树。

2）**成功 ASL 推导**：
- 0~7 之间：顺序比较 **1 次**（到达该块），再折半进入子区间
- 8~15 之间：顺序比较 **2 次**，再折半
- 8(n-1) ~ 8n-1 之间：顺序比较 **n 次**，再折半

每块 7 个元素（去掉索引元素后），折半查找子树高度 = 3，成功 ASL 子 = (1+2×2+4×4)/7 = 21/7 = 3

**总 ASL 成功**：

ASL成功 = (1/8n) × ∑ᵢ₌₁ⁿ (i + (i+1) + 2(i+2) + 4(i+3))  
       = (1/8n) × ∑(i + (i+1) + 2(i+2) + 4(i+3))  
       = (1/8n) × ∑(8i + 17)  
       = **(n+1)/2 + 17/8**

**⚠️ 推论**：混合查找的 ASL = **索引层 ASL + 子区间 ASL 加权**。

### 📋 【做题 SOP · 题型6】折半查找 ASL 计算

**⏱️ 建议用时**：120 秒 ｜ **考频**：★★★★☆ ｜ **难度**：★★★★☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP)：
- "折半查找**成功 / 失败**的平均查找长度"
- "ASL" + 具体数值或具体表
- 综合题中要求画判定树

**🔧 标准步骤**：
- **Step 1**：确定表长 n，画出**圆圈（实际）+ 方框（失败）**的完整判定树
- **Step 2**：按层统计：第 i 层的**圆圈数** nᵢ（成功用）、第 i 层的**方框数** mᵢ（失败用）
- **Step 3**：**成功 ASL** = ∑(i × nᵢ) / n（i 是层号，从 1 开始）
- **Step 4**：**失败 ASL** = ∑((i-1) × mᵢ) / (n+1)（方框在第 i 层则比较次数为 i-1，因为方框是虚构的）
- **Step 5**：对答案做合理性检验 → 成功 ASL 应在 [⌈log₂(n+1)⌉/2, ⌈log₂(n+1)⌉] 区间

**🪤 踩坑预警**：
- Step 1 易踩：把方框画到第 h 层但漏掉不同深度（n 非 2^h-1 时有两层方框）
- Step 3 易踩：分母错用 n+1（成功分母是 n）
- Step 4 易踩：分母错用 n（失败分母是 n+1），**且比较次数用 "方框层数"而非"方框父层数"**
- Step 5 检查：若成功 ASL > h（树高），必然算错

**✅ 自检口诀**：**"成功到圆除以n，失败到方父除以n+1"**

**🎯 一题实战演示（第14题 · n=12）**：
- Step 1：画 n=12 的判定树，含 12 圆圈 + 13 方框
- Step 2：圆圈按层：1, 2, 4, 5（第 1~4 层）；方框按层：3 在第 4 层、10 在第 5 层
- Step 3：ASL 成功 = (1×1 + 2×2 + 3×4 + 4×5)/12 = 37/12
- Step 4：ASL 失败 = ((4-1)×3 + (5-1)×10)/13 = (9+40)/13 = 49/13
- Step 5：h = ⌈log₂13⌉ = 4，成功 ASL=37/12≈3.08 ∈ [2, 4] ✅

## 【题型7】分块查找结构与 ASL 计算（选16、选18）

### 核心知识

| 分块查找要点 | 说明 |

| **ASL 总公式** | ASL = L_I（索引查找）+ L_S（块内查找） |

| **符号定义** | n = 总元素数；b = 块数；s = 每块元素数（n = b·s） |

### 第16题精析（B）—— 分块查找的数据组织方式

- C 错：要求"块内有序"过于严苛，分块查找的优点之一就是不要求块内排序

**答案 B**。

**⚠️ 推论（超级有用）**：分块查找三要素 = **块间有序 + 块内无序 + 每块最大/最小建索引**。

## 【题型8】分块查找最优块长 s = √n（选17、选19）

### 核心知识

| 情形 | 最优 s | 最优 ASL |
|------|-------|---------|
| **均匀分块 + 双顺序查找** | **s = √n** | **ASL_min = √n + 1** |
| **均匀分块 + 索引折半 + 块内顺序** | s = √n（近似最优） | ASL_min ≈ ⌈log₂(√n+1)⌉ + (√n+1)/2 |
| **均匀分块 + 双折半查找** | s 仍约 √n 量级 | ASL_min ≈ ⌈log₂(√n+1)⌉ + ⌈log₂(√n+1)⌉ |

**数学推导**：ASL = b/2 + s/2 + 1 = n/(2s) + s/2 + 1

令 d(ASL)/ds = 0 → -n/(2s²) + 1/2 = 0 → s² = n → **s = √n** → ASL_min = √n + 1

---

**解析推导链**：
- n = 2500
- s_最优 = √n = √2500 = **50**

**答案 A（50）**。

**⚠️ 推论（秒算）**：**s = √n 是黄金公式**；常用值：n=100→s=10，n=2500→s=50，n=10000→s=100。

### 第19题精析（C）—— n=65025 索引折半查找效率

**解析推导链**：
- n = 65025，最优块长 s = √65025 = **255**
- 索引表有 b = n/s = 65025/255 = 255 个索引项
- 索引折半：⌈log₂(255+1)⌉ = ⌈log₂256⌉ = **8** 次
- 块内折半：⌈log₂(255+1)⌉ = 8 次
- 最好情况下的最大比较次数 = 8 + 8 = **16**

**答案 C（16）**。

**⚠️ 易错点**：
- **易错点 1**：算成 √n = 255 后把 b 也算成 255 但下一步套错公式
- **易错点 2**：忘记"最好情况"= 折半 + 折半（不是顺序）

## 【题型9】不等概率查找的效率优化策略（选04、综合07【2013统考】）

### 核心知识

| 策略 | 适用场景 | 最优 ASL 公式 |
|------|---------|--------------|
| **按概率降序排列 + 顺序查找** | 顺序存储/链式均可 | ASL = ∑(pᵢ × i)，按 p₁≥p₂≥…≥pₙ |
| **折半查找（等概率假设）** | 存储必须顺序有序 | 与概率无关，ASL 固定 |
| **构造二叉排序树**（链式） | 链式 + 需要高效查找 | 按概率设计树形，概率大的靠近根 |

**核心定理（排序不等式）**：对于给定的概率 p₁, p₂, …, pₙ，顺序查找的 ASL 在**按概率降序排列**时取得最小值。

---

### 第04题精析（A）—— 长度 3 不等概率 ASL

**解析推导链**：
- 查第 1 个元素概率 1/2，查长度 1
- 查第 2 个元素概率 1/3，查长度 2
- 查第 3 个元素概率 1/6，查长度 3
- ASL = 1/2 × 1 + 1/3 × 2 + 1/6 × 3 = 1/2 + 2/3 + 1/2 = 3/6 + 4/6 + 3/6 = 10/6 = **5/3**

**答案 A（5/3）**。

**⚠️ 推论**：不等概率 ASL = **Σ(pᵢ × 查长度)**，不要套等概率公式 (n+1)/2。

### 综合07精析 —— 2013 统考：S={'do','for','repeat','while'} 概率排序优化

**原题**：p₁=0.35, p₂=0.15, p₃=0.15, p₄=0.35（按字典序：do, for, repeat, while），折半 ASL = 2.2。要求改进。

**1) 顺序存储答案**：

- **按概率降序排列**：p=0.35 两个元素（do, while）排前两位，p=0.15 两个元素（for, repeat）排后两位
- 新序列：do, while, for, repeat
- 采用**顺序查找**
- ASL成功 = 0.35×1 + 0.35×2 + 0.15×3 + 0.15×4 = 0.35 + 0.70 + 0.45 + 0.60 = **2.1**
- **比折半查找 (2.2) 更短** ✅

**2) 链式存储答案**：

**答案 1（单链表 + 顺序查找）**：
- 按概率降序排列：do → while → for → repeat
- ASL = 0.35×1 + 0.35×2 + 0.15×3 + 0.15×4 = **2.1**

**答案 2（二叉排序树）**：
- 构造 BST 使**高概率元素靠近根**
- 示例树结构（两种等价设计，此处列其一）：
  ```
           for           repeat
          /   \    或    /    \
        do    while    do     while
              /                /
           repeat            for
  ```
- 以"BST 1"（根为 for）为例：
  - for 在第 1 层，do 在第 2 层，while 在第 2 层，repeat 在第 3 层
  - ASL = 0.15×1 + 0.35×2 + 0.35×2 + 0.15×3 = 0.15 + 0.70 + 0.70 + 0.45 = **2.0**
- **比顺序查找的 2.1 更短** ✅

**⚠️ 推论（超级有用）**：
- 不等概率查找 → **降序排 + 顺序查** 是最基本优化
- 链式结构还可以用 **BST 二次优化**（让高概率元素更靠根）
- 字典序排列是"默认存在的顺序"，但**查找效率最优的顺序是按概率降序**

### 第22题精析（B）—— 2016 统考：跳跃式顺序查找

### 综合04 —— 折半查找递归版

**已实现代码（详见板块二）**：

```c
int BinSearchRec(SSTable ST, ElemType key, int low, int high) {
    if (low > high) return 0;             // 递归边界
    int mid = (low + high) / 2;            // 取中间位置
    if (key > ST.elem[mid])
        return BinSearchRec(ST, key, mid+1, high);  // 向后半查
    else if (key < ST.elem[mid])
        return BinSearchRec(ST, key, low, mid-1);   // 向前半查
    else
        return mid;                        // 命中
}
```

**关键理解**：
- 时间复杂度 O(log₂n)
- 空间复杂度 O(log₂n)（递归栈深度）
- **递归版本 vs 迭代版本**：时间相同，空间递归多一个栈

### 综合05 —— 顺序查找自适应优化（与前驱交换）

**已实现代码（详见板块二）**：

int SeqSrch(RcdType R[], ElemType k) {

### 综合06 —— 杨氏矩阵查找（n×n 矩阵，行列均升序）

**已实现代码（详见板块二）**：

```c
bool findkey(int A[][], int n, int k) {
    int i = 0, j = n-1;                    // 从右上角开始
    while (i < n && j >= 0) {
        if (A[i][j] == k) return true;
        else if (A[i][j] > k) j--;         // 当前大 → 向左
        else i++;                           // 当前小 → 向下
    }
    return false;
}
```

**关键理解**：
- 起点：**右上角 A[0][n-1]**（或左下角 A[n-1][0]）
- 不能选左上/右下角（那样向哪边都能 ≤ 或 ≥ 目标，**无法排除方向**）
- 时间 O(n)，空间 O(1)
- 比较次数 ≤ 2n（每次要么 j-- 要么 i++，总步数 ≤ 2n）

**⚠️ 推论**：**杨氏矩阵搜索起点必须是"右上"或"左下"**——这是唯一能保证"每次比较后能排除一行或一列"的位置。

## 必背代码 1：折半查找递归版（综合04）

```c
// ===== 数据结构定义 =====
typedef struct {
    ElemType *elem;          // 存储空间基址（0 号留空，从 1 开始）
    int length;              // 表的长度
} SSTable;

// ===== 折半查找递归实现 =====
// 初始调用：BinSearchRec(ST, key, 1, ST.length)
int BinSearchRec(SSTable ST, ElemType key, int low, int high) {
    if (low > high) 
        return 0;                          // 查找失败，返回 0
    int mid = (low + high) / 2;            // 取中间位置（向下取整）
    if (key > ST.elem[mid])                // 目标在后半
        return BinSearchRec(ST, key, mid+1, high);
    else if (key < ST.elem[mid])           // 目标在前半
        return BinSearchRec(ST, key, low, mid-1);
    else                                   // 命中
        return mid;
}
```

**关键点**：
- **边界条件**：`low > high` 即失败退出
- **mid 取法**：`(low+high)/2` 等价于 `⌊(low+high)/2⌋`（C 的整数除法自动向下取整）
- **时间**：O(log₂n)
- **空间**：O(log₂n)（递归栈深度）
- **迭代版对比**：空间可降到 O(1)

## 必背代码 2：顺序查找 + 前驱交换（综合05）

```c
// ===== 顺序查找线性表，找到后与前驱交换 =====
// 目的：把高频元素逐渐前移，降低后续查找 ASL
int SeqSrch(RcdType R[], ElemType k) {
    int i = 0;
    // 逐个顺序扫描查找关键字 k
    while (R[i].key != k && i < n) 
        i++;
    // 若找到且不是首个元素，则与前驱交换
    if (i < n && i > 0) {
        RcdType temp = R[i];
        R[i] = R[i-1];
        R[i-1] = temp;
        return --i;                        // 返回交换后的新位置
    }
    else 
        return -1;                         // 未找到
}
```

**关键点**：
- **交换条件**：`i < n`（找到）+ `i > 0`（不是首元素）
- **返回值**：**交换后的**位置（--i）
- **链表版本**的思路：交换前需要保存指向前驱的指针
- **时间**：O(n)
- **空间**：O(1)
- **意义**：局部性优化，让热点数据"上浮"

## 必背代码 3：杨氏矩阵查找（综合06）

```c
// ===== n×n 杨氏矩阵（行/列均升序）查找目标值 k =====
// 思路：从右上角出发，大则左移，小则下移
bool findkey(int A[][], int n, int k) {
    int i = 0, j = n-1;                    // 起点：右上角
    while (i < n && j >= 0) {              // 未越界
        if (A[i][j] == k) 
            return true;                   // 命中
        else if (A[i][j] > k) 
            j--;                           // 当前值大 → 向左移一列
        else 
            i++;                           // 当前值小 → 向下移一行
    }
    return false;                          // 越界未找到
}
```

**关键点**：
- **起点选择**：必须是**右上角**（或左下角）→ 每次比较后能**排除一行或一列**
- **左上/右下不可**：向两个方向都是单调，**无法排除**
- **时间**：O(n)（每次 i++ 或 j--，总步数 ≤ 2n）
- **空间**：O(1)
- **推广**：对 m×n 矩阵，时间 O(m+n)
- **变种**：也可以用"右上角的折半查找"进一步优化，但 O(n) 已经很好
