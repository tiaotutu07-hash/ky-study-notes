## 📚 板块一 · 题型分类与解题方法

### 【题型 1】B 树 / B+ 树基本性质与概念辨析（题号：1、2、3、12、13、14、18、24）

#### 核心知识：m 阶 B 树的七条性质（必背）

| # | 性质 | 备注 |
|---|---|---|
| 1 | **树中每个结点至多有 m 棵子树**（即至多含 m-1 个关键字） | "m 阶"的"阶"= 子树数上限 |
| 2 | **若根结点不是终端结点，则根结点至少有 2 棵子树** | 根可以只有 1 个关键字 |
| 3 | **除根结点外的所有非叶结点至少有 ⌈m/2⌉ 棵子树**（即至少含 ⌈m/2⌉-1 个关键字） | 这是"分裂/合并"的底线 |
| 4 | **所有非叶结点的结构**：`(n, P₀, K₁, P₁, K₂, P₂, …, Kₙ, Pₙ)`，n = 关键字个数 | Kᵢ 有序，Pᵢ 指向子树 |
| 5 | **所有叶结点都出现在同一层次上**，且不带信息（在王道语境下叶结点=失败结点） | 统考 2023 例外：叶=最底层终端 |
| 6 | **结点内关键字有序** | 升序或降序均可 |
| 7 | **B 树是所有结点的平衡因子均等于 0 的多路查找树** | B 树天然绝对平衡 |

#### B 树 vs B+ 树四大核心差异

| 维度 | B 树 | B+ 树 |
|---|---|---|
| **关键字与子树数关系** | n 个关键字 → n+1 棵子树 | n 个关键字 → n 棵子树（一一对应） |
| **非叶结点作用** | 存储关键字 + 指向数据的指针 | **仅起索引作用**，不存数据记录 |
| **叶结点** | 不含信息（失败结点） | **包含全部关键字**，且按顺序链接成链表 |
| **查找方式** | 仅支持多路（随机）查找，成功可在内部结点终止 | **支持顺序查找 + 随机查找**，成功必到叶 |

#### 第 01 题精析（答案 D）—— 题目核心主旨：由图反推 B 树阶数是否唯一

**推导链**：
- 图中根结点有 2 个关键字 → 3 棵子树；叶子层每个结点有 2~3 个关键字 → 3~4 棵子树。
- **关键字数 < 子树数，首先排除 B+ 树**（B+ 树关键字数 = 子树数）。
- 对 4 阶 B 树：根至少 2 子树（✅），非叶至少 ⌈4/2⌉=2 子树、至多 4 子树（✅，2~3 满足）
- 对 5 阶、6 阶 B 树同样满足（分别要求 ⌈5/2⌉=3 最少、⌈6/2⌉=3 最少 —— 题中 2~3 恰好满足 5、6 阶的最少要求）
- **同时满足 4 阶、5 阶、6 阶的定义，无法唯一确定 → 选 D**。

**⚠️ 易错点**：见到图就急着数关键字最多是多少然后反推 m ——**错！B 树定义只给出上下界，一个具体图可能满足多个 m**。

**⚠️ 推论（超级有用）**：**给定一棵 B 树图形，一般无法唯一判定其阶数**，除非题干给出阶数或图形特征触发唯一下界（如某结点子树数 = 某一具体值）。

#### 第 02 题精析（答案 C）—— 题目核心主旨：找"关键字数下界"的错误表述

**推导链**：
- **A**：根至多 m 棵子树 ✅（性质1）
- **B**：所有叶结点在同一层 ✅（性质5）
- **C**：**非叶结点至少 ⌈m/2⌉ 棵子树**，不是 m/2 或 (m+1)/2 —— 这是关于**子树数**的条件，不是"关键字数"。若把"关键字数"读成"子树数"，选项 C 数字也对不上定义 → ❌
- **D**：结点内关键字有序 ✅（性质6）

**⚠️ 易错点**：选项 C 的"m/2（m 为偶数）或 (m+1)/2（m 为奇数）"乍看像 ⌈m/2⌉ 的变形，但题干说的是**叶结点**（王道定义下叶结点无关键字）—— **叶结点本身没有"子树数"概念**，此说法与 B 树性质对不上号。

**⚠️ 推论**：**⌈m/2⌉ 永远是"除根外非叶结点"的子树数下界**，不要把它套到根或叶身上。

#### 第 03 题精析（答案 B）—— 题目核心主旨：验证高度为 3 的 3 阶 B 树四条叙述

**推导链**（3 阶、高度 3）：
- **Ⅰ**：每个结点至少 2 棵非空子树 —— 每个非根非叶至少 ⌈3/2⌉=2 子树 ✅；根若不是叶至少 2 子树 ✅ → Ⅰ 正确
- **Ⅱ**：每个结点至多 2 个关键字 —— 每个结点至多 m-1=2 关键字 ✅ → Ⅱ 正确
- **Ⅲ**：最多存储 26 个关键字 —— 满 3 叉树有 1+3+9=13 结点，每结点最多 2 关键字 → 26 关键字 ✅ → Ⅲ 正确
  - 推导：`关键字数 ≤ (m-1)(1+m+m²+…+mʰ⁻¹) = mʰ-1`，代入 m=3、h=3 → 26
- **Ⅳ**：插入一个元素引起 B 树结点分裂后，树的高度变为 4 —— **❌错**，分裂不一定传导到根
  - 反例见答案图 1：3 阶 B 树中间结点分裂只让父结点多一个关键字，不增高

**⚠️ 易错点**：以为**分裂 = 树高 +1**，实际上**仅当分裂一路传到根**才会增高。

**⚠️ 推论（超级有用）**：**最多关键字数 = mʰ - 1**（满 m 叉树，每结点塞满 m-1 关键字）。

#### 第 12 题精析（答案 A）—— 题目核心主旨：B 树是否支持顺序查找

**推导链**：
- **A**：B 树和 B+ 树都能有效支持顺序查找 —— **B 树不支持顺序查找**（只能多路/随机），B+ 树因叶子链表支持 → ❌
- **B**：都支持随机查找 ✅
- **C**：都是平衡的多叉树 ✅
- **D**：都可用于文件索引 ✅（但 B+ 树更适合，因读写代价更低）

**⚠️ 易错点**：想当然认为"关键字有序 = 支持顺序查找"，但 **B 树的顺序查找需要中序遍历整个树**，效率低，实际不使用。

**⚠️ 推论**：**B 树仅支持多路查找；B+ 树的叶子链表 = B+ 树支持顺序查找的硬件基础**。

#### 第 13 题精析（答案 C）—— 题目核心主旨：B 树/B+ 树查找操作细节

**推导链**：
- **A**：B 树查找成功时，若目标在内部结点则立即返回，**不必查到叶子** ✅
- **B**：B 树查找失败时，必须沿路径查到叶结点（失败结点）✅
- **C**：B+ 树查找成功时，**不一定要查到叶结点** —— **❌错**，B+ 树所有数据记录都只在叶子层，**成功必到叶**
- **D**：B+ 树每次查找长度都相等 ✅（因为路径必从根到叶）

**⚠️ 易错点**：把 B 树的"内部结点可终止"的性质错误地套到 B+ 树身上。

**⚠️ 推论（超级有用）**：**B 树 = 成功可能在任一层；B+ 树 = 成功必到叶，查找长度恒定**。

#### 第 14 题精析（答案 D · 2009 统考）—— 题目核心主旨：识别 B+ 树特有属性

**推导链**：m 阶 B 树**不要求叶结点之间用指针链接**，这是 B+ 树才有的特征 → D 描述的其实是 B+ 树 → 选 D。

**⚠️ 易错点**：把"叶结点同一层"和"叶结点链接"混为一谈 —— 前者是 B 树 B+ 树共有，后者仅 B+ 树。

**⚠️ 推论**：**"叶间链接指针" = B+ 树识别符**，一旦看到这个特征，绝对是 B+ 树。

#### 第 18 题精析（答案 A · 2016 统考）—— 题目核心主旨：B+ 树独有特性

**推导链**：
- **A**：B+ 树能支持顺序查找 —— 因叶结点含全部关键字且链表相连 ✅（独有）
- **B**：结点中含有关键字 —— B 树 B+ 树都含 ❌
- **C**：根结点至少有两个分支 —— B 树 B+ 树都如此 ❌
- **D**：所有叶结点都在同一层 —— B 树 B+ 树都如此 ❌

**⚠️ 易错点**：与 2009 真题对比记忆：**2009 答"链接指针"，2016 答"顺序查找"，其实指的是同一件事的因与果**。

**⚠️ 推论（超级有用）**：**"B+ 树特有 = 顺序查找 + 叶间指针链接 + 叶子含全部关键字"三位一体**。

#### 第 24 题精析（答案 B · 2023 统考）—— 题目核心主旨：非空 B 树性质判断

**四条叙述逐一分析**（注意：**本题"叶结点"指最底层终端结点**，与王道通常定义不同）：

- **Ⅰ 插入操作可能增加树的高度** —— ✅ 分裂传到根时增高
- **Ⅱ 删除操作一定会导致叶结点的变化** —— ✅
  - 若删的是叶子关键字，直接变化
  - 若删的是内部结点关键字，也要与前驱/后继交换后转成删叶 → 最终叶子仍变
- **Ⅲ 查找某关键字总是要查找到叶结点** —— ❌ B 树内部结点存关键字，找到即返回
- **Ⅳ 插入的新关键字最终位于叶结点中** —— ❌ 初始插入在叶，但**分裂后可能被移到父结点**

**⚠️ 易错点（极其重要！）**：本题的 **"叶结点"= 最底层终端结点**（与 B+ 树定义一致）。**王道书一贯将"叶结点"定义为失败结点（不含关键字）**，但**统考真题中"叶结点"指最底层含关键字的终端结点**。**审题时看上下文判定！**

**⚠️ 推论（超级有用）**：统考与教材"叶结点"定义存在差异 → 凡见"叶结点"字样，**先看题是否为统考真题**，再结合上下文判定。

#### 📋 【做题 SOP · 题型 1】B 树 / B+ 树性质辨析

**⏱️ 建议用时**：60 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★★☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP）：
- "下列关于 m 阶 B 树的说法中，错误的是"
- "B 树和 B+ 树的区别 / 特点"
- "B+ 树不同于 B 树的特点"
- 给出性质陈述 I/II/III/IV 判断正误

**🔧 标准步骤**：
- **Step 1**：**定位"被测对象"** —— 是 B 树还是 B+ 树？是根/非根非叶/叶？ → 目的：避免把 B+ 性质套 B 树
- **Step 2**：**调用核心性质速查卡**（见速查表1）逐项比对 → 目的：排除表述正确的选项
- **Step 3**：**特别警惕四大高频陷阱**（见踩坑预警）→ 目的：识别出题人故意混淆
- **Step 4**：**若涉及"叶结点"字样**，判定是王道定义（失败结点）还是统考定义（终端结点）→ 目的：避免 2023 式陷阱
- **Step 5**：**输出选项 + 用一条性质/反例自证** → 目的：二次校验

**🪤 踩坑预警**：
- Step 1 易踩：把"m 阶"当成"m 个关键字"（实际是 m 棵子树）
- Step 2 易踩：⌈m/2⌉ 是"非根非叶"下界，不是根的下界（根下界 = 2 子树）
- Step 3 易踩：把"叶结点同一层" + "叶结点链接"混为一谈
- Step 4 易踩：统考 2023 叶结点 = 终端结点，与教材不一致
- Step 5 易踩：顺序查找只有 B+ 树支持，B 树不支持！

**✅ 自检口诀**：**"阶看子树，叶分两义"**

**🎯 一题实战演示**（第 18 题 · 2016 统考）：
- **Step 1**：被测对象 = B+ 树独有特性
- **Step 2**：调用 B vs B+ 对比表 —— B/ 含关键字 / 根 2 分支 / 叶同层 三者共享，**顺序查找**为 B+ 独有
- **Step 3**：警惕 B、D 两项是"共有"不是"独有"，易被误选
- **Step 4**：本题无叶结点定义陷阱
- **Step 5**：选 A，自证：B+ 叶子按关键字链接为链表 → 支持顺序查找 → 仅 B+ 树如此 ✅

### 【题型 2】B 树最大 / 最小高度公式计算（题号：8、9、10）

#### 核心知识：两条高度公式（必须刻进 DNA）

| 方向 | 公式 | 记忆要诀 |
|---|---|---|
| **最小高度**（结点最密、每结点塞满） | `h ≥ log_m (n+1)` | **底 m** —— 让每层塞到最满 |
| **最大高度**（结点最稀、每结点刚好最少） | `H ≤ log_⌈m/2⌉ ((n+1)/2) + 1` | **底 ⌈m/2⌉，参数先除2再 +1** |

**底层逻辑**：
- **最小高度**：每结点 m-1 关键字 → 层层塞满 → `n ≤ mʰ - 1` → `h ≥ log_m(n+1)`
- **最大高度**：每结点最少 ⌈m/2⌉-1 关键字（根只有1）+ 每层结点最少 → 推导得 `(n+1)/2 ≤ ⌈m/2⌉^(H-1)` → `H ≤ log_⌈m/2⌉((n+1)/2) + 1`

#### 第 08 题精析（答案 C、B）—— 题目核心主旨：5 阶 B 树 53 关键字

**推导链**：
- **最大高度**：`H ≤ log_⌈5/2⌉((53+1)/2) + 1 = log_3 27 + 1 = 3 + 1 = 4` → 最大高度 **4**
- **最小高度**：`h ≥ log_5(53+1) = log_5 54 ≈ 2.5` → 最小高度 **3**（向上取整）

**⚠️ 易错点**：两个公式的底（m vs ⌈m/2⌉）和参数（n+1 vs (n+1)/2）极易混淆。

**⚠️ 推论**：**"最大高度公式底小，最小高度公式底大"** —— 因为底越小算出来越大。

#### 第 09 题精析（答案 A、D）—— 题目核心主旨：3 阶 B 树 2047 关键字

**推导链**：
- **最大高度**：`H ≤ log_⌈3/2⌉((2047+1)/2) + 1 = log_2 1024 + 1 = 10 + 1 = 11` → **11**
- **最小高度**：`h ≥ log_3 2048 ≈ 6.9` → 向上取整 **7**

**⚠️ 注意**：答案解析中特别提醒 —— 某些辅导书把"不含关键字的最底层叶结点"也算进高度，会多算 1。**王道/统考以含关键字的最底层结点为最底层**。

**⚠️ 推论**：**高度定义以"含关键字结点"为准**，失败结点层不计入高度。

#### 第 10 题精析（答案 B）—— 题目核心主旨：7 阶 B 树第 2016 个关键字的最大深度

**推导链**（最坏情况 = 每结点关键字最少）：
- 根至少 1 关键字、2 子树
- 第 2 层至少 2 结点，每结点至少 ⌈7/2⌉-1 = 3 关键字 → 2×3 = 6 关键字
- 第 3 层至少 2×⌈7/2⌉ = 2×4 = 8 结点，每结点 3 关键字 → 8×3 = 24 关键字
- 第 h 层至少 `2×4^(h-2)` 结点 → `2×4^(h-2)×3` 关键字
- 前 h 层关键字总数：`n = 1 + 2×3×(1+4+…+4^(h-2)) = 1 + 2×(4^(h-1) - 1)`
- h=5：n=511；h=6：n=2047
- 2016 > 511 且 2016 < 2047 → **第 2016 个关键字最早出现在第 6 层** → **启动 5 次 I/O**（根已读入内存）

**⚠️ 易错点**：I/O 次数 = 层数 - 1（根已在内存）。

**⚠️ 推论（超级有用）**：**B 树搜索的 I/O 次数 = 目标所在层 - 1**（若根驻留内存）。

#### 📋 【做题 SOP · 题型 2】B 树最大/最小高度计算

**⏱️ 建议用时**：45 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★☆☆☆

**🎯 识别特征**：
- "含 n 个关键字的 m 阶 B 树，最大/最小高度是"
- "至少需要 ___ 次 I/O"
- "高度为 h 的 m 阶 B 树至少/至多 ___"

**🔧 标准步骤**：
- **Step 1**：**读题圈出** m、n、方向（最大/最小）→ 目的：确定套哪个公式
- **Step 2**：**求最小高度**套 `h ≥ log_m(n+1)`，向上取整 → 目的：对应"塞满"场景
- **Step 3**：**求最大高度**套 `H ≤ log_⌈m/2⌉((n+1)/2) + 1`，向下取整 → 目的：对应"最稀"场景
- **Step 4**：**若求 I/O 次数**：I/O = 层数 - 1（根驻留内存时）→ 目的：避免多算 1
- **Step 5**：**校验结果合理性**（最大 ≥ 最小）→ 目的：防计算错位

**🪤 踩坑预警**：
- Step 1 易踩：把"关键字数 n"错算成"结点数"
- Step 2 易踩：底数用成 ⌈m/2⌉（那是最大高度公式的底）
- Step 3 易踩：忘记 +1 或把 (n+1)/2 写成 n+1
- Step 4 易踩：忘记根驻留内存，多算 1 次 I/O
- Step 5 易踩：log 取整方向搞反（min 向上取整、max 向下取整）

**✅ 自检口诀**：**"min 用 m，max 用 半阶"**

**🎯 一题实战演示**（第 08 题）：
- **Step 1**：m=5，n=53，求最大和最小高度
- **Step 2**：最小 h ≥ log_5 54 ≈ 2.5 → **向上取整 = 3**
- **Step 3**：最大 H ≤ log_3 27 + 1 = 3 + 1 = **4**
- **Step 4**：本题不求 I/O，跳过
- **Step 5**：最大 4 ≥ 最小 3 ✅ → 选 **C、B**

### 【题型 3】B 树结点数 / 关键字数 / 高度极值计算（核心防坑重构版）

#### 🔴 前置断言：计算极值前，先判断是“定高度”还是“定总数”！

**场景 A：【定高度 $h$ 】（房东视角：楼层已定，算能住多少人/开多少房）**
*物理常识：楼层锁死时，每个房间塞满，总人数和总房间数就最多；每个房间只住最低限额，总人数和房间数就最少。*

| 极值类型 | 底层内存状态 (物理形态) | 408 核心推导公式 ($m$阶, $h$层) |
| :--- | :--- | :--- |
| **关键字 / 结点 最多** | **【贪吃蛇满载模式】**：结点全塞满 $m-1$ 个键，生 $m$ 个杈 (满 $m$ 叉树) | 结点最多 = $(m^h-1)/(m-1)$ <br> 关键字最多 = $m^h-1$ |
| **关键字 / 结点 最少** | **【穷鬼卡低保模式】**：根节点保底 1 个键 (2 杈)；其余所有结点全卡下限 $\lceil m/2 \rceil - 1$ 个键 (瘦竹竿树) | 关键字最少 = $2(\lceil m/2 \rceil)^{h-1} - 1$ <br> *(注：通常直接考这棵最瘦树的具体节点长啥样，少考总节点数公式)* |

---

**场景 B：【定关键字总数 $N$ 】（租客视角：100人必须住下，算要盖多高/开几个房）**
*物理常识：总人数锁死时，每个房间塞满，开的房间就少，楼就矮；每个房间非要卡最低限额，就逼着你疯狂开新房，楼就得盖得极高！*

| 极值类型 | 底层内存状态 (物理形态) | 408 核心推导公式 ($m$阶, $N$个键) |
| :--- | :--- | :--- |
| **结点最少 / 高度最低** | **【满载低楼层】**：每个房间全按上限 $m-1$ 塞满，最省空间。 | 高度 $h \ge \log_m(N+1)$ |
| **结点最多 / 高度最高** | **【低保高层塔】**：每个房间只住最少人数，逼着树疯狂向上“生长”。 | 高度 $h \le \log_{\lceil m/2 \rceil}(\frac{N+1}{2}) + 1$ |

#### 🟢 B 树关键字与叶结点（失败结点）物理映射关系
**核心铁律：$N$ 个关键字 $\rightarrow$ 必定对应 $N+1$ 个叶结点（失败空指针）**
*物理推演：切香肠定理。把整个查找范围看作一根香肠，你在树里插入了 $N$ 个有效关键字（切了 $N$ 刀），必定会切出 $N+1$ 个落空的区间（空指针）。*

**⚠️ 易错点**：把失败结点（叶）与终端结点（最底层含关键字结点）混淆。

**⚠️ 推论**：**任何含 n 关键字的 B 树都有 n+1 个叶结点**（叶=失败结点语境下）。

**⚠️ 推论**：涉及"最少关键字"时，**根贡献 1，其余每个 ⌈m/2⌉-1**。

#### 📋 【做题 SOP · 题型 3】B 树关键字/结点数极值计算

**⏱️ 建议用时**：60 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★★☆☆

**🎯 识别特征**：
- "高度为 h 的 m 阶 B 树至少 / 至多含 ___ 个关键字 / 结点"
- "含 n 个关键字的 m 阶 B 树最多 / 最少结点数"
- "第 k 层最多 / 最少关键字数"

**🔧 标准步骤**：
- **Step 1**：**圈出四要素**：m、h（或 n）、问"多"还是"少"、问"关键字"还是"结点"→ 目的：选对极端场景
- **Step 2**：**若求"最少关键字"**：根=1，非根每结点=⌈m/2⌉-1 → 推导得瘦高型（类 ⌈m/2⌉ 叉满树）→ 目的：用下界
- **Step 3**：**若求"最多关键字"**：每结点=m-1，按 mʰ-1 上限估算 → 目的：用上界
- **Step 4**：**逐层累加**：第 1 层 1 结点、第 2 层 ≥2 或 =m 结点、第 k 层 ≥2×⌈m/2⌉^(k-2) 或 =m^(k-1) 结点 → 目的：精确计数
- **Step 5**：**关键字 vs 结点**：关键字 ≈ 结点数 × 每结点关键字数；**n 关键字 → n+1 叶子**

**🪤 踩坑预警**：
- Step 1 易踩：把"结点"读成"关键字"，或反之
- Step 2 易踩：以为非根结点最少关键字是 ⌈m/2⌉（其实是 ⌈m/2⌉-1）
- Step 3 易踩：忘记每结点 m-1 关键字上限
- Step 4 易踩：第 2 层结点数受根约束（根子树数 ≤ m）
- Step 5 易踩：问"叶子数"时答成"终端结点数"（叶=失败结点=关键字数+1）

**✅ 自检口诀**：**"少瘦高，多满塞"**

**🎯 一题实战演示**（第 20 题 · 2018 统考）：
- **Step 1**：m=3，h=5，问最少关键字数
- **Step 2**：根=1 关键字，其他非叶=⌈3/2⌉-1=1 关键字 → 每结点都是 1 关键字、2 子树
- **Step 3**：跳过（本题问最少）
- **Step 4**：5 层满二叉树 = 2⁵-1 = 31 结点
- **Step 5**：每结点 1 关键字 → 共 **31 关键字** → 选 **B**

### 【题型 4】B 树插入操作与分裂传递（题号：4、11、21、综合01、综合02-part1）

#### 核心知识：插入流程与分裂传递规则

**B 树插入标准流程**：
1. **定位**：从根搜到目标关键字应插入的**最底层终端结点**
2. **直接插入**：若该结点关键字数 < m-1，直接插入并保持有序 → 结束
3. **触发分裂**：若插入后关键字数 = m，**溢出**，分裂该结点
4. **分裂规则**：取中间关键字（第 ⌈m/2⌉ 位置）**上升到父结点**，左右两侧分别成为两个新结点
5. **递归传递**：若父结点因上升也溢出 → 继续分裂向上传递
6. **极端情况**：分裂传递到根 → 根也分裂 → **树高 +1**（唯一增高途径）

#### 分裂触发条件表

| m 阶 B 树 | 结点最多关键字 (m-1) | 插入后分裂阈值 | 分裂点（第几关键字上升） |
|---|---|---|---|
| 3 阶 | 2 | 插入后 = 3 | 第 2 个（⌈3/2⌉=2） |
| 4 阶 | 3 | 插入后 = 4 | 第 2 个（⌈4/2⌉=2） |
| 5 阶 | 4 | 插入后 = 5 | 第 3 个（⌈5/2⌉=3） |

#### 第 11 题精析（答案 B）—— 题目核心主旨：插入引起高度为 h 的 B 树变化的最坏情况

**推导链**（最坏情况 = 分裂传到根 + 根也分裂 → 树高 +1）：
- **A**：若插入导致高度 +1，必然从最底层传到根 ✅

### 【题型 5】B 树删除操作与借兄弟/合并（题号：4（删除部分）、15、23、综合02-part2）

#### 核心知识：删除流程与下溢处理规则

**B 树删除标准流程**：
1. **定位关键字**
2. **情况 A：关键字在叶结点（最底层含关键字结点）**
   - 直接删除
   - 若删后该结点关键字数 < ⌈m/2⌉-1 → **下溢**，进入处理流程
3. **情况 B：关键字在内部结点** —— **先换再删**
   - 用该关键字的前驱（左子树最大）或后继（右子树最小）**替代**它
   - 然后转化为"删除叶结点的前驱/后继"→ 回到情况 A

**下溢处理三选一**：
- **借左兄弟**：若左兄弟关键字数 > ⌈m/2⌉-1（即有富余），把父的分隔关键字下移到当前结点，把左兄弟最大关键字上移到父
- **借右兄弟**：同理，对称操作
- **合并**：若左右兄弟都无富余 → 把当前结点、父的分隔关键字、某一兄弟**合并成一个新结点**；父少一个关键字，可能**递归下溢**

#### 删除触发条件表

| m 阶 B 树 | 最少关键字 (⌈m/2⌉-1) | 删除后下溢阈值 | 判断兄弟可借 |
|---|---|---|---|
| 3 阶 | 1 | 删后 = 0 | 兄弟关键字数 ≥ 2 |
| 4 阶 | 1 | 删后 = 0 | 兄弟关键字数 ≥ 2 |
| 5 阶 | 2 | 删后 = 1 | 兄弟关键字数 ≥ 3 |

#### 📋 【做题 SOP · 题型 5】B 树删除与借/合并

**⏱️ 建议用时**：120 秒（综合题） ｜ **考频**：★★★★☆ ｜ **难度**：★★★★★

**🎯 识别特征**：
- "删除关键字 ___，画出新 B 树"
- "删除后根结点中关键字序列是"
- "下列 ___ 不可能是 T₁ 的根结点序列"

**🔧 标准步骤**：
- **Step 1**：**定位关键字所在结点**（叶 or 内部）→ 目的：分情况处理
- **Step 2**：**若在内部结点 → 先换再删**：用前驱（左子最大）或后继（右子最小）替代，然后删替代的原位置（转化为叶情况）→ 目的：统一到叶处理
- **Step 3**：**执行叶结点删除**，检查是否下溢（关键字数 < ⌈m/2⌉-1）→ 目的：决定是否处理
- **Step 4**：**下溢处理三步判断**：
  - 左兄弟关键字数 ≥ ⌈m/2⌉ → **借左兄弟**（父分隔下移、兄弟最大上移）
  - 右兄弟关键字数 ≥ ⌈m/2⌉ → **借右兄弟**（对称）
  - 都不够 → **合并**（当前 + 父分隔 + 某兄弟 → 新结点；父减 1 关键字）
  → 目的：保持 B 树性质
- **Step 5**：**若合并导致父下溢，递归重复 Step 4** → 目的：处理连锁合并
- **Step 6**：**若根结点变空，新根 = 唯一子结点，树高 -1** → 目的：处理高度变化

**🪤 踩坑预警**：
- Step 1 易踩：忘记删内部结点需先换再删，直接在内部删会破坏性质
- Step 2 易踩：用前驱就统一用前驱、用后继就统一用后继，别混用
- Step 3 易踩：下溢阈值记错（3 阶下溢 = 0 关键字，5 阶下溢 = 1 关键字）
- Step 4 易踩：借兄弟时**不能直接把兄弟的关键字搬过来**，必须通过父中转
- Step 5 易踩：合并后忘记检查父是否下溢
- Step 6 易踩：根下溢不合并，直接变新根（若根只剩 0 关键字）

**✅ 自检口诀**：**"能借则借，不借则并，并传至根"**

**🎯 一题实战演示**（综合题 02 · 删 80）：
- **Step 1**：80 在叶 `[80]`
- **Step 2**：跳过（不在内部）
- **Step 3**：删 80 → 空结点，下溢
- **Step 4**：左兄弟 `[45]` 仅 1 关键字（3 阶需 ≥ 2 才能借）→ 不够借；右兄弟 `[100]` 仅 1 关键字 → 不够借 → **合并**：空 + 父分隔 90 + 右兄弟 `[100]` → 新结点 `[90, 100]`
- **Step 5**：父 `[90]` 变空（下溢）→ 递归处理：父的左兄弟 `[40]` 仅 1 关键字 → 不够借 → 再合并：空 + 祖父分隔 50 + 左兄弟 `[40]` → 新结点 `[40, 50]`；祖父 `[30, 50]` 变 `[30]`
- **Step 6**：根 `[30]` 1 关键字，未下溢（3 阶根下界 = 1），无需高度调整
- **最终**：根`[30]`，子`[20][40,50]`，叶`[8][25][35][45][90,100]`

#### 第 19 题精析（答案 B · 2017 统考）—— 题目核心主旨：B+ 树适合的应用

**推导链**：
- **A 编译器词法分析** —— 用自动机和语法树，不是 B+ 树 ❌
- **B 关系数据库系统中的索引** —— B+ 树**最经典应用**，磁盘读写代价低、支持范围查询 ✅
- **C 网络路由表快速查找** —— 主要靠高速缓存、压缩、专用查找算法 ❌

### 模板 1：B 树插入与分裂传递（伪代码）

```c
// 输入：B 树 T，待插关键字 k；m 阶 B 树
void BTreeInsert(BTree T, KeyType k) {
    Node *leaf = SearchLeaf(T, k);        // ① 从根搜到应插入的叶（最底层终端结点）
    InsertKeyInOrder(leaf, k);            // ② 保持升序插入到 leaf
    
    Node *cur = leaf;
    while (cur->keyNum == m) {            // ③ 关键字数达到 m → 溢出
        int mid = (m + 1) / 2;            //    分裂点 = 第 ⌈m/2⌉ 个关键字
        KeyType upKey = cur->keys[mid-1]; //    上升关键字
        Node *rightNew = SplitRight(cur, mid); // ④ 右半新结点
        // cur 保留左半（前 mid-1 个关键字）
        
        if (cur == root) {                // ⑤ 根溢出 → 生成新根，树高 +1
            Node *newRoot = NewNode();
            newRoot->keys[0] = upKey;
            newRoot->children[0] = cur;
            newRoot->children[1] = rightNew;
            root = newRoot;
            break;
        } else {                          // ⑥ 把 upKey 插入父结点
            Node *parent = cur->parent;
            InsertKeyAndChild(parent, upKey, rightNew);
            cur = parent;                 // 继续检查父是否溢出
        }
    }
}
```

**关键要点**：
- **每次分裂都在中间位置切开**，上升的关键字是**第 ⌈m/2⌉ 个**
- **分裂链式传递**，从叶一路向上，最极端情况传到根
- **唯一使树高 +1 的途径** = 根结点也发生分裂

### 模板 2：B 树删除与借/合并（伪代码）

```c
void BTreeDelete(BTree T, KeyType k) {
    Node *cur = SearchNode(T, k);         // ① 定位 k 所在结点
    
    if (!IsLeaf(cur)) {                   // ② 内部结点 → 先换再删
        Node *predLeaf = FindPredecessor(cur, k);  // 找 k 的前驱（左子树最大）
        Swap(k, predLeaf);                //    交换到叶
        cur = predLeaf;                   //    转成删叶情况
    }
    
    RemoveKey(cur, k);                    // ③ 从叶直接删除
    
    while (cur != root && cur->keyNum < ceil(m/2.0) - 1) {  // ④ 下溢处理
        Node *parent = cur->parent;
        Node *leftSib = LeftSibling(cur);
        Node *rightSib = RightSibling(cur);
        
        if (leftSib && leftSib->keyNum > ceil(m/2.0) - 1) {
            BorrowFromLeft(cur, parent, leftSib);       // ⑤ 借左兄弟
            break;
        } else if (rightSib && rightSib->keyNum > ceil(m/2.0) - 1) {
            BorrowFromRight(cur, parent, rightSib);     //    借右兄弟
            break;
        } else {                          // ⑥ 合并
            if (leftSib) Merge(leftSib, parent, cur);   // 合并左兄+父分隔+当前
            else        Merge(cur, parent, rightSib);   // 合并当前+父分隔+右兄
            cur = parent;                 // 递归向上
        }
    }
    
    if (root->keyNum == 0 && !IsLeaf(root)) {  // ⑦ 根变空，新根=唯一子，树高 -1
        root = root->children[0];
    }
}
```

**关键要点**：
- **借兄弟 = 父与兄弟的一次关键字交换**（不是直接搬运）
- **合并会消耗父的一个关键字**，可能引发父下溢 → 连锁向上
- **仅当根变空时树高 -1**

### 模板 3：B 树作文件索引的阶数计算（综合题 03 推导）

**题目要点**：磁盘页 4000B，指针 5B，关键字 5B，记录 200B，总 2×10⁷ 条记录。

**推导步骤**：
- 一个 B 树结点（=一个磁盘页）容量：`(2×(m-1) + m) × 5 + 2 ≤ 4000`
  - `2×(m-1)` = `m-1 个关键字 + m-1 个记录地址`（每个 5B）
  - `m` = m 个子树指针（每个 5B）
  - `+ 2` = 存储当前结点关键字数 n 的 2B 整数
- 化简：`15m - 8 ≤ 4000` → `m ≤ 267.2` → **m = 267**

**计算索引结点数**：
- 一个索引结点最多存 m-1 = 266 个索引项，最少存 ⌈m/2⌉-1 = 133 个索引项
- 2×10⁷ 条记录，每个页块装 4000/200 = **20 条记录** → 共 2×10⁷/20 = **10⁶ 个页块**
- **最多需要** 10⁶ / 133 ≈ **7519** 个磁盘页块作索引（每索引项对应一个页块，一个索引结点最少装 133 项时索引块最多）
- **最少需要** 10⁶ / 266 ≈ **3760** 个磁盘页块作索引（每索引结点装满时索引块最少）

**⚠️ 易错点**：
1. 一个结点的空间公式必须包含**关键字数 + 记录地址 + 子指针 + n 自身**四部分
2. 注意 B 树的索引项分布在各层次（不同于 B+ 树集中在叶）

**⚠️ 推论（超级有用）**：**B 树阶数 m 由一次磁盘 I/O 读入的页块容量决定** —— 硬件决定理论上界。

| **根下溢** | 根剩 0 关键字 | 合并后的唯一子 = 新根 | 树高 -1 |
