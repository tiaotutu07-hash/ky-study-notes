## ═══ Part A · BST 二叉排序树 ═══

## 【题型1】BST 基本性质与定义（中序有序、大小关系、最值）（选01、选02、选04、选30【2018统考】、选34【2024统考】）

### 核心知识

| BST 性质 | 说明 |
|---------|------|
| **定义（递归）** | 左子树所有值 < 根 < 右子树所有值；左右子树也是 BST |
| **中序遍历结果** | 严格递增序列（无重复） |
| **最小关键字** | **最左下结点**（一路沿 lchild 到底） |
| **最大关键字** | **最右下结点**（一路沿 rchild 到底） |
| **最大值结点的右指针** | **一定为空**（否则右边还有更大的值） |
| **最小值结点的左指针** | **一定为空** |
| **插入新结点** | 总是作为**新叶子**加入，不引起树的"分裂重组" |

**关键推论**：
- BST **中序有序**是判断一棵树是否 BST 的充要条件
- 任一结点 v 的**左子树**最大值 < v < **右子树**最小值

---

### 第01题精析（C）—— BST 性质辨析

**解析推导链**：
- A 错：BST 插入新结点**不会**引起树的"分裂组合"（那是 B 树的操作）→ BST 插入只是挂一个新叶子
- B 错：BST 进行**中序遍历**（不是层序）才能得到有序序列
- C 对：当插入的关键字**有序**时，BST 退化为**单支树**（线性链），深度最大 = n
- D 错：BST 最坏情况（单支树）查找要比较 n 次，**可能超过**结点数的 1/2

**⚠️ 推论（超级有用）**：**关键字有序插入 → BST 必定退化为单支树** → 深度 n，查找 O(n)。

---

### 第02题精析（B）—— BST 遍历得到有序序列

**解析推导链**：由 BST 定义（左 < 根 < 右）→ 中序遍历 L-N-R 的结果必为升序。

**⚠️ 推论**：**"BST 中序 = 有序序列"** 是出镜率最高的铁律。反过来：任给有序数组，可**唯一**确定其"按折半查找方式"构造的 BST。

---

### 第04题精析（B）—— BST 最大值结点的指针状况

**解析推导链**：
- BST 中**最大值结点位于最右下** → 其右指针**一定为空**
- **反证法**：若右指针不为空，则右子树中存在比"原最大值"更大的值 → 矛盾
- 左指针不一定为空（最大值可能有左子树）

**⚠️ 易错点**：错选 C "左右指针均为空" → 认为最大值一定是叶子。**错**：最大值只需"右指针为空"，它可以有左孩子。

**⚠️ 推论**：**BST 中最大值 → 右指针必空**；**最小值 → 左指针必空**。

### 第30题精析（C）—— 2018 统考：给定 BST 图求大小关系

**原题**：BST 如下图所示，求元素大小关系。

```
        x₁
          \
           x₂
          /
         x₃
          \
           x₄
           /
          x₅
```

**解析推导链**：BST 中序遍历 LNR → 递增序列。

对本图做中序遍历：
- 从 x₁ 开始：左空 → 访问 x₁ → 进入右子树（x₂ 为根）
- x₂ 的左子树根是 x₃ → 进入 x₃ 的左子树（空）→ 访问 x₃ → x₃ 的右子树（x₄ 为根）
- x₄ 的左子树根是 x₅ → 进入 x₅ 的左子树（空）→ 访问 x₅ → x₅ 右空 → 回退
- 访问 x₄ → x₄ 右空 → 回退 → 访问 x₂ → x₂ 右空 → 结束

**中序序列：x₁, x₃, x₅, x₄, x₂**

由 BST 中序升序 → **x₁ < x₃ < x₅ < x₄ < x₂** → 局部关系 **x₃ < x₅ < x₄** → 选 C。

**⚠️ 推论（超级有用）**：**遇 BST 图推大小关系 → 闭眼做中序遍历 → 中序序列就是升序序列**。

### 第34题精析（D）—— 2024 统考：BST 子树抽象大小关系

**原题**：BST 根为 k₁，k₂ 是 k₁ 的右孩子，k₃ 是 k₂ 的左孩子；子树 T 挂在 k₃ 的右孩子。求 T 中任一结点 x 与 k₁、k₂、k₃ 的关系。

**解析推导链**：
- k₂ 在 k₁ 的右子树 → **k₂ > k₁**
- k₃ 在 k₂ 的左子树 → **k₃ < k₂**
- k₃ 又在 k₁ 的右子树下 → **k₃ > k₁**
- 综合：**k₁ < k₃ < k₂**

- T 是 k₃ 的**右子树** → T 中任一 x **> k₃**
- T 又在 k₂ 的左子树内 → T 中任一 x **< k₂**
- 综合：**k₃ < x < k₂** → 选 D

**⚠️ 易错点**：直接写 **x > k₁** 或 **x < k₂** 都不够精细——题目要的是**最紧的区间**。

**⚠️ 推论（超级有用）**：**抽象 BST 推理 → 沿从根到 T 的路径逐层缩窄区间**。左拐时 x < 当前根；右拐时 x > 当前根。T 最终落在 `(max 左拐前祖先, min 右拐前祖先)` 区间内。

### 📋 【做题 SOP · 题型1】BST 性质推理

**⏱️ 建议用时**：40 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★☆☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP）：
- "二叉排序树" / "BST" / "二叉搜索树"
- 给一张抽象 BST 图求元素关系
- 求 BST 中最大值 / 最小值 / 某结点指针状况

**🔧 标准步骤**：
- **Step 1**：确认是 BST（题目已给定或隐含）→ 立即调用"中序递增"铁律
- **Step 2**：求大小关系 → 做**中序遍历** LNR → 所得序列即升序序列
- **Step 3**：求最值 → **最小一路向左、最大一路向右**；最值结点的对应侧指针**必空**
- **Step 4**：抽象推理（如 2024 真题）→ 沿根到目标路径**逐层缩窄区间**：左拐缩上界、右拐缩下界

**🪤 踩坑预警**：
- Step 1 易踩：题干说"平衡二叉树"不等于 BST（2015 真题陷阱）
- Step 2 易踩：做成先序或后序遍历
- Step 3 易踩：最值结点误认为"叶子"（最大值可以有左孩子）
- Step 4 易踩：漏掉"同时需满足所有祖先的约束"，只看直接父亲

**✅ 自检口诀**：**"BST 中序升序，左小右大记牢"**

**🎯 一题实战演示（第34题 · 2024 统考）**：
- Step 1：题目明确是 BST ✅
- Step 4（抽象推理）：
  - T 沿 k₁ → k₂：右拐 → x > k₁
  - T 沿 k₂ → k₃：左拐 → x < k₂
  - T 沿 k₃ → T：右拐 → x > k₃
  - 综合三条约束：**k₃ < x < k₂**（k₁ 的约束 x > k₁ 已被 x > k₃ 吸收，因为 k₃ > k₁）
- **答案 D：k₃ < x < k₂** ✅

## 【题型2】BST 查找路径合法性判别（选05、选25【2011统考】、选32【2020统考】）

### 核心知识

| 判据 | 说明 |
|------|------|
| **查找路径性质** | 沿 BST 自顶向下，每次比较后向左或向右 |
| **合法路径充要条件** | 每次转向后的下一个结点必须在**当前允许区间**内 |
| **区间维护** | 初始区间 (-∞, +∞)；向左走则 high=当前根；向右走则 low=当前根 |
| **违规信号** | 出现"左拐后的结点 > 刚刚走过的某个右拐祖先"等矛盾 |

**核心方法：画图法** —— 把给定序列按"根→路径"画出来，检查是否出现违规。

---

### 第05题精析（C）—— 在 BST 中查 363，不可能的比较序列

**解析推导链**：BST 中查找时，先与根比较，小则转左子树，大则转右子树。检查每个选项对应的路径是否合法（每一步转向必须符合 BST 性质）。

- C: 925, 202, 911, 240, 912, 245, 363
  - 925 → 查 363 < 925 → 转左子树
  - 202 < 363 → 转 202 的右子树
  - 911 → 363 < 911 → 转左子树（911 的左子树所有值应 < 911）
  - 240 < 363 → 转 240 的右子树（240 的右子树所有值应 > 240 且 < 911）
  - **912 出现**！但 912 > 911，而 912 位于 911 的左子树中（违反 BST）→ **错误** ❌

- A、B、D 均可构造合法路径，不违反 BST。

**答案 C**。

**⚠️ 推论**：**BST 查找路径每一步转向后，新结点必须在"祖先约束区间"内**——只要出现越界，即为非法序列。

### 第25题精析（A）—— 2011 统考：不可能构成 BST 查找路径

**原题四个序列**：
- A: 95, 22, 91, 24, 94, 71
- B: 92, 20, 91, 34, 88, 35
- C: 21, 89, 77, 29, 36, 38
- D: 12, 25, 71, 68, 33, 34

**解析推导链**：画查找路径图逐个检查。

**选项 A**：
- 95 → 查小于 95 → 转左子树
- 22 → 查大于 22 → 转右子树（22 的右子树所有值应在 (22, 95) 内）
- 91 → 查小于 91 → 转左子树（此时路径中应该都比 91 小，且比 22 大）
- 24 → 查大于 24 → 转右子树（24 的右子树所有值应在 (24, 91) 内）
- 94 → ❌ **94 > 91**，与"此处应 < 91"矛盾 → **违反 BST**

选项 B、C、D 检验均合法（可画出 BST）。

**答案 A**（不可能）。

**⚠️ 易错点（高频）**：只看"序列是否递增/递减"，忽略"路径转向后区间约束"。

**⚠️ 推论（超级有用）**：**维护一个"允许区间" (low, high)**；遇大转右 → low 更新为当前值；遇小转左 → high 更新为当前值。**违规 = 后续数值越界**。

### 第32题精析（B）—— 2020 统考：不能生成给定 BST 的输入序列

**原题**：给定目标 BST（根=4，左子树=2-1-3，右子树=5），问哪个输入序列**不能**生成该 BST。

**四个选项**：
- A: 4, 5, 2, 1, 3
- B: 4, 5, 1, 2, 3
- C: 4, 2, 5, 3, 1
- D: 4, 2, 1, 3, 5

**解析推导链**：对每个序列按 BST 插入规则构建，看是否与目标 BST 一致。

**选项 B**：4, 5, 1, 2, 3
- 插 4 → 根
- 插 5 → 4 的右孩子（5 > 4）✓
- 插 1 → 4 的左孩子（1 < 4）✓
- 插 2 → 1 的右孩子（2 > 1；在 4 的左子树，在 1 的右孩子位）✓
- 插 3 → 2 的右孩子（3 > 2）✓

但目标 BST 要求 **2 是 4 的左孩子**、**1 是 2 的左孩子**、**3 是 2 的右孩子**。选项 B 生成的树是：**1 是 4 的左孩子**、**2 是 1 的右孩子**、**3 是 2 的右孩子** → **结构不同** ❌

**答案 B**。

**⚠️ 推论**：**BST 同构判别 → 比较"每个结点的父子关系"**，不能只看中序序列（中序一致只能说明都是 BST，不代表同构）。

### 📋 【做题 SOP · 题型2】BST 路径合法性判别

**⏱️ 建议用时**：60 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★★☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP）：
- "下列哪个序列**不可能**是 BST 的查找路径"
- "下列输入序列**不能**生成此 BST"
- "**可能/不能**构成二叉排序树…"

**🔧 标准步骤**：
- **Step 1**：设定初始允许区间 `(low, high) = (-∞, +∞)`
- **Step 2**：读序列第一个元素 x₁（根），验证 x₁ ∈ (low, high)
- **Step 3**：比较 x₂ 与 x₁：若 x₂ < x₁ → 向左转，new (low, high) = (low, x₁)；若 x₂ > x₁ → 向右转，new (low, high) = (x₁, high)
- **Step 4**：继续读下一元素 xᵢ，**两种可能**：
  - xᵢ 在**当前路径的延续**上：若 xᵢ 在当前区间内，则继续按 Step 3 更新区间
  - xᵢ 是**祖先兄弟位置**：此时 xᵢ 必须违反区间——立即判"非法"
- **Step 5**：若序列走完都没越界 → 合法；某一步越界 → **不可能是 BST 路径** → 答这个选项

**🔧 标准步骤（"能否生成此 BST" 版，针对第32题）**：
- **Step 1**：按序列顺序**逐个**插入到初始空 BST
- **Step 2**：每次插入都按 BST 规则（小走左、大走右，空位则挂上）
- **Step 3**：构造完成后，**逐个结点**比较父子关系与目标 BST 是否一致
- **Step 4**：若全等 → 可生成；有一处父子关系不同 → **不能生成**

**🪤 踩坑预警**：
- Step 3 易踩：只更新 low 或 high 其中一个，漏掉另一个
- Step 4 易踩：把"同中序"当成"同 BST"（它俩是两件事）
- 通用易踩：画路径时把"兄弟结点"画成"父子结点"

**✅ 自检口诀**：**"区间逐步缩，越界即违规"**

**🎯 一题实战演示（第25题 · 2011 统考 · 选项 A）**：
- Step 1：初始 (low, high) = (-∞, +∞)
- Step 2：x₁=95 ∈ (-∞, +∞) ✓
- Step 3：查 363 > 95？不，题目未给目标；按路径"95→22"看：22 < 95 → 左转 → 新区间 (-∞, 95)
- Step 4：22 → 91：91 > 22 → 右转 → 新区间 (22, 95)
- 91 → 24：24 < 91 → 左转 → 新区间 (22, 91)
- 24 → 94：94 > 24 → 右转 → 新区间 (24, 91)
- **验证 94 ∈ (24, 91)？94 > 91，不在区间内 → 越界 ❌**
- **答案 A：不可能是 BST 查找路径** ✅

## 【题型3】BST 构造与查找过程模拟（选06、选07）

### 核心知识

**BST 构造规则**：依次读取序列，每个元素按"小走左、大走右"规则插入，空位即为其位置。

**关键性质**：
- 不同输入序列**可能**生成**相同** BST（中序都升序但插入顺序不同）
- 所谓"不同 BST"往往指**形态不同**（父子关系不同）

---

### 第06题精析（C）—— 哪个序列构造的 BST 与其他 3 个不同

**解析推导链**：

构造 A、B、D 序列的 BST，再与 C 对比。

**序列 A: (100, 80, 90, 60, 120, 110, 130)**：
```
       100
      /   \
     80   120
      \   / \
      90 110 130
     /
    60
```

**序列 B: (100, 120, 110, 130, 80, 60, 90)**：构造过程与 A 产生的是**相同的树结构**（中序+结构都一致）。

**序列 C: (100, 60, 80, 90, 120, 110, 130)**：
- 插 100 → 根
- 插 60 → 左（60 < 100）
- 插 80 → **60 的右孩子**（80 > 60，在 100 的左子树）
- 插 90 → 80 的右孩子
- ...
结果：60 位置与 A 的 80 在 100 左子树下作**根**不同 → 树形完全不同

**序列 D: (100, 80, 60, 90, 120, 130, 110)**：结构与 A 相同。

**答案 C**。

**⚠️ 推论**：**BST 的形态由"插入序列中每个元素所遇的第一个比它大/小的祖先"决定**——只要每个元素的"第一个比它大祖先"与"第一个比它小祖先"不变，树形就不变。

---

### 第07题精析（A）—— 插入序列后查 60 的比较次数

**原题**：从空开始依次插入 52, 26, 14, 32, 71, 60, 93, 58, 24, 41，构造 BST，查 60 的比较次数。

**解析推导链**：

构造 BST：

```
                52
              /    \
            26      71
           / \     /  \
          14  32  60   93
           \   \  /
           24  41 58
```

查找 60：从根 52 开始 → 60 > 52 → 右转；71 → 60 < 71 → 左转；60 == 60 → 命中

**共 3 次比较** → 选 A。

**⚠️ 推论**：**BST 查找比较次数 = 从根到目标结点的路径长度 + 1**（含根）。

### 📋 【做题 SOP · 题型3】BST 构造与查找

**⏱️ 建议用时**：90 秒 ｜ **考频**：★★★☆☆ ｜ **难度**：★★☆☆☆

**🎯 识别特征**（看到这些关键词 → 立即调用此 SOP）：
- "依次插入…构造二叉排序树"
- "在此 BST 中查找 X 的比较次数"
- "构造结果不同 / 相同"的多选判断

**🔧 标准步骤**：
- **Step 1**：以序列第一个元素作为**根**
- **Step 2**：依次读取下一个元素 x：与当前根比较，小则进左子树，大则进右子树
- **Step 3**：递归地重复 Step 2，直到遇到空位 → 在此处挂上 x
- **Step 4**：完成后验证——中序遍历应得升序（自检）
- **Step 5**：查找 X：从根起比较，路径上每个结点都算一次比较，直到命中或空

**🪤 踩坑预警**：
- Step 2 易踩：漏掉"在子树根处比较"那一步
- Step 4 易踩：不做中序校验，直接画完就答

## 【题型4】BST 查找效率与形态（选03、选08、选09、选10）

### 核心知识

| 量 | 公式 | 备注 |
|------|------|------|
| **查找效率决定因素** | **树的深度**（高度） | 非结点数 |
| **平均查找长度** | 与树形**密切相关** | 同样 n 个结点不同形态 ASL 不同 |
| **最坏查找次数** | n | 单支树（输入有序） |
| **最理想深度** | ⌈log₂(n+1)⌉ | 等价于"类完全二叉树"形态 |
| **n 个不同结点的 BST 形态数** | **Catalan(n) = C(2n, n) / (n+1)** | Catalan 数 |
| **Catalan(5) 典型值** | **42** | 常考 |

**Catalan 数速查**：C₀=1, C₁=1, C₂=2, C₃=5, C₄=14, C₅=42, C₆=132。

---

### 第03题精析（A）—— BST 查找效率的决定因素

**解析推导链**：BST 查找路径自顶向下，平均查找长度主要取决于**树的高度（深度）**——因为最坏路径长度就是深度。结点数、存储结构、某结点的度等都不是决定因素。

**⚠️ 推论**：**相同结点数的 BST 可能有多种形态**（从链到近似完全二叉树），查找效率跟"形态"强相关。

---

### 第08题精析（D）—— n 个结点 BST 查找某关键字的最多比较次数

**解析推导链**：
- 最坏情况：输入有序 → BST 退化为**单支树** → 查找一个不存在的值或最后一个结点 → 需比较 **n 次**
- A (n/2)、B (log₂n)、C (log₂n+1) 都是"最好/理想"情况，不是"最多"

**答案 D（n）**。

**⚠️ 推论**：**BST 最坏查找次数 = n**（单支链时发生）。

---

### 第09题精析（D）—— 5 个不同结点的 BST 形态数

**解析推导链**：
- 5 个不同结点的 BST，其中序序列是**固定**（因为 BST 中序必须升序）
- 给定固定中序，BST 的形态数 = 先序序列数 = **Catalan(n)**
- Catalan(5) = C(10, 5) / 6 = 252 / 6 = **42**

**答案 D（42）**。

**⚠️ 易错点**：误用 n! 或 2ⁿ → 这是"排列数"而非"Catalan 数"。

**⚠️ 推论（超级有用）**：**n 个不同结点的 BST 形态数 = Catalan(n)**；考场可死记：**C₃=5, C₄=14, C₅=42**。

---

### 第10题精析（D）—— n 结点 BST 最理想深度

**解析推导链**：
- 当 BST 叶结点全部分布在相邻两层时，深度最小（类完全二叉树）
- 类比完全二叉树深度公式：**深度 = ⌈log₂(n+1)⌉**

**答案 D**。

**⚠️ 推论**：**"最理想 BST 深度公式 = 折半查找判定树高度公式"**——两者本质都是"平衡的 BST"。

## 【题型5】AVL 定义与性质（含"反 BST"陷阱）（选11、选12、选23【2009统考】、选29【2015统考】）

### 核心知识

| AVL 性质 | 说明 |
|---------|------|
| **平衡因子 BF** | BF(v) = 左子树高度 - 右子树高度 ∈ {-1, 0, 1} |
| **平衡二叉树定义** | **每个**结点的平衡因子绝对值 ≤ 1 |
| **AVL 是否一定是 BST** | **不一定！** 王道教材将 AVL 定义为 "高度平衡的二叉排序树"，但 2015 真题题干却将其放宽为"只要求平衡" |
| **中序遍历** | 一般得到升序（AVL 默认带 BST 性质）；但若题目**明确**反 BST（如 2015 题），则中序可能降序 |
| **最小结点数递推** | nₕ = nₕ₋₁ + nₕ₋₂ + 1，n₀=0, n₁=1, n₂=2, n₃=4, n₄=7, n₅=12, n₆=20 |

### 第11题精析（C）—— 20 结点 AVL 的最大深度

**解析推导链**：问"20 个结点的 AVL **最大**深度" → 等价于问"给定深度 h 的 AVL **最少**结点数"中，最大的 h 使得 nₕ ≤ 20。

**递推计算**：
- n₁ = 1
- n₂ = 2
- n₃ = n₂ + n₁ + 1 = 2 + 1 + 1 = 4
- n₄ = n₃ + n₂ + 1 = 4 + 2 + 1 = 7
- n₅ = n₄ + n₃ + 1 = 7 + 4 + 1 = 12
- n₆ = n₅ + n₄ + 1 = 12 + 7 + 1 = **20** ✓
- n₇ = n₆ + n₅ + 1 = 20 + 12 + 1 = 33 > 20 ❌

所以 20 个结点构造的 AVL 高度**最多为 6**。

**答案 C（6）**。

**⚠️ 推论（超级有用）**：**AVL 最大深度题 → 递推查表 nₕ**，n₆=20 是常考锚点。

### 第29题精析（D）—— 2015 统考：AVL 中序降序的反常规陷阱 ★

**原题**：**现有一棵无重复关键字的平衡二叉树（AVL），对其进行中序遍历可得一个降序序列**。求叙述正确的是。

**解析关键**：题目设定 AVL 中序为**降序** → 相当于**反常规 BST**（左 > 根 > 右）。

**四个选项分析**：
- A：**根结点的度一定为 2** → 错。只有两个结点的 AVL 根度为 1。
- B：**树中最小元素一定是叶结点** → 错。中序降序 → 最小元素出现在**中序最后** → 即"最右下结点"，但它不一定是叶子（可能有左孩子）。
- C：**最后插入的元素一定是叶结点** → 错。最后插入的结点**可能触发调整**，经过旋转后变为非叶结点。
- D：**树中最大元素一定是无左子树** → 对！中序降序 → 最大元素在中序最**前** → 即"最左下结点" → 左子树必空（否则还有更大的在左边），**但可以有右子树**。

**答案 D**。

**⚠️ 易错点（极高频）**：
- 想当然认为 AVL 中序**一定升序** → 套常规 BST 性质做题 → 全错
- 把"叶结点"与"无左/右子树"混淆——无左子树 ≠ 叶子

**⚠️ 推论（超级有用 · 反常规）**：
- **AVL 不一定是 BST**！当题目明确给出反常规条件，必须按题目的"大小关系"重新推理
- **"最大值无左子树" ≠ "最大值是叶结点"**：最大值可以有右子树

## 【题型6】AVL 构造与旋转（LL/RR/LR/RL）★ 超高频（选14、选15、选24【2010统考】、选33【2021统考】）

### 核心知识

| 四种旋转模式 | 插入位置（相对最小不平衡子树根 A）| 调整方法 | 调整后根 |
|------------|---------------------------------|---------|---------|
| **LL 型**（左孩子的左子树插入）| A 的**左孩子 B** 的左子树 | **对 A 做右旋** | B |
| **RR 型**（右孩子的右子树插入）| A 的**右孩子 B** 的右子树 | **对 A 做左旋** | B |
| **LR 型**（左孩子的右子树插入）| A 的**左孩子 B** 的右孩子 C 之下 | **先 B 左旋，再 A 右旋** | C |
| **RL 型**（右孩子的左子树插入）| A 的**右孩子 B** 的左孩子 C 之下 | **先 B 右旋，再 A 左旋** | C |

**关键记忆**：
- **"最小不平衡子树"** = 离新插入结点**最近**、|BF| 首次变为 2 的祖先
- 确定旋转类型的"两字母代码" = 从最小不平衡子树根走到新插入结点的**前两步方向**
- LL/RR 只需**1 次旋转**；LR/RL 需要**2 次旋转**
- 旋转后**整个子树的高度不变**（关键：相邻祖先的 BF 不再变化）

---

### 第14题精析（A）—— 哪些操作可能引起 2 次旋转

**解析推导链**：
- 插入 → 可能引起 LR 或 RL 平衡旋转 → 2 次旋转
- 删除 → 也可能引起 LR 或 RL → 2 次旋转

**答案 A（添加、删除结点都可能）**。

**⚠️ 推论**：**插入、删除**都可能引起 2 次旋转；只是单纯的"平衡结点"操作才**不会**。

---

### 第15题精析（C）—— 1...1024 依次插入 AVL 的高度

**解析推导链**：
- 按关键字升序依次插入时，若插入的数是 **2^k - 1**（如 1, 3, 7, 15, 31, 63, ..., 1023），此时构建的 AVL **恰好是满二叉树**
- 插入 1023 时，AVL 为满二叉树 → 共 **1023** 个结点 → 高度 = **10**（从第 1 层到第 10 层，满树）

**注意**：王道原解析（第15题）说"插入 1023 时高度 9"——这是因为王道把**根算第 0 层**的惯例。按此惯例：高度 9 表示 10 层。

- 插入第 **1024 个**关键字后，AVL 高度 **+ 1** = **10**（按根第 0 层算）→ 这就是选 C

**答案 C（10）**。

**⚠️ 推论**：**连续 2^k - 1 个升序插入 → 恰为满二叉树**；再插一个就**+1 层**。

### 第24题精析（C）—— 2010 统考：插入 48 后 37 的左右孩子

**原题 AVL**：
```
       24
      /  \
    13    53
         /  \
        37   90
```

**插入 48**：

**Step 1**：按 BST 插入规则 → 48 > 24 → 53 → 48 < 53 → 37 → 48 > 37 → 37 的右孩子位 → 插入

**Step 2**：检查平衡因子自下而上：
- 48 BF=0
- 37 BF=0-1=-1
- 53 BF=(37的高度=2) - (90的高度=1) = 1
- 24 BF=1-3=**-2** ❌ **失衡！**

**Step 3**：24 是最小不平衡子树根。从 24 走到 48 的前两步是 **R（53）→ L（37）** → **RL 型**

**Step 4**：RL 调整 = 先对 53 右旋，再对 24 左旋

完整过程（王道图示）：
```
调整前:                    53右旋:                  24左旋最终:
      24                        24                       37
     /  \                      /  \                     /  \
    13   53                   13   37                  24   53
         / \                       / \                /  \  / \
        37  90                    NIL  53            13  NIL 48 90
         \                             / \
          48                          48  90
```

**Step 5**：最终 AVL 中 37 作为新根，其**左孩子 = 24**，**右孩子 = 53**。

**答案 C（24, 53）**。

**⚠️ 推论**：**RL 型旋转 → 调整后根 = B 的左孩子 C**（本题中 C = 48 是插入值，但 37 才是 B，注意别搞错）——**等等，本题中是 53 的左孩子 37 旋上去当根**，这是一个更细的版本，原解析是用"RL"描述但实际旋转后是**37**作根（因为 37 位于"53 的左子树"路径的前段）。

**⚠️ 超重要细节**：实际上本题 RL 旋转后根是 37（并非 48），因为 RL 型旋转时，B 的**左孩子 C** 成为新根——这里 B=53，C=37，新根=37 ✓。

### 第33题精析（D）—— 2021 统考：AVL 插入 23 后根的关键字

### 📋 【做题 SOP · 题型6】AVL 插入与旋转（四种类型判别）★★★★★

**⏱️ 建议用时**：120 秒 ｜ **考频**：★★★★★ ｜ **难度**：★★★★☆

**🎯 识别特征**：
- "平衡二叉树中插入关键字 X 后…"
- "调整后**根结点的关键字**是"
- "插入新结点后所在结点的**左右孩子**分别是"
- AVL 失衡调整

**🔧 标准步骤（死记）**：
- **Step 1**：**按 BST 规则插入新结点**（小走左，大走右），定位到叶子挂上
- **Step 2**：**自下而上**沿插入路径更新每个祖先的平衡因子 BF = 左高 - 右高
- **Step 3**：找到**第一个** |BF| = 2 的祖先 A → 这是**最小不平衡子树根**
- **Step 4**：**定旋转类型**（关键）：从 A 走到新插入结点的**前两步方向**：
  - 前两步 L-L → **LL 型**（对 A 右旋）
  - 前两步 R-R → **RR 型**（对 A 左旋）
  - 前两步 L-R → **LR 型**（先对 B 左旋，再对 A 右旋）
  - 前两步 R-L → **RL 型**（先对 B 右旋，再对 A 左旋）
- **Step 5**：执行旋转，记住**新根**：
  - LL → 新根 = B（A 的左孩子）
  - RR → 新根 = B（A 的右孩子）
  - LR → 新根 = C（B 的右孩子）
  - RL → 新根 = C（B 的左孩子）
- **Step 6**：旋转后子树高度**不变**（关键性质）→ 不需再向上传播

**🪤 踩坑预警**：
- Step 1 易踩：插入位置找错（忘记 BST 规则）
- Step 2 易踩：BF 方向搞反（BF = 左 - 右，不是右 - 左）
- Step 3 易踩：找"最大"不平衡而不是"最小"（应是**离插入点最近**的那个）
- Step 4 易踩：把 LR 当 RL（方向记反）
- Step 5 易踩：RL 的新根答成 B 而非 C（C 才是对的）

**✅ 自检口诀**：**"找最小不平衡，前两步定类型，LL/RR 新根 B，LR/RL 新根 C"**

**🎯 一题实战演示（第33题 · 2021 统考）**：
- Step 1：插入 23 → 位置在 25 的左孩子
- Step 2：BF 自下而上：25 (+1) → 30 (+1) → 20 (**-2**)
- Step 3：最小不平衡根 A = 20（离插入点 23 最近的 |BF|=2 祖先）
- Step 4：从 20 到 23 的前两步 = R（30）→ L（25）→ **RL 型**
- Step 5：RL 型 → 新根 = B=30 的左孩子 = **C=25**
- **答案 D：25** ✅

## 【题型7】AVL 最少结点数与形态数（斐波那契型递推）（选13、选26【2012统考】、选28【2013统考】）

| h | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |

### 第13题精析（D）—— 高度 3 的 AVL 形态数

**解析推导链**：高度为 3 的 AVL，左右子树高度组合有三种情况：
1. 左=2, 右=2
2. 左=1, 右=2
3. 左=2, 右=1

分别计算：
- 情况 ①：两棵高度 2 的 AVL 形态 × = **3 × 3 = 9**
- 情况 ②：1 种 × 3 种 = **3**
- 情况 ③：3 × 1 = **3**

**合计：9 + 3 + 3 = 15**。

**答案 D（15）**。

**⚠️ 推论**：**高度 h 的 AVL 形态数 Tₕ 递推：Tₕ = Tₕ₋₁ × Tₕ₋₂ × 2 + Tₕ₋₁²**

## 【题型8】AVL 插入/删除后形态关系（T1 vs T3）（选27【2013统考】、选31【2019统考】）

### 核心知识

**核心命题**：在一棵非空 BST/AVL T₁ 中删除某结点 v 后形成 T₂，再将 v 插入 T₂ 形成 T₃。问 T₁ 与 T₃ 的关系。

| 树类型 | v 是叶结点 | v 不是叶结点 |
|--------|-----------|------------|
| **BST**（2013 题27）| T₁ = T₃ ✓ | T₁ ≠ T₃（因为非叶被删后用其他结点填补） |
| **AVL**（2019 题31）| T₁ **可能不相同**（删后可能失衡调整）| T₁ **一定不相同**（非叶删除必引发结构变化） |

**关键区别**：AVL **每次操作都可能触发旋转调整** → T₁ 与 T₃ 的关系比 BST 更"易变"。

---

### 第27题精析（C）—— 2013 统考：BST 删 v 再插 v 的 T1 与 T3 关系

**四个叙述**：
- I：若 v 是 T₁ 的叶结点，则 T₁ 与 T₃ 不同
- II：若 v 是 T₁ 的叶结点，则 T₁ 与 T₃ 相同
- III：若 v 不是 T₁ 的叶结点，则 T₁ 与 T₃ 不同
- IV：若 v 不是 T₁ 的叶结点，则 T₁ 与 T₃ 相同

**解析推导链**：
- v 是叶子：在 BST 中删叶子不改变其他结点的位置；再按 BST 规则插入 v，v 又回到原位（叶子位置） → **T₁ = T₃**。故 **II 对，I 错**。
- v 不是叶子：删除非叶时需用"右子树最小值"或"左子树最大值"来顶替 v；再插入 v 时会作为**新叶子**挂上某个位置，与原位置不同 → **T₁ ≠ T₃**。故 **III 对，IV 错**。

**答案 C（II、III 对）**。

**⚠️ 推论（超级有用）**：**BST 删叶再插 = 不变**；**BST 删非叶再插 = 变**。

---

### 第31题精析（A）—— 2019 统考：AVL 删 v 再插 v 的 T1 与 T3 关系

**三个叙述**：
- I：若 v 是 T₁ 的叶结点，则 T₁ 与 T₃ **可能不相同**
- II：若 v 不是 T₁ 的叶结点，则 T₁ 与 T₃ **一定不相同**
- III：若 v 不是 T₁ 的叶结点，则 T₁ 与 T₃ 一定相同

**解析推导链**（与 BST 完全不同）：

**I 验证**：v 是叶子，删 v 后 AVL 可能失衡 → 触发旋转 → 整树结构变化；再插入 v 时再次触发调整 → **T₃ ≠ T₁**，但也**可能相同**（若删+插都没触发调整）。所以 "**可能不相同**" 是对的 ✓

**举反例**（王道原图）：
```
T₁:                    删除 0:               插入 0:
       1                    1                      2
      / \      删 0          \       调整后         / \
     0   2                    2       →         1   3
          \                    \                 \
           3                    3                 0
```
→ **T₁ ≠ T₃**（同一初始 AVL 和同一结点 v=0，经过删+插后形态不同）。

**II 验证**：v 不是叶子，删除非叶必用其他结点顶替，位置变动 + 可能引发调整；再插入 v 作为新叶子。**一般不同**，但**可能相同**（若调整恰好恢复原形态）。

**举反例**：
```
T₁:                 删除 2 + 调整:          插入 2:
      2                  3                      3
     / \        删 2    / \       调后        / \
    1   3               1   NIL              1   NIL
                                              \
                                               2
```
→ T₃ ≠ T₁ ✓

**但另一例**：也有删非叶后再插回恰好恢复 T₁ 的情况（题目的反例图）。所以 **II 错**（不一定不同），**III 错**（不一定相同）。

**只有 I 正确** → **答案 A（仅 I）**。

**⚠️ 易错点（极高频）**：把 AVL 的规律**套 BST 的规律**回答（2013 的 II、III → 2019 的"II 对、III 错"）→ 错！因 AVL 多了"旋转"这一层变化。

**⚠️ 推论（超级有用）**：
- **BST**：删叶再插=原样；删非叶再插=变
- **AVL**：无论删叶还是非叶，**都可能变或不变**（取决于是否触发旋转）→ **只有"可能不同"的叙述才对**

### 📋 【做题 SOP · 题型8】BST/AVL 删插关系判别

**⏱️ 建议用时**：60 秒 ｜ **考频**：★★★★☆ ｜ **难度**：★★★★☆

**🎯 识别特征**：
- "在 T₁ 中删除 v 形成 T₂，再插入 v 形成 T₃"
- "T₁ 与 T₃ 的关系"
- 选项组合 "若 v 是/不是叶结点…"

**🔧 标准步骤**：
- **Step 1**：**先辨树类型**：题干是 BST 还是 AVL？这决定完全不同的答题路径
- **Step 2**：**BST 情况**：
  - v 是叶 → T₁ = T₃（铁律）
  - v 非叶 → T₁ ≠ T₃（铁律）
- **Step 3**：**AVL 情况**：
  - 任何情况下都要考虑**旋转调整**的影响 → 结论必须用"**可能**"而非"**一定**"
  - v 是叶 → T₁ 与 T₃ **可能不相同**（删/插都可能触发调整）

- **Step 4**：遇到带"一定"的叙述，**立即警惕**——AVL 几乎所有"一定"叙述都是错的（除非是定义层面）
- **Step 5**：用**反例法**快速否证强陈述

**🪤 踩坑预警**：
- Step 1 易踩：没看清是 BST 还是 AVL，套错规律
- Step 3 易踩：BST 的规律机械搬到 AVL 上（2013 → 2019 的陷阱）
- Step 4 易踩：对"一定相同"和"一定不同"都没警觉

**✅ 自检口诀**：**"BST 叶同非叶异，AVL 全看调整否"**

**🎯 一题实战演示（第31题 · 2019 统考）**：
- Step 1：树类型 = AVL（题目明示）
- Step 2-3：AVL 规律 → 任何叙述带"一定"的几乎都错
- Step 4：检验三项：
  - I "叶结点则可能不同" → 对（符合"可能"原则）
  - II "非叶则一定不同" → 错（可能相同）
  - III "非叶则一定相同" → 错（可能不同）
- Step 5：用王道提供的反例验证
- **答案 A：仅 I** ✅

## 【题型9】红黑树基本性质与判别（选16、选17、选18、选19）

### 核心知识

**红黑树五大性质**（必背）：
1. 每个结点**非黑即红**
2. 根结点是**黑色**
3. 所有叶子（NIL）是**黑色**
4. **红结点的孩子必为黑**（不红红）
5. 任一结点到其所有**叶子（NIL）的路径上黑色结点数相同**（黑高相等）

| 红黑树性质推论 | 说明 |
|--------------|------|
| 高度上界 | **h ≤ 2 log₂(n+1)** |
| 红结点数 vs 黑结点数 | 红结点数 ≤ **2 × 黑结点数**（不是 ≤ 黑结点数！）|
| 查询效率 | 比 AVL **略低**（AVL 更严格平衡） |
| 插入/删除效率 | 比 AVL **更高**（调整次数有上界） |
| 红黑树 vs AVL | 红黑树 = **适度平衡**；AVL = **严格平衡** |

---

### 第16题精析（D）—— 红黑树与 AVL 说法不正确的

**四个叙述**：
- I：**一棵含 n 个结点的红黑树的高度至多为 2log₂(n+1)** ✓（红黑树性质）
- II：**若一个结点是红色的，则它的父结点和孩子结点都是黑色的** ✓（性质 4）
- III：**红黑树的查询效率一般优于含相同结点数的 AVL 树** ✗（AVL 更严格平衡，查询更快）
- IV：**若 AVL 某结点的左右孩子的平衡因子都是零，则该结点的平衡因子也是零** ✗（左右孩子 BF=0 只说明左右孩子**自身**是平衡的，但说不出"左右孩子的高度"——有可能高度差 1 或 2）

**答案 D（III、IV 错）**。

**⚠️ 推论**：**AVL 查询快于红黑树**；**AVL 子树 BF 不传递**。

---

### 第17题精析（C）—— 红黑树与 AVL 描述不正确的

**四个叙述**：
- A：两者都属于**自平衡二叉树** ✓
- B：**查找、插入、删除的时间复杂度都相同**（都是 O(log₂n)）✓
- C：**红黑树插入和删除过程至多有 2 次旋转操作** ✗（删除时情况 1 可能变为情况 2/3/4，情况 2 会变情况 3 → 旋转次数可超过 2）
- D：**红黑树任一结点的左右子树高度（含叶结点）之比不超过 2** ✓（由性质 5 推出）

**答案 C**。

---

### 第18题精析（B）—— 红黑树说法正确的

**四个叙述**：
- A：**红结点数最多 = 黑结点数** ✗（最多 = 2 × 黑结点数）
- B：**若红黑树所有结点都是黑的，则一定是满二叉树** ✓（黑高相等+无红结点 → 每层都满）
- C：**红黑树任何一个分支结点都有两个非空孩子** ✗（分支结点可以有一个空叶孩子）
- D：**红黑树的子树也一定是红黑树** ✗（子树的根可能是红色，违反"根黑"性质）

**答案 B**。

### 第19题精析（A）—— 哪个选项满足红黑树定义

**解析推导链**：验证四个选项是否满足五大性质。一般答题方法：
- 根是否为黑（排除 B 若根为红）
- 是否有连续红结点（排除违反性质 4 的）
- 黑高是否处处相等（最后检查）

**答案 A**（其余选项各有违反）。

**⚠️ 推论**：**红黑树判别 → 五大性质逐条校验，根黑 / 不红红 / 黑高相等是三大重点**。

## 【题型10】红黑树插入构造与调整操作（选20、选21、选22）

### 核心知识

**红黑树插入四步**：
1. 按 BST 规则找到插入位置

## 【题型11】BST 与 AVL 综合构造（画树 + ASL + 调整过程）（综合01、综合02、综合03、综合04）

### 核心知识

综合题"标准答题套路"：
1. **根据给定序列构造树**（BST 按插入序列；AVL 每步检查平衡并调整）
2. **画出判定树/形态图**（圆圈实结点+方框失败结点）
3. **计算 ASL**：
   - 成功 ASL = ∑(层数 × 圆圈数)/ n
   - 失败 ASL = ∑(层数 × 方框数)/ (n+1)

---

### 综合01精析 —— 先序遍历构造 BST，求 ASL

**原题**：BST 先序序列 (50, 38, 30, 45, 40, 48, 70, 60, 75, 80)，画树并求 ASL。

**解析完整推演**：

BST 的**中序**必为升序，可得：(30, 38, 40, 45, 48, 50, 60, 70, 75, 80)。配合先序序列，可唯一确定 BST：

```
              50
            /     \
          38       70
         / \       / \
       30   45    60  75
           / \          \
         40   48         80
```

**查找成功 ASL**：
- 层 1：1 个（50）
- 层 2：2 个（38, 70）
- 层 3：4 个（30, 45, 60, 75）
- 层 4：3 个（40, 48, 80）

ASL 成功 = (1×1 + 2×2 + 3×4 + 4×3)/10 = (1+4+12+12)/10 = **29/10 = 2.9**

**查找失败 ASL**：

方框（NIL）总数 = 11 个（n+1=11）。
- 层 4 的方框（父在层 3）：**5 个**（30 的左右、45 无、60 的左右、75 左）—— 按王道给出的图为 5 个
- 层 5 的方框（父在层 4）：**6 个**（40 的左右、48 的左右、80 的左右）

ASL 失败 = (3×5 + 4×6)/11 = (15+24)/11 = **39/11**

（注：这里的层号按"方框位置所在层"计；比较次数 = 方框父圆圈层数 = 方框层 - 1；所以成功层号 k 对应比较 k 次，失败方框位层 m+1 对应比较 m 次）

**王道原解**：**ASL 成功 = 2.9 · ASL 失败 = 39/11**（两结果均为按"比较次数"而非按"方框位置"，即失败时根到方框父圆圈结点数）。

## 必背代码 1：判断一棵二叉树是否为 BST（综合05）

```c
// ===== 核心思想 =====
// BST 的中序遍历 = 严格升序序列
// 用一个全局 predt 保存中序前驱值，每次访问新结点时比较 predt < 当前值
// 若出现 predt >= 当前值，则不是 BST

KeyType predt = -32767;                    // 全局变量，初值设为足够小（-∞）
                                           // 保存当前结点的中序前驱值

int JudgeBST(BiTree bt) {
    int b1, b2;
    if (bt == NULL) 
        return 1;                          // 空树视为 BST
    else {
        b1 = JudgeBST(bt->lchild);         // 判断左子树是否 BST
        // 左子树返回 0 说明左不是 BST，或前驱 >= 当前 → 非 BST
        if (b1 == 0 || predt >= bt->data) 
            return 0;                      // 则不是 BST
        predt = bt->data;                  // 保存当前结点的关键字作为下一次的前驱
        b2 = JudgeBST(bt->rchild);         // 判断右子树
        return b2;                         // 返回右子树的结果
    }
}
```

**关键点**：
- **核心套路**：中序遍历 + 检查前驱 < 当前值
- **为什么用全局变量 predt**：中序遍历过程中要跨递归调用保存前驱值
- **时间**：O(n)（每个结点访问一次）
- **空间**：O(h)（递归栈，h 是树高）
- **坑**：predt 初值不能是 0（若 BST 允许负值），要足够小

## 必背代码 2：求 BST 中指定结点的层次（综合06）

```c
// ===== 核心思想 =====
// BST 查找路径是自顶向下的，每次下降一层
// 从根开始，按 BST 规则（小走左，大走右）查找目标
// 比较次数 = 查找深度 = 层次

int level(BiTree bt, BSTNode *p) {
    int n = 0;                             // 统计查找次数（层次）
    BiTree t = bt;
    if (bt != NULL) {
        n++;                               // 从根开始，第 1 层
        while (t->data != p->data) {
            if (p->data < t->data)         // 目标值小于当前 → 向左子树查
                t = t->lchild;
            else                            // 目标值大于当前 → 向右子树查
                t = t->rchild;
            n++;                           // 层次加 1
        }
    }
    return n;                              // 返回结点层次
}
```

**关键点**：
- 非递归实现，空间 O(1)
- 时间 O(h)，h 为树高；最坏 O(n)（单支树）
- 假设目标 p 必定在树中（若不在需另加判断）
- 层次 = 比较次数 = 从根到目标的路径结点数

## 必背代码 3：判断二叉树是否为平衡二叉树（AVL）（综合07）

```c
// ===== 核心思想 =====
// 利用后序遍历（先判子树，再判根）
// 递归同时返回两件事：高度 h 和平衡标记 balance
// 用 int& 引用参数返回多个值

void Judge_AVL(BiTree bt, int &balance, int &h) {
    int bl = 0, br = 0, hl = 0, hr = 0;    // 左右子树的平衡标记和高度
    if (bt == NULL) {                      // 空树，高度为 0，平衡
        h = 0;
        balance = 1;
    }
    else if (bt->lchild == NULL && bt->rchild == NULL) {
        // 仅有根结点，高度为 1，平衡
        h = 1;
        balance = 1;
    }
    else {
        Judge_AVL(bt->lchild, bl, hl);     // 递归判断左子树
        Judge_AVL(bt->rchild, br, hr);     // 递归判断右子树
        h = (hl > hr ? hl : hr) + 1;       // 当前树高 = 最高子树 + 1
        if (abs(hl - hr) < 2)              // 若左右子树高度差 < 2（即 ≤ 1）
            balance = bl && br;            // 则当前是否平衡 = 左右子树都平衡
        else
            balance = 0;                   // 否则不平衡
    }
}
```

**关键点**：
- **后序遍历**：必须先判子树，才能判根
- **双参数返回**：C++ 用引用；C 用指针
- 时间 O(n)，空间 O(h)
- 常见变体：直接返回"平衡则返回高度，不平衡返回 -1"单参数版

## 必背代码 4：求 BST 最小值 / 最大值（综合08）

```c
// ===== 核心思想 =====
// BST 中最左下结点 = 最小值；最右下结点 = 最大值
// 只需遍历指针一直向左/向右即可，无需比较关键字

KeyType MinKey(BSTNode *bt) {
    while (bt->lchild != NULL)             // 一路向左到底
        bt = bt->lchild;
    return bt->data;                       // 返回最左下结点的关键字
}

KeyType MaxKey(BSTNode *bt) {
    // 求 BST 中最大关键字结点
    while (bt->rchild != NULL)             // 一路向右到底
        bt = bt->rchild;
    return bt->data;                       // 返回最右下结点的关键字
}
```

**关键点**：
- 无需比较任何关键字，只看指针是否为空
- 时间 O(h)，空间 O(1)
- **退出条件**：lchild/rchild 为空（而非 data 的某条件）

## 必背代码 5：从大到小输出 BST 中 ≥ k 的所有关键字（综合09）

```c
// ===== 核心思想 =====
// 从大到小 → 逆中序遍历 (RNL)：先右子树，再根，再左子树
// 剪枝优化：遇到小于 k 的结点无需再访问（但左子树还要剪）

void OutPut(BSTNode *bt, KeyType k) {
    if (bt == NULL) return;                // 空树直接返回
    if (bt->rchild != NULL)
        OutPut(bt->rchild, k);             // 先递归输出右子树的结点
    if (bt->data >= k)                     // 若当前值 >= k 则输出
        printf("%d", bt->data);
    if (bt->lchild != NULL)
        OutPut(bt->lchild, k);             // 再递归输出左子树的结点
}
```

**关键点**：
- **逆中序 RNL**：为了达到"从大到小"输出顺序
- **剪枝思路**：也可判断若 `bt->data < k` 则不进入 lchild（因为左子树所有值更小，都 < k）——王道代码未做此剪枝
- 时间 O(n)，空间 O(h)

## 必背代码 6：求 BST 第 k 小元素（综合10）

```c
// ===== 核心思想 =====
// 每个结点增设 count 字段，记录以该结点为根的子树结点总数
// 利用 count 快速定位第 k 小元素所在子树，递归查找
// 时间 O(log₂n)（平均，BST 高度为 log₂n）

// ===== 增强的 BST 结点定义 =====
// struct BSTNode {
//     KeyType data;
//     BSTNode *lchild, *rchild;
//     int count;       // 以该结点为根的子树结点数
// };

BSTNode *Search_Small(BSTNode *t, int k) {
    // 在以 t 为根的子树上寻找第 k 小的元素
    if (k < 1 || k > t->count) 
        return NULL;                       // k 非法
    if (t->lchild == NULL) {
        // 左子树为空：t 就是该子树的最小值
        if (k == 1) return t;              // k=1 则返回 t
        else return Search_Small(t->rchild, k - 1);
        // 否则第 k 小在右子树中（k-1 是因为排除 t 自己）
    }
    else {
        // 左子树不空：根据左子树 count 判断
        if (t->lchild->count == k - 1)     // 左子树大小 = k-1 → t 就是第 k 小
            return t;
        if (t->lchild->count > k - 1)      // 左子树大小 > k-1 → 第 k 小在左子树
            return Search_Small(t->lchild, k);
        if (t->lchild->count < k - 1)      // 左子树大小 < k-1 → 第 k 小在右子树
            return Search_Small(t->rchild, k - (t->lchild->count + 1));
        // 右子树中找第 k-(左子树大小+1) 小（因为要扣掉左子树和 t 自己）
    }
}
```

**关键点**：
- 结构体 **必须增加 count 字段**，插入/删除时需同步维护
- 递归思路：按 count 判断第 k 小在左/当前/右
- 时间 O(log₂n)（平均），最坏 O(n)（单支树）
- **超高频考点**：2013 年综合题曾考
