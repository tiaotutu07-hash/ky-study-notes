## 一、概念默写(蓝笔对应)  25 题

**Q1.** Cache 的物理实现是什么?集成位置?
→ **答:** SRAM 组成,集成在 CPU 内部。

**Q2.** 时间局部性的定义?
→ **答:** 某条指令/数据被访问后,不久很可能再次被访问(来源:循环、子程序重复、同一变量反复读写)。

**Q3.** 空间局部性的定义?
→ **答:** 某存储单元被访问后,邻近单元很可能也被访问(来源:指令顺序存放、数组连续布局)。

**Q4.** Cache 中"块"、"行"、"组"严格定义?
→ **答:** 块=Cache↔主存搬运的信息包(数据);行=Cache 内装一个块的容器(数据+辅助位);组=多行的集合(组相联的组内自由度单位)。

**Q5.** Cache 辅助位中哪些永远必备?
→ **答:** 有效位(V, 1 bit) + 标记位(Tag, t bit)。

**Q6.** LRU 位的位数怎么算?
→ **答:** LRU 位数 = log₂(组内 E) bit/行。按"组内"E,不按总行数。2/4/8 路分别 1/2/3 位。

**Q7.** Cache 命中率公式?平均访问时间公式(默认串行版)?
→ **答:** H = N_c / (N_c + N_m);T_a = T_c + (1 − H) · T_m。

**Q8.** 三种映射方式的本质区别?
→ **答:** 通用组织 (S, E, B, m) 中 E(每组行数/路数)取值不同:直接 E=1,组相联 1<E<C/B,全相联 E=C/B(S=1)。

**Q9.** CSAPP 的 (S, E, B, m) 四参数 + 标记位/容量公式?
→ **答:** S = 组数, E = 每组行数, B = 块大小(字节), m = 主存物理地址位数。Tag t = m − s − b;数据区 C = S × E × B。

**Q10.** 直接映射:E 值/地址结构/比较器数?
→ **答:** E = 1。地址 | Tag (t) | 行号 (c = s) | 偏移 (b) |。比较器 1 个。

**Q11.** 全相联:S 值/地址结构/比较器数?
→ **答:** S = 1。地址 | Tag | 偏移 |(**无组索引**)。比较器 N(总行数,昂贵相联存储器)。

**Q12.** 组相联(r 路):地址结构/比较器数?
→ **答:** | Tag (t) | 组号 (s) | 偏移 (b) |。比较器 E = r 个(组内并行)。

**Q13.** 为什么组索引必须用中间位?
→ **答:** 主存连续块的高位相同。若用高位做组索引,连续访问的多个块会全部映射到同一组,严重冲突。CSAPP 图6-31。

**Q14.** 组相联的两种退化方式?
→ **答:** S = 1 → 全相联;E = 1 → 直接映射。

**Q15.** 直接映射是否需要替换算法?为什么?
→ **答:** 不需要。目标行唯一,新块直接覆盖。

**Q16.** LRU 命中时的更新规则?
→ **答:** 被命中行计数器清 0;**比原值小的**计数器全部 +1;比原值大的不变。

**Q17.** LRU 缺失且无空闲行时的规则?
→ **答:** 替换计数最大的行(最久未用);新装入行计数 0;其余全部 +1。

**Q18.** LRU 命中规则的最易错点?
→ **答:** **只有"比原值小的"+1**,比原值大的不动(否则计数会失序)。

**Q19.** 什么是抖动?何时发生?
→ **答:** 工作集 > 组内路数 E 时,组里的行被反复加载和驱逐,极端命中率为 0。

**Q20.** 写命中两种策略的全部叫法?
→ **答:** ① 全写法 = 直写法 = Write Through;② 回写法 = 写回法 = Write Back。

**Q21.** 回写法需要什么辅助位?为什么?
→ **答:** 脏位(dirty bit, 1 bit/行)。标记该 Cache 行是否被 CPU 改过——替换时只有脏块需要写回主存,干净的可直接丢弃。

**Q22.** 写策略两种现实搭配?
→ **答:** ① 全写 + 非写分配(配写缓冲,安全优先);② 回写 + 写分配(配脏位,性能优先)。

**Q23.** 现代 CPU L1 d-cache 主流的写策略?
→ **答:** 回写 + 写分配。

**Q24.** Cache 总容量公式?
→ **答:** 总容量 = 总行数 × (数据位 + 辅助位)。辅助位 = 1(V) + t(Tag) + 1(脏,仅回写) + log₂E(LRU,仅 LRU)。

**Q25.** 多级 Cache 全局命中率公式?
→ **答:** 全局命中率 = H₁ + (1 − H₁) · H₂(L2 局部命中率)。

## 二、易错默写(红笔对应)  18 题

**Q26.** ⚠️ 组相联场景下"行"、"块"、"组"为什么不能混用?
→ **答:** 组相联定位是先选组再在组内找行,混用会算错组号/行号。严格上:块=信息包(数据),行=容器(含辅助位),组=多行的集合。

**Q27.** ⚠️ 脏位什么情况下出现?直写法需要吗?
→ **答:** 脏位**仅回写法**需要(标记是否要写回)。**直写法不需要**(主存永远与 Cache 同步)。

**Q28.** ⚠️ LRU 位按什么算?总行数还是组内路数?
→ **答:** **按组内路数 E**,位数 = log₂E。**不是按总行数**。1024 行 4 路 → 2 位(不是 10 位)。

**Q29.** ⚠️ 为什么 Cache 组索引必须在中间位,不能用高位?
→ **答:** 主存连续块的地址高位相同;用高位会让连续访问全部映射到同一组,严重冲突。用中间位则连续块分散到不同组。

**Q30.** ⚠️ 全相联映射的地址有组索引位吗?
→ **答:** **完全没有**(根本没有,不是填 0)。只有 Tag + 块内偏移。因为只有 1 组,无须选择。

**Q31.** ⚠️ 比较器数量是总行数还是组内行数?
→ **答:** **组内行数 E**(不是总行数)。直接 1,r 路 r,全相联才是 N(因为它只有 1 组,组内 = 总行数)。

**Q32.** ⚠️ 直接映射为什么不需要替换算法?
→ **答:** 直接映射下每个主存块对应的 Cache 行**唯一**。新块装入时直接覆盖该行的旧块,没有"选哪行"的问题。

**Q33.** ⚠️ 抖动是什么?命中率会到多少?
→ **答:** 工作集 > 组内路数 E 时组内被反复加载/驱逐,极端**命中率为 0**。修复:加大相联度 / 数组填充(CSAPP) / 改变访问模式。

**Q34.** ⚠️ 写策略反向搭配为什么不考?
→ **答:** 全写+写分配自相矛盾(全写已保证一致性,还调入做什么);回写+非写分配也矛盾(回写本想少访主存,却又不调入)。逻辑上自洽性差,几乎不考。

**Q35.** ⚠️ Cache 容量两种口径如何区分?
→ **答:** ① CSAPP 数据区 C = S × E × B(只算数据);② 王道总容量 = 行数 × (数据位 + 辅助位)(含 V/Tag/脏位/LRU位)。题目问"总位数"通常是王道口径。

**Q36.** ⚠️ 408 默认的平均访问时间公式是哪个?
→ **答:** **串行版** T_a = T_c + (1 − H) · T_m。并行版极少考。

**Q37.** ⚠️ 多级 Cache 全局命中率为什么不能直接相加?
→ **答:** L2 只在 L1 不命中时才查。全局 = L1 命中 + L1 不命中但 L2 命中 = H₁ + (1 − H₁) · H₂。直接相加会重复计算 L1 已命中的部分。

**Q38.** ⚠️ CPU 用什么地址访问 Cache?存在"Cache 地址"吗?
→ **答:** 用**主存物理地址**。**不存在"Cache 地址"**——Cache 按 Tag 内容比对判断命中,不是按"自己的地址"寻址。

**Q39.** ⚠️ LRU 命中规则的具体细节?
→ **答:** 被命中行计数器 **清 0**;**只有"比原值小的"** 计数器 +1;**比原值大的不动**。若全部 +1 会破坏"0 = 最新"的序关系。

**Q40.** ⚠️ 🟣 按字编址下 Cache 块内偏移位数怎么算?
→ **答:** b = log₂(块内**编址单元数**)。按字节编址 b = log₂(块字节数);按字编址 b = log₂(块字数)。例:32 位地址按字编址,块 8 字 → b = 3(不是 5);相应主存空间 = 4G 字 = 16 GB。

**Q41.** ⚠️ 🟣 多核环境下写策略能解决一致性问题吗?
→ **答:** **不能**。写策略(全写/回写)只解决"Cache 和主存"两者的一致性。多核场景下,核 A 改了它的 Cache 副本,核 B 的副本仍是老值——这需要专门的核间一致性协议。

**Q42.** ⚠️ 🟣 MESI 协议的 4 个状态含义?
→ **答:** **M**(Modified, 已修改+独占,主存旧)、**E**(Exclusive, 独占,主存新)、**S**(Shared, 共享,主存新)、**I**(Invalid, 无效)。任一核写时,广播让其他核的副本变 I。

**Q43.** ⚠️ 🟣 Bélády 异常是什么?哪些算法存在?
→ **答:** FIFO 算法下"增加缓存反而降低命中率"的反直觉现象(经典反例:页面序列 1,2,3,4,1,2,5,1,2,3,4,5,FIFO 在 3 槽位命中 3 次,4 槽位反而只命中 2 次)。**LRU 和 RAND 不存在**(LRU 是栈算法,有"槽位多的内容包含槽位少的"性质)。

## 三、跨章关联默写  5 题

**Q44.** Cache 由什么物理存储器构成?和第 3 章(一)的关系?
→ **答:** **SRAM**(静态随机存取存储器,易失/随机访问/双稳态触发器/不需要刷新/快但贵)。主存用 DRAM。Cache 选 SRAM 是为了追求速度,即使贵也只用少量。

**Q45.** Cache 在存储器层次结构中扮演什么角色?
→ **答:** "上层缓存下层"思想的最典型实现。层次链:寄存器 → L1 → L2 → L3 → 主存 → 外存,逐层变大/变慢/变便宜。每一层都是下一层热点数据的缓冲。

**Q46.** 全相联映射和虚拟存储器的什么部件硬件思路一致?
→ **答:** **TLB(快表)**。TLB 用全相联(或高路数组相联)+ 按内容寻址,核心是**相联存储器**——同时并行比较多个表项的 Tag,与全相联 Cache 是同一硬件思路。

**Q47.** Cache 的 LRU 算法和抖动概念在虚拟存储器中如何复用?
→ **答:** **虚存页替换**也常用 LRU(替换最久未用的页)。Cache 的"抖动"在虚存中对应"**缺页抖动**"——工作集超过分配页框数时,页面被反复换入换出。**Bélády 异常**在虚存页替换中同样存在。

**Q48.** 为什么 L1 Cache 要分离 i-cache 和 d-cache?和第 5 章什么概念有关?
→ **答:** **第 5 章指令流水线**。流水线中取指和访存可能在同一周期都要访问 Cache。统一 Cache 只有一个端口 → **结构冒险**(资源冲突)。分离 i/d-cache 让两者**并行访问**,消除该冒险。
