## 一、概念默写(蓝笔对应)  20 题

**Q1.** 进位计数制的核心规则是什么?

→ **答案**:基数为 r,**逢 r 进 1**;每位数码取值 0 ~ r-1。

**Q2.** "位权"的定义是?

→ **答案**:第 i 位的权值 = r^i,**离小数点越远,权值越大**。

**Q3.** 二进制 ↔ 八/十六进制 的快速转换方法?

→ **答案**:**3 位一组**(二↔八),**4 位一组**(二↔十六),从小数点向两侧分组。

**Q4.** 十进制 → 任意进制 的方法?

→ **答案**:整数部分**除基取余**,先余为低后余为高;小数部分**乘基取整**,先整为高后整为低。

**Q5.** ASCII 中 `'0'`、`'A'`、`'a'` 的十进制值分别是?

→ **答案**:`'0'` = 48,`'A'` = 65,`'a'` = 97(大小写差 32)。

**Q6.** 大端存储和小端存储的区别?

→ **答案**:**大端**——高字节存低地址(网络协议、TCP/IP);**小端**——低字节存低地址(Intel x86)。

**Q7.** 真值和机器数的区别?

→ **答案**:**真值**是数学上的真实数值(可带正负号、小数点),**机器数**是计算机内部的纯 0/1 位串(只有 0 和 1)。

**Q8.** 原码的定义是?

→ **答案**:**最高位为符号位**(0 正 1 负),其余位写**真值绝对值**的二进制。

**Q9.** n+1 位原码整数的范围?

→ **答案**:**-(2^n - 1) ≤ x ≤ 2^n - 1**,关于原点对称。

**Q10.** 补码的定义?

→ **答案**:正数同原码;负数 = 其原码**除符号位外逐位取反、末位加 1**(即"反码 +1")。

**Q11.** n+1 位补码整数的范围?

→ **答案**:**-2^n ≤ x ≤ 2^n - 1**,比原码**多 1 个最小负数 -2^n**。

**Q12.** 补码的位权法公式?(零壹独家亮点 ⭐)

→ **答案**:D = **-x_{n-1} · 2^{n-1} + Σ_{i=0}^{n-2} x_i · 2^i**,即最高位权值取负、其余位权值正常累加。

**Q13.** 由 [x]补 求 [-x]补 的方法?

→ **答案**:**连同符号位**,**每位取反、末位加 1**(无论 x 正负,机械操作即可)。

**Q14.** 变形补码(双符号位 / 模 4 补码)是什么?

→ **答案**:**用 2 位符号位**——00 表示正,11 表示负;运算后若两符号位**不一致**(01 或 10)即**溢出**。

**Q15.** 移码的定义(默认偏置 2^n)?

→ **答案**:**[x]移 = 2^n + x**,即把真值整体向右平移 2^n,使结果非负。

**Q16.** 移码与同一真值的补码的关系?

→ **答案**:**仅符号位相反**,其余位完全相同。

**Q17.** 移码的"保序性"指什么?

→ **答案**:真值越大,移码位串越大,可以**直接按无符号数比较大小**——这就是浮点阶码用移码的根本原因。

**Q18.** 四种码各自的实际用途?

→ **答案**:**整数→补码**,**浮点尾数→原码**,**浮点阶码→移码**,**反码**主要作为教学桥梁、几乎不单独使用。

**Q19.** C 语言 `int` 类型的表示范围?

→ **答案**:**[-2^31, 2^31 - 1]** = [-2147483648, 2147483647]。

**Q20.** C 语言不同字长间整数转换的核心规则?

→ **答案**:**先字长转换,再符号转换**;扩展时:无符号→**零扩展**,有符号→**符号扩展**。

## 二、易错默写(红笔对应)  11 题

**Q21.** ⚠️ 哪几种码对 0 的表示**不唯一**?

→ **答案**:**原码、反码** 各有 [+0]/[-0] 两种表示;**补码、移码** 对 0 唯一表示。

**Q22.** ⚠️ 为什么十进制 0.3 不能精确转换为二进制?

→ **答案**:二进制小数位的权值只能是 1/2、1/4、1/8、...、1/2^n,**(0.2)^N 概率**的十进制小数才能精确表示;0.3 等不在此列,只能近似——**这就是浮点舍入误差的根源**。

**Q23.** ⚠️ 二↔八/十六分组时,补 0 规则?

→ **答案**:**整数部分高位补 0**(向左凑齐),**小数部分低位补 0**(向右凑齐)。

**Q24.** ⚠️ 8 位原码能表示的不同数值有多少个?

→ **答案**:**255 个**(2^8 = 256 种位串,但 ±0 重复算 1 个,所以 256 - 1 = 255)。

**Q25.** ⚠️ 真值 -2^n 在哪几种 n+1 位编码中能表示?

→ **答案**:**只有补码、移码**能表示;原码、反码因关于原点对称、最小负数为 -(2^n-1),**表不出 -2^n**。

**Q26.** ⚠️ 8 位补码 1000 0000 对应真值是什么?它的原码呢?

→ **答案**:对应 **-128**;**原码无法表示 -128**(8 位原码范围 -127 ~ 127),所以这个特殊值**找不到原码**。

**Q27.** ⚠️ 移码的偏置量一定是 2^n 吗?

→ **答案**:**不一定**——王道默认 2^n,但题目可能指定其他值。**IEEE 754 单精度阶码偏置 = 127 = 2^7 - 1**,做题时务必看清题目。

**Q28.** ⚠️ 求 [-x]移 时什么情况会溢出?

→ **答案**:当 x = -2^n(8 位时 -128)时,-x = 2^n,**超出 n+1 位移码上界(2^n - 1)**,**发生溢出**。

**Q29.** ⚠️ 同一个 8 位位串 "1000 0000" 在不同解释下的真值?

→ **答案**:无符号 = **128**;补码 = **-128**;原码 = **-0**;移码(偏置 128)= **0**。**位串相同、解释不同、真值不同**。

**Q30.** ⚠️ C 语言中 signed 和 unsigned 数混合运算时如何处理?

→ **答案**:**统一按 unsigned 处理**;所以 `if (-1 < 0u)` **不成立**(因为 -1 强转无符号变成 0xFFFFFFFF,远大于 0u)。

**Q31.** ⚠️ 8421 BCD 码加法什么时候需要修正?

→ **答案**:相加结果 ≥ 10(即落入 1010 ~ 1111 这 6 个非法码,或产生进位)时,**结果加 6**(0110)修正。

## 三、跨章关联默写  5 题

**Q32.** 补码"减法变加法"的具体形式是?(→ 2.2 节)

→ **答案**:**A - B = A + [-B]补**;模运算下 [A]补 - [B]补 = [A]补 + [-B]补,**所以 ALU 只需一套加法器**就能做减法。

**Q33.** 双符号位(变形补码)如何判断溢出?(→ 2.2 节)

→ **答案**:运算后若**两个符号位不相等**(01 或 10),则**溢出**;**01 表示正溢**(上溢),**10 表示负溢**(下溢)。

**Q34.** 浮点数中尾数和阶码分别用什么编码?(→ 2.3 节)

→ **答案**:**尾数用原码**(便于规格化和判零),**阶码用移码**(便于硬件直接比较两浮点数的大小)。

**Q35.** 0.3 转二进制不精确这一性质,在浮点数中体现为什么?(→ 2.3 节)

→ **答案**:浮点数的**舍入误差**——这就是为什么 `0.1 + 0.2 != 0.3` 在 float / double 计算中是必然现象。

**Q36.** 大端/小端会在哪些场景中再次出现?(→ 第 4 章 / CN / OS)

→ **答案**:**第 4 章存储系统**(主存按字节寻址时如何拼字)、**计算机网络**(网络字节序统一为大端)、**操作系统**(union / 指针类型转换的字节解释)。
