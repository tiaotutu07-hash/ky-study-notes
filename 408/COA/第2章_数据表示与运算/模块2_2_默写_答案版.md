## 一、概念默写(蓝笔对应)  25 题

**Q1.** 一位全加器(FA)的接口是?

→ 三个输入:加数 A_i、加数 B_i、低位进位 C_{i-1};两个输出:本位和 S_i、进位 C_i。

**Q2.** FA 的逻辑表达式是?

→ **S_i = A_i ⊕ B_i ⊕ C_{i-1}**;**C_i = A_i·B_i + (A_i ⊕ B_i)·C_{i-1}**。

**Q3.** 串行进位加法器和并行进位加法器(CLA)的延迟差异?

→ 串行 O(n)(进位逐位传递),并行 O(log n)(进位一次性算出)。

**Q4.** 并行进位的核心思想?

→ 引入生成函数 **G_i = A_i·B_i** 和传播函数 **P_i = A_i⊕B_i**,所有 C_i 一组逻辑电路同时算出。

**Q5.** OF 标志位的计算公式与作用?

→ **OF = C_n ⊕ C_{n-1}**,**判有符号数运算溢出**(无符号下无意义)。

**Q6.** CF 标志位的计算公式与作用?

→ **CF = Sub ⊕ C_out**,**判无符号数运算溢出**(有符号下无意义)。

**Q7.** SF 标志位的定义与作用?

→ **SF = F_{n-1}**(结果最高位),仅在**有符号**运算下表示结果正负。

**Q8.** ZF 标志位的定义?

→ 结果 F == 0 时 ZF = 1,否则 ZF = 0;有符号、无符号下都有意义。

**Q9.** ALU 的接口?

→ 两个 n 位操作数输入端 A、B,一个进位输入端 C_in,一个 ALUop 控制端;输出 F 和 4 个标志位。

**Q10.** 逻辑移位的规则?

→ 视位串为无符号数,**左移低位补 0,右移高位补 0**;移出位丢弃。

**Q11.** 算术左移的规则与溢出条件?

→ 低位补 0;若**移出位与新符号位不同**,**溢出**。

**Q12.** 算术右移的规则与精度丢失?

→ **高位补符号位**;若**最低位 1 移出**,**精度丢失**(带余除以 2)。

**Q13.** ROL/ROR 与 RCL/RCR 的区别?(零壹)

→ **ROL/ROR** 不经过 CF,首尾直接相接;**RCL/RCR** 经过 CF,移位时 CF 进入位串。

**Q14.** 补码加减运算的核心公式?

→ **[A+B]补 = [A]补 + [B]补**;**[A-B]补 = [A]补 + [-B]补**(均 mod 2^(n+1))。

**Q15.** 补码运算 4 个特点?

→ ① 按二进制相加逢二进一;② **符号位与数值位一起参与运算**;③ 减法变加法;④ 高位进位丢弃,保留 n+1 位。

**Q16.** 溢出判断法 1(单符号位)?

→ V = A_s·B_s·~S_s + ~A_s·~B_s·S_s;**同号相加结果异号 = 溢出**。

**Q17.** 溢出判断法 2(双符号位 / 变形补码)?

→ 结果两符号位:**00 正不溢、01 上溢、10 下溢、11 负不溢**;不一致 = 溢出。

**Q18.** 溢出判断法 3(进位异或)?

→ V = C_n ⊕ C_{n-1};**最高数值位进位与符号位进位不同 = 溢出**。

**Q19.** 加减运算电路如何用一套加法器同时支持加和减?

→ 加 Sub 控制信号:**Sub=0 直接相加**;**Sub=1 把 Y 经反相器+末位进位 1**(即 [-Y]补)送入加法器。

**Q20.** 无符号大小比较看哪些标志位?

→ 看 **ZF 和 CF**:ZF=1 → A=B;ZF=0 且 CF=0 → A>B;CF=1 → A<B。

**Q21.** 有符号大小比较看哪些标志位?

→ 看 **ZF、OF、SF**:ZF=1 → A=B;ZF=0 且 OF=SF → A>B;ZF=0 且 OF≠SF → A<B(即 OF⊕SF=1)。

**Q22.** 原码一位乘法的两个步骤?

→ ① 符号位 = X 符号 ⊕ Y 符号;② **数值位用绝对值相乘**(n 次加法 + n 次右移)。

**Q23.** Booth 比较法的 4 种操作?

→ 在乘数末尾附 y_{n+1}=0,看 **y_n y_{n+1}**:**00→加0、01→加[X]补、10→加[-X]补、11→加0**;每次操作后右移一位(算术右移)。

**Q24.** 乘法运算电路的关键寄存器?

→ **被乘数寄存器 X**、**乘积寄存器 P**(初始 0)、**乘数寄存器 Y**;P 和 Y **同步右移**,Y 最低位送控制逻辑决定加 X 还是加 0。

**Q25.** 补码除法(加减交替)的核心规则?

→ **余数与除数同号 → 商 1,余数左移 - 除数**;**异号 → 商 0,余数左移 + 除数**;末位常用恒置 1 法。

## 二、易错默写(红笔对应)  14 题

**Q26.** ⚠️ 减法运算时 CF 公式有什么特殊?

→ 减法时 Sub=1,**CF = ~C_out**(无借位时 Cout=1,故 CF=0;有借位时 Cout=0,故 CF=1)。

**Q27.** ⚠️ 为什么 OF 在无符号运算下无意义?

→ OF 是基于"两同号数加结果异号"判断,而无符号数没有"符号"概念,所以即使位串最高位 1→0 改变,也不代表溢出。

**Q28.** ⚠️ 同样,CF 在有符号运算下为什么无意义?

→ CF 检测的是"无符号意义下的加法是否产生进位",有符号补码加法的进位/借位**与"是否溢出"无直接对应**。

**Q29.** ⚠️ 什么情况下加减运算一定不会溢出?

→ **异号相加**(一正一负,绝对值不会变大)、**同号相减**(等价于异号相加)——这两种情况一定不溢出。

**Q30.** ⚠️ 双符号位补码中,哪一位才是结果的"真符号位"?

→ **高位(第 1 位)**永远代表真正的符号;低位(第 2 位)只用来配合高位判溢出。

**Q31.** ⚠️ 加减运算电路硬件能否区分有符号 vs 无符号?

→ **不能!** 同一套加法器对位串"无意识",**全靠 OF/CF 标志位事后区分语义**。所以 C 程序中 signed 和 unsigned 的加减用的是同一条机器指令。

**Q32.** ⚠️ 有符号比较时,为什么不能光看 SF 判正负?

→ 因为**溢出时 SF 会翻转**(例如两个大正数加导致 SF=1 看似负),需要用 **OF 校正**:有效正负判断是 SF ⊕ OF。

**Q33.** ⚠️ 原码乘法过程中的右移是哪种移位?

→ **逻辑右移**(因为参与运算的是绝对值,即数值位,无符号语义)。

**Q34.** ⚠️ 无符号乘法溢出判断的具体规则?

→ 2n 位乘积的**高 n 位全为 0** → 不溢出;**任何一位为 1** → 溢出。

**Q35.** ⚠️ 有符号乘法溢出判断的具体规则?

→ 2n 位乘积的**高 n 位每一位都等于低 n 位的最高位**(即高 n 位**全 0** 或**全 1**)→ 不溢出;否则溢出。

**Q36.** ⚠️ 32 位 int 整数除法,什么情况下会发生溢出?

→ **仅 -2^31 ÷ -1 一种情况**(结果 +2^31 超 int 上限),其他不溢出。

**Q37.** ⚠️ 整数除法时除数为 0 会怎样?

→ 触发**硬件异常**(Linux 为 SIGFPE),由操作系统的异常处理程序接管。

**Q38.** ⚠️ 不恢复余数法(加减交替)和恢复余数法的区别?

→ 恢复余数法:不够减时要"加回除数恢复余数";加减交替法:**直接做下一步加除数**,通过下一步的方向调整来等价完成,**省一步**。

**Q39.** ⚠️ 算术移位 vs 逻辑移位:对于无符号数,算术右移和逻辑右移结果是否一样?

→ **不一定**——王道答案:无符号数的"符号位"看作 0,所以算术右移在无符号语义下**等同于**逻辑右移(高位都补 0);但这只在编译器允许时成立,严格区分时**仍按各自规则**。

## 三、跨章关联默写  5 题

**Q40.** ALU 在 CPU 数据通路中的位置?(→ 第 5 章)

→ ALU 是**运算器的核心**,接收来自通用寄存器/立即数的两个输入,执行算逻运算,结果回写到目的寄存器,**同时把 4 个标志位送入状态寄存器(PSW)**。

**Q41.** OF/CF/SF/ZF 在指令系统中如何被使用?(→ 第 5 章)

→ **cmp 指令**实际执行减法但不保存结果,**只更新标志位**;后续 **jcc 指令**(jz/jnz/ja/jb/jg/jl 等)根据标志位决定是否跳转——**这是高级语言 if/while 编译后的根本机制**。

**Q42.** 补码加减为什么"减法变加法"?(→ 2.1 节)

→ 因为补码本身是 mod 2^(n+1) 的等价类,**[A]补 - [B]补 ≡ [A]补 + [-B]补 (mod 2^(n+1))**,所以减法可以用 -B 的补码加法实现,**硬件不需要单独的减法器**。

**Q43.** 除数为 0 异常如何被 CPU 处理?(→ 第 5 章)

→ 除法器检测到除数为 0 时,**置异常请求位**,在当前指令周期末尾 CPU 响应异常,**保存断点 → 跳转到中断向量表对应处理程序入口**(类似中断处理流程)。

**Q44.** 位级等价性(零壹亮点)在浮点编码中如何体现?(→ 2.3 节)

→ IEEE 754 的位串可以**直接按整数比较大小**(对正数有效),这就是位级等价性的延伸——位串的"无符号语义"和"特定语义"在某些场景下能直接互用,硬件设计大幅简化。
