## 一、概念默写(蓝笔对应)  25 题

**Q1.** 浮点数的通用表达公式?

→ **N = (-1)^S · M · R^E**;S 是符号位,M 是尾数(原码),R 是基数(IEEE 754 隐含为 2),E 是阶码(移码)。

**Q2.** 浮点数中尾数和阶码各决定什么?

→ **尾数决定精度**(有效数字位数),**阶码决定范围**(指数大小)。

**Q3.** 浮点数表示范围数轴上的 4 个不可表示区?

→ **正上溢、正下溢、负下溢、负上溢**;上溢中断,下溢按机器零处理。

**Q4.** 浮点数规格化的数学定义(基数为 2)?

→ **1/2 ≤ |M| < 1**;尾数最高位必须是有效位 1,小数点紧贴最高有效位之后。

**Q5.** 原码规格化的正/负数尾数形式?

→ 正数 **0.1xx...x**,负数 **1.1xx...x**(数值位最高为 1)。

**Q6.** 补码规格化的正/负数尾数形式?

→ 正数 **0.1xx...x**,负数 **1.0xx...x**(数值位最高与符号位异)。

**Q7.** 左规和右规的规则与适用场景?

→ **左规**:尾数高位连续 0,**尾左移阶减**,**可多次**;**右规**:尾数溢出形如 ±1x.xxx,**尾右移阶加**,**仅一次**。

**Q8.** IEEE 754 单精度浮点数的格式?

→ **1 位符号 + 8 位阶码 + 23 位尾数**,共 32 位;偏置 = 127。

**Q9.** IEEE 754 双精度浮点数的格式?

→ **1 位符号 + 11 位阶码 + 52 位尾数**,共 64 位;偏置 = 1023。

**Q10.** IEEE 754 阶码偏置量的公式?

→ **bias = 2^(k-1) - 1**,其中 k 是阶码位数;单精度 k=8 → 127,双精度 k=11 → 1023。

**Q11.** IEEE 754 的"隐藏位"是什么?

→ 规格化数的尾数最高位必为 1,**这个 1 不存储**(隐藏);所以单精度尾数实际有 **24 位有效**(23 显式 + 1 隐藏)。

**Q12.** IEEE 754 阶码全 0、尾数全 0 表示什么?

→ **±0**(由符号位 S 决定正负);+0 和 -0 在数值上相等。

**Q13.** IEEE 754 阶码全 0、尾数非 0 表示什么?

→ **非规格化数**,真实指数 **E = 1 - bias**(单精度 -126,双精度 -1022),尾数无隐藏位 1,M = 0.f。

**Q14.** IEEE 754 阶码全 1、尾数全 0 表示什么?

→ **±∞**(无穷大);用于上溢标志。

**Q15.** IEEE 754 阶码全 1、尾数非 0 表示什么?

→ **NaN**(Not a Number),表示未定义运算结果(如 0/0、∞ - ∞)。

**Q16.** float 类型的最大正数和有效十进制位数?

→ 最大正规格化数 ≈ **3.4×10³⁸**,有效十进制位数 **6 ~ 7 位**。

**Q17.** double 类型的最大正数和有效十进制位数?

→ 最大正规格化数 ≈ **1.8×10³⁰⁸**,有效十进制位数 **15 ~ 16 位**。

**Q18.** 浮点数加减运算的五个步骤?

→ ① **对阶** ② **尾数加减** ③ **规格化** ④ **舍入** ⑤ **判溢出**。

**Q19.** 浮点对阶的规则?

→ **小阶向大阶看齐**:小阶尾数右移、阶码加 1,直到两数阶码相等。

**Q20.** IEEE 754 的默认舍入方式?

→ **就近舍入到偶**(类似四舍五入,但当结果恰好在两可表示数中间时,选最低有效位为偶数的那个)。

**Q21.** 浮点运算的真溢出由什么决定?

→ **由阶码是否上溢/下溢决定**,**不是看尾数**;尾数溢出可通过右规修正。

**Q22.** 浮点乘除运算的核心公式?

→ X·Y = (M_x · M_y) · 2^(E_x + E_y);X/Y = (M_x / M_y) · 2^(E_x - E_y);**无需对阶**。

**Q23.** 浮点乘除步骤与加减的关键差异?

→ ① **无需对阶**;② 阶码做加/减,尾数做乘/除;③ **规格化只需检查一次**(因输入已规格化);④ 仍需舍入和判溢出。

**Q24.** 大端方式和小端方式的区别?

→ **大端**:高字节存低地址(网络字节序、TCP/IP);**小端**:低字节存低地址(Intel x86/x64、最常见)。

**Q25.** struct 边界对齐的两条规则?

→ ① **每个成员的起始地址 % 该成员大小 = 0**;② **整个 struct 总大小 % 最大成员大小 = 0**(不够则补 padding)。

## 二、易错默写(红笔对应)  14 题

**Q26.** ⚠️ 浮点数发生上溢时的处理?

→ **必须中断运算**,触发硬件异常;无法用 ±∞ 继续运算。

**Q27.** ⚠️ 浮点数发生下溢时的处理?

→ **通常按机器零处理**(数值精度过低,直接当 0);不中断程序。

**Q28.** ⚠️ 浮点加减规格化时,右规和左规分别可能引发什么溢出?

→ **右规阶码 +1 → 可能上溢**;**左规阶码 -1 → 可能下溢**;运算结束必须再判一次溢出。

**Q29.** ⚠️ IEEE 754 偏置量为什么不是经典移码的 2^(k-1)?

→ IEEE 标准取 **bias = 2^(k-1) - 1**,这样单精度阶码真值范围是 [-126, +127],对称且可与无符号数比较等价。

**Q30.** ⚠️ 非规格化数的真实指数 E 为什么不是 0 - bias?

→ IEEE 754 规定非规格化数的 **E 真 = 1 - bias**(单精度 = -126),不是 -127;**这是为了让最大非规格化数和最小规格化数平滑衔接**(零壹独家亮点)。

**Q31.** ⚠️ IEEE 754 的隐藏位在哪些数中存在?

→ **仅规格化数有隐藏位 1**;**非规格化数没有**(尾数 M = 0.f);±0 也没有。

**Q32.** ⚠️ 十进制 0.1 能否精确转二进制?

→ **不能**;0.1 = 0.0001100110011...(无限循环),float 只能保留 24 位有效,**必有舍入误差**。

**Q33.** ⚠️ 浮点对阶为什么不能"大阶向小阶"看齐?

→ 大阶向小阶意味着大数尾数**左移**,**最高有效位会被移出**,数值出错;只能"小阶向大阶"。

**Q34.** ⚠️ 尾数溢出是否一定意味着浮点结果真的溢出?

→ **不一定**!尾数溢出形如 ±1x.xxx 时,**右规一次**(尾右移阶加)即可修正;**只有规格化后阶码超界,才是真溢出**。

**Q35.** ⚠️ 浮点乘除的规格化只需检查一次,加减为什么要多次?

→ **乘除输入已规格化,结果幅度变化小**,最多右规 1 次或左规 1 次;**加减运算可能让尾数严重失衡**(连续 0 或溢出),需多次左规或右规一次。

**Q36.** ⚠️ int 转 float 是否会精度损失?

→ **可能!** float 尾数仅 24 位有效,int 是 32 位;**当 int 超过 2²⁴ 时**,转 float 会丢失低位精度(经典例:16777217 → 16777216)。

**Q37.** ⚠️ float 转 int 时数据如何处理?

→ ① 小数部分**向 0 截断**(只保留整数部分);② **可能溢出**(若 float 超出 int 范围);③ 不像 int→float 隐式安全,实践中要小心。

**Q38.** ⚠️ struct 中成员的顺序对 sizeof 的影响?

→ **顺序不同 sizeof 不同**!典型例:`{int, char, short}` = 8 字节,`{char, int, short}` = 12 字节(因为 char 后跟 int 需要 3 字节 padding)。

**Q39.** ⚠️ Intel x86 是大端还是小端?网络字节序呢?

→ **Intel x86/x64 是小端**(低字节→低地址);**网络字节序(TCP/IP)是大端**;ARM 默认小端但可配置。

## 三、跨章关联默写  4 题

**Q40.** 浮点数的舍入误差和 2.1 节哪个性质直接相关?(→ 2.1)

→ 2.1 节"**任意二进制小数都能精确转十进制,但十进制小数不一定能精确转二进制**";浮点的舍入误差就是这一性质在 IEEE 754 编码中的具体体现——0.1 + 0.2 ≠ 0.3 的根因。

**Q41.** IEEE 754 阶码用移码而不是补码,本质好处是什么?(→ 2.1)

→ **移码具有保序性**,真值越大移码位串越大,**可以直接按无符号数比较两浮点数大小**(对正数有效),硬件比较器无需特殊改造。

**Q42.** 大小端和 struct 对齐在主存中如何影响指令执行?(→ 第 4 章)

→ 主存按字节寻址,**对齐数据可一次读取完成**(高效);**未对齐数据需多次访问**(降速);现代 RISC 强制对齐。大小端则决定了 union/指针类型转换时的字节解释。

**Q43.** 浮点指令在 CPU 中如何处理?(→ 第 5 章)

→ 早期 CPU 配独立**浮点协处理器(FPU,如 x87)**;现代 CPU **集成 FPU + SSE/AVX 向量浮点单元**;浮点运算指令(FADD、FMUL 等)走独立的浮点流水线,与整数指令并行执行。
