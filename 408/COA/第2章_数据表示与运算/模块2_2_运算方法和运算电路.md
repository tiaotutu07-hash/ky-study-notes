# 模块 2.2 · 运算方法和运算电路

> **本节解决的问题:** 2.1 节我们把数"装"进了机器,这一节要让机器"算"。如何用一堆与/或门拼出能加减乘除的电路?如何让一套加法器同时支持有符号、无符号?如何判断"算溢出了"?
> **核心脉络:** 全加器 FA(基本细胞)→ 加法器(串行/并行)→ 带标志加法器(OF/CF/SF/ZF)→ ALU(总成)→ 移位/加减/乘除四类运算。
> **学完能做什么题:** 各类移位运算的位串变化、补码加减+溢出判断三法、CF/OF/SF/ZF 的填表、有符号无符号大小比较、Booth 比较法乘法、加减交替除法、乘除溢出判断。这是 408 综合题最爱出的章节,2024 大题就考了 CF 的具体含义。

---

## 一、运算的"原子":全加器与加法器

### 1.1 一位全加器(FA · Full Adder)

**定义:** 三入两出。三入为加数 A_i、加数 B_i、低位进位 C_{i-1};两出为本位和 S_i、向高位进位 C_i。

```
逻辑表达式:
   S_i = A_i ⊕ B_i ⊕ C_{i-1}     ← 三个里有奇数个 1 → S=1
   C_i = A_i·B_i + (A_i ⊕ B_i)·C_{i-1}

逻辑结构(简版):
   A_i ─┬───┐
        │ XOR ──┬─ XOR ── S_i
   B_i ─┴───┘   │   ↑
                │ C_{i-1}
   两个 AND + 一个 OR → C_i
```

⚠️ **本质理解:** **加法器 = 一堆 FA 串起来**;**减法器 = 加法器 + 反相器 + 进位补 1**(把 -B 转成 [B反]+1)。这就是为什么"减法可以由加法做"——硬件层面,减法器**根本就是加法器**。

### 1.2 串行进位加法器(行波进位)

n 个 FA 的进位首尾相连:C_0 → FA0 → C_1 → FA1 → ... → C_n。**信号像水波一样从低位"波"到高位**,所以也叫**行波进位**(Ripple Carry)。

⚠️ **缺点:** 高位必须等低位算完,**最长延迟 = n × 单 FA 延迟**,字长 n 越大越慢。这就是为什么需要"并行进位"。

### 1.3 并行进位加法器(CLA · Carry Look-Ahead)

**核心思想:** 既然进位 C_i 完全由 A、B、C_0 决定,那就**用一组逻辑电路一次性算出所有 C_i**,不必等。

```
对每位 i:
   生成函数 G_i = A_i · B_i        // 两位都为 1,本位必产进位
   传播函数 P_i = A_i ⊕ B_i        // 一位为 1,可以传递低位进位
   则 C_i = G_i + P_i · C_{i-1}    // 展开后所有 C_i 仅依赖 G/P/C_0
```

**所以才能** 让所有进位**同时产生**,延迟从 O(n) 降到 O(log n)。8 位、16 位、32 位机的加法器都靠这套实现。

### 1.4 带标志加法器(必考)

n 位加法器除了和,还要输出 4 个标志位,**占 4 个寄存器位**:

| 标志 | 名称 | 计算公式 | 作用 |
|---|---|---|---|
| **OF** | 溢出 (Overflow) | **OF = C_n ⊕ C_{n-1}** | 判**有符号数**溢出(无符号下无意义) |
| **CF** | 进位/借位 (Carry) | **CF = Sub ⊕ C_out** | 判**无符号数**溢出(有符号下无意义) |
| **SF** | 符号 (Sign) | **SF = F_{n-1}**(最高位) | 结果正负;有符号才有意义 |
| **ZF** | 零 (Zero) | **ZF = 1** ⟺ F = 0 | 结果是否为 0;两边都有意义 |

⚠️ **CF 公式细节(命题追踪 2024):** 加法时 Sub=0,CF = C_out;**减法时 Sub=1,CF = ~C_out**(无借位则进位输出为 1,有借位为 0,所以要"翻转")。

> **零壹补充:** 零壹用一句话总结——"**OF 判有符号、CF 判无符号、SF 看符号位、ZF 看是不是 0**"。背这一句话即可应付一半选择题。

### 1.5 算术逻辑单元(ALU)

ALU = 带标志加法器 + 多路选择器 + 移位 + 与/或/非/异或。
**两个 n 位输入端 A、B,一个 ALUop 控制端**(决定执行什么操作),输出 F 和 4 个标志位。

```
        A(n位)             B(n位)
          │                   │
          ▼                   ▼
        ┌─────────────────────┐
        │       ALU           │ ◀── ALUop(操作选择)
        │  加/减/与/或/非/移位  │
        └──────┬──────────────┘
               │       │ │ │ │
               ▼       ▼ ▼ ▼ ▼
              F(n位)  ZF OF SF CF
```

⚠️ **ALUop 位数 = ⌈log₂ 操作种类⌉**——做 8 种操作至少需要 3 位 ALUop。

> **跨章关联:** ALU 是 **第 5 章 CPU 数据通路**的核心部件,所有指令的算术/逻辑操作都通过 ALU。

**【一节收束】** FA 是细胞,串行慢/并行快;带标志加法器是"加法器+4 个标志位",ALU 是把这一切再加上逻辑运算和移位的总成。**记住 4 个标志位的公式和适用范围**,就拿下了 2024 真题的核心概念。

## 二、移位运算:乘除二的硬件实现

### 2.1 三种移位的对照(命题追踪 2012 / 2017 / 2018)

| 移位 | 把谁当成什么 | 左移空位 | 右移空位 | 用途 |
|---|---|---|---|---|
| **逻辑移位** | 无符号数 | **低位补 0** | **高位补 0** | unsigned 的 ×2 / ÷2 |
| **算术移位** | 有符号数(补码) | **低位补 0** | **高位补符号位** | signed 的 ×2 / ÷2 |
| **循环移位** | 位串(可带 CF 也可不带) | 末位接首位 | 首位接末位 | 多字长串、密码学 |

```
原数 1011 0010(8 位):

逻辑左移 1 位:0110 0100  (高位 1 移出,低位补 0)
算术左移 1 位:0110 0100  (同上,但若移出"位"≠新符号位,则溢出)
逻辑右移 1 位:0101 1001  (高位补 0)
算术右移 1 位:1101 1001  (高位补符号位 1 — 因为原最高位是 1)
循环左移 1 位:0110 0101  (移出的最高位 1 进了最低位)
```

⚠️ **算术移位的两条致命陷阱(命题追踪 2018 题 2):**
- **左移:** 若**移出的位与新符号位不同**,**溢出**。例如补码 0100 0000 左移变 1000 0000——符号从正变负,溢出。
- **右移:** 若**最低位是 1 移出**,**精度丢失**(相当于做了带余除以 2)。

⚠️ **零壹独家:循环移位还分"带进位"和"不带进位"两种。** ⭐
- **不带进位循环左移(ROL):** 最高位 → CF,**同时** → 最低位
- **带进位循环左移(RCL):** 原 CF → 最低位,最高位 → 新 CF
- 右移版本(ROR / RCR)同理对调

```
RCR(带进位循环右移)演示:
   原:CF=0,寄存器 1011 0100
       ↓
   新:CF=0(原最低位),寄存器 0101 1010
       (原 CF=0 进入最高位,原最低位 0 进入 CF)
```

**做题口诀:** 看到"ROL/ROR" → 不经过 CF;看到"RCL/RCR" → 经过 CF。

**【二节收束】** 三种移位都是"位串移动 + 空位补什么"的差异。**逻辑补 0,算术左补 0 右补符号,循环首尾相接**。把这一句话刻进框架图,移位题秒杀。

## 三、加减运算:补码统一了一切

### 3.1 补码加减:**一个公式打天下**

$$[A+B]_{补} = [A]_{补} + [B]_{补} \pmod{2^{n+1}}$$
$$[A-B]_{补} = [A]_{补} + [-B]_{补} \pmod{2^{n+1}}$$

**特点 4 件套:**
1. 按二进制规则运算,**逢二进一**
2. **符号位与数值位一起参与运算**(这就是补码的精髓)
3. 加法直接相加;减法转成"被减数 + 减数的负补码"
4. **最终结果的高位进位被丢弃**,保留 n+1 位即为 [A±B]补

```
例 2.3(王道 P41):字长 8 位,A=+15,B=+24,求 [A+B]补 和 [A-B]补
   [A]补 = 0000 1111
   [B]补 = 0001 1000
   [-B]补 = 1110 1000

   [A+B]补 = 0000 1111 + 0001 1000 = 0010 0111 → 真值 +39 ✅
   [A-B]补 = 0000 1111 + 1110 1000 = 1111 0111 → 真值 -9  ✅
```

### 3.2 溢出判断三法 ⭐ 综合题考点

> **正是因为** 字长有限,**两个同号数相加** 才可能"挤"出范围,**异号数相减** 也可能溢出(异号相减等价于同号相加)。**异号相加、同号相减一定不溢出**。

**法 1 · 单符号位法**(王道首选)

设 A 符号位 As,B 符号位 Bs,结果符号位 Ss:
$$V = A_s \cdot B_s \cdot \overline{S_s} + \overline{A_s} \cdot \overline{B_s} \cdot S_s$$

**口诀:同号相加而结果异号 → 溢出。**

**法 2 · 双符号位法(变形补码)**

把符号位扩为 2 位,运算时双符号一起参加。结果符号位 S_{s1}S_{s2}:
- **00 = 正且不溢出**
- **01 = 正溢(上溢)** ⚠️
- **10 = 负溢(下溢)** ⚠️
- **11 = 负且不溢出**

**口诀:两符号位相同则不溢出,01 上溢、10 下溢。**

**法 3 · 进位异或法**

最高数值位的进位 C_{n-1} 与符号位的进位 C_n 异或:
$$V = C_n \oplus C_{n-1}$$

**口诀:两个进位一致则不溢出,不一致则溢出。**

```
例:8 位补码 A=0100 0000(+64), B=0100 0000(+64)
                     ↑双正
   A+B = 1000 0000   ↓双负 → 溢出!
   
   法1验证:As=0, Bs=0, Ss=1 → V = 0·0·0(非) + 1·1·1 = 1 → 溢出 ✓
   法2(双符号):00,1000000 + 00,1000000 = 01,0000000 → 01 上溢 ✓
   法3(进位):C_n=0(从最高位向上无进位), C_{n-1}=1 → 0⊕1=1 → 溢出 ✓
```

⚠️ **三法等价**,题目里看哪个数据齐全用哪个。综合题常给"双符号位运算结果",直接套法 2 最快。

### 3.3 加减运算电路(图 2.9)

```
                Sub(加=0/减=1)
                    │
                    ▼
   X ───────────► [加法器]──── F(结果)
                    ▲
                    │ ┌─Y─┐
                    │ │   │
                    │ │   ▼ 反相器
                    └─MUX─┘
                       ↑
                      Sub
   
   减法时 Sub=1:Y 经反相器变 ~Y,加法器算 X + ~Y + 1 = X + (-Y)补
   加法时 Sub=0:Y 直接进加法器,算 X + Y
```

⚠️ **关键洞察(命题追踪 2011):** **同一个加法器**既可以做有符号补码加减,也可以做无符号加减——电路根本不知道**位串到底是有符号还是无符号**,**全靠 OF / CF 标志位事后区分**。

### 3.4 标志位详解 + 大小比较 ⭐ 2024 大题原题

**回看 4 个标志(本节再深化):**
- **OF = C_n ⊕ C_{n-1}** —— 判有符号溢出
- **CF = Sub ⊕ C_out** —— 判无符号溢出
- **SF = F_{n-1}** —— 结果最高位
- **ZF = (F == 0)** —— 结果是否为 0

**无符号数比较 A vs B(执行 A-B):**

| 关系 | ZF | CF |
|---|---|---|
| A = B | 1 | 0 |
| A > B | 0 | 0 |
| A < B | 0 | **1** ⚠️(借位) |

**有符号数比较 A vs B(执行 [A]补 - [B]补):**

| 关系 | ZF | OF | SF |
|---|---|---|---|
| A = B | 1 | 0 | 0 |
| A > B(无溢出) | 0 | 0 | **0** |
| A > B(有溢出导致 SF 看似为负) | 0 | 1 | **1** |
| A < B(无溢出) | 0 | 0 | **1** |
| A < B(有溢出导致 SF 看似为正) | 0 | 1 | **0** |

**统一判 A > B**:`OF == SF`(异或为 0)且 `ZF == 0`,即 `(OF ⊕ SF) | ZF == 0`。

> **本质理解:** 当**没溢出**时,SF 直接就是结果的真实正负;**溢出**时,SF 翻转——所以要用 OF 把 SF "校正回来"。**这就是为什么** 有符号比较必须看 `OF ⊕ SF`,不能光看 SF。

⚠️ **2024 真题原型(CF 的作用):** 题目问"CF 表示什么、有何作用",标准答案就是上面"判无符号数运算溢出"。如果光答"进位/借位"会扣分。

> **跨章关联:** 这套标志位会在 **第 5 章指令系统**的 cmp / jcc 系列指令中再次出现。

### 3.5 原码加减(了解即可)

原码加减规则非常复杂:**同号求和、异号求差**,差时还要比较绝对值,绝对值大的当被减数。**这就是为什么** 计算机抛弃了原码加减、只用补码——一条规则全包。

**【三节收束】** 补码加减只有"按二进制相加,丢高位进位"一条规则;**溢出判断三法等价**,做题挑数据齐全的那个;有符号比较看 `OF⊕SF`,无符号比较看 `CF`。

## 四、乘法运算:加法 + 移位的迭代

### 4.1 原码一位乘法(命题追踪 2020 / 2024)

**思想:** **符号位** 与 **数值位** **分开处理**。

```
步骤 1:符号位 = X 符号 ⊕ Y 符号(异或)
步骤 2:数值位 = |X| × |Y|(两个无符号数相乘)
步骤 3:n 次"加法 + 右移"循环

递推式: P_{i+1} = 2^{-1} (P_i + X · y_{n-i})  (i = 0, 1, ..., n-1)
最终乘积:P_n = X × Y(2n 位)
```

**手算演示(王道 P44):被乘数 X=0.1101, 乘数 Y=0.1011, 求 X·Y**

```
   被乘数 X = 0.1101  (绝对值)
   乘数  Y = 0.1011

   逐位看 Y(从最低位 y_4=1 开始):
   y_4 = 1 → 加 X     → 加 0.1101
   y_3 = 1 → 加 X<<1  → 加 0.1101·2¹
   y_2 = 0 → 加 0     → 加 0
   y_1 = 1 → 加 X<<3  → 加 0.1101·2³

   累加:
       0.0000 1101  ← y_4=1, X<<0
     + 0.0011 0100  ← y_3=1, X<<1, 实际操作右移1次
     + 0.0000 0000  ← y_2=0
     + 0.0110 1000  ← y_1=1
     ─────────────
       0.1000 1111

   符号位 = 0 ⊕ 0 = 0
   [X·Y]原 = 0.1000 1111
```

⚠️ **关键操作细节:**
- **被乘数和乘数的"绝对值"参与运算**,符号位最后单独决定
- 每次循环:**判断乘数最低位 → 加 0 或加 X → 部分积右移一位 → 乘数也右移一位**
- 总共 **n 次循环**(乘数有 n 位有效数值位)
- **右移用"逻辑右移"**(数值位运算,不带符号)

### 4.2 补码 Booth 比较法(零壹独家清晰版)⭐

**和原码乘法的本质区别:符号位参与运算**,不再单独处理。

**操作规则:在乘数末尾附加 y_{n+1} = 0**,然后看 **y_n y_{n+1}** 决定操作:

| y_n y_{n+1} | 操作 |
|---|---|
| **0 0** | 部分积加 0,右移一位 |
| **0 1** | 部分积加 [被乘数]补,右移一位 |
| **1 0** | 部分积加 [-被乘数]补,右移一位 |
| **1 1** | 部分积加 0,右移一位 |

**右移规则:** 算术右移(高位补符号位)。

```
例(零壹):[X]补 = 1.0101, [Y]补 = 1.0011, 求 [X·Y]补

[-X]补 = 0.1011

部分积      乘数+附加位     说明
0.0000      1.0011  0       初始,看 30 = "10"  
+ 0.1011                    操作: +[-X]补
0.1011                      
0.0101 1    1.001  10       右移
+ 0.0000                    看 11,操作:+0
0.0010 11   1.00  110       右移
+ 0.0101                    看 01,操作:+[X]补?
                            (略,跟随表格机械操作即可)
...

[X·Y]补 = 0.0001 0111(具体演算见王道答案)
```

⚠️ **Booth 法的精髓:** 把"连续的 1"用"加+减"代替——例如乘以 0111 的连续 1,等价于乘以 (1000 - 1),只需 1 次加 + 1 次减,而不必 3 次加。**这就是为什么** Booth 法在硬件上比原码乘更快。

### 4.3 乘法运算电路(王道 图 2.10)

```
   [被乘数寄存器 X · 32 位]
              │
              │ 32 位
              ▼
   ┌─[ 32 位 ALU ]── 加法 ──────────┐
   │              ▲                  │
   │              │ 32 位             │
   │            ┌─┴───┐               │
   │            │     │               │
   ▼ 32 位      │     │     ┌────────┐
  [C]│       [乘积寄存器 P · 32 位] [乘数寄存器 Y · 32 位]
     ▼          ▲                       │
     └─ 进位 ────────── 右移(同步)───────┘
                       ↑
                  控制逻辑/计数器 C_n
                  (每次循环 C_n 减 1,Y 最低位送控制逻辑)
```

**总线宽:** P 寄存器和 Y 寄存器**同步右移**——P 的最低位被移到 Y 的最高位,Y 的最低位移出送给控制逻辑。

⚠️ **零壹独家亮点 · 阵列乘法器:** 一种"硬件叠加"的快速乘法器,所有部分积**同时**产生,排成阵列再多操作数相加,**1 个时钟周期**就能算完。考研只需知道概念。

### 4.4 乘法溢出判断(命题追踪 2019 / 2020 / 2021)

> **本质:** n 位 × n 位的真实乘积是 **2n 位**;若硬件只保留**低 n 位**作为结果,可能丢掉高位 → 溢出。

**无符号乘法溢出判断:**
> **乘积高 n 位全为 0** → 不溢出;否则溢出。

**有符号乘法溢出判断:**
> **乘积高 n 位每一位都等于乘积低 n 位的符号位**(即高 n 位**全为 0** 或 **全为 1**)→ 不溢出;否则溢出。

```
8 位无符号:  x=200, y=2
   x×y = 400 = 0000 0001 1001 0000(16 位完整)
   高 8 位 = 0000 0001 ≠ 全 0 → 溢出 ⚠️

8 位有符号:  x=100, y=2
   x×y = 200 → 16 位补码 = 0000 0000 1100 1000
   低 8 位最高位 = 1(看似负),高 8 位 = 0000 0000(全 0)
   高位 ≠ 低位符号位 → 溢出 ⚠️
   (实际 200 超出 8 位有符号上限 127)
```

⚠️ **零壹独家 · 位级等价性 ⭐:** 无符号乘法和有符号乘法的**低 n 位结果完全相同**(只要操作数位串相同)——这就是为什么 C 语言 `int * int` 和 `unsigned * unsigned` 在硬件上**用同一条乘法指令**,只在溢出判断上不同。**该考点 2024 已直接命中**。

```c
// 同一段位串,在不同语义下:
int          a = -5;          // 1111 1111 1111 1111 1111 1111 1111 1011
unsigned int b = (unsigned)a; // 同样位串,值 = 4294967291
int          x = a * a;       //   = +25
unsigned int y = b * b;       // = ?(高 32 位丢失,低 32 位 = +25)

// 低 32 位完全一样!这就是位级等价性。
```

## 五、除法运算:试减 + 左移

### 5.1 原码除法 · 加减交替法(不恢复余数法)

**思想:被除数 - 除数,够减上 1,不够减上 0;不够减时不"恢复余数",而是直接做下一步。**

```
规则(简化版):
   符号位 Q_s = X 符号 ⊕ Y 符号
   数值位:被除数减除数 → 商一位 → 余数左移 → 加/减除数 → ...
   - 余数为正 → 商 1, 余数左移 - 除数
   - 余数为负 → 商 0, 余数左移 + 除数
   重复 n 次;若第 n+1 步余数为负,需加 |Y| 修正
```

⚠️ **要点:**
- 商共 n 位(数值位)
- 第一次试减若商 1 → 表示**除数过小**,数值会溢出 → 需要预处理(被除数缩半 / 除数扩大)

### 5.2 补码除法 · 加减交替法

**和原码除的核心区别:符号位参与运算**,商位由"余数与除数同号 → 商 1,异号 → 商 0"决定。

```
规则:
1. 第一步:被除数与除数同号 → 减(被除数 - 除数);异号 → 加(被除数 + 除数)
2. 余数与除数同号 → 商 1, 余数左移一位 - 除数
3. 余数与除数异号 → 商 0, 余数左移一位 + 除数
4. 重复 n 次(机器字长 n+1)
5. 末位恒置 1(精度允许时)
```

⚠️ **末位恒置 1 法**:为了避免最后一步的修正复杂性,直接把商的最低位强制设为 1,误差仅 ±2^(-n),实用足够。

### 5.3 除法运算电路(王道 图 2.11)

**关键寄存器:**
- **R(余数寄存器):** 初始存被除数高 32 位,运行中存中间余数高位,结束存余数
- **Q(商/被除数低位):** 初始存被除数低 32 位,运行中存中间余数低位+商,结束存商
- **Y(除数寄存器):** 始终存除数

每次循环:R+Q 联动**左移**(Q 最高位移入 R 最低位,Q 空出最低位上商)。

⚠️ **除法溢出(王道明确):** **32 位 int 除法只有 -2^31 ÷ -1 会溢出**(因为结果 +2^31 超出 int 上界),其他情况**都不溢出**。这条是 408 真题的硬核细节。

### 5.4 被除数为什么要扩展

```
n 位 ÷ n 位,商最多 n 位。
若被除数和除数都 n 位、按 2n 位输入除法器:
  - 定点小数:被除数低位补 n 个 0(扩成 2n 位)
  - 定点整数:被除数高位补 n 个 0(扩成 2n 位)
  - 否则商位数不够
```

⚠️ **除数为 0 异常:** 整数除法时若 Y=0,触发"除数为 0"硬件异常,操作系统接管处理(Linux 为 SIGFPE)。

> **跨章关联:** 异常处理 → **第 5 章 CPU 控制器**,除零信号会进入异常处理流程。

**【五节收束】** 除法 = 试减 + 左移;原码加减交替不恢复余数;补码加减交替按"同号商 1 异号商 0";只有 -2^31 ÷ -1 在 int 除法中溢出。

## 六、模块小结表 · 一查就会

| 知识点 | 核心要点 | 易错 |
|---|---|---|
| 全加器 FA | S=A⊕B⊕C, Cout=AB+(A⊕B)C | 三入两出 |
| 串行 vs 并行 | 串行 O(n),并行 O(log n) | 32 位机用 CLA |
| OF | C_n ⊕ C_{n-1},判有符号 | 无符号下无意义 |
| CF | Sub ⊕ Cout,判无符号 | 减法时 CF=~Cout |
| SF | 最高位 F_{n-1} | 仅有符号下有意义 |
| ZF | F==0 时为 1 | 两边都有意义 |
| 逻辑移位 | 空位补 0(双向) | 视位串为无符号 |
| 算术左移 | 移出位≠新符号位即溢出 | 补码语义 |
| 算术右移 | 高位补符号位,低位 1 移出丢精度 | 不是补 0 |
| 循环移位 | ROL/ROR 不带 CF;RCL/RCR 带 CF | 零壹考点 |
| 补码加减 | [A±B]补 = [A]补 + [±B]补 | 符号位参与运算 |
| 溢出三法 | 单符号 / 双符号 / 进位异或 | 三法等价 |
| 双符号位 | 00正/01上溢/10下溢/11负 | 不一致 = 溢出 |
| 加减电路 | 1 个加法器 + Sub 控制 | 同电路兼无符号 |
| 无符号比较 | ZF / CF | A<B → CF=1 |
| 有符号比较 | ZF / OF / SF | A>B → OF==SF, ZF=0 |
| 原码乘 | 异或符号 + 绝对值乘 | n 次加+右移 |
| Booth 比较法 | 看 y_n y_{n+1},末位附 0 | 4 种操作 |
| 乘法溢出 | 高 n 位与低 n 位符号位关系 | 有符号要全等 |
| 位级等价性 | 同位串 unsigned/int 乘低 n 位结果一致 | 零壹亮点 |
| 加减交替除法 | 同号商 1 异号商 0 | 末位恒置 1 |
| int 除法溢出 | **仅 -2^31 ÷ -1** | 唯一情况 |
