**对照速记:**

| 码制 | 8 位 00000000 解释 |
|---|---|
| 原码 | +0 |

### 第 22 题 · [陷阱+争议] · 王道印刷可能有错

**王道印的答案:** A 原码 — **但解析自相矛盾**:题面"机器表示 1.0010"按解析自己的算法 = 反码 = C
**任逸选的:** C(从计算讲是对的)
**错因本质:** 这题答案有问题,你的真正困惑(偏置 127 vs 128)指向的是 T24,串题了。

**-0.1101 四码对照(必背):**

| 码制 | -0.1101 表示 | 备注 |
|---|---|---|
| 原码 | 1.**1101** | 符号位 1 + 绝对值 |
| 反码 | 1.**0010** | 数值位每位取反 |
| 补码 | 1.**0011** | 反码 +1 |
| 移码 | **不适用** | 只用于阶码整数,不用于定点小数 |

→ 1.0010 = 反码,正确答案应为 C。**A 大概率是王道印刷错。** 不死磕,把这张表背熟。

**附:偏置量 127 vs 128(T24 真正的考点):**
- **课本默认偏置 = 2ⁿ = 128**(8 位)→ 范围 -128 ~ +127,与补码差符号位
- **IEEE 754 单精度阶码偏置 = 127 = 2ⁿ-1** → 让 0000..0 和 1111..1 留作"特殊值"(非规格化数 / 无穷大)

**口诀:** **课本 128,IEEE 127。** 题给哪个用哪个,不要硬套。

**框架图标注:**
- 🔵 蓝笔 → [移码 · 偏置量] → "**默认 2ⁿ,IEEE 用 2ⁿ-1**"
- 🔴 红笔 → [移码] → "**⚠️ 移码不用于定点小数**"

**识别信号:** 题目给"偏置 127" → 别填 128;给"偏置 128" → 别填 127。**先看题给值再算。**

### 第 29 题 · [陷阱] · 选对但答案记错

**正确答案:** B = -125(2015 真题)
**任逸选的:** B ✓(任逸记成"答案 D"是错的)
**错因本质:** 选对了但解题路径不稳,需要刻进 SOP。

**SOP — N 个 1/0 摆位求最大最小:**

| 目标 | 符号位 | 剩余 1 怎么放 |
|---|---|---|
| 最小整数(最负) | **1** | **全堆末位**(数值位越小,绝对值越大) |
| 最大整数(最正) | **0** | **全堆高位**(数值位越大,真值越大) |

**本题(3 个 1 + 5 个 0,求最小):**
- 符号位 1 用掉 1 个 1 → 剩 2 个 1 堆末位 → **1000 0011**
- 反推真值:取反末位 +1 → 0111 1100+1 = 0111 1101 = 125 → **真值 -125**

**框架图标注:**
- 🔵 蓝笔 → [补码 · 大小判断] → "**N 个 1 摆位:负堆末位,正堆高位**"
- 🔵 蓝笔 → [2015 真题] → "**3 个 1 + 5 个 0 → 最小 -125**"

**识别信号:** "X 个 1 + Y 个 0 组成 N 位补码,求最大/最小" → 走 SOP。

### 第 30 题 · [漏洞] · 真错

**正确答案:** D = 32769(2016 真题)
**任逸选的:** B = 32767
**错因本质:** T2U(同字长 signed→unsigned)当成"取绝对值"了 — 实际是**位串守恒、解释翻倍**。

**三定律:**
> **同字长 signed ↔ unsigned 转换:位串完全不变,只换解释方式。**

**本题机械执行:**
- `short si = -32767` → 16 位补码:
  - +32767 原码 = 0111 1111 1111 1111
  - -32767 反码取反 = 1000 0000 0000 0000
  - 补码 +1 = **1000 0000 0000 0001**
- 强转 unsigned short:位串不变 = 1000 0000 0000 0001
- 按无符号解释:2¹⁵ + 1 = **32769** ✓

**T2U / U2T 完整速查表(必背):**

| 方向 | 原值 | 16 位位串 | 新解释 | 新值 |
|---|---|---|---|---|
| **T2U** | -32767 | 1000 0000 0000 0001 | 无符号 | **32769** |
| **T2U** | -1 | 1111 1111 1111 1111 | 无符号 | **65535** |
| **T2U** | +X(≥0) | 0XXX... | 无符号 | 不变 |
| **U2T** (T32 题) | 65535 | 1111 1111 1111 1111 | 补码 | **-1** |
| **U2T** | 32768 | 1000 0000 0000 0000 | 补码 | **-32768** |

**口诀:** **T2U 负→大正(+2ⁿ);U2T 大正→负(-2ⁿ)。** 正数 T2U 不变。

**框架图标注:**
- 🔴 红笔 → [C · 同字长转换] → "**⚠️ T2U 不是取绝对值**"
- 🔵 蓝笔 → [T2U/U2T] → "**位串守恒,翻一圈 = 2ⁿ**"
- 🔵 蓝笔 → 速查 → "**-1↔65535,-32768↔32768**"

**识别信号:** `unsigned X = signed_var;` 或反向 → "**位串不变,看新解释**"。

**正确答案:** B = -2³¹ ~ 2³¹ - 1(2022 真题)

- 真题写 "**n 位补码**"(总位数)→ 范围 -2ⁿ⁻¹ ~ 2ⁿ⁻¹-1

**默认题目的 n = 总位数(包括符号位)。**

**框架图标注:**

### 第 11 题 · [陷阱] · 不会

**正确答案:** C r1×r2
**任逸选的:** 不会(描述串到 T12 的"模 4 补码")
**错因本质:** r1=F5H, r2=EEH(8 位补码),要先转真值再判范围 [-128, 127],不要硬乘补码。

**机械算法(SOP):**
1. F5H = 1111 0101 → 负数,取反+1 = 0000 1011 = 11 → **r1 = -11**
2. EEH = 1110 1110 → 负数,取反+1 = 0001 0010 = 18 → **r2 = -18**
3. 逐项算:
   - r1+r2 = -29 ∈ [-128,127] → 不溢出
   - r1-r2 = 7 → 不溢出
   - **r1×r2 = (-11)×(-18) = 198 > 127** → **溢出 ✓**
   - r1/r2 = 0 → 不溢出

**口诀:补码硬乘费力不讨好,先转十进制真值,看积是否 ±128 内。**

**框架图标注:**
- 🔴 红笔 → [乘法溢出] → "**⚠️ 转真值再判,别硬乘**"

### 第 21 题 · [漏洞] · 加减电路低位进位

**正确答案:** B (0100 0101, 1101 1001, 1)
**错因本质:** 加法器没有"减"指令——**减法 = X + ~Y + 1**,这个 +1 就是**低位进位输入 Cin = Sub**。即"Sub 信号既控制 Y 取反,又作为 Cin 加上去"。

**电路工作流程:**
```
       X(8位)         Y(8位)
        │              │
        │           [反相器]── Sub=1 时取反
        ▼              ▼
       [加法器]── Cin = Sub
        │
        ▼
      Sum

   Sub=0(加):Y'=Y, Cin=0 → X+Y
   Sub=1(减):Y'=~Y, Cin=1 → X+~Y+1 = X+(-Y)补
```

**本题机械执行(x=69, y=38, 算 x-y):**
- Sub = 1(减法)
- 输入 X = 69 = **0100 0101**
- 输入 Y' = ~38 = ~(0010 0110) = **1101 1001**
- 低位进位 Cin = Sub = **1**

**口诀:加减电路只有 1 个加法器,减法靠"反相 + Cin=1"实现。Cin 就是 Sub 信号。**

**框架图标注:**
- 🔵 蓝笔 → [加减电路] → "**Sub 同时充当 Cin**"
- 🔴 红笔 → [加减电路] → "**⚠️ 减法:Y 取反,Cin=1**"

**识别信号:** "加法器低位进位 Cin / x-y 时输入信息" → Cin = Sub = 1(减),Y 取反。

### 第 32 题 · [漏洞] · 蒙的 · OF/CF 公式

**正确答案:** A CF=0, OF=0
**任逸:** 做对但不清楚本质
**错因本质:** 只会查表,不会用 **OF = Cn ⊕ Cn-1, CF = Sub ⊕ Cout** 公式 + 双视角验证。

**双视角法(最稳):**
- **OF**:把 R1, R2 当**有符号**,看 R1-R2 真值是否超 [-2³¹, 2³¹-1]
- **CF**:把 R1, R2 当**无符号**,看 R1 是否够减 R2(够 → 0,不够 → 1)

**本题(R1=FFFF FFFFH, R2=FFFF FFF0H,算 R1-R2):**
- 有符号视角:R1=-1, R2=-16, R1-R2 = 15 → 在 int 范围 → **OF=0**
- 无符号视角:R1=2³²-1, R2=2³²-16, R1>R2 够减,**不借位 → CF=0**
- 答案:CF=0, OF=0 → A

**公式法验证:**
- R1+(-R2)补 = R1+(~R2+1) = FFFF FFFF + 0000 0010 = (1) 0000 000F
- C_out=1, C_n-1=1(因最高数值位也产生进位)→ **OF = 1⊕1 = 0** ✓
- 减法 CF = ~C_out = ~1 = **0** ✓

**口诀:**
- **OF 公式:Cn ⊕ Cn-1**
- **减法 CF = ~C_out**(够减不借,Cout=1 → CF=0)
- **双视角法做后备验证**

**框架图标注:**
- 🔵 蓝笔 → [标志位本质] → "**双视角:OF 看有符号,CF 看无符号**"
- 🔴 红笔 → [CF] → "**⚠️ 减法 CF = ~Cout(翻转)**"
- 🔵 蓝笔 → [公式速查] → "**OF=Cn⊕Cn-1, CF=Sub⊕Cout**"

**识别信号:** "sub R1 R2 / 求 OF, CF" → 双视角 + 公式互验。

### 第 34 题 · [陷阱] · 2024 真题 · int↔short 反复

**正确答案:** B -32759
**错因本质:** "int → short → int" 不是恒等映射:**截断会改值,再扩展也回不去**。

**机械执行:**
```
int i = 32777
       = 32768 + 9
       = 0000 0000 0000 0000 1000 0000 0000 1001
       = 0x0000 8009

short si = i  ← 截断,只留低 16 位
       = 1000 0000 0000 1001 = 0x8009
       解释:最高位 1 → 负数 → -2^15 + 9 = -32768 + 9 = -32759

int j = si    ← short→int 符号扩展(高 16 位补 1)
       = 1111 1111 1111 1111 1000 0000 0000 1001
       = 0xFFFF 8009
       = -32759
```

**对照表(必背):**

| 转换 | 操作 | 例 |
|---|---|---|
| **大→小**(int→short) | **截低位**(高位扔) | 32777 → -32759 |
| **小→大,有符号** | **符号扩展**(高位补符号) | -32759 → -32759 |
| **小→大,无符号** | **零扩展**(高位补 0) | (unsigned) → 不变 |

**为啥不是 -32777?** 因为 32777 = 32768+9 = 0x8009,**截断后最高位变成符号位**,值翻为 -2^15+9 = -32759,**而不是简单"丢 1 位变 -32777"**。

**口诀:int→short 截低位,看截后最高位决定符号。**

**框架图标注:**
- 🔵 蓝笔 → [int→short] → "**截低 16 位,看新最高位**"
- 🔵 蓝笔 → [2024 真题] → "**32777→-32759→-32759**"
- 🔴 红笔 → [反复转换] → "**⚠️ 大小回转不是恒等**"

**识别信号:** "int → short → int / 反复字长转换" → 截断改值 → 符号扩展回不去。

### 第 35 题 · [漏洞] · 2024 真题 · 三类乘法实现 ⭐

**口诀:**
- **阵列 = 1 周期**(部分积同时产生)
- **循环 = n 周期**(每位移 1 加 1)
- **常数 = 编译期展开成移位+加**(变量也能循环,只是慢)

- 🔵 蓝笔 → [乘法实现] 节点新增三个子节点:

## SOP 卡片(模块 2.2 产出)

### SOP-1 · 补码运算溢出判断(选择题秒杀)
1. 看到"4 个 hex 补码 + 哪个运算溢出" → **全部转十进制真值**
2. 算 +/-/×/÷ 的真值结果
3. 看哪个超 [-2ⁿ⁻¹, 2ⁿ⁻¹-1]
4. **不要硬算补码,极易出错**

### SOP-2 · 加减电路 / 减法机械执行
- 减法 Sub=1 → Y 全部取反 → Cin=1 → 加法器算 X+~Y+1
- 输入端三件套:**X 原值, ~Y, Cin=1**

### SOP-3 · 标志位双视角法
- **OF**:把操作数当**有符号**,看真值是否越 [-2ⁿ⁻¹, 2ⁿ⁻¹-1]
- **CF**:把操作数当**无符号**,看够不够减(够=0,不够=1)
- ALU **同时**输出两个标志,与变量类型无关

### SOP-4 · 大小比较选标志
- **比较中有 unsigned 参与** → 走无符号路径 → 看 ZF/CF
- **纯 signed** → 走有符号路径 → 看 ZF/OF/SF
- C 表达式 `j-1` 当 j 是 unsigned 时 = 0xFF...FF(最大正)

### SOP-5 · 字长转换三类
- **大→小**:截低位(可能改值)
- **小→大,有符号**:符号扩展
- **小→大,无符号**:零扩展
- 反复转换不是恒等

### SOP-6 · 三类乘法实现速查
| 类型 | 速度 | 用途 |
|---|---|---|
| 阵列 | 1 周期 | 通用 CPU |
| ALU+移位循环 | n 周期 | 简易 CPU / 软件 |
| 编译器常数优化 | 1-2 移位+加 | 乘数是常数 |
