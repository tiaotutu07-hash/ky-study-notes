### 第 3 题 · [陷阱] · 选对补反例

**正确答案:** D
**任逸选的:** D ✓
**错因本质:** 选对但说不清楚为什么。

**关键鉴别点:** 二进制小数位权只能是 1/2、1/4、…、1/2ⁿ,所以二进制小数能精确表示的十进制数 = **分母只能是 2 的幂**的有限小数。

**反例口诀:** **分母带 5 → 二进制必死循环**

**框架图标注:**
- 🔵 蓝笔 → [进位计数制 / 十→任意进制] → "**分母非 2 幂 → 二进制无限**"
- 🔵 蓝笔 → 跨章 [2.3 浮点精度] → "**0.1+0.2≠0.3 根源**"

**识别信号:** "任何十进制小数都能精确..." → 立刻 ❌

### 第 13 题 · [陷阱] · 选对补反例

**正确答案:** C
**任逸选的:** C ✓
**错因本质:** 凭感觉选对,没用特殊值代入系统验证。

**核心规则:** **负数补码:数值位部分越小,绝对值越大,即真值越小(更负)。**

**机械步骤:**
1. 写界值 -32 的补码 → 7 位字长(1 符号 + 6 数值): 1, 100000
2. 要 x > -32 → 数值位按无符号比 > 100000
3. → x₁ 必为 1,且 x₂...x₆ 不全为 0

**特殊值排除表:**

| 选项 | 取值 | 真值 | x > -32? |
|---|---|---|---|
| A 1, 000000 | -64 | ❌ |
| B 1, 000000 | -64 | ❌ |
| C 1, 100001 | -31 | ✅ |
| D 1, 000001 | -63 | ❌ |

**框架图标注:**
- 🔴 红笔 → [补码 · 性质] → "**⚠️ 负补:数值位越小绝对值越大**"
- 🔵 蓝笔 → [补码 / 大小判断] → "**特殊值代入法**"

**识别信号:** "若 x > / x < 某值,数值位应满足..." → 走"先写界值补码 + 特殊值代入"

### 第 21 题 · [漏洞] · 真错

**正确答案:** **D 移码**(王道答案就是 D,任逸记成 B 是错的)
**任逸选的:** C 反码
**错因本质:** 选项里看到移码当没看见。**移码"全 0 串 = -2ⁿ 最负值"特性没建立反射。**

**对照速记:**

| 码制 | 8 位 00000000 解释 |
|---|---|
| 原码 | +0 |
| 补码 | +0 |
| 反码 | +0 |
| **移码** | **-128** ← [-128]移 = 2⁷ + (-128) = 0 |

**核心公式:** $[x]_移 = 2^n + x$ → **全 0 = 最负值,100..0 = 0**,与补码刚好符号位相反。

**框架图标注:**
- 🔴 红笔 → [移码 · 4 性质] → "**⚠️ 移码 00..0 = -2ⁿ**"
- 🔴 红笔 → [移码] → "**⚠️ 与补码符号位相反**"
- 🔵 蓝笔 → 选择题策略 → "**4 选 1 含移码 → 优先验 D**"

**识别信号:** ABCD 出现"原/补/反/移"四码并列 → 优先用移码 100..0 / 00..0 两个特殊串验证。

### 第 22 题 · [陷阱+争议] · 王道印刷可能有错

**王道印的答案:** A 原码 — **但解析自相矛盾**:题面"机器表示 1.0010"按解析自己的算法 = 反码 = C
**任逸选的:** C(从计算讲是对的)
**错因本质:** 这题答案有问题,你的真正困惑(偏置 127 vs 128)指向的是 T24,串题了。

**-0.1101 四码对照(必背):**

| 码制 | -0.1101 表示 | 备注 |
|---|---|---|
| 原码 | 1.**1101** | 符号位 1 + 绝对值 |
| 反码 | 1.**0010** | 数值位每位取反 |
| 补码 | 1.**0011** | 反码 +1 |
| 移码 | **不适用** | 只用于阶码整数,不用于定点小数 |

→ 1.0010 = 反码,正确答案应为 C。**A 大概率是王道印刷错。** 不死磕,把这张表背熟。

**附:偏置量 127 vs 128(T24 真正的考点):**
- **课本默认偏置 = 2ⁿ = 128**(8 位)→ 范围 -128 ~ +127,与补码差符号位
- **IEEE 754 单精度阶码偏置 = 127 = 2ⁿ-1** → 让 0000..0 和 1111..1 留作"特殊值"(非规格化数 / 无穷大)

**口诀:** **课本 128,IEEE 127。** 题给哪个用哪个,不要硬套。

**框架图标注:**
- 🔵 蓝笔 → [移码 · 偏置量] → "**默认 2ⁿ,IEEE 用 2ⁿ-1**"
- 🔴 红笔 → [移码] → "**⚠️ 移码不用于定点小数**"

**识别信号:** 题目给"偏置 127" → 别填 128;给"偏置 128" → 别填 127。**先看题给值再算。**

### 第 27 题 · [漏洞] · 真错

**正确答案:** B 0xFFFF 8FA0
**任逸选的:** D 0x8000 8FA0
**错因本质:** 题目明明白白写"16 位补码整数",当成无符号去做了零扩展;且 D 选项的"符号位飞到第 31 位"是**不存在的规则**。

**SOP 两步:**
1. 看原数有/无符号(题给)
2. 有符号 → **符号扩展**(高位补符号位的值);无符号 → **零扩展**(高位补 0)

**本题:** 0x8FA0 = **1**000 1111 1010 0000 → 负数 → 高 16 位全补 1 → **0xFFFF 8FA0**

**对照排雷:**

| 选项 | 操作 | 错在哪 |
|---|---|---|
| A 0x0000 8FA0 | 零扩展 | 把补码当无符号 |
| **B 0xFFFF 8FA0** | **符号扩展** ✓ | |
| C 0xFFFF FFA0 | 数值位被改 | 不存在的规则 |
| D 0x8000 8FA0 | 符号位飞到第 31 位 | 不存在的规则 |

**框架图标注:**
- 🔴 红笔 → [C 转换 · 不同字长] → "**⚠️ 先看原数有/无符号**"
- 🔵 蓝笔 → [小转大] → "**有符号补符号位,无符号补 0**"

**识别信号:** "16 位补码 → 32 位" / "short → int" → 立刻问"原数正还是负",负数高位全 1。

### 第 29 题 · [陷阱] · 选对但答案记错

**正确答案:** B = -125(2015 真题)
**任逸选的:** B ✓(任逸记成"答案 D"是错的)
**错因本质:** 选对了但解题路径不稳,需要刻进 SOP。

**SOP — N 个 1/0 摆位求最大最小:**

| 目标 | 符号位 | 剩余 1 怎么放 |
|---|---|---|
| 最小整数(最负) | **1** | **全堆末位**(数值位越小,绝对值越大) |
| 最大整数(最正) | **0** | **全堆高位**(数值位越大,真值越大) |

**本题(3 个 1 + 5 个 0,求最小):**
- 符号位 1 用掉 1 个 1 → 剩 2 个 1 堆末位 → **1000 0011**
- 反推真值:取反末位 +1 → 0111 1100+1 = 0111 1101 = 125 → **真值 -125**

**框架图标注:**
- 🔵 蓝笔 → [补码 · 大小判断] → "**N 个 1 摆位:负堆末位,正堆高位**"
- 🔵 蓝笔 → [2015 真题] → "**3 个 1 + 5 个 0 → 最小 -125**"

**识别信号:** "X 个 1 + Y 个 0 组成 N 位补码,求最大/最小" → 走 SOP。

### 第 30 题 · [漏洞] · 真错

**正确答案:** D = 32769(2016 真题)
**任逸选的:** B = 32767
**错因本质:** T2U(同字长 signed→unsigned)当成"取绝对值"了 — 实际是**位串守恒、解释翻倍**。

**三定律:**
> **同字长 signed ↔ unsigned 转换:位串完全不变,只换解释方式。**

**本题机械执行:**
- `short si = -32767` → 16 位补码:
  - +32767 原码 = 0111 1111 1111 1111
  - -32767 反码取反 = 1000 0000 0000 0000
  - 补码 +1 = **1000 0000 0000 0001**
- 强转 unsigned short:位串不变 = 1000 0000 0000 0001
- 按无符号解释:2¹⁵ + 1 = **32769** ✓

**T2U / U2T 完整速查表(必背):**

| 方向 | 原值 | 16 位位串 | 新解释 | 新值 |
|---|---|---|---|---|
| **T2U** | -32767 | 1000 0000 0000 0001 | 无符号 | **32769** |
| **T2U** | -1 | 1111 1111 1111 1111 | 无符号 | **65535** |
| **T2U** | +X(≥0) | 0XXX... | 无符号 | 不变 |
| **U2T** (T32 题) | 65535 | 1111 1111 1111 1111 | 补码 | **-1** |
| **U2T** | 32768 | 1000 0000 0000 0000 | 补码 | **-32768** |

**口诀:** **T2U 负→大正(+2ⁿ);U2T 大正→负(-2ⁿ)。** 正数 T2U 不变。

**框架图标注:**
- 🔴 红笔 → [C · 同字长转换] → "**⚠️ T2U 不是取绝对值**"
- 🔵 蓝笔 → [T2U/U2T] → "**位串守恒,翻一圈 = 2ⁿ**"
- 🔵 蓝笔 → 速查 → "**-1↔65535,-32768↔32768**"

**识别信号:** `unsigned X = signed_var;` 或反向 → "**位串不变,看新解释**"。

### 第 34 题 · [陷阱] · 选对但答案记错

**正确答案:** B = -2³¹ ~ 2³¹ - 1(2022 真题)
**任逸选的:** B ✓(任逸记成"答案 D"是错的)
**错因本质:** 概念对,但对自己不自信。**关键是要分清楚 n 是"总位数"还是"数值位数"。**

**铁律:**

> **n 位补码范围(n 含符号位)= [-2ⁿ⁻¹, 2ⁿ⁻¹ - 1]**
> 32 位补码 → -2³¹ ~ 2³¹ - 1 → -2,147,483,648 ~ 2,147,483,647(`int` 范围)

**两套写法都对(看 n 怎么定义):**
- 模块 MD 写 "**n+1 位字长**"(含符号位)→ 范围 -2ⁿ ~ 2ⁿ-1
- 真题写 "**n 位补码**"(总位数)→ 范围 -2ⁿ⁻¹ ~ 2ⁿ⁻¹-1

**默认题目的 n = 总位数(包括符号位)。**

**框架图标注:**
- 🔴 红笔 → [补码 · 范围] → "**⚠️ 看 n 是总位数还是数值位**"
- 🔵 蓝笔 → [int 范围] → "**32 位:-2³¹~2³¹-1**"

**识别信号:** "N 位补码范围" → 总位数 → -2^(N-1) ~ 2^(N-1) - 1。
