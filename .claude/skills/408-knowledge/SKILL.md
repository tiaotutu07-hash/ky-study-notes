## 6 阶段博览群书工作流

### Phase A · 多源扫描

```
Step A1 · 读考纲骨架
  read 408/{科目}/_考纲.md
  定位目标考纲节点及其子项
  
  if 考纲该节有子项:
    直接用子项做骨架
  if 考纲该节无子项(只有大标题):
    综合 5 份材料目录,自动拆出子项
    写入 outline 等任逸确认
    例: (四) 控制器的功能和工作原理
    → AI 拆: (四)-1 基本结构 / (四)-2 硬布线 / (四)-3 微程序 / ...

  ⭐ 考纲展开检查(v6.1 新增):
  考纲用词可能比实际考查范围窄。对每个考纲子项,AI 主动思考:
  - 这个子项在多核/多处理器场景下有没有延伸?(如"写策略"→多核一致性)
  - 有没有经典 trade-off 没列出来?(如"块大小"→块太大太小的权衡)
  - 有没有反直觉/异常现象?(如"FIFO"→Bélády 异常)
  - 有没有编址方式的隐含前提?(如"按字节编址"vs"按字编址"影响偏移位数)
  发现的展开点写入对齐表"考纲隐含"行,在 outline 里供任逸确认。

  🔥 大题级考点标记(v6.2 新增):
  if 原材料中有大题集:
    read 大题集,标记哪些考纲子项是大题级(综合计算/手动模拟):
    🔥 = 必须在 §3 How 带完整逐步推演表
        + §5 Example 带 ⭐⭐⭐ 手动模拟题
        + §8 Quiz 带"自己从头模拟"的题
    普通 = §2 讲清概念 + §5 有例题即可
  if 没有大题集:
    AI 凭考频和题型判断,在 outline 里标注供任逸确认

Step A2 · 扫描原材料目录
  read 原材料/ 文件列表
  .md 优先于 .pdf
  扫描版 .pdf 只在必要时按页码范围读

Step A3 · 建对齐表
  对每份材料,按考纲子项逐个匹配:

  | 考纲子项 | 王道 | 竟成 | 零壹讲义 | 零壹PPT | 课外 |
  |---------|------|------|---------|--------|------|
  | 子项1   | X.X  | X.X  | 讲义N §X| PPT §X | X.X  |
  | 子项2   | ...  | ...  | ...     | ...    | ...  |
  | 王道独有 | ...  |      |         |        |      |
  | 竟成独有 |      | ...  |         |        |      |
  | 课外独有 |      |      |         |        | ...  |

  规则:
  - 一份材料的一个段落只归入一个考纲子项(不重复)
  - 考纲没有但某源有的 → "XX独有"行
  - 考纲有但所有源都没的 → ⚠️ 缺失

Step A4 · ① 覆盖度自检
  - [ ] 考纲每个子项至少 2 源覆盖?
  - [ ] 5 份材料都扫过了?
  - [ ] .md 优先于 .pdf 读取?
  - [ ] 对齐表无空行?
  - [ ] "XX独有"内容已标注?
  未通过 → 补读遗漏材料后重跑 A3
```

```

  - 分类体系

Step B2 · 建多源概念对照表
  | 概念 | 王道 | 竟成 | 零壹 | 课外 |
  |-----|------|------|------|------|
  | XX  | 叫法A| 叫法B| 叫法C| 英文 |

Step B3 · ② 术语统一自检
  - [ ] 同一概念各源叫法都列出了?
  - [ ] study.md 里用哪个叫法已决定?(默认: 王道为主,括号注英文)
  - [ ] 公式符号统一了?
  - [ ] 无遗漏概念?
  未通过 → 补提取后重跑 B2
```

### Phase C · 冲突识别

```
Step C1 · 逐概念比对
  找:
  - 定义矛盾(A说X,B说Y)
  - 公式差异(符号不同 vs 实质不同)
  - 分类差异(A分3类,B分4类)
  - 重点差异(A强调这个,B强调那个)
  - 深度差异(A只提一句,B讲了两页)

Step C2 · 冲突分级
  🔴 实质冲突: 结论不同,必须选边 → 写入 §7 Pitfalls
  🟡 表述冲突: 说法不同本质一样 → 括号注明
  🟢 深度差异: 不矛盾只是详略不同 → 标 [XX补充]

Step C2.5 · 真题盲区扫描(v6.1 新增) ⭐
  if 原材料中有 真题集:
    read 真题集,逐题检查:
    该真题考查的知识点,在 5 源材料中被重点讲了吗?
    
    分类:
    🟣 真题盲区: 真题考过,但 5 源材料都只是一笔带过或没提
      例: Cache 多核一致性(2015/2019真题)、Bélády异常、按字编址
      → 必须写入 §6 Variants 或 §7 Pitfalls,标 [真题补充 20XX]
    🔵 真题强调: 真题反复考,材料也重点讲 → 确认 §5 Example 有对应题
    
  if 没有真题集:
    AI 凭自身知识,对每个考纲子项主动回忆常见真题考点
    标注为 [AI推测,待真题验证],置信度较低但胜于不查

Step C3 · ③ 冲突暴露自检
  - [ ] 所有 🔴 冲突都标了选边理由?
  - [ ] 选边优先级: 考纲 > 王道 > 竟成 > 零壹 > 课外
  - [ ] 🟢 深度差异标注了来源?
  - [ ] 没有假装没冲突?
  - [ ] 🟣 真题盲区都识别了?(v6.1 新增)
  未通过 → 重新比对
```

### Phase D · 逻辑重组(10 步法)

```
Step D1 · 分配内容到 10 步
  §0 考纲对齐 ← Phase A 对齐表精简版
  §1 Why      ← 各源引言/动机段,3-5句
  §2 What     ← Phase B 统一后的定义+分类
  §3 How      ← 各源计算方法/推导/代码,融合
  §4 📺 推荐  ← 标视频推荐+图引用(AI不复述图)
  §5 Example  ← 各源例题按难度排(见 Phase E)
  §6 Variants ← "XX独有"中值得纳入的内容
  §7 Pitfalls ← Phase C 的 🔴 冲突 + 各源 ⚠️ 易错
  §8 Quiz     ← AI 自编 ≥2 题 + 题目本引用 ≥1 题
  §9 知识树   ← 结构化数据块(给自动加肉用)

Step D2 · 确定认知顺序
  §2 §3 内部:
  - 先具体后抽象(先举例再定义)
  - 先简单后复杂
  - 有因果链(因为X → 所以才有Y)

Step D3 · 嵌入八条铁律检查点
  标注: 哪里用讲述式引入(铁律1) / 过渡句(铁律2) /
       全景导图(铁律3) / 收束小结(铁律4) / 因果链(铁律5) /
       先严格定义再简化(铁律6) / 新概念说why(铁律7) / 新术语展开(铁律8)

Step D4 · ④ 认知顺序自检
  - [ ] §2 先具体后抽象?
  - [ ] §3 先简单后复杂?
  - [ ] 10 步之间逻辑连贯?(Why→What→How 递进)
  - [ ] 八条铁律检查点都标了?
  - [ ] 跨章关联 ≥3 处已预埋?
  - [ ] §2→§3 过渡桥已写?(v6.2 新增)
  - [ ] 🔥 大题级考点已在 §3 §5 安排足够篇幅?(v6.2 新增)
  未通过 → 重排后重跑
```

### 八条铁律(在 §2 §3 §6 中强制执行)

```

铁律3 · 模块开头必须有全景导图(3-5句)

  用 > **收束小结:** 格式

铁律5 · 强制因果链连接词(≥5处)
  "因为...所以..." / "正是因为...才需要..."
  "还记得吗..." / "这不就是第X章的Y嘛..."

铁律6 · 易混概念先严格定义,再说简化用法(v6.2 新增)
  ❌ "王道把行=块等价使用,但严格来说..."(先说糊涂话再纠正)
  ✅ "行 = 块 + 辅助位(严格定义)。王道日常混用,
      考试只有算容量时才需要区分。"(先给正确的,再说哪里可以偷懒)

铁律7 · 新概念首次出现必须解释"为什么是这样"(v6.2 新增)
  ❌ "辅助位有: 有效位、标记、脏位、LRU位"(只说有什么)
  ✅ "辅助位有: 有效位、标记、脏位、LRU位。
      为什么没有组索引和块内偏移?因为它们在 CPU 地址里,
      用来'找到这一行'后就扔掉了,只有标记需要存下来做身份证。"

铁律8 · 新术语首次出现必须展开解释,不能只提名字(v6.2 新增)
  ❌ "全写法通常配写缓冲(write buffer)缓解。"(写缓冲是什么?没说)
  ✅ "全写法通常配写缓冲——一个小队列,CPU 把要写的数据丢进去就走,
      队列慢慢往主存写,这样 CPU 不用等。队列满了 CPU 还是得等。"
  适用于: §2 §3 §6 中所有首次出现的专业术语
```

## G4 · 默写空白版 PDF

Claude Code 用 bash 运行 reportlab 生成:

```python
from reportlab.lib.pagesizes import A4
from reportlab.lib.styles import ParagraphStyle
from reportlab.lib.units import cm
from reportlab.platypus import SimpleDocTemplate, Paragraph
from reportlab.pdfbase import pdfmetrics
from reportlab.pdfbase.cidfonts import UnicodeCIDFont

pdfmetrics.registerFont(UnicodeCIDFont('STSong-Light'))

def make_blank_pdf(module_name, questions, output_path):
    """
    questions = [
        {"category": "一、概念默写", "items": ["Q1. ...", "Q2. ..."]},
        {"category": "二、易错默写", "items": ["QN. ⚠️ ..."]},
        {"category": "三、跨章关联", "items": ["QN. ..."]},
    ]
    """
    doc = SimpleDocTemplate(
        output_path, pagesize=A4,
        leftMargin=2*cm, rightMargin=2*cm,
        topMargin=2.5*cm, bottomMargin=2.5*cm,
    )
    title_s = ParagraphStyle('T', fontName='STSong-Light', fontSize=20, spaceAfter=20)
    head_s = ParagraphStyle('H', fontName='STSong-Light', fontSize=14,
                            spaceAfter=10, spaceBefore=15)
    q_s = ParagraphStyle('Q', fontName='STSong-Light', fontSize=12,
                         spaceAfter=80, leading=18)
    meta_s = ParagraphStyle('M', fontName='STSong-Light', fontSize=11,
                            spaceAfter=20)
    story = [
        Paragraph(f"默写测试 · {module_name}", title_s),
        Paragraph("日期:______年___月___日   得分:______/100", meta_s),
    ]
    for cat in questions:
        story.append(Paragraph(cat['category'], head_s))
        for q in cat['items']:
            story.append(Paragraph(q, q_s))
    doc.build(story)
    print(f"✅ PDF 生成: {output_path}")
```

**执行规则:**
- 题干只显示,不显示答案
- 每题间至少 80pt 空白(给手写)
- 题号与答案版完全一致

| ⑧ | 全章跑完 | 一致性 | 术语/关联/知识树填充率/题目本全章覆盖 |

**每层自检格式统一:**
```
### 自检 {编号} · {名称}
- [ ] 检查项1
- [ ] 检查项2
...
⚠️ 未通过项: [列出] → 回退修复后再前进
```

**自检 ①-⑤ 是 AI 内部执行,不输出文件。**
**自检 ⑥ 写入 outline 供任逸 review。**
**自检 ⑦ 结果附在 study.md 头部注释里。**
**自检 ⑧ 输出 consistency_report 文件。**

## 通用约束

```
1. 不越界: 只处理本考纲节点,不涉及其他节点
2. 不注水: 宁可少写几条高质量,不凑数
3. 不废话: 每句话回答"这对考试有什么用"
4. 不替代: 不替代视频讲动态过程,不替代书上的图
5. 不隐瞒: 不确定的必须标注,冲突必须暴露
6. 科目自适应:
   - DS 侧重算法步骤 + 伪码
   - COA 侧重计算方法 + 推导
   - OS 侧重概念对比 + PV 操作
   - CN 侧重协议 + 报文字段
```
