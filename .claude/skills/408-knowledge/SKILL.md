## 阶段 1 · MD 正文

**文件名:** `模块{章}_{节}_{标题}.md`(例:`模块8_1_插入排序.md`)

### 必备元素(全部必须有)

| 元素 | 要求 |
|---|---|
| **ASCII 树形/结构图** | 每个数据结构、每次操作变化都配图,保真到空格缩进 |
| **带中文注释的 C 代码** | 关键算法必须逐行注释,不能光贴代码 |
| **手动推演(trace)** | 重要算法必须用具体数据跑一遍,逐步给出执行过程 |
| **Markdown 对比表** | 易混知识点必须用表格对比 |
| **⚠️ 易错警告** | 陷阱、特例、408 真题考点,统一 ⚠️ + 加粗 |
| **跨章关联标签** | `**跨章关联:**` 引出,本模块至少 3 处 |
| **本质理解** | 重要结论必须深挖"为什么",不能只列"是什么" |

### 写法五条铁律(完整版,不省略)

**1. 禁止开门甩定义 · 必须讲述式引入**

```
❌ "## 线索二叉树
线索二叉树是..."

✅ "## 线索二叉树
还记得上一节说的 n+1 个空指针浪费吗?今天我们变废为宝..."
```

**2. 相邻知识点必须有过渡句**

```
❌ "### 先序遍历(略)
### 中序遍历(略)"

✅ "讲完先序,只需把 visit 从最前面挪到左递归之后,就得到中序——代码一字不改,顺序一变。"
```

**3. 模块开头必须有全景导图(3-5 句)**

写明:本节解决什么问题、核心脉络是什么、学完能做什么题。

**4. 每个 ## 大块末尾必须有收束小结(2-3 句)**

把这个大块的核心结论用 2-3 句话扣住,让任逸抬眼就知道"刚才学的精华是这三点"。

**5. 强制使用因果链连接词(≥5 处)**

- "因为...所以..."
- "正是因为...我们才需要..."
- "还记得吗..."
- "这不就是第 X 章的 Y 嘛..."
- "所以才会有..."

### 模块末尾固定两件套

**件套 1 · 总结表**
```
| 知识点 | 核心要点 |
|--------|---------|
| ... | ... |
```

**件套 2 · 串联助记框架(ASCII 树状图)**

把本小节所有知识点串成一张树状图。

### 阶段 1 自检清单

```
- [ ] 三份材料都翻过并融合?

- [ ] 每个 ## 末尾有收束小结?
```

## 阶段 2 · 框架图标注 + 默写双版本

### Part A · 框架图标注清单(MD)

**格式(严格遵守):**

```markdown
## 框架图标注清单 · [模块名]

### 🔵 蓝笔标注(知识点/规律)  共 X 条
1. 标在 [框架图节点名] 旁 → "[≤15字的知识点/规律]"
2. 标在 [框架图节点名] 旁 → "[≤15字]"
...

### 🔴 红笔标注(易错/陷阱)  共 X 条
1. 标在 [框架图节点名] 旁 → "⚠️ [≤20字的易错描述]"
2. 标在 [框架图节点名] 旁 → "⚠️ [≤20字]"
...

### ↔️ 跨章关联标注  共 X 条
1. 标在 [框架图节点名] 旁 → "→第X章·[关联知识点名]"
...
```

**铁律:**
- 蓝笔每条 ≤15 字(写不下就是你的锅)
- 红笔每条 ≤20 字
- 必须指明框架图上的具体节点名
- 总标注数 = MD 中所有核心知识点 + 所有 ⚠️ 的数量

### Part B · 默写答案版(MD)

**格式:**

```markdown

> 空白版见同名 PDF 文件。

## 一、概念默写(蓝笔对应)

**Q1.** [知识点名] 的定义/性质是什么?

→ **答案**:[标准答案,与框架图蓝笔条目一致]

**Q2.** [算法名] 的 3 个关键步骤是?

→ **答案**:① ... ② ... ③ ...

## 二、易错默写(红笔对应)

**Q5.** ⚠️ [概念A] 和 [概念B] 的核心区别是?

→ **答案**:...

## 三、跨章关联默写

**Q8.** [本节知识点] 和第 X 章的 [某知识点] 有什么联系?

→ **答案**:...
```

**铁律:**
- 题数 = 蓝笔标注数 + 红笔标注数 + 跨章标注数(一一对应)
- 每题答案 = 标注清单里对应的内容(任逸自测)

### Part C · 默写空白版(PDF) ⭐ Claude Code 直接执行

**Claude Code 用 bash 工具运行下面这段代码生成 PDF**,不要丢给任逸让他跑:

```python
from reportlab.lib.pagesizes import A4
from reportlab.lib.styles import ParagraphStyle
from reportlab.lib.units import cm
from reportlab.platypus import SimpleDocTemplate, Paragraph
from reportlab.pdfbase import pdfmetrics
from reportlab.pdfbase.cidfonts import UnicodeCIDFont
from reportlab.lib.enums import TA_LEFT

pdfmetrics.registerFont(UnicodeCIDFont('STSong-Light'))

def make_blank_pdf(module_name, questions, output_path):
    """
    questions = [
        {"category": "一、概念默写", "items": ["Q1. ...", "Q2. ..."]},
        {"category": "二、易错默写", "items": ["Q5. ⚠️ ..."]},
        {"category": "三、跨章关联", "items": ["Q8. ..."]},
    ]
    """
    doc = SimpleDocTemplate(
        output_path, pagesize=A4,
        leftMargin=2*cm, rightMargin=2*cm,
        topMargin=2.5*cm, bottomMargin=2.5*cm,
    )

    title_s = ParagraphStyle('T', fontName='STSong-Light', fontSize=20, spaceAfter=20)
    head_s = ParagraphStyle('H', fontName='STSong-Light', fontSize=14,
                            spaceAfter=10, spaceBefore=15)
    q_s = ParagraphStyle('Q', fontName='STSong-Light', fontSize=12,
                         spaceAfter=80, leading=18)
    meta_s = ParagraphStyle('M', fontName='STSong-Light', fontSize=11,
                            spaceAfter=20)

    story = [
        Paragraph(f"默写测试 · {module_name}", title_s),
        Paragraph("日期:______年___月___日   得分:______/100", meta_s),
    ]
    for cat in questions:
        story.append(Paragraph(cat['category'], head_s))
        for q in cat['items']:
            story.append(Paragraph(q, q_s))

    doc.build(story)
    print(f"✅ PDF 生成: {output_path}")

# 调用示例(根据本模块实际内容填充)
questions = [
    {"category": "一、概念默写", "items": [...]},
    {"category": "二、易错默写", "items": [...]},
    {"category": "三、跨章关联", "items": [...]},
]
make_blank_pdf("模块8.1 · 插入排序", questions,
               "/绝对路径/模块8_1_默写_空白版.pdf")
```

**执行规则:**
- 用 `reportlab`,Windows 环境若报字体缺失:`pip install reportlab` 即可
- 题目内容**只显示题干,不显示答案**
- 每题之间至少 80pt 空白(给手写)
- 题号编号要与答案版完全一致
- 横向 A4 布局可考虑(题多时);题少时纵向

## 批处理模式

任逸说"跑第X章所有小节"时:

```
1. view 第X章目录,列出所有原材料(王道PDF的所有小节)

   ✅ 已完成: 8.1, 8.2, 8.3
```

**批处理时不要中间询问任逸**,一次跑完。出错就跳过继续下一个,最后汇报。

## 通用约束

2. **不注水:** 宁可少写几条高质量,不凑数

4. **科目自适应:**
   - DS 侧重算法步骤 + 伪码
   - COA 侧重计算方法 + 推导
