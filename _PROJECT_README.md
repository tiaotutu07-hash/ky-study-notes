## 4 · 标注颜色系统(物理框架图)

任逸的物理框架图用 4 色笔标注,你产出"框架图标注清单"时必须严格遵守:

| 颜色 | 用途 | 字数限制 |
|---|---|---|
| 🔵 蓝笔 | 知识点 / SOP / 规律 | ≤15 字 |
| 🔴 红笔 | 易错点 / 陷阱 / 真题坑 | ≤20 字 |
| 🟣 紫笔 | 跨章关联 / 二级结论 | ≤15 字 |
| 🟢 绿笔 | 例题位置编号 | 仅编号(如 `E14-2`) |
| ★ 星号 + 橙荧光 | 秒杀技巧 | ≤10 字 |

字数限制是**物理写作约束**(框架图留白少),不是审美建议。超字数 = 写不下 = 你的锅。

## 6 · AI 工作模式约定

### 模式 A · 主动产出(skill 调用)
任逸用具体命令触发,如"跑模块 8.1"。
你按 skill 流程**一次跑完**,中间不问任逸。

### 模式 B · 答疑(任逸做题撞墙)
任逸贴题或截图问"为啥选 X"。
**直接给答案,不要复述题目。**
结构:
1. 正确答案(1 行)
2. 错误选项错在哪(每个选项一行)
3. 这题考点本质(1-2 行)
4. 识别信号(下次怎么秒判)≤15 字

### 模式 C · 规划/复盘
任逸问"接下来怎么安排""这个数据说明什么"。
给数据驱动的方案,不给鸡汤。
"基于你的 X 和 Y,应该这样:1/2/3"
