## 二、Day 1 下午:skill v6.0 改造(3-4h)

### Step 6: 改造 408-knowledge skill(2-3h)

**核心改动点**(对比 v5.2):

#### A. 加 6 阶段博览群书工作流
- Phase A: 多源扫描
- Phase B: 概念抽取(三材料对照表)
- Phase C: 冲突识别
- Phase D: 逻辑重组(7步法)
- Phase E: 难度梯度规划
- Phase F: 输出 outline + 等审批

#### B. 加多层自检
- Phase A 后: 覆盖度自检
- Phase B 后: 术语统一自检
- Phase C 后: 冲突暴露自检
- Phase D 后: 认知顺序自检
- Phase E 后: 难度梯度自检
- 最终: 综合自检报告

#### C. 加 7 步法输出格式
1. 考纲对齐
2. Why
3. What
4. How
5. 📺 动态过程(视情况插)
6. Example
7. Variants
8. Pitfalls
9. Quiz
10. 知识树更新

#### D. 加自动加肉机制 ⭐ 新增
跑完每个小节,自动:
- 提取 study.md 关键要点
- 填到 `_知识树/CO/第N章.md` 对应叶子的字段(本质 / 关键参数 / 🔴红坑 / 📝讲义路径)
- 更新进度跟踪表

#### E. 加批量跑章逻辑
- Round 1: 章节级总览(整章读)
- Round 2: 串行跑小节(每节独立 prompt + 章节锚点)
- Round 3: 全章一致性自检

#### F. 加 outline 审批门
- 跑章节前先输出 outline.md
- 等任逸审批"全部同意 / 调整 X"
- 同意后才深入

## 六、常见问题排查(预备)

### Q1: skill 跑章时崩了 / 半路停了
A: 当前对话上下文太长。新开 Claude Code 会话,从断点继续("从 §3.4 继续跑")。

### Q2: 加肉字段填错了 / 想撤销
A: git 上一次 commit 是兜底,任逸或妻子手动用 git reset 即可。

### Q3: 知识树叶子和考纲对不上
A: 优先以王道章节为准(主教材),考纲为粗骨架。如果差异大,在叶子 metadata 里标注。

### Q4: 验证时第3章产出质量低
A: 不要硬刚,先看 _自检报告.md,找出哪个 Phase 有问题,针对性调 prompt。

### Q5: 妻子 OCR 进度跟不上
A: 接受降级:用王道整本 PDF 直接给 AI 读,不强求 MD 转换。质量稍降但能跑。
